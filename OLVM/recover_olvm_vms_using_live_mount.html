Recovering OLVM virtual machines using live mount

Recover an Oracle Linux Virtualization Manager virtual machine by creating a Live Mount from a snapshot to provide immediate access to data.

About this task

A Live Mount creates a new virtual machine from a point-in-time copy of a source virtual machine. This process does not affect the original virtual machine, making it a reliable method for rapid data access, disaster recovery testing, or database validation. The live-mounted virtual machine uses the Rubrik cluster as its primary storage domain and is initially set to the Don't Protect state.

Procedure

  1. Log in to RSC.
  2. Open and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click the OLVM VMs inventory card.
    The OLVM VMs page appears.
  5. Select the OLVM virtual machine that includes the virtual machine for live mount.
    The virtual machine details page appears.
  6. On the Snapshots card, select a day marked with a green dot.
    The green dot indicates that at least one successful snapshot was created on that day.
    The Snapshots card displays the Day view.
  7. Open for the snapshot and select Recover.
    The VM Recovery wizard appears.
  8. Select Live Mount and click Next.
    The VM Recovery wizard displays the Select compute resource page.
  9. Select the host to recover and click Next.
    Click items in the Name field until you reach the host to recover.
  10. Optional: Expand Advanced Export Options and select Power on the virtual machine after export.
    Select this option to start the virtual machine once the live mount is complete.
  11. Click Next.
  12. Select the virtual disk and click Next.
  13. Select the network for exporting the virtual machine.
  14. Review the confirmation details and click Export.

Results

You successfully live mounted and recovered an OLVM virtual machine snapshot. RSC creates a new virtual machine from the selected snapshot.

What to do next

You can monitor the progress of the live mount task from the Events page.
