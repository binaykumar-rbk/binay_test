Manage reports based on System Tags

You can use the System Tags filter in RSC to generate reports and view metrics based on the created tags.

This topic describes a feature or features that Rubrik is gradually rolling out to customers.

You can generate customizable reports based on the types of objects that you group using System Tags. You can either create and schedule custom reports or utilize predefined reports using RSC.

You can use Reports for the following benefits:
  • Audits: Using reports containing data for Rubrik clusters or other sources for audits
  • Data management: Using report metrics for data management and planning
  • Customizable views: Establishing views of the most common system metrics using various filters
  • Operational visibility: Tracking compliance, monitoring capacity, and overseeing the status of protection and recovery tasks across the environment

Reports available with the System Tags filter

The following table lists reports available with the System Tags filter:
Report Name Description Use case
Object Capacity report Displays the amount of transferred local data and used storage. To gain insights on storage consumption for effective resource management and planning of objects associated with a particular System Tag.
Object Capacity Over Time Displays the amount of transferred local data and used storage on a monthly, daily, or hourly basis. RSCsupports creating custom reports for specific objects. To gain insights on storage consumption over a period of time for effective resource management and planning of objects associated with a particular System Tag.
Event Report Displays the RSC events status. Reports use colors to indicate Critical, Warning, and Information severity levels. You can view a column for the reason for failed events, and each event is a hyperlink to the event series in RSC. To monitor and troubleshoot event failures by using System Tag filters for isolating RSC events based on objects associated with a particular System Tag.
Indexing Report Displays information about the indexing status of objects. To monitor the indexing status of objects associated with a particular System Tag.
Compliance Report Displays the compliance status of the selected workload. To determine whether the snapshots taken for an object are in compliance with the policy schedule of the objects associated with a particular System Tag.
Protection Report Displays protection information for the selected workload. To check the health of a backup for data integrity and safety and view whether a protection policy is assigned to the objects associated with a particular System Tag.
Protection Tasks Details Displays the protection tasks for each cluster and a summary table with detailed information. Task types include Backup, Archival, Index, Replication, and Log operations. To monitor the status and details of protection tasks across clusters with objects associated with a particular System Tag.
Recovery Tasks Details Displays the status of the recovery tasks for each cluster and a summary table with detailed information. Task types include Export, File Download, File Restore, In-Place Recovery, Instant Recovery, Live Mount, Restore, and Validation. To monitor the status and details of recovery tasks across clusters with objects associated with a particular System Tag.
Note:
When generating a report, you can filter the results by selecting one System Tag at a time. This ensures optimal performance when aggregating data across multiple workloads.

For more information about reporting, see Reports in RSC User Guide.
