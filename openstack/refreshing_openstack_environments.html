      • AHV virtual machines

      • File systems

        RSC provides management and protection of file system data on supported Linux, Unix, and Windows hosts through filesets.

        RSC provides data management, SLA Domain protection, and recovery for OpenStack virtual machines.

        • Add an OpenStack virtual machine

          Add an OpenStack virtual machine to RSC so you can perform snapshots to protect your data.

        • OpenStack virtual machine management

          You can use RSC to onboard, refresh, edit, monitor, and delete OpenStack virtual machines.

          • Refreshing OpenStack environments

            Refresh OpenStack environments to display the current settings.

          • Removing OpenStack environments

            Remove an OpenStack environment from RSC when you no longer need it.

          • Editing OpenStack environments

            Edit the Keystone authentication settings for an OpenStack environment that RSC manages.

          • Editing an OpenStack proxy virtual machine environment

            Edit the network, flavor, and security group settings for an OpenStack proxy virtual machine.

          • Monitoring OpenStack proxy virtual machines

            Monitor the proxy OpenStack virtual machines to determine their status.

        • Set up protection for OpenStack virtual machines

          RSC protects OpenStack virtual machines and regions using SLA Domains and on-demand snapshots.

        • Manage protection for OpenStack virtual machines

          After completing the protection setup for OpenStack virtual machines, RSC takes automated snapshots of those virtual machines and disks based on the configured policies in SLA Domains. You can also take on-demand snapshots, recover files and folders from OpenStack virtual machines, and export OpenStack virtual machine snapshots.

        • Recover OpenStack virtual machines

          You can recover files and folders used by OpenStack virtual machines by mounting snapshots from local clusters or remote archival locations. You can also export an OpenStack virtual machine snapshot to create a point-in-time copy for backup or migration.

Refreshing OpenStack environments

Refresh OpenStack environments to display the current settings.

Procedure

  1. Log in to RSC.
  2. Open and select Settings.
    The Settings page appears.
  3. Expand the Data Sources menu and select OpenStack.
    The OpenStack details page appears.
  4. Select and click Refresh for the OpenStack environment.

Results

You have successfully refreshed the OpenStack environment. RSC now displays the current settings for that environment.
