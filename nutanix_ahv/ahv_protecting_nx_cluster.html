Protecting Nutanix clusters

Specify an SLA Domain setting for Nutanix clusters to have the setting applied to the objects and virtual machines contained by the clusters and server.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click the AHV VMs inventory card.
    The Nutanix Virtual Machines page appears.
  5. Click Clusters.
    The Clusters tab appears.
  6. Select a Nutanix cluster.
    Select multiple Nutanix clusters to assign the same setting to all of them. To help find Nutanix clusters, use the filters, sort the entries by the Name column, or use the Search by name field.
  7. Click Manage Protection.
    The Manage Protection wizard appears.
  8. Choose a protection option.
    OptionDescription
    SLA Domain The SLA Domain is assigned to the selected Nutanix clusters.

    Use the Search by name field to find an SLA Domain.
    Clear existing assignment The SLA Domain is assigned from the next higher level object based on inheritance rules.
    Don't protect The selected Nutanix clusters are excluded from all further SLA Domain assignments and protection.
  9. (Don't protect) In Existing snapshot retention, choose an option for the existing snapshots.
    OptionDescription
    Use current SLA Domain for retention The retention is derived from the current SLA Domain.

    This is the default option.
    Keep forever The snapshots are retained until they are manually deleted.
    Expire immediately The snapshots are deleted from the Rubrik cluster.
  10. Click Next.
    The Confirmation page appears.
  11. Review the summary information and click Assign.

Results

RSC processes the specified assignments and begins protecting the selected Nutanix clusters according to the assignments.
