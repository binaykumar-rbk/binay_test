Adding Nutanix Prism Central to Rubrik clusters

To protect AHV virtual machines managed by Nutanix Prism Central, add Nutanix Prism Central to one or more Rubrik clusters by providing the required details, such as IP address, login credentials, and root certificate.

Before you begin

Ensure that you perform the following actions:
  • Complete the prerequisites outlined in Prerequisites for onboarding Nutanix workloads.
  • Obtain a user credential to add that has cluster administration privileges for the Nutanix Prism Central instance.

    Ensure that the user credentials are the same for Nutanix Prism Central instance and the Nutanix clusters when you add Nutanix Prism Central to a Rubrik cluster running Rubrik CDM version 9.2.0 or earlier. The user credentials can be different for the Nutanix Prism Central instance and the Nutanix clusters when you add Nutanix Prism Central to a Rubrik cluster running CDM version 9.2.1 or later.

About this task

You can add a Nutanix Prism Central instance only to a Rubrik cluster that is running CDM version 9.0 or later. Additionally, you can add a Nutanix Prism Central instance to more than one Rubrik cluster if the Rubrik clusters are running CDM version 9.1.1 or later. You can enable disaster recovery for a Nutanix Prism Central instance if the Rubrik clusters are running CDM version 9.2.0 or later.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Data Sources and then click Nutanix.
    The Nutanix page appears.
  4. Click Add Prism Central.
    The Add Prism Central wizard appears.
  5. Review the What to expect? page and click Next.
  6. In Prism Central, enter the IP address or hostname of the Nutanix Prism Central instance to add.
  7. In Username, enter a username that has cluster administration privileges for the Nutanix Prism Central instance to add.
  8. In Password, enter the password of the user account.
  9. In the certificate field, paste the text of the trusted Certificate Authority (CA) root certificate for Nutanix Prism Central for TLS validation.
    This is the root certificate you used for the Nutanix Prism Central and associated Nutanix clusters or Nutanix Cloud Clusters (NC2), but if the root certificate was not set up, you can provide the Nutanix Prism Central certificate. RSC then automatically retrieves the certificate for each Nutanix cluster or NC2 when establishing a connection with them.
    The following command retrieves the Nutanix Prism Central certificate:

    openssl s_client -connect IP:port -tls1_2

    Where IP is the IP address of Nutanix Prism Central and port is the web port of Nutanix Prism Central.

  10. In Rubrik cluster, select a Rubrik cluster, and then click Next.
    RSC uses the selected Rubrik cluster to establish a connection with Nutanix Prism Central and verify the network connectivity and credentials.
    The Select Nutanix Clusters to Add to RSC page appears, listing the Nutanix clusters managed by the specified Nutanix Prism Central instance.
  11. Select the Nutanix clusters to add and click Next.
    The Map Prism Central Clusters page appears and displays the selected Nutanix clusters.
  12. Optional: Select Add different credentials for each Nutanix cluster to connect with RSC.
    Use this option if different login credentials exist in Nutanix Prism Central and the Nutanix clusters.
  13. From Rubrik CDM Cluster, select a Rubrik cluster to map to the Nutanix cluster.
  14. In Username and Password, enter the username and password for each Nutanix cluster.

    The Username and Password option appears when you choose to add different credentials for each Nutanix cluster and select a Rubrik cluster running CDM version 9.2.1 or later. RSC auto-populates the same credentials provided for Nutanix Prism Central when you select a Rubrik cluster running CDM version 9.2.0 or earlier.

    You can use the Bulk Add option to add multiple Nutanix clusters with the same credentials to connect with RSC. You can select a Rubrik cluster and provide the credentials to apply to all or selected Nutanix clusters.

  15. Click Run Prechecks.
    RSC performs connectivity tests and verifies the credentials to ensure smooth communication between Nutanix and Rubrik clusters. You cannot change the mapping once you map a Rubrik cluster to a Nutanix cluster.
    The Prechecks Status page appears and displays the failed and successful prechecks.
  16. Click Next.
    If prechecks succeed for one or more Nutanix clusters and fails for other clusters, retry prechecks or continue with a partial configuration.
    The Enable Backup Support for Nutanix DR page appears.
  17. Optional: Turn on Enable Data Protection Support for Nutanix DR.
    This option enables disaster recovery, to ensure uninterrupted protection of the AHV virtual machines when the virtual machines migrate between sites during a planned failover.
  18. Click Add.

Results

RSC adds Nutanix Prism Central and maps the Nutanix clusters to the selected Rubrik clusters.
