        RSC provides data management and protection for virtual machines deployed in a Nutanix Acropolis environment.

        • AHV virtual machine protection

          RSC provides protection for AHV virtual machines either by individually assigning the virtual machine to an SLA Domain or through automatic protection.

        • Nutanix Cloud Clusters on AWS and Azure

          Nutanix NC2 provides a platform for running applications in public clouds and for extending your on-premises Nutanix infrastructure in AWS and Azure. With RSC, you can protect Nutanix NC2 workloads on AWS and Azure.

        • Disaster Recovery support for Nutanix Prism Central

          Disaster Recovery for Nutanix Prism Central ensures uninterrupted protection of the AHV virtual machines during a planned failover.

        • Addition of Nutanix clusters to RSC

          By adding Nutanix clusters, Nutanix Cloud Clusters (NC2), or Nutanix Prism Central to RSC, you can protect the AHV virtual machines deployed on Nutanix clusters or NC2.

        • AHV virtual machine protection

          SLA Domains can be used to apply the data protection policies to an individual virtual machine or a selected set of virtual machines.

        • Excluding AHV virtual machine disk files

          Exclude specific managed disks while protecting other managed disks attached to a virtual machine instance.

        • AHV virtual machine scripts

          You can configure the Rubrik cluster to run scripts on a guest OS before creating a snapshot, after creating a snapshot, and after the Rubrik cluster completes the backup process.

        • Rubrik Backup Service installation for AHV

          The Rubrik Backup Service is a software installed on a host that coordinates activity between the Rubrik cluster and the host.

        • Network segmentation on Nutanix clusters

          Rubrik supports AHV virtual machine backup from Nutanix clusters that have network segmentation enabled.

        • AHV snapshot consistency

          With the AHV snapshot consistency feature, you can select the type of snapshot you want RSCto take. Depending on your specific requirements, RSCcan take either application-consistent snapshots or crash-consistent snapshots.

        • On-demand snapshots of AHV virtual machines

          In addition to policy-based snapshots, you can create AHV virtual machine snapshots by using the on-demand snapshot process.

        • Local host page for AHV virtual machine

          The local host page provides detailed information about the virtual machine, protection of the virtual machine, available snapshots, and tasks related to the virtual machine and snapshots.

        • Recovery of AHV virtual machines

          You can recover an AHV virtual machine or specific disks within a virtual machine from snapshots that are locally available on a Rubrik cluster.

          • Recovering AHV files or folders

          • AHV virtual machine recovery using Live Mount

            Rubrik supports the Live Mount operation for AHV virtual machine snapshots from Rubrik clusters on to Nutanix clusters.

            • AHV virtual machine Live Mount operations

              The operations available for a Live Mounted AHV virtual machine on the Live Mounts page vary depending on the type of the Live Mount.

            • Creating a live mount for AHV

              Live mount an AHV virtual machine on a Nutanix cluster from a snapshot on the Rubrik cluster. The live mounted virtual machine uses the Rubrik cluster as its datastore.

            • Live Mounts page for Nutanix virtual machines

              The Live Mounts page for Nutanix virtual machines displays details about the mounted virtual machines and virtual disks and provides options to unmount virtual machines and virtual disks, change the power status, and migrate the data source.

            • Unmounting an AHV virtual machine

              Unmount the live mounted AHV virtual machine from the Rubrik cluster.

            • Powering off an AHV virtual machine after live mount

              Power off a mounted AHV virtual machine from the Live Mounts page.

            • Migrating a live mounted AHV virtual machine

              Migrate the data source of a live mounted AHV virtual machine from a Rubrik cluster to a Nutanix storage container.

          • Exporting an AHV virtual machine snapshot

            Perform an export operation to create a virtual machine from a point-in-time copy of the source virtual machine.

          • AHV virtual disk recovery by mounting

            You can use RSC to recover specific virtual disks in an AHV virtual machine by mounting the disks from a snapshot on to a target AHV virtual machine.

          • Downloading an AHV virtual disk to a CDM cluster

            Download a specific virtual disk from an AHV virtual machine snapshot to a CDM cluster.

          • Downloading an AHV virtual disk to a local machine

            After you have downloaded the AHV virtual disks to your CDM cluster, download the virtual disks to a local machine for offline analysis.

          • Recovery of a batch of AHV virtual machines

            You can recover a batch of AHV virtual machines in a single operation from the Nutanix Virtual Machines inventory page.

          • AHV virtual machine in-place recovery

            Nutanix AHV virtual machine in-place recovery provides a faster and easier option for recovering an AHV virtual machine from snapshots, compared to other recovery options.

          • AHV virtual machine instant recovery

            You can use the instant recovery option to quickly replace the source virtual machine with a new virtual machine created from a snapshot.

Powering off an AHV virtual machine after live mount

Power off a mounted AHV virtual machine from the Live Mounts page.

About this task

The Live Mounts page lists all recovered virtual machines that were mounted using Live Mount from the local Rubrik cluster.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Live Mounts.
    The Live Mounts page appears.
  4. Click Nutanix Virtual Machine.
    A list of the mounted virtual machines appears.
  5. Locate the recovered virtual machine to power off.
    To help find a specific virtual machine, use the filters, sort the entries by column heading, or use the search field.
  6. Open the ellipsis menu for the recovered virtual machine and select Power Off.
    A confirmation message appears.
  7. Click Power Off.

Results

RSC initiates a task for the Rubrik cluster to gracefully power down the selected virtual machine.
