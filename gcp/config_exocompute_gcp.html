Configuration of Exocompute on GCP

You can configure Exocompute on GCP to launch Exocompute resources to perform tasks such as restoring files and leveraging storage tiering.

When you configure Exocompute on GCP, RSC launches an Exocompute resource to perform tasks, such as restoring files, performing export and restore operations using an archived snapshot, and leveraging storage tiering. You can configure Exocompute when adding a GCP project to RSC. You can configure only one Exocompute host for GCP. If an Exocompute host already exists, the Exocompute host option is not available for selection while adding a GCP project. For more information on how to add a GCP project for the Exocompute use case, see Adding GCP projects for protection using OAuth or Manual onboarding for GCP projects.
Note:
The Exocompute configuration process requires you to define the networking mode when adding a GCP project to RSC for the Exocompute use case. You can select Rubrik-managed or customer-managed networking mode. For more information about these modes, see Exocompute on GCP networking modes. If you select the customer-managed networking mode, you must ensure all networking prerequisites are met, as described in Exocompute on GCP customer-managed networking prerequisites.

Run Health Checks

The Run Health Checks screen appears during the initial configuration workflow. The health check runs:
  • Before the Google Kubernetes Engine (GKE) is created to verify that RSC has all the necessary permissions, network access, and prerequisites needed to successfully build the GKE cluster.
  • After the GKE cluster is successfully created to verify required connectivity for data management tasks, such as archival location connectivity and other workload-specific checks.

Running health checks helps you proactively verify cluster functionality and essential connectivity for all Exocompute features (such as snapshot indexing, file recovery, and storage tiering).

To ensure best outcomes and full functionality, it is recommended that you select and enable all available health checks.

Exocompute health check options Description
Default connectivity Runs automatically to verify essential resources, including connectivity to the Google Artifact Registry (GAR).

The Rubrik cluster uses the GAR to store necessary container images. These container images are downloaded and used by the GKE cluster when executing various Exocompute jobs.

Archival locations (optional) Verifies connectivity to specific archival locations.
GCP (optional) Verifies connectivity and permissions required for uploading files to GCP buckets.
Note:
When using customer-managed networking during manual onboarding of GCP projects, running health checks is critical. The health checks verify the connectivity and firewall rules you manually configured, ensuring the cluster can reach the GAR and RSC endpoints.
