RSC user management for GCP protection

Designating specific users with appropriate privileges and roles in RSC enhances security for managing GCP projects.

Designating specific users with the appropriate privileges and roles for managing GCP projects within RSC enhances security. To achieve this, you create user accounts in RSC for individuals you identify as part of your data management team and assign appropriate roles to each of those accounts.

Privileges are assigned to specific roles, and roles are then assigned to user accounts. There are two ways to create a role:
  • Creating a role by selecting from pre-configured role templates provided by RSC
  • Creating a custom role to select individual privileges based on your requirements

Pre-configured role template

The pre-configured role templates relevant for GCP protection are:
  • GCP Cloud Account Owner: An RSC user with this role can manage all aspects of GCP projects.
  • Backup Administrator: An RSC user with this role can manage every aspect of backup and recovery operations on RSC for all supported workloads.

Custom role

A custom role provides the flexibility to select specific privileges for GCP workloads according to your requirements. You can create custom roles for different operations, such as:
  • GCP project management
  • Recovery operations for all GCP workloads, which include GCE instances and persistent disks
Related concepts
  • Roles
    Roles define privileges for user accounts in a Rubrik cluster. With the administrator account, you can assign administrator roles that grant different access permissions to user accounts for specific purposes.
