Adding GCP projects for cloud-native storage

Add a GCP project to RSC to create a cloud-native archival location used for GCP storage tiering. RSC archives snapshots of objects from other GCP projects to this project.

Before you begin

Assign the roles and permissions to the GCP IAM user account that you have designated to add a GCP project to RSC, as described in Permissions to add GCP projects.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Cloud Accounts and select GCP.
    The GCP page appears.
  4. Click Add GCP Projects.
    The Add GCP Projects wizard appears.
  5. Select Cloud-native storage.
  6. Click Manage Granular Permissions to include permissions you want RSC to have.
    Only the Basic permission category is selected by default to adhere to the principle of least privilege. You can select the Encryption permission category if your scenarios demand it.
  7. Optional: Select Encryption and click Save.
  8. Click Next
    The Authorize access page appears.
  9. Click Sign in with Google.
  10. Sign in with the Google account to authorize RSC to protect GCE instances and persistent disks across the projects associated with the Google account.
    After a successful login, the Authorize access page reappears.
  11. Click Next.
    The Select Projects for Protection page appears, displaying the list of projects.
  12. Select the projects that you want to use as a storage destination to archive snapshots and click Next.
    The Permissions summary page appears.
  13. Review the list of projects and click Submit.

Results

You have successfully added the GCP project for cloud-native storage to RSC. You can use this project as a storage destination to archive snapshots from your cloud resources.
