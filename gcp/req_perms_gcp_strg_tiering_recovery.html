        • GCP project setup

          You can set up a GCP project by adding the project to RSC and start protecting your GCP workloads.

        • Exocompute on GCP

          With the Exocompute on GCP feature, RSC launches a Google Kubernetes Engine cluster to perform various operations to protect GCP workloads. RSC uses the Google Kubernetes Engine cluster to serve as a compute resource in your GCP project.

        • GCP storage tiering setup

          You can use GCP storage tiering to archive GCE instance or persistent disk snapshots that are older than a specified threshold to GCS storage classes. By archiving snapshots to these storage classes, you can optimize costs.

        • GCP workloads protection setup

          You can set up SLA Domains to apply data protection policies, ensuring automatic and hierarchical protection of GCP projects and individual GCP workloads, with granular control overriding higher-level assignments.

        • GCP workloads recovery

          After RSC takes snapshots of your GCE instances and persistent disks, you can recover the required instances and disks using these snapshots.

        • GCE instances and persistent disks

          After you set up protection for GCE instances and persistent disks, RSC takes automated snapshots of those instances and disks based on the policies configured in SLA Domains. You can also take on-demand snapshots and refresh GCP projects.

Required permissions for GCP storage tiering and recovery

Information about the permissions that RSC requires for the GCP storage tiering and recovery operations.

The following table describes the permissions that RSC requires for GCP storage tiering and recovery operations.
Permission Use case
storage.buckets.create To enable a user or service account to create Google Cloud Storage (GCS) buckets within a GCP project.
storage.buckets.get To enable a user or service account to read metadata about a GCS bucket.
storage.buckets.update To enable a user to modify the settings or configurations of a GCS bucket.
storage.objects.create To enable a user or service account to upload new objects or files to a GCS bucket.
storage.objects.get To enable a user or service account to read metadata about an object in a GCS bucket.
storage.objects.update To enable a user or service account to modify the settings or configuration of an object in a GCS bucket.
storage.objects.delete To enable a user or service account to delete objects within a GCS bucket.
storage.objects.list To enable a user or service account to list the objects within a GCS bucket.
storage.buckets.enableObjectRetention To enable a user or service account to set object retention policies on a GCS bucket.
storage.objects.setRetention To enable a user or service account to set retention parameters on individual objects in a GCS bucket.
cloudkms.keyRings.list To enable a user to view all the key rings within a GCP project in Cloud Key Management Service (Cloud KMS), giving them the ability to view a list of all the containers where cryptographic keys are stored.
cloudkms.cryptoKeys.list To enable a user to view and access a list of all the Cryptographic Keys (CryptoKeys) stored within their Cloud KMS.
storage.multipartUploads.create To create a multipart upload operation for a large object.
storage.multipartUploads.list To list all multipart upload operations.
storage.multipartUploads.listParts To list all the parts in a multipart upload operation.
storage.multipartUploads.abort To stop and clean up an in-progress multipart upload session.
