        • GCP project setup

          You can set up a GCP project by adding the project to RSC and start protecting your GCP workloads.

        • Exocompute on GCP

          With the Exocompute on GCP feature, RSC launches a Google Kubernetes Engine cluster to perform various operations to protect GCP workloads. RSC uses the Google Kubernetes Engine cluster to serve as a compute resource in your GCP project.

        • GCP storage tiering setup

          You can use GCP storage tiering to archive GCE instance or persistent disk snapshots that are older than a specified threshold to GCS storage classes. By archiving snapshots to these storage classes, you can optimize costs.

        • GCP workloads protection setup

          You can set up SLA Domains to apply data protection policies, ensuring automatic and hierarchical protection of GCP projects and individual GCP workloads, with granular control overriding higher-level assignments.

        • GCP workloads recovery

          After RSC takes snapshots of your GCE instances and persistent disks, you can recover the required instances and disks using these snapshots.

        • GCE instances and persistent disks

          After you set up protection for GCE instances and persistent disks, RSC takes automated snapshots of those instances and disks based on the policies configured in SLA Domains. You can also take on-demand snapshots and refresh GCP projects.

Required permissions for native GCP protection

RSC requires certain permissions to protect GCP resources.

The following table lists the permissions required by RSC for native GCP protection.
Permission Access level Use case
cloudkms.cryptoKeys.list Read To get the list of crypto keys in the key ring for selection when exporting a GCE instance or a persistent disk snapshot.
cloudkms.keyRings.list Read To get the list of key rings in the GCP project for the KMS key selection when exporting a GCE instance or a persistent disk snapshot.
compute.disks.create Write To create disks attached to the exported GCE instance when performing the export task or creating disks when exporting persistent volumes.
compute.disks.createSnapshot Write To create a disk snapshot during the GCE instance or persistent disk snapshot creation task.
compute.disks.delete Write To delete a disk created by RSC when rolling back a task that involves disk creation, such as restoring a GCE instance in the event of a failure.
compute.disks.get Read To read the metadata of an attached disk while creating a GCE instance snapshot or a persistent disk.
compute.disks.list Read To discover inventory for persistent disks and disks attached to GCE instances.
compute.disks.setLabels Write To assign labels to a newly created disk during the recovery task or a detached disk during the restore task.
compute.disks.use Write To attach a disk to a GCE instance.
compute.disks.useReadOnly Write

To attach a read-only disk to a GCE instance.
compute.firewalls.list Read To read network names for suggesting network configurations while exporting a GCE instance.
compute.globalOperations.get Read To check the progress of global operations, such as deleting a snapshot.
compute.instances.attachDisk Write To attach new disks to the GCE instance to be restored when performing the restore task.
compute.instances.create Write To create a GCE instance for the export task of a GCE instance snapshot.
compute.instances.delete Write To delete a newly created GCE instance if the export instance job fails and needs to be rolled back.
compute.instances.detachDisk Write To detach existing disks from a GCE instance during the restore task.
compute.instances.get Read To get a GCE instance during the GCE instance snapshot creation task.
compute.instances.list Read To get GCE instances for the GCP project refresh task.
compute.instances.setLabels Write To assign labels to a GCE instance when creating an instance during the export instance task or restoring labels of the GCE instance during the restore instance task.
compute.instances.setTags Write To assign tags to the GCE instances during the export task.
compute.instances.setServiceAccount Write To add service accounts during the export task.
compute.instances.start Configuration To restore GCE instances from a snapshot.
compute.instances.stop Configuration To stop the GCE instance before replacing existing attached volumes with new volumes during the restore task.
compute.machineTypes.list Read To display the list of types of machines compatible with the GCE instance snapshot during the export task.
compute.networks.list Read To read network names for suggesting network configurations while exporting GCE instance snapshots.
compute.projects.get Read To get the list of shared VPC host projects for the specified GCP project.
compute.regionalOperations.get Read To find out if a regional operation has been completed.
compute.regions.list Read To update regions during the GCP project setup.
compute.snapshots.create Write To create snapshots during the GCE instance snapshot creation task.
compute.snapshots.delete Write To delete snapshots when reverting the GCE instance snapshot creation task and performing the garbage collection task.
compute.snapshots.get Read To get a snapshot for use during the export or restore recovery task.
compute.snapshots.list Read To get the list of snapshots for the snapshot integrity task.
compute.snapshots.setLabels Write To set labels on the snapshots created during the GCE instance snapshot creation task.
compute.snapshots.useReadOnly Read To create a disk from the source snapshot when exporting a GCE instance
compute.subnetworks.use Read To set the subnetwork resource for the exported GCE instance during the export task.
compute.subnetworks.useExternalIp Read To set the subnetwork resource for the exported GCE instance during the export task.
compute.zoneOperations.get Read To check the progress of a zonal operation, such as creating a disk snapshot.
iam.serviceAccounts.actAs Write To attach disks when restoring a GCE instance.
resourcemanager.projects.get Read To add GCP projects for testing purposes.

Permission Access level Use case
resourcemanager.projects.getIamPolicy Read To verify policy bindings after addition.
iam.roles.get Read To check if custom roles have the correct permissions.
