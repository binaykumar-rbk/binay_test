      • Google Cloud Platform workloads

        RSC protects the following types of Google Cloud Platform workloads: Google Compute Engine instances and persistent disks.

        • Key features for protecting GCP workloads

          RSC provides unified data management, automated global data protection, and various recovery options to protect your GCP workloads.

        • Common GCP protection terms

          It is important to understand some common terms when protecting GCP workloads with RSC.

        • High-level architecture for GCP protection

          A high-level architectural overview of how RSC protects GCP workloads and RSC and GCP IAM roles provisioning process.

        • Planning for GCP protection

          Before setting up a GCP project in RSC, you need to plan user and access management for designated RSC users. This planning also involves reviewing the permissions that RSC requires in your GCP environment to protect your GCP workloads.

        • GCP project setup

          You can set up a GCP project by adding the project to RSC and start protecting your GCP workloads.

        • Exocompute on GCP

          With the Exocompute on GCP feature, RSC launches a Google Kubernetes Engine cluster to perform various operations to protect GCP workloads. RSC uses the Google Kubernetes Engine cluster to serve as a compute resource in your GCP project.

        • GCP storage tiering setup

          You can use GCP storage tiering to archive GCE instance or persistent disk snapshots that are older than a specified threshold to GCS storage classes. By archiving snapshots to these storage classes, you can optimize costs.

        • GCP workloads protection setup

          You can set up SLA Domains to apply data protection policies, ensuring automatic and hierarchical protection of GCP projects and individual GCP workloads, with granular control overriding higher-level assignments.

        • GCP workloads recovery

          After RSC takes snapshots of your GCE instances and persistent disks, you can recover the required instances and disks using these snapshots.

          • Privilege elevation for GCP recovery

            RSC provides you an option to secure GCP projects by using a least permission model, granting only necessary read permissions. For recovery operations, you can temporarily elevate permissions by authorizing a service account role.

          • File Indexing for GCE instances and disks

            RSC uses Exocompute on GCP to extract file system index metadata from the persistent disk snapshots created when protecting GCE instances and persistent disks.

          • Evaluation of options for GCP recovery

            Understanding the recovery options for GCP is essential to ensure alignment with business needs for data consistency and availability. Thorough evaluation of these options, including disk encryption strategies, enables informed decision-making and effective data protection during recovery.

          • Recovery of GCE instances and persistent disks

            You can recover your GCE instances and persistent disks using options such as recovering files or folders, exporting, and restoring.

            • Restoring a GCE instance

              Use a GCE instance snapshot to replace the original GCE instance with a fully functional point-in-time copy.

            • Exporting a GCE instance snapshot

              Export a snapshot of a GCE instance to create a point-in-time copy of that instance.

              • Sharing encryption key with a different GCP project

                To export a GCE instance snapshot to a different target GCP project, share the disk encryption key from the original project with the target project.

            • Exporting a persistent disk snapshot

              Export a snapshot of a persistent disk to create a point-in-time copy of that disk.

            • Recovering files or folders from a GCP snapshot

              Recover files or folders from GCE instances or persistent disks to a GCS bucket if the files that you want to access are deleted or corrupted.

        • GCE instances and persistent disks

          After you set up protection for GCE instances and persistent disks, RSC takes automated snapshots of those instances and disks based on the policies configured in SLA Domains. You can also take on-demand snapshots and refresh GCP projects.

Sharing encryption key with a different GCP project

To export a GCE instance snapshot to a different target GCP project, share the disk encryption key from the original project with the target project.

About this task

You can share the encryption key by running the gcloud command in the original project of the snapshot before you export the snapshot to a different target project.

Procedure

  1. Open the Google Cloud Platform website from your web browser.
    The Google Cloud webpage appears.
  2. Click Go to my console.
    The Google Cloud Console appears.
  3. Click Create or select a project.
    The Select a project dialog box appears.
  4. Select the original project of the snapshot.
    The Welcome page of the project appears.
  5. Open the Cloud Shell Terminal and run the following command: 
    gcloud kms keys add-iam-policy-binding <source-project-key-name> \
--location <source-project-key-location> \
--keyring <source-project-key-keyring> \
--member <target-project-default-compute-service-account> \
--role roles/cloudkms.cryptoKeyEncrypterDecrypter
    Where:
    • <source-project-key-location> is the location where the key is stored.
    • <source-project-key-keyring> is the name of the key ring that contains the key. Key rings are used to organize keys within a project and location.
    • <target-project-default-compute-service-account> is the default Compute Engine service account of the target project. It must be in the following format: service-<target-project-number>@compute-system.iam.gserviceaccount.com

Results

You have successfully shared the disk encryption key with the target GCP project.

What to do next

Export the GCE instance snapshot, as described in Exporting a GCE instance snapshot.
