PostgreSQL protection prerequisites

Information about the specific prerequisites that RSC requires for protecting PostgreSQL clusters.

The following table describes the requirements for protecting PostgreSQL clusters:
Component Prerequisites
Rubrik CDM
  • Rubrik CDM version 9.2.1 or later for Community PostgreSQL.
  • Rubrik CDM version 9.2.3 or later for EDB PostgreSQL Advanced Server (EPAS).
  • Rubrik CDM version 9.3.1 or later for EDB PostgreSQL Extended Server (PGE).
Ports

You must ensure that the following ports are open for communication and data transfer between Rubrik CDM and PostgreSQL clusters.

  • Ports 12800 and 12801 TCP: Required for communication between Rubrik CDM and Rubrik Backup Service.

    All the Rubrik CDM nodes must be able to reach these ports on the PostgreSQL hosts.

  • Ports 111, 2049, and 32764 to 32769 TCP/UDP: Required for data backup through NFS connection.

    PostgreSQL hosts must be able to reach these ports on all Rubrik CDM nodes.

  • Port 9639 TCP: Required for Write-Ahead Log (WAL) archiving or log backup.

    PostgreSQL hosts must be able to reach this port on all Rubrik CDM nodes.

  • Port 5432 TCP: Required for establishing a connection with the PostgreSQL server.

    All the Rubrik CDM nodes must be able to reach this port on the PostgreSQL hosts.

    Note:
    Port 5432 is the default port. If you configure a custom port, you must use that specific port to establish a connection with the PostgreSQL server.
Operating system on host Linux is the only supported operating system.

The Rubrik Compatibility Matrix guide provides information about the supported Linux OS versions.

For PostgreSQL clusters running on RHEL, the PostgreSQL user requires access permission when SELinux is enabled. Using the sudo semanage permissive -a postgresql_t command provides the access to the PostgreSQL user.
PostgreSQL version
  • Community PostgreSQL
  • EDB PostgreSQL Advanced Server (EPAS)
  • EDB PostgreSQL Extended Server (PGE)

The Rubrik Compatibility Matrix guide provides information about the supported PostgreSQL versions.
PostgreSQL archive mode You must turn on the archive mode to enable continuous archiving.

Setting the wal_level configuration parameter to replica or logical performs WAL archiving to capture log data. This enables you to restore your data to a specific point in time.
PostgreSQL user permission The PostgreSQL user requires Superuser privileges to perform data backup and WAL backup.

Rubrik connects to the PostgreSQL cluster using peer authentication.
NFS client Rubrik recommends installing the latest version of the NFS client on the host because the NFS protocol is used for data backup.
