Managing protection for a PostgreSQL cluster

To protect your PostgreSQL clusters, manage protection by assigning data protection policies.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click the PostgreSQL inventory card.
    The PostgreSQL page appears.
  5. Select a PostgreSQL cluster and click Manage Protection.
    You can also select multiple PostgreSQL clusters.
    The Manage Protection wizard appears.
  6. Select a protection option.
    SLA Domain Assigns the SLA Domain to the selected objects.

    A Confirmation wizard appears, displaying a summary of the SLA Domain, including the frequency and retention details of existing snapshots.
    Don't protect Excludes the selected objects from all further SLA Domain assignments and protection.

    The Don't Protect wizard appears, displaying a message that the selected objects have been excluded from any future assignments to the SLA Domain.
    Clear existing assignment Derives the SLA Domain from the next higher level object, based on inheritance rules.

    The Inherit SLA Domain wizard appears, displaying a message that the selected objects inherit the SLA Domain assignment of the next higher level object.
  7. (Don't protect) In Existing snapshot retention, select an option for the existing unprotected snapshots.
    Use current SLA Domain for retention Derives the retention period from the current SLA Domain.

    This is the default option.
    Keep forever Retains the snapshots until you manually delete them.
    Expire immediately Deletes the snapshots from the cluster.
  8. Click Next.
  9. Review the information.
  10. Optional: In Add note, enter a description for assigning or modifying data protection policies.
  11. Click Assign.

Results

RSC processes the specified assignments and begins protecting the selected PostgreSQL clusters according to the assignments.

You can view the latest user note in the PostgreSQL cluster overview page.
