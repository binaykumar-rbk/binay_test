  • Workloads

    Rubrik Security Cloud - Data Protection securely manages and protects data from a wide variety of workload types.

    • Rubrik Security Cloud workloads

      RSC securely manages and protects data from a wide variety of workload types.

    • Cloud Workloads
    • Data Center Workloads
      • AHV virtual machines

        RSC provides data management and protection for virtual machines deployed in a Nutanix Acropolis environment.

      • Hyper-V virtual machines

        A Rubrik cluster provides data management and protection for virtual machines that are deployed in a Microsoft Hyper-V environment. The Rubrik cluster can manage and protect virtual machines in an environment with multiple Hyper-V servers and virtual machines.

      • vSphere virtual machines

        Rubrik Security Cloud provides data management and SLA Domain protection for vSphere virtual machines, including vSphere VM templates that are managed either through a VMware vSphere environment or a content library.

        • vSphere VM templates

          Rubrik Security Cloud support selected backup and recovery operations for vSphere VM templates and content library items. These operations include file recovery, export, live mount, mounting virtual disks, and recovering virtual machine images as a template.

        • Application consistency

          You can set application consistency for vSphere virtual machines individually or in batch at different levels in the vSphere hierarchy.

        • vSphere virtual machine protection

          RSC provides protection for virtual machines through individual assignment of the virtual machine to an SLA Domain or through automatic protection.

          • Protecting a vSphere virtual machine

            Protect a virtual machine by assigning an SLA Domain.

          • Protecting vSphere virtual machines by object

            RSC classifies vSphere virtual machines according to virtual machines, folders, clusters, hosts, content libraries, and tags. You can assign an SLA Domain to a virtual machine, folder, cluster, host, content library, or tag category to protect all virtual machines in that group.

          • Automatic protection

            To provide consistency when applying automatic protection, the Rubrik cluster adheres to a specific set of rules.

          • SLA Domain assignment options for virtual machines

            Protect a set of virtual machines by assigning the selected set to an SLA Domain. Assigning virtual machines to an SLA Domain protects the virtual machines by applying the data protection policies of the SLA Domain.

          • Assigning SLA Domains to vSphere tags

            Assign an SLA Domain to the virtual machines with the assigned vSphere tags.

          • vSphere virtual machine protection states

            Depending on the state of a virtual machine, there are several states of virtual machine protection.

          • Find protected vSphere objects

            RSC provides several methods for finding protected objects in the vSphere Inventory list page.

          • Creating advanced tags

            Create advanced tags by combining multiple tags with conditional logic.

          • Editing advanced tags

            Edit the advanced tag to modify the backup setup assigned to the appropriate SLA Domain.

          • Deleting advanced tags

            Delete an advanced tag from the Rubrik cluster when it is no longer required.

        • vSphere management

          The Rubrik cluster accesses virtual machine data through a connection with the VMware vCenter Server or ESXi host that manages the hypervisor running the virtual machine.

        • vSphere diagnostics

          Rubrik Security Cloud vSphere diagnostics confirm connectivity and access permissions for a vCenter server connected to a Rubrik cluster. You can use vSphere diagnostics to verify and troubleshoot connectivity, backup, and recovery of a vSphere object or component.

        • VMware Cloud on AWS

          A Rubrik Cloud Cluster can protect virtual machines deployed on VMware Cloud on AWS.

        • Microsoft Azure VMware Solution

          A Rubrik cluster can protect virtual machines deployed on the Microsoft Azure VMware Solution.

        • Google Cloud VMware Engine

          A Rubrik cluster can protect virtual machines deployed on the Google Cloud VMware Engine.

        • VMDK files

          Virtual machines can include VMDK files that do not need to be protected.

        • vSphere virtual machine scripts

          RSC allows you to configure the Rubrik cluster to run scripts on a guest OS before a snapshot, after a snapshot, and after the Rubrik cluster completes the backup process.

        • Rubrik Backup Service

          The Rubrik Backup Service is software installed on a host that coordinates activity between the Rubrik cluster and the host.

        • On-demand snapshots of vSphere virtual machines

          In addition to policy-based snapshots, create virtual machine snapshots by using the on-demand snapshot process.

        • vSphere virtual machine details page

          The vSphere virtual machine details page provides detailed information about the virtual machine, protection of the virtual machine, available snapshots, and tasks related to the virtual machine and snapshots.

        • Recovery of vSphere virtual machines

          To recover a vSphere virtual machine, a Rubrik cluster mounts a point-in-time copy of that virtual machine.

        • Storage array integration

          Rubrik Security Cloud provides a user interface for integrating Rubrik clusters with storage arrays, reducing the time a virtual machine is quiescent during snapshot operations.

        • Continuous Data Protection

          Continuous Data Protection uses VMware vSphere APIs for I/O Filtering to continuously capture changes to data on a vSphere virtual machine and sends them to a Rubrik cluster.

        • Standalone ESXi host protection

          Rubrik supports protecting and restoring virtual machines on standalone ESXi hosts, providing continuous data security in environments that do not include a vCenter server.

      • VMware Cloud

        A Rubrik Cloud Cluster effectively protects virtual machines deployed on VMware Cloud on AWS, Microsoft Azure VMware Solution, and Google Cloud VMware Engine.

      • VMware Cloud Director vApps

        The Rubrik cluster provides SLA Domain protection and data management for VMware Cloud Director vApps.

Find protected vSphere objects

RSC provides several methods for finding protected objects in the vSphere Inventory list page.

Search method Description
Search field
  • Use the search field to find a specific virtual machine at the object level.
  • Enter a portion of the virtual machine name to search.
  • RSC begins a predictive search and updates the results as letters are entered.
Rubrik Cluster filter Use the Rubrik Cluster filter to find specific virtual machines on the selected Rubrik cluster.
SLA Domain filter Use the SLA Domain filter to find specific virtual machines that belong to the selected SLA Domain.
VM Type Use the VM Type filter to find a specific virtual machine type. RSC provides the following categories:
  • Virtual Machine: Selects vSphere objects saved as vSphere virtual machines
  • Template: Selects vSphere virtual machine templates managed through a VMware vSphere environment
  • Content Library item: Selects vSphere virtual machine templates that are managed through a content library
Status filter Use the Status filter to find virtual machines that have the selected protection state.
Object tabs

Use the different object tabs on the vSphere VMs page to define a hierarchical view to search and to browse. Then, use the search field to find entities within the defined view, or to browse entities within the defined view. RSC provides the following tabs:

  • Virtual Machines: Provides a virtual machine only view, with the hierarchical location of each virtual machine displayed in the location column
  • Folders: Provides the vCenter Server folder hierarchy view, starting at the vCenter Server
  • Clusters & Hosts: Provides the vCenter Server cluster and host hierarchy view, starting at the vCenter Server
  • Content libraries: Provides a hierarchical view of all content libraries on the VMware vSphere server
  • Tags: Enables you to create tag rules to protect vSphere virtual machines
The same vCenter Server is displayed more than once when RSC lists the Servers by vCenter Server:Rubrik cluster pairs.
