Updating advanced Exocompute settings for Rubrik clusters

Update additional advanced Exocompute settings for Rubrik-managed clusters for better performance, scalability, and security.

Before you begin

Before configuring advanced settings associated with User Defined Routing, it is essential to configure the routing table associated with your Exocompute VNet. Refer to the topic for customizing cluster egress with a user-defined routing table in the Microsoft Azure documentation for information about configuring a routing table.

About this task

When editing Exocompute details for Rubrik-managed clusters, you can optionally configure some advanced Exocompute settings, such as enabling encryption at the host, enabling a paid tier for an AKS cluster, increasing the max nodes, and so on. These advanced settings optimize performance, scalability, and security.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Cloud Accounts and select Azure.
    The Azure page appears.
  4. From the Azure dropdown list, select Rubrik Compute.
    A list of Microsoft Entra domains appears.
  5. Select a Microsoft Entra domain.
    The list of Exocompute host subscriptions appears.
  6. Click for an Exocompute host subscription and select Edit Exocompute.
    The Edit Exocompute wizard appears. The Private Container Registry option is turned off.
  7. Click Next.
  8. From the Regions dropdown list, select a region for which to launch the Exocompute instance.
  9. For Rubrik-managed clusters, update the configurations for VNet, Subnet, and Pod CIDR on the next page of the wizard.
  10. Click Edit Advanced settings to configure advanced Exocompute settings for the AKS cluster.
  11. In AKS Tier, select Free or Paid.
    The Free option is selected by default.
  12. Turn on User Defined settings to specify a custom route to manage the incoming traffic for your virtual network and AKS cluster.
  13. Turn on Encrypt Exocompute Disk at Host for Azure to encrypt the data on node disks in the data center.
  14. Enter the maximum number of nodes to deploy as part of an AKS cluster in Max Exocompute Node Count to scale the Exocompute cluster.
  15. Enter AKS Resource Group Prefix to specify the prefix associated with the nodeResourceGroup that an AKS cluster uses to manage AKS-related resources, such as a load balancer.
  16. Turn on Allowlist Exocompute API server IP and enter additional IP addresses in Allowlist Additional IP Addresses to restrict access to the API server associated with the Azure Kubernetes Service to only the specified IP addresses.
  17. In Private DNS Zone ID for private Exocompute, enter the Azure resource ID for the DNS zone if you want to use a custom private DNS zone for the private AKS cluster.
    Before configuring a custom private DNS zone, you must ensure that specific prerequisites are met. The Private AKS clusters topic provides more information about the prerequisites.
  18. Click Done to close the Edit advanced settings page.
  19. Click Finish.

Results

RSC updates the advanced Exocompute settings for the Rubrik-managed cluster.
Related concepts
  • Private AKS clusters
    Private AKS clusters enable secure in-VNet communication between the control plane and node pools, restricting API server access to the VNet. They manage a private DNS zone by default but can be configured with a custom DNS zone for specific network needs.
