Azure User Defined Routing configuration

To manage outbound traffic for an Azure Kubernetes Service cluster without using public IP addresses, you can configure User Defined Routes in your Azure VNet used for Exocompute.

If providing an Azure policy exemption for public IP addresses is not feasible, you can alternatively configure User Defined Routing (UDR) for outgoing traffic from your Azure Kubernetes Service (AKS) clusters. This approach enables you to manage outgoing traffic effectively while complying with Azure policies that restrict the creation of public IP addresses.

When Rubrik Exocompute on Azure deploys an AKS cluster, Azure provisions a standard load balancer with a public IP address to manage outgoing traffic to the internet. This IP address is essential for the AKS nodes to access external resources. Alternatively, you can use UDRs to define custom routes for managing outbound traffic in your Azure Virtual Network (VNet). These custom routes ensure compliance with the policies and requirements of your organization.

Note:
Depending on your network setup, such as having an Azure firewall, a third-party firewall, or no firewall configured, you may need to configure UDRs differently.
