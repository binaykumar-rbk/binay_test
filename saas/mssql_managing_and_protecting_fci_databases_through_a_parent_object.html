Managing and protecting FCI databases through a parent object

Protect databases in a failover cluster instance by assigning an SLA Domain to the parent failover cluster, or to the parent Microsoft SQL Server database.

Before you begin

Add each Windows Server host that is a node in the failover cluster to the Rubrik cluster.

About this task

A derived SLA Domain assignment provides a way to uniformly manage and protect a group of FCI databases. A derived SLA Domain assignment applies to the databases that exist at the time of the assignment and to databases that are added after the assignment.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click the SQL Server Databases inventory card.
    The SQL Server Databases page appears.
  5. Click Failover Clusters.
    The Failover Clusters tab appears. The page lists by name each failover cluster. For each failover cluster, the page provides the number of Microsoft SQL Server databases and the SLA Domain assigned.
  6. Select a failover cluster or a Microsoft SQL Server database.
    Click the name of a failover cluster to view the Microsoft SQL Server databases on that failover cluster.

    Select multiple failover clusters or Microsoft SQL Server databases to apply the same SLA Domain protection to databases on all of the selections.

  7. Click Manage Protection.
    The Manage Protection wizard appears.
  8. In the SLA Domain section, select an SLA Domain.
  9. Optional: Turn on the Take copy only backups toggle.
    The Rubrik cluster performs copy only backups for the policy-driven backups of the databases in the selection group.
  10. Click Next.
    The Confirmation page appears.
  11. Optional: Select Include on-demand and downloaded snapshots.
    The retention for on-demand snapshots and snapshots downloaded from an archival location is updated.
  12. Click Assign.

Results

The Rubrik cluster assigns the SLA Domain and other settings to all existing databases and their children within the selection group.
