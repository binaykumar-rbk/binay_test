Transitioning to using Azure managed identities

For enhanced security, transition your cluster from using connection strings to using managed identities as the authentication type to your storage container.

About this task

Rubrik CDM versions 9.0.2 and later support transitioning to managed identities.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
  3. On the navigation bar, click Infrastructure > Clusters.
  4. Click a cluster.
  5. Click and select Transition to managed identities.
  6. From the Account dropdown list, select the Azure subscription.
  7. From the Managed identities dropdown list , select a managed identity.
  8. Click Update.

Results

A message is displayed when the transition starts. If the transition succeeds, on the details page for the cluster, the Authentication Type field displays Managed Identities. After this, the cluster will use managed identities as the mechanism by which to authenticate with the Azure storage account.

A message is displayed if the transition fails. If the transition fails, the Rubrik cluster continues to function as expected, and there is no impact when running the cluster. Backups and SLA Domains function as expected.

You can also check the audit logs to learn whether the transition succeeded or failed.

What to do next

Wait 24 hours after a successful transition to rotate any storage account keys.
