Common Fileset Use Cases

Use cases or examples to help you understand how Rubrik interprets the syntax provided in the include and exclude fields within filesets. These examples are created with a Windows share, which uses the (\) as the directory separator. When working with Linux, (/) serves as the directory separator.

Recursive backup of everything under the root path

Include: \, \**, \*.

Backup data:
  • passwords.txt
  • Finance
  • Userdata
  • Userdata\template.txt
  • Userdata\Alice
  • Userdata\Alice\file1.txt
  • Userdata\Bob
  • Userdata\Bob\file2.txt
  • Userdata\Bob\folder
  • Userdata\Bob\folder\file3.txt

Recursive backup of everything under \Userdata\Bob

Include: \Userdata\Bob, \Userdata\Bob\, \Userdata\Bob\**, \Userdata\Bob\*.

Backup data:
  • Userdata\Bob\file2.txt
  • Userdata\Bob\folder
  • Userdata\Bob\folder\file3.txt

Recursive backup of everything that starts with capital B under \Userdata

Include: \Userdata\B*, \Userdata\B*\, \Userdata\B*\**, \Userdata\B*\*.

Backup data:
  • Userdata\Bob
  • Userdata\Bob\file2.txt
  • Userdata\Bob\folder
  • Userdata\Bob\folder\file3.txt

Non-recursive backup of files that contain the . character, under all directories that start with a capital B, within \Userdata

Include: \Userdata\B*\*.*

Backup data: Userdata\Bob\file2.txt

Understanding the impact of inclusion and exclusion rules on backup size

The inclusion rule home/ubuntu includes all paths in the share, such as the following:
  • /home/ubuntu/
  • /home/user/asdf/huge_folder/home/ubuntu
  • /ifs/data/.snapshot/<snapshot date>/home/ubuntu
Note:
This rule makes the backup multiple times larger than the intended /home/ubuntu backup because it includes all folders named .snapshot. You can use an exclusion rule to omit these folders.

Symbolic Links and Junction Points

The Rubrik cluster does not follow symbolic links or junction points during a fileset backup. When a symbolic link is included in a fileset, the Rubrik cluster backs up the link as a file. The Rubrik cluster does not follow the link and does not back up the file or folder to which the symbolic link points. To back up a symbolic link, you must directly include the file or folder it points to in a fileset.
