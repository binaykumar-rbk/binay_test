        • Point-in-time recovery

          For a database that uses the full recovery model or the bulk-logged recovery model, the Rubrik cluster uses a combination of a snapshot of the database and the database transaction log backups to recover a database.

          A Live Mount creates a new database from a point-in-time copy of the source database. The Rubrik cluster provides a Samba share of the new database directly from the Rubrik cluster storage layer.

Restoring a Microsoft SQL Server database instance

Restore a Microsoft SQL Server instance, including its associated databases, to a recovery point.

About this task

You cannot use this task to recover a system database.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click the SQL Server Databases inventory card.
    The SQL Server Databases page appears.
  5. In the Hosts/Instances tab, select a host and an underlying Microsoft SQL Server instance.
    Alternatively, you can perform this selection from the Failover Clusters tab.
  6. Open the ellipsis menu for the selected Microsoft SQL Server instance and select Recover Databases.
    The SQL Server Database Recovery wizard appears.
  7. Select a restore strategy to use and click Next.
    Restore strategyDescription
    Latest point in time Selects the latest snapshot for the database recovery.
    Closest point in time Selects the snapshot for database recovery that is closest to the time of the outage. The closest point in time is the latest time up to and including the requested time.
  8. Select In-place Recovery and click Next.
  9. Optional: Determine whether you want to keep the restored databases in a restoring state and click Next.
  10. Verify the details on the confirmation screen and click Recover.

Results

You have restored a Microsoft SQL Server instance, including its associated databases.
