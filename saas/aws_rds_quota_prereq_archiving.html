Quotas prerequisites for Amazon RDS archiving

Information about the Amazon RDS instance quotas prerequisites for archiving Amazon RDS PostgreSQL, Aurora PostgreSQL, and RDS SQL Server databases.

To successfully archive Amazon RDS PostgreSQL, Aurora PostgreSQL, and RDS SQL Server databases, you must provision adequate Amazon RDS quotas for specific Amazon RDS instance types in your AWS account. As part of the archival process, RSC creates temporary PostgreSQL, Aurora PostgreSQL, or RDS SQL Server database instances from the manual snapshot. For RSC to create these temporary instances successfully, your AWS account must have sufficient quotas for the specific Amazon RDS instance types. If the quotas are too low, the temporary instance creation and the archival process fail.

Note:
RSC supports archiving only for Standard RDS SQL Server databases. Amazon RDS Custom for SQL Server is not supported.

db.m6in.2xlarge Equal to the number of RDS instances that you want to archive in parallel. Required only when PostgreSQL database archival protection is configured. This is applicable only for archival.
