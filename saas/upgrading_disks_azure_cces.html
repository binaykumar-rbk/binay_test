Upgrading Azure CCES clusters to use PremiumV2_LRS disks

Upgrade existing CCES clusters for Azure from PremiumV1_LRS disks to PremiumV2_LRS disks using the upgrading disks workflow.

About this task

This procedure describes upgrading from PremiumV1_LRS to PremiumV2_LRS disks for Azure CCES. RSC enables seamless disk upgrades that improve performance. The upgrade process runs in batches, adding new nodes and removing old ones, with each batch taking approximately three hours. The upgraded disks have a throughput of 300 MBPS and IOPS of 3000. Successful upgrades complete without downtime or performance degradation.

Refer to Limitations with Upgrading Disks for Azure CCES for more information.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Dashboard page appears.
  3. On the navigation bar, select Infrastructure > Clusters.
    The Clusters page appears.
  4. Select the Azure cluster.
  5. Click and select Upgrade Disks.
    The Upgrade Disks Confirmation page appears.
  6. Select the Azure account that contains the disks to upgrade.
  7. In Batch Size, enter a batch size.

    This value defines the number of nodes that you can upgrade in each migration iteration. By default, RSCpopulates this field with the total number of nodes in the Rubrik cluster. Rubrik recommends this default value as it is generally quicker and more cost-effective.

    Alternatively, you can select a custom batch size, with a minimum of one node, to control how many nodes are added and removed for each iteration.

  8. Optional: Select Update Instance Type to ensure migration of the nodes to the new instance type configuration.
  9. Optional: In New Instance Type, select an instance.
    This option appears only when selecting a new instance type.
  10. Optional: In New Node Count, enter the total number of nodes.

    By default, this value is the current node count in the cluster. You can update this value to the desired new total node count after the migration completes.

    This option appears only when selecting a new instance type.

  11. Click Upgrade.
  12. Click the notification that appears when the upgrade is complete.

    Azure uses a replace node workflow. It first adds new nodes to PremiumV2_LRS disks and then removes old nodes from the cluster.

    Use the MigrateDisksonCluster search filter to trace upgrade job logs. This event type supports text-based searches in events to check the upgrade status.

Results

The Event Logs page appears, displaying the upgrade details.
