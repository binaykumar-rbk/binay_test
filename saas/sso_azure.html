    • Azure SSO

      Configuring single sign-on with Azure involves several individual tasks, which must be performed in the correct order.

      • Retrieving service provider SAML configuration

        Retrieve the entity ID and callback URL values from RSC for preparing the federation metadata file in Azure.

      • Creating a non-gallery enterprise application

        Create a non-gallery enterprise application for RSC in the Azure portal.

      • Configuring Azure SSO

        Configure SSO in RSC by uploading the Azure metadata file and downloading the Rubrik metadata file.

      • Uploading the Rubrik metadata file

        Upload the Rubrik metadata file to the enterprise application in Azure.

      • User attributes and claims in Azure

        For SSO authentication, RSC requires that Azure issue specific claims and attributes.

      • Configuring user attributes and claims

        In the Azure portal, configure the claims for RSC by assigning values to the email address and group attributes.

      • Creating custom app roles

        In the Azure portal, create custom app roles to determine the permissions for user accounts that log in to RSC using Azure SSO.

      • Assigning app roles to users and groups

        In the Azure portal, assign the custom app role to the user accounts who need single sign-on access to RSC.

      • Authorizing RSC SSO groups for Azure SSO

        After creating Azure custom app roles and assigning the roles to user accounts in Azure SSO groups, create RSC SSO group roles or use existing roles and authorize RSC SSO groups by mapping Azure custom app roles to RSC SSO group roles.

      • Testing the SSO configuration

        Test the SSO configuration to verify that authentication requests are handled by the identity provider.

      • Unauthorized access condition

        Investigating the Azure SSO configuration can help find the cause of an unauthorized access error.

Azure SSO

Configuring single sign-on with Azure involves several individual tasks, which must be performed in the correct order.

Setting up single sign-on (SSO) on Azure involves several discrete tasks, which you perform through the Azure portal and RSC. The order in which you complete these tasks is important because the later tasks build on the results of earlier tasks.

The following diagram shows the sequence of tasks involved in setting up Azure SSO with RSC.



  1. Retrieve SAML info: Azure requires you to retrieve the entity ID and callback URL values from RSC.
  2. Create app: In Azure, configuring SSO requires you to create a non-gallery enterprise application through the Azure portal using the SAML 2.0 protocol.
  3. Configure Azure SSO: RSC requires you to upload the Azure metadata file. With the non-gallery enterprise application, you can download the Azure SAML metadata file, which is required by RSC to configure SSO.
  4. Upload Rubrik info: Azure requires you to upload the Rubrik metadata file to the enterprise application. You can download the Rubrik metadata file from RSC.
  5. Configure attributes: RSC requires you to configure the attributes, claims, and name ID in the Azure portal for SSO authentication. You must ensure to use your email address to configure the name ID in Azure portal.
  6. Create roles: Azure requires you to create custom app roles to determine the permissions for user accounts that log in to RSC using Azure SSO.
  7. Assign roles: Azure requires you to assign the custom app roles to user accounts who need SSO access to RSC.
  8. Authorize SSO: After creating Azure custom app roles and assigning the roles to user accounts in Azure SSO groups, RSC requires you to create RSC SSO group roles or use existing roles and authorize RSC SSO groups by mapping Azure custom app roles to RSC SSO group roles.
  9. Test connection: To verify whether the SSO connection has been established and that the appropriate SAML attributes have been correctly received from Azure, RSC requires you to test the SSO connection.
