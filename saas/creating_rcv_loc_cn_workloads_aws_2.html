Creating RCV locations for cloud native workloads on AWS

Create a Rubrik Cloud Vault location to archive the backups of cloud-native workloads.

Before you begin

Perform the following tasks:
  • Review the Rubrik Cloud Vault (RCV) specifications for cloud-native workloads, as described in Rubrik Cloud Vault specifications for cloud-native workloads.
  • Review the best practices, as described in Rubrik Cloud Vault best practices.

About this task

You can create an RCV location in AWS on clusters running Rubrik CDM version 9.3.1 or later.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Infrastructure > Rubrik Cloud Vault.
    The Rubrik Cloud Vault dashboard appears.
  4. Click RCV Locations.
    A list of RCV locations appears.
  5. Click Create RCV Location.
    The Create an RCV Location wizard appears, displaying the supported workload types.
  6. Select Cloud native and click Next.
  7. In Location Name, enter a display name for the archival location.
  8. In Cloud Provider, select AWS.
  9. From Tier, select a storage tier.
    TierDescription
    Backup The Backup tier in RCV offers a minimum retention period of 30 days, providing fast access to data for immediate recovery needs.
    Archive The Archive tier in RCV offers a minimum retention period of 180 days to access data, making it suitable for long-term data retention and compliance purposes.
    The Rubrik Cloud Vault storage tiers topic provides more information and a comparison between the storage tiers to help you make an informed decision based on your data storage requirements.
  10. From Redundancy, select a redundancy type.
    Redundancy typeDescription
    Single zone Stores your data redundantly across multiple devices within a single Availability Zone. This option is available only for the backup tier.
    Multi-zone Stores your data on multiple devices across a minimum of three Availability Zones in a region. This option is available only for both backup and archive tiers.
  11. Select a region type.
    OptionDescription
    Source region

    Creates an RCV location in the same region as the source workload.

    This is a cost-effective option due to less data transfer charges.
    Specific region

    Creates an RCV location in a specific region.

    Using this option will cost more due to the egress charges. Choose this option only if there is a need to maintain the snapshot copy in a different region.

    Note:
    This option is only available with customer-hosted exocompute. It is not available with Rubrik-hosted exocompute.
  12. For specific region, select a geographical region for the RCV location.
    The list of regions appears based on the selected storage tier.
  13. Click Next.
  14. Review the summary information and click Add.

Results

RSC creates an RCV location for the cloud-native workloads on AWS.

The newly created RCV location appears on the Rubrik Cloud Vaults page and on the Archival Locations page.
