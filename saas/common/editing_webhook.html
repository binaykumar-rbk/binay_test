Editing a webhook

Update the settings of a webhook added to RSC.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Notifications.
  4. Select Webhooks.
    On this page, you can see a list of webhooks already added to RSC.
    The Webhooks page appears.
  5. Click next to the webhook you want to edit and then select Edit.
    The Update Webhook wizard appears.
  6. In the Webhook configuration page, update the webhook provider, webhook name, endpoint URL, Endpoint Certificate, Service Account, and API Key details as needed. If updating a Generic provider type, update the authentication settings as needed.
  7. Click Next.
    The Select event and severity page appears.
  8. Edit the selected events and their severity levels.
    The next window that displays depends on your webhook configuration. If you configured to trigger alerts for both events and audit logs, the Event Export Template window appears. If you configured to trigger alerts for only audit logs, the Audit Export Template window appears.
  9. In the Event Export Template window, update your previously configured settings for the desired output format by either selecting to use the default template or creating your own custom template.
  10. Click Next.
    If you configured to trigger alerts for only audit logs, the Audit Export Template window appears.
  11. In the Audit Export Template window, , update your previously configured settings for the desired output format by either selecting to use the default template or creating your own custom template.
  12. Click Next.
    The Confirmation window appears.
  13. Review the details for the webhook configuration and the selected events, objects, and severity and then click Save.

Results

RSC saves the updated webhook settings.
