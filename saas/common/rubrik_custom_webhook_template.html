Rubrik custom webhook template

You can create custom webhook templates in RSC by defining JSON payloads with variables supported by Rubrik.

Defining custom webhook templates

RSC allows you to define the format of webhook payloads by creating custom JSON templates. Custom JSON templates allow you to select variables supported by Rubrik and structure webhook notifications to meet your specific requirements.

By default, webhooks are configured with a predefined and standardized PD-CEF format.

You can define a custom webhook template when adding or updating a webhook.

The Variables column lists all the available variables supported by Rubrik. In the Define Template pane, which appears when defining a custom webhook template, placeholder text provides a sample template. The placeholder text disappears as you enter your own content.

Entering key-value pairs triggers a message at the bottom of the screen to explain any encountered errors. You must resolve the errors before continuing.

The preview and output of the webhook variables in the Preview pane are automatically arranged in alphabetical order.

Webhook template syntax and examples

In the Define Template pane, you must enter key-value pairs for each webhook notification variable you want to use. You must use the double curly braces {{}} notation to specify each variable supported by Rubrik dynamically in your custom template. You must enter your customized keys for each key-value pair.

You can create a simple JSON template with a single value. The following example creates a simple JSON template using the Message key from a Rubrik event object structure:
{
  "MyCustomMessageKey" : "{{.Message}}"
}
You can modify this simple template by adding text before and after it. For example:
"Hello my: {{.Message}}"
You can use more than one key to create a template. For example:
"MyCustomMessageKey" : "Message: {{.Message}} for account: {{.Account}}"
You can use sub-structures, such as the Error key from a Rubrik event object structure. For example:
"MyErrorId": "{{.Error.ID}}"
You can also add custom dates in the following format:
// 1. Default results into UTC ex: 2025-02-05 23:13:20.916705803 +0000 UTC
{
  "time": "{{.Time}}",
}
// 2. will result into RFC3339 format: ex: "2006-01-02T15:04:05Z07:00"
{
  "time": "{{.Time.UTC.Format "2006-01-02T15:04:05Z"}}",
}
// 3. will result into RFC1123 format: ex: "Mon, 02 Jan 2006 15:04:05 MST"
{
  "time": "{{.Time.UTC.Format "Mon, 02 Jan 2006 15:04:05 MST"}}",
}
// 4. ...... they can create formate on their own

RSC supports the following custom templates for events and audits by default.

Sample Event Custom Template:
{
  "summary": "{{.Message}}",
  "source": "Rubrik Security Cloud",
  "severity": "{{.Severity}}",
  "timestamp": "{{.Time}}",
  "class": "{{.ActivityType}}",
  "component": "{{.ObjectType}}",
  "group": "Event",
  "custom_details": {
    "seriesId": "{{.ActivitySeriesID}}",
    "type": "Event",
    "id": "{{.UUID}}",
    "objectId": "{{.ObjectID}}",
    "objectName": "{{.ObjectName}}",
    "objectType": "{{.ObjectType}}",
    "status": "{{.Status}}",
    "clusterName": "{{.ClusterName}}",
    "clusterId": "{{.ClusterUUID}}",
    "eventName": "{{.MessageTemplate}}",
    "errorId": "{{.Error.ID}}",
    "errorCode": "{{.Error.ErrorCode}}",
    "errorRemedy": "{{.Error.Remedy}}",
    "errorReason": "{{.Error.Reason}}",
    "errorMessage": "{{.Error.Message}}",
    "location": "{{.Location}}",
    "url": "{{.URL}}",
    "customerID": "{{.Account}}",
    "logicalSizeInBytes": "{{.Metrics.LogicalSizeInBytes}}",
    "dataTransferredInBytes": "{{.Metrics.DataTransferredInBytes}}",
    "effectiveThroughput": "{{.Metrics.EffectiveThroughputInBytesPerSecond}}"
  }
}
Sample Audit Custom Template:
{
  "summary": "{{.Message}}",
  "source": "Rubrik Security Cloud",
  "severity": "{{.Severity}}",
  "timestamp": "{{.Time}}",
  "class": "{{.AuditType}}",
  "component": "{{.ObjectType}}",
  "group": "Audit",
  "custom_details": {
    "seriesID": "{{.AuditSeriesID}}{{.EventSeriesID}}",
    "type": "Audit",
    "id": "{{.UUID}}",
    "objectId": "{{.ObjectID}}",
    "objectFid": "{{.ObjectFid}}",
    "objectName": "{{.ObjectName}}",
    "objectType": "{{.ObjectType}}",
    "status": "{{.EventStatus}}",
    "clusterName": "{{.ClusterName}}",
    "clusterId": "{{.ClusterUUID}}",
    "eventName": "{{.MessageTemplate}}",
    "auditUserName": "{{.UserName}}",
    "auditUserId": "{{.UserID}}",
    "ipAddress": "{{.IPAddress}}",
    "customerID": "{{.Account}}",
    "userNote": "{{.UserNote}}",
    "isPolarisAudit": "{{.IsPolarisAudit}}",
    "orgID": "{{.OrgID}}"
  }
}
