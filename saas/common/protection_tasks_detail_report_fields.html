    • Reports

      You can use the Reports feature to perform audits and data management planning by creating customizable reports for application data protection and for the underlying infrastructure of Rubrik Security Cloud.

      • Find reports using the navigation panel

        You can use the navigation panel on the Reports page to find reports by category, search, or pinned status.

      • Create, customize, and manage reports

        You can customize and manage reports using a variety filter and grouping options, and creating report schedules.

      • Backup strikes reports

        A backup strike is an indication that an object could not successfully complete all backups scheduled on a single day.

      • Create reports with report templates

        Information about the report templates in RSC to create reports that provide insights into resource usage and related operations.

        • Anomalies report

          The Anomalies report in RSC displays data for systems impacted by attacks such as ransomware encryption and mass deletion.

        • Audit report

          For daily operations and auditing purposes, you can view the Audit report in RSC, which displays a comprehensive log of system activities, user actions, and internal jobs.

        • Compliance report

          For daily operations and auditing purposes, you can view the Compliance report in RSC, which displays information about the adherence of protected objects to established policies and regulatory requirements.

        • Cloud native object capacity report

          The Cloud Native Object Capacity report offers a unified view of your cloud-native object capacity usage across your cloud workloads, such as Microsoft Azure, AWS, and GCP.

        • Infrastructure report

          For daily operations and auditing purposes, you can view the Infrastructure report in RSC, which displays information about the status, health, and utilization of the data protection infrastructure components.

        • Object Audit report

          For daily operations and auditing purposes, you can view the Object Audit report in RSC which displays a historical record of actions and changes performed on protected objects and the users who initiated those actions.

        • Object Backup Task Summary report

          The Object Backup Task Summary report provides information about scheduled backup tasks that are triggered by the protection policy of the assigned SLA Domain.

        • Object Capacity report

          For daily operations and auditing purposes, you can view the Object capacity report in RSC, which displays the current storage consumption and size attributes of protected objects.

        • Object Capacity Over Time report

          For daily operations and planning purposes, you can view the Object Capacity Over Time report in RSC.

        • Protection report

          For daily operations and auditing purposes, you can view the Protection report in RSC, which displays the current protection status and key attributes of protected objects.

        • Protection Tasks Details report

          For daily operations and auditing purposes, you can view the Protection Tasks Details report in RSC, which displays detailed information about the tasks related to data protection activities.

          • Protection Tasks Detail report fields

            Information about the fields in the Protection Tasks Details report.

        • Quorum Authorization Roles report

          The Quorum Authorization Roles report displays a list of users who can approve specific policies associated with a Quorum Authorization request. Using this report, you can easily find the users with the QAuth reviewer role and the policies they can approve.

        • Recovery Tasks Details report

          For monitoring service level agreements and evaluating the efficiency of your disaster recovery plan, you can view the Recovery Tasks Details report in RSC.

        • Roles report

          For daily operations and auditing purposes, you can view the Roles report in RSC, which displays the roles created by the administrator and their descriptions.

        • Service Account report

          For daily operations and auditing purposes, you can view the Service Account report in RSC, which displays service accounts and their roles.

        • SSO Groups report

          For daily operations and auditing purposes, you can view the SSO Groups report in RSC, which displays SSO groups and their roles.

        • Threat Monitoring report

          For proactive security monitoring and auditing purposes, you can view the Threat Monitoring report in RSC, which displays details about identified security threats and their status.

        • Users report

          For daily operations and auditing purposes, you can view the Users report in RSC, which displays users and their roles.

Protection Tasks Detail report fields

Information about the fields in the Protection Tasks Details report.

The following table lists the fields in the Protection Tasks Details report.
Field Description
Task Type The type of task.
Task Status The status of the task: succeeded, succeeded with warnings, failed, or canceled.
Object Name The name of object for which the task ran.
Object Type The type of object for which the task ran.
Start Time The start time of the task in local time.
End Time The end time of the task in local time.
SLA Domain The SLA domain of the object for which the task ran.
Replication Source The Rubrik cluster location that sends replicas of source snapshots or backups.
Replication Target The destination Rubrik cluster where the replicated data is stored.
Archival Target The storage location where data is archived for long-term retention and compliance purposes.
Direct Archive This allows NAS backups to be directly uploaded to archial without generating a local snapshot on the Rubrik cluster. This is currently applicable to Filesets only.
Recovery Point The specific point-in-time to which a database can be restored.
Recovery Point Type The type of the recovery point (e.g. Point In Time, Snapshot).
Username The user that requested the recovery.
Status Reason The reason for the task status column. It could be the reason for failure or warnings.
Object Count The total number of objects per cluster, object type, cluster type, or location.
Task Count The total number of tasks per cluster.
Success Rate The success rate of the task, filtered by object, cluster, location, or time.
