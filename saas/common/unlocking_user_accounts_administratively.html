Unlocking user accounts administratively

Unlock a local user account or an SSO user account, regardless of whether the account was locked out due to administrative actions, brute force, or user account inactivity, if you have administrator or RSC account owner privileges.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Users & Access and select Users.
    You can view the SSO user accounts by clicking SSO. A list of locked and unlocked SSO user accounts displays. You can view specific user accounts by using the Roles and Lockout filters.
    The Local tab of the Users page appears by default, displaying a list of locked and unlocked local user accounts.
  4. From the filter options, under Lockout, select one or more of the following locked account options:
    OptionAccount
    Administrative Locked by an administrator.
    Breached password Locked due to a breached password.
    Inactive account Locked due to user inactivity.
    Suspected brute-force Locked due to suspected brute force attempts.
    The Users page displays a list of locked user accounts based on the selected option.
  5. Select a user account to unlock.
    You can unlock multiple user accounts simultaneously. If you have administrator privileges in a tenant organization, and if you attempt to unlock user accounts that belong to multiple organizations, RSC displays the following message: Tenant is unauthorized to perform the action for users belonging to multiple organizations. Please contact global organization administrators.
  6. Click and select Unlock User Accounts.
    A confirmation prompt appears.
  7. Click Continue.

Results

RSC unlocks the selected user account administratively.
