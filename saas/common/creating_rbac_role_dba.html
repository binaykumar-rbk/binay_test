Creating an RBAC role for the database administrator

Create a new database administrator role with role-based access control privileges. These privileges provide finer granularity for determining what permissions database administrators have for performing which actions and on which objects.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Users and Access.
  4. Select Roles.
    The list of roles appear.
  5. Click Create Role.
    The Create Role wizard appears.
  6. Click Custom Role.
  7. In Role name, enter a name of the role.
  8. Optional: Click Add Description.
    Enter a description for the custom role.
  9. In Data Management Permissions, click Configure to set the data management privileges for the role.
    The wizard displays the Data Management Permissions page.
  10. Select By object type and click Next.
  11. Select the objects for which the database administrator role requires permission.
    Clicking Object type selects all the objects. The default permissions grants the database administrator role full access to all objects on RSC of the specified type. For example, selecting Oracle DBs grants full access to all Oracle database objects on RSC.
    Selected objects display the All protectable objects, All recovery targets, and Full privileges buttons.
  12. Optional: Determine which objects, targets, and privileges for which you are granting permission.
    SelectionDescription
    All protectable objects Choose this to protect all existing and future objects or to protect specific objects.

    If you are protecting specific objects, select these objects from the object columns.

    You can narrow the search by specifying the Rubrik cluster you are searching.
    All recovery targets Choose this to protect all existing and future recovery targets or to protect specific recovery targets.

    If you are protecting specific recovery targets, select the recovery targets from the target columns.

    You can narrow the search by specifying the Rubrik cluster you are searching.
    Full privileges Choose this to set up data management privileges for objects.

    Select the objects for which you are granting permission.

    Select the View, Protection, Recovery, and Data source management headings to select all the objects in these groups.
  13. Click Next.
  14. Determine which archival location you are selecting.
    SelectionDescription
    Select specific archival locations Choose this to select specific archival locations. Select the archival locations from the list you want to use.

    Filter the search by selecting from the Storage type, Archival Location Type, Rubrik Cluster, and Status lists.
    Select all existing and future archival locations Choose this to select all existing and future archive locations.

    Additionally, you can add permissions for this database administrator role to add, edit, or delete archival locations.
  15. Click Next.
  16. Optional: Turn on Self-Service.
    Enabling self-service turns off the ability to select specific SLA Domains and specifies the manage permission for all existing and future SLA Domains.
    Turning on self-service enables the database administrator role you are adding to create and manage all existing and future SLA Domains.
  17. Determine which SLA Domains you are selecting.
    SelectionDescription
    Select specific SLA Domains Choose this to select specific SLA Domains. Select the SLA Domains from the list you want this database administrator to have access to.

    Filter the search by selecting from the Assigned Orgs, Created By, and Managed By lists.
    Select all existing and future SLA Domains Choose this to select all existing and future archive locations.

    Additionally, you can add permissions for this database administrator to add, edit, or delete archival locations.
  18. Optional: (Specific SLA Domains) Select Don't protect to include SLA Domains with the Don't Protect setting in the list of SLA Domains.
  19. Optional: (Specific SLA Domains) Select Clear existing assignment to include SLA Domains with this setting in the list of SLA Domains.
  20. Click Done.

Results

RSC creates a database administrator role with RBAC privileges and refreshes the Roles page.

What to do next

After you have created the DBA role, you can assign the DBA Active Directory Group to be a member of this role. After you do this, member access is controlled through Active Directory instead of through RSC as added or removed from the DBA role, which saves you from having to update RSC with these changes.
