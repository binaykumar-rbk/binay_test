              Update permissions for an individual AWS cloud account to ensure that the required permissions for the new features are included in RSC. You can use the AWS CloudFormation template to update the permissions. You can also remove permissions that are no longer needed.

              Update permissions for all the AWS cloud accounts that are added using an AWS Organization to ensure that the required permissions for the new features are included in RSC.               Update permissions for an AWS cloud account to ensure that the required permissions for the new features are included in RSC. You can update the permissions manually. You can also remove unneeded permissions.

              Upgrade an AWS cloud account in the "Update permissions" state to retrieve the current set of permissions.

Deleting permissions for an Amazon RDS cloud account

Delete permissions for an Amazon RDS account project and delete the RSC cloud account.

About this task

Disable the RSC cloud account for AWD RDS before removing the related RSC cloud account.

When the cloud account is removed, AWS removes the cross-account role from the CloudFormation template and all CloudFormation stack resources of the AWS account.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Cloud Accounts and select AWS.
  4. Open the drop-down menu and select Protection.
    The EC2, EBS, and Application Protection tab appears.
  5. Click RDS Protection.
    The list of AWS cloud accounts appears.
  6. Open the ellipsis menu for a cloud account and select Delete.
    The Delete account page appears.
  7. Click Launch CloudFormation Stack to launch the CloudFormation template on the AWS CloudFormation console.
    RSC redirects the user to the CloudFormation > Stacks > Stack Detail page on the AWS CloudFormation console.

    The Select Template page appears. RSC initiates a redirection URL and the browser adds that URL to the Specify an Amazon S3 template URL field.

  8. Open the Other Actions menu and select Delete Stack.
    Delete the stack only after disabling protection for the RSC cloud accounts for AWS.
  9. When prompted, click Yes, Delete.
    AWS deletes the stack and the IAM role that Rubrik created while adding the cloud account.
  10. In the Delete cloud account page on RSC, click Delete.

Results

RSC removes the RSC cloud account.

When the cloud account is removed, RSC deletes all credentials associated with that cloud account.
