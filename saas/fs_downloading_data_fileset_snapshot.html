          • Locating a fileset snapshot for recovery

            Locate a fileset snapshot to recover data from the entire snapshot or from a selection of files and folders within the snapshot.

          • Searching for a file or folder for recovery

            Use the search capability to find a specific file or folder from a fileset snapshot for recovery.

          • Downloading data from fileset snapshots

            You can use RSC to recover files or folders by downloading the backed-up data.

          • Restoring fileset snapshot data to the source location

            Restoring files or folders from the fileset snapshot to the original location of the source data overwrites the original data.

          • Restoring fileset snapshot data to a new location

            Restore files or folders from the fileset snapshot to a location that is different from the original location of the data on the source host.

          • Export path for fileset snapshot data

            The path to export the fileset snapshot data must comply with the rules that RSC enforces.

          • Exporting fileset snapshot data

            Export files or folders from the fileset snapshot to a host that is different from the source host.

Downloading data from fileset snapshots

You can use RSC to recover files or folders by downloading the backed-up data.

Before you begin

Locate a fileset snapshot to recover, as described in Locating a fileset snapshot for recovery, or search for a file or folder to recover, as described in Searching for a file or folder for recovery.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click a fileset card based on the OS type of the host.
    • Windows filesets
    • Linux & Unix filesets
    The corresponding filesets page appears.
  5. In the Name column, click a host name.
    The local host page appears.
  6. In the calendar view of the Snapshots card, click a date.
    The Day view of the Snapshots card lists all the backups created on the selected date.
  7. In the Day view, click Recover files for the fileset backup to recover.
    The Fileset Recovery wizard opens the Select folders and files page, displaying the list of folders and files in the fileset.
  8. On the Select Recovery Type page of the recovery wizard, select Download.
  9. Optional: In Add a note, enter text to attach to the object type for this task.
    You can enter notes up to 1024 characters.
  10. Click Recover.

Results

RSC starts preparing the data to be downloaded and creates an event of Recovery type with all the details of the download request.

What to do next

Check the status of the download request in the Events tab for the host. To see the events in the Recovery category, filter the events by Event type. Download the data by clicking the download link that appears in the Event timeline when the download link is ready.
