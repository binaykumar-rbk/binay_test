        • Locating a fileset snapshot for recovery

          Locate a fileset snapshot to recover data from the entire snapshot or from a selection of files and folders within the snapshot.

        • Searching for a file or folder for recovery

          Use the search capability to find a specific file or folder from a fileset snapshot for recovery.

        • Downloading data from fileset snapshots

          You can use RSC to recover files or folders by downloading the backed-up data.

        • Restoring fileset snapshot data to the source location

          Restoring files or folders from the fileset snapshot to the original location of the source data overwrites the original data.

        • Restoring fileset snapshot data to a new location

Export path for fileset snapshot data

The path to export the fileset snapshot data must comply with the rules that RSC enforces.

When a backup copy of a file, folder, or fileset is exported, the Rubrik cluster writes the exported data to a location on the target host. The location where the data is written consists of the path on the target that is provided through the Export Path value combined with the path of the exported object relative to the root of the backup.

The path specified in Export Path must already exist on the target. The Rubrik cluster will create the rest of the path, starting at the specified Export Path value, if it does not already exist.

For a Linux or Unix host, the root directory can be specified by a single forward slash character.

For a Windows host, the root directory of a drive can be specified by the drive letter, a colon, and a backslash. For example, D:\ represents the root of the ‘D’ drive.

Use case Fileset backup for a Windows host Fileset backup of a Linux or Unix host
Description A fileset backup of a Windows host includes the test_example.txt file. The full path of test_example.txt on the source Windows host is: C:\Users\Atom\testing\test_example.txt A fileset backup of a Linux or Unix host includes the test_example file. The full path of test_example on the source host is: /usr/local/tmp/test_example
Example 1 The path specified in Export Path is: C:\

The Rubrik cluster writes the file to the following path on the target host: C:\Users\Atom\testing\test_example.txt

 The path specified in Export Path is: /

The Rubrik cluster writes the file to the following path on the target host: /usr/local/tmp/test_example
Example 2 The path specified in Export Path is: G:\testing\temp

The Rubrik cluster writes the file to the following path on the target host: G:\testing\temp\C_\Users\Atom\testing\test_example.txt

 The path specified in the Export Path is: /usr/local/tmp

The Rubrik cluster writes the file to the following path on the target host: /usr/local/tmp/usr/local/tmp/test_example
