      • Hyper-V virtual machines

        A Rubrik cluster provides data management and protection for virtual machines that are deployed in a Microsoft Hyper-V environment. The Rubrik cluster can manage and protect virtual machines in an environment with multiple Hyper-V servers and virtual machines.

        • Hyper-V virtual machine protection

          Rubrik clusters can protect virtual machines based on a direct SLA Domain assignment or based on inheritance of the SLA Domain assigned to the parent object.

        • Rubrik Backup Service for Hyper-V

          The Rubrik Backup Service provides enhanced integration with protected resources and host systems.

        • Prerequisites for System Center Virtual Machine Manager

          Microsoft provides SCVMM hosts to manage virtual machines across multiple hosts. The Rubrik Backup Service should be installed on all hosts running SCVMM.

        • Hyper-V host management

          After installing the Rubrik Backup Service software on a Hyper-V host, adding a host to the Rubrik cluster establishes a secure connection between the Rubrik cluster and the Rubrik Backup Service that is running on the host.

        • Hyper-V SLA Domain assignment

          Provide protection for a virtual machine through an SLA Domain.

        • Local Hyper-V host page

          The local host page provides detailed information about the protection of a virtual machine, and tasks related to the virtual machine.

        • Hyper-V virtual machine snapshots

          The Rubrik cluster provides protection for virtual machines by combining native snapshot technology with the fast and scalable cloud data management platform of the Rubrik cluster.

        • Excluding Hyper-V virtual machine VHD files

          Among the VHD files on a virtual machine, some VHD files might not require a backup. Exclude these VHD files from the backup, thereby reducing the size of the snapshots.

        • Hyper-V archival snapshot retention

          The retention period assigned to the archival snapshot by the associated SLA Domain determines the expiration of an archival snapshot.

        • Unmanaged data

          RSC supports managing file system and application data that is not subject to a retention policy.

        • Hyper-V virtual machine recovery

          You can recover virtual machines and restore data by using any Rubrik data protection object and using an available recovery action.

          • Hyper-V virtual machine recovery actions

            You can recover a source virtual machine for a Rubrik cluster by mounting a point-in-time copy of the source virtual machine.

            • Hyper-V Instant Recovery

              Using Instant Recovery, you can quickly bring a full Hyper-V virtual machine online while also deprecating and replacing the original virtual machine.

            • Performing an instant recovery of Hyper-V virtual machines

              Use RSC to perform an instant recovery.

            • Hyper-V virtual machine recovery using Live Mount

              You can recover a Hyper-V virtual machine using the Live Mount recovery option and choose to migrate the data from the Rubrik cluster to a Hyper-V host.

            • Hyper-V export

              An export creates a new virtual machine from a point-in-time copy of the source virtual machine.

            • Performing a Hyper-V virtual machine snapshot export

              Perform an export to create a virtual machine from a point-in-time copy of the source virtual machine.

            • Hyper-V virtual machine recovery by mounting virtual disks

              You can recover a Hyper-V virtual machine by mounting virtual disks from a snapshot on the source Rubrik cluster onto a target virtual machine.

            • Recovering Hyper-V virtual machines by mounting virtual disks

              Recover a Hyper-V virtual machine by mounting virtual disks from a snapshot onto a target virtual machine. The target virtual machine is on the same Rubrik cluster as the source virtual machine.

            • Hyper-V virtual machine recovery by downloading virtual machine files

              You can recover a Hyper-V virtual machine by downloading virtual disks and virtual machine files without downloading the entire virtual machine. You can fully restore the original virtual machine by downloading all files and disks, or you can download specific disks and configuration files to restore a virtual machine with only the data from the disks.

            • Recovering Hyper-V virtual machines by downloading virtual machine files

              Recover a Hyper-V virtual machine by downloading specific virtual disks and virtual machine files without downloading the entire virtual machine.

            • Hyper-V batch recovery

              To recover multiple Hyper-V virtual machines at the same time, you can perform a batch recovery operation.

            • Hyper-V in-place recovery

              Hyper-V in-place recovery offers a faster and more efficient option for virtual machine recovery compared to other Hyper-V recovery methods. In-place recovery transfers only the changed blocks in a Virtual Hard Disk (VHDX) file, significantly reducing the number of blocks transferred and the latency in recovering virtual machines.

            • Performing an in-place recovery of Hyper-V virtual machines

              Perform an in-place recovery using RSC for faster recovery of data in Hyper-V virtual machines.

          • Recovering a file or folder

            The Rubrik cluster provides file-level restore of files and folders from any local snapshot, replica, or archival snapshot that was successfully indexed.

Hyper-V export

An export creates a new virtual machine from a point-in-time copy of the source virtual machine.

When creating a copy of the source virtual machine, the datastore of the selected Hyper-V host is the datastore for the recovered virtual machine. Rubrik can export the resulting virtual machine disk (VMDK) as either Thick Provisioned or Thin Provisioned.

The Rubrik cluster assigns a new name to the recovered virtual machine and powers it up. The Rubrik cluster does not connect the recovered virtual machine to a network. The Rubrik cluster sets the protection state of the new virtual machine to Don't Protect.
