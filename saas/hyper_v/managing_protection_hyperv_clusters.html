Managing protection for Hyper-V clusters and hosts

Choose a data protection policy for a Hyper-V cluster or host.

About this task

Assigning Hyper-V clusters and hosts to an SLA Domain protects the clusters and hosts by applying the data protection policies of the SLA Domain.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
  3. On the navigation bar, click Inventory.
  4. Click the Hyper-V VM card.
  5. Click Clusters and Hosts.
  6. Select one or more clusters or hosts.
  7. Click Manage Protection.
    The Manage Protection wizard starts, and the Assign SLA Domain page appears.
  8. Choose a protection option.
    OptionDescription
    SLA Domain The SLA Domain is assigned to the selected objects.
    Clear existing assignment The SLA Domain is derived from the next higher level object based on inheritance rules.
    Don't protect The selected objects are excluded from all further SLA Domain assignments and protection.
  9. (Don't protect) In Existing snapshot retention, choose an option for the existing snapshots.
    OptionDescription
    Use current SLA Domain for retention The retention is derived from the current SLA Domain.

    This is the default option.
    Keep forever The snapshots are retained until they are manually deleted.
    Expire immediately The snapshots are deleted from the Rubrik cluster.
  10. Click Next.
    (SLA Domain) A Confirmation page appears in the wizard displaying the summary of SLA Domain changes and options to apply changes to existing snapshots.

    (Clear existing assignment) The Inherit SLA Domain page appears and displays options to apply changes to existing snapshots.

    These options are not displayed when the Don't protect option is selected.
  11. Optional: For Apply changes to existing snapshots, choose from the following options.
    • Turn on to apply changes to the existing snapshots.
    • Turn off to keep the retention policy of existing snapshots unchanged.
  12. Optional: When Apply changes to existing snapshots is turned on, configure the Include on-demand and downloaded snapshots setting.
    • Select the setting to apply changes to the retention policy of on-demand and downloaded snapshots.
    • Clear the setting to keep the retention policy of on-demand and downloaded snapshots unchanged.
  13. Optional: In Add a note, enter text to attach to the object type for this task.
    You can enter notes up to 1024 characters.
  14. Click Assign.

Results

RSC processes the specified SLA Domain assignment options for Hyper-V virtual machines and manages protection for the virtual machines.
