Recovering a batch of vSphere virtual machines using exports

Perform exports for recovering a batch of vSphere virtual machines in a single operation using Anomaly Detection.

Procedure

  1. Log in to RSC.
  2. Click and select Data Threat Analytics.
    The Data Threat Analytics dashboard appears.
  3. On the navigation bar, click Anomaly Detection.
    The Anomaly Detection page appears.
  4. Select the vSphere virtual machines to recover.
    • The selected machines must belong to a single cluster for batch recovery.
    • You cannot perform batch recovery if you select virtual machines along with Recovery Plans.
    • You can select a maximum of 10 virtual machines at once.
  5. Click Recover.
    The Virtual Machine Recovery wizard opens and displays the Select a recovery approach page.

    If you selected only a single virtual machine for recovery, the Anomaly Recovery wizard opens and displays the Select a recovery approach page.

  6. Enter a date and time.
    The Restore to closest snapshot in time option is selected by default and uses the closest snapshot before or after a specified date and time. This option applies to all the selected virtual machines.
    When you specify a Before date but do not specify the time, the closest snapshot used is the snapshot taken before 00:00:00 on the specified date. When you specify an After date but do not specify the time, the closest snapshot used is the snapshot taken after 23:59:59 on the specified date.
  7. Click Next.
    The Confirm snapshot selection page opens.
  8. To change the snapshot of a virtual machine, click on the respective row.
    A graph appears on this page and displays the suspicious, added, modified, and deleted files.
  9. Click Change Selected Snapshot.
    The Select a snapshot dialog box appears.
  10. Select a snapshot to recover and click Close.
  11. Click Next.
    The Select a recovery option page opens.
  12. Select Export and click Next.
    The Select storage page opens.
  13. Select a compute resource to recover the virtual machines.
    You can search for compute resources, if required, in the search bar.
  14. Optional: In Advanced settings, select an option.
    OptionDescription
    Exported virtual machine prefix Enter a prefix for the exported virtual machines.
    Remove virtual network device Select this option when networking changes or issues prevent the virtual machine from starting.
    Preserve Mac address Select this option to use the MAC addresses from the snapshot instead of assigning new MAC addresses. When the snapshot contains manually assigned MAC addresses, those MAC addresses are used even when this option is not selected.
    Recovery with tags Select this option to restore tags assigned to the original virtual machine.
    You can select multiple options.
  15. Click Next.
    The Select a datastore page opens.
  16. Select a datastore and click Export.
    The notification on the page confirms the recovery. This is also indicated in the Events page.

Results

Anomaly Detection creates new virtual machines from the snapshot on the selected compute resource.
