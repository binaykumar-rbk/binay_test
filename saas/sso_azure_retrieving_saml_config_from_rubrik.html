  • Security

    Configure and manage RSC security features, including user access, SSO, Quorum Authorization, and data encryption.

    • Users and Access

      RSC provides role-based access control, and several methods for authenticating a user account.

    • Quorum authorization

      Quorum authorization is a security feature that enforces the requirement of getting additional approvals before a requester can perform data-modifying actions in RSC.

    • Single sign-on

      Rubrik Security Cloud supports SSO using the SAML 2.0 standard. With SSO, you can log in to RSC using credentials associated with an identity provider.

    • SAML 2.0 metadata files

      SAML 2.0 uses information from metadata files to exchange information between an identity provider and a service provider, such as RSC, to establish a trust relationship.

    • RSC SSO workflow

      RSC supports configuring single sign-on with an identity provider for secure authentication.

    • Okta integration workflow

      The integration workflow provides a high-level view of the requirements for configuration of SSO with Okta.

    • ADFS integration workflow

      The workflow provides a high-level view of the tasks involved in configuring SSO with ADFS.

    • Azure SSO

      Configuring single sign-on with Azure involves several individual tasks, which must be performed in the correct order.

      • Retrieving service provider SAML configuration

        Retrieve the entity ID and callback URL values from RSC for preparing the federation metadata file in Azure.

      • Creating a non-gallery enterprise application

        Create a non-gallery enterprise application for RSC in the Azure portal.

      • Configuring Azure SSO

        Configure SSO in RSC by uploading the Azure metadata file and downloading the Rubrik metadata file.

      • Uploading the Rubrik metadata file

        Upload the Rubrik metadata file to the enterprise application in Azure.

      • User attributes and claims in Azure

        For SSO authentication, RSC requires that Azure issue specific claims and attributes.

      • Configuring user attributes and claims

        In the Azure portal, configure the claims for RSC by assigning values to the email address and group attributes.

      • Creating custom app roles

        In the Azure portal, create custom app roles to determine the permissions for user accounts that log in to RSC using Azure SSO.

      • Assigning app roles to users and groups

        In the Azure portal, assign the custom app role to the user accounts who need single sign-on access to RSC.

      • Authorizing RSC SSO groups for Azure SSO

        After creating Azure custom app roles and assigning the roles to user accounts in Azure SSO groups, create RSC SSO group roles or use existing roles and authorize RSC SSO groups by mapping Azure custom app roles to RSC SSO group roles.

      • Testing the SSO configuration

        Test the SSO configuration to verify that authentication requests are handled by the identity provider.

      • Unauthorized access condition

        Investigating the Azure SSO configuration can help find the cause of an unauthorized access error.

    • OneLogin integration workflow

      The OneLogin integration workflow provides a high-level view of the tasks involved in configuring single sign-on with OneLogin.

    • IP allowlist

      RSC administrators can specify IPv4 addresses for the IP allowlist. RSC blocks unauthorized login attempts and unauthenticated API calls that originate from addresses that are not on the allowlist.

    • Authentication

      Rubrik Security Cloud authentication restricts access to a specified set of users.

    • Certificate management

      RSC Certificate Management provides a centralized dashboard to manage certificates across all connected Rubrik clusters and for RSC workflows.

    • Encryption management

      With the encryption management feature in RSC, you can view information about data-at-rest encryption for the Rubrik clusters and archives, access to your key management services, and enhance data security through automated key rotation every 30 days.

    • User account lockout

      The user account lockout feature enhances system security by providing global administrators the ability to lock and unlock user accounts. This feature also prevents unauthorized access from brute force attacks and safeguards local user accounts by automatically locking them after a configurable period of inactivity.

    • Session management

      The session management feature enhances system security in the following ways: ending inactive user sessions, ending active user sessions after a specific duration, and limiting the number of concurrent sessions allowed for each user.

    • Password policy

      RSC administrators can enhance the security of their RSC domain by using the password policy feature to configure password complexity rules for local users.

    • New device login notification

      RSC notifies you when a new device is used to log in to your RSC account, providing details such as device name, IP address, and location.

Retrieving service provider SAML configuration

Retrieve the entity ID and callback URL values from RSC for preparing the federation metadata file in Azure.

Before you begin

Use an account assigned an Administrator, or any role that has permission to view and manage security settings.

About this task

Retrieving SAML configuration from RSC is required for creating an enterprise application in Azure and is the first step in a series of steps required to configure Azure SSO.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Security and select SSO.
    The SSO page appears.
  4. In the Provide Rubrik information to the Identity Provider section, click Copy to Clipboard to copy the Service Provider Entity ID.
  5. Paste the clipboard content into a text file.
  6. Click Copy to Clipboard to copy the Application Callback URL and paste the value into the text file.

Results

The SAML configuration values are ready for use in the Azure portal.

What to do next

Create a non-gallery enterprise application in the Azure portal, as described in Creating a non-gallery enterprise application.
