Enabling Data Security Posture for Azure cloud accounts

Enable Data Security Posture for an existing Azure cloud account in RSC.

Before you begin

Ensure that all the workloads that you want to enable Data Security Posture for are added to the Azure cloud account. To add additional workloads for scanning, you can follow the process to add an Azure account with Data Security Posture enabled.

Procedure

  1. Log in to the Azure subscription that contains the workloads to be protected.
    The Azure portal page appears.
  2. Click and select Settings.
  3. Click Cloud Accounts and select Azure.
    The Azure page appears.
  4. Click Add Azure Subscription.
    The Add Azure Subscription wizard appears and displays the Select use case page.
  5. In the row for the specific Azure subscription, in the DSPM column, click Configure.
    The Data Security Posture wizard appears.
  6. Click Log in with Azure.
    The Microsoft Azure portal login page appears.
  7. Log in to complete the steps to select an outpost subscription for scanning and select Consent on behalf of your organization.
  8. Review the displayed information and click Accept.
  9. Select the Azure subscription to designate as the outpost account and click Next.
    The outpost account is the account that accesses the Azure account to perform sensitivity scans and generate metadata for data classification and sensitivity. RSC uses this outpost subscription to perform data discovery and run classification scans.

    When you have already designated and configured an outpost account, the outpost account ID appears, and you can go to the next page in the wizard.

  10. In Resource group name, enter the name of the resource group.
  11. In Resource group region, select the region for the resource group.
  12. Optional: Click Add Resource Group Tag to add key value pairs for the resource group.
  13. Click Submit.

Results

RSC enables Data Security Posture for the Azure cloud account.
