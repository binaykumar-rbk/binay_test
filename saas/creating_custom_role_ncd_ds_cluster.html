Creating custom roles for NAS CD data sets by cluster

Create a custom role in RSC with Role Based Access Control privileges for NAS CD objects with permissions configured by clusters. These privileges provide granularity for determining the permissions to perform actions, and on which objects.

About this task

Creating custom roles by cluster allows configuring access permissions for data sources depending on the clusters you select.

Performing this task is necessary if you want the user role with NAS CD UI access.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Users & Access, then select Roles.
    The list of roles appears.
  4. Click Create Role.
    The Create Role wizard appears.
  5. Select Custom role.
  6. In Role name, enter the name of the role.
  7. Optional: Click Add Description, to enter a description of the role.
  8. In Data Management Privileges, click Configure to set the data management privileges for the role.

    The wizard displays the Data Management Privileges page.

  9. Select By cluster and click Next.
  10. Select the NAS CD clusters containing the NAS CD objects for which you want to assign the permissions and then click Next.
  11. In Assign Data Management Privileges, select the required permissions.
    PermissionDescription
    View Provides permission to view a NAS CD cluster.

    This is one of the required permissions for accessing NAS CD UI.
    Data source management Provides permission to access the NAS CD UI.

    This is one of the required permissions for accessing NAS CD UI.
    Recovery Provides permission to perform recovery operation for a NAS CD cluster.
    RSC does not support Protection of NAS CD objects from RSC UI.
  12. Click Next and then click Done.
    You can click Next to skip the following pages, which are not applicable for NAS CD objects:
    • Assign Datacenter Archival Locations
    • Select SLA Domains
  13. In Assign System Configuration Privileges, click Configure to set the system configuration privileges for the role.
  14. Optional: Select Federated access, View clusters, and then click Done.
    Note:
    The Federated access and View clusters are required permissions for accessing the NAS CD UI. If you do not want to provide access permissions to the NAS CD UI, select only View clusters.
  15. In Rubrik Application Privileges, click Configure.
  16. Optional: Select the necessary permissions for managing Rubrik application features.
    The Data Observability for NAS CD data sets section provides information on the supported Rubrik application features.
  17. Click Create.
    A message appears indicating that the role is created.
  18. Optional: Click Go to Users Page.
    The Users page appears, displaying a list of available users.
  19. Click Done.

Results

RSC creates a new role based on your selections and refreshes the Roles page.
