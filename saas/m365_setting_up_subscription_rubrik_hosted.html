Setting up a Microsoft 365 subscription for Rubrik hosting

Set up integration between a Microsoft 365 subscription and RSC for the Rubrik-hosted deployment mode.

Before you begin

Complete the requirements for Microsoft 365 subscriptions and user accounts listed in Rubrik-hosted deployment mode requirements.

About this task

This topic describes a feature or features that Rubrik is gradually rolling out to customers.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
  3. On the navigation bar, click Inventory.
  4. On the Inventory page, click the Microsoft 365 inventory card.
  5. Select the subscription configuration type.
    OptionDescription
    Setup This choice appears when setting up the first subscription.
    Add Subscription This choice appears when setting up subsequent subscriptions.
    The Microsoft 365 Setup page appears.
  6. Click Add Subscription.
  7. Select an option to store Microsoft 365 backup data.
    SelectionDescription
    Retain the same geographical regions specified in the Microsoft 365 Admin Center. Select the option to Store data in its original geographic regions. Then select the central data location (CDL) from the drop-down menu.
    Use RSC to specify an Azure region for backup data. Select the region from the drop-down menu.
    RSC configures the subscription to store the backup data in the specified location.
  8. Click Next.
  9. Log into your Microsoft 365 account.
  10. Log into each Microsoft 365 App to review and accept the list of permissions.
  11. Click Add Subscription.
  12. When the success prompt appears, click Done.

Results

RSC creates the new Microsoft 365 subscription.

The setup of Azure resources takes about 20 minutes. For some Microsoft 365 subscriptions, the number of objects in the subscription may cause RSC to take several additional minutes to update the Inventory page.
