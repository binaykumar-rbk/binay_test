      • Microsoft Entra ID

        RSC supports backup and recovery of Microsoft Entra ID.

        • Known limitations for Microsoft Entra ID on RSC

          Microsoft Entra ID on RSC includes some known limitations for recovering groups, users, relationships, and Microsoft Entra ID properties.

        • Non-recovery of soft-deleted objects by RSC

          RSC does not recover higher-privileged, soft-deleted Microsoft Entra ID user or group objects. Instead, RSC creates new objects, leaving the soft-deleted ones in the Microsoft Recycle Bin.

        • Microsoft Entra ID just-in-time access

          Just-in-time access for Microsoft Entra ID separates read and write access to align with the principle of least privilege, ensuring that critical write permissions are active only during necessary recovery operations. Just-in-time access is available for Microsoft Entra ID domains using OAuth authentication.

          Add a Microsoft Entra ID directory as a protected workload using OAuth for authentication.

        • Adding a Microsoft Entra ID directory without OAuth

          Add a Microsoft Entra ID directory as a protected workload without OAuth, using your existing Microsoft Entra ID domain application registration for authentication.

        • Microsoft Entra ID object and relationship recovery

          RSC recovers various Microsoft Entra ID objects and object relationships, including users, groups, roles, enterprise apps, and app registrations.

        • Microsoft Entra ID cross-tenant recovery

          Microsoft Entra ID cross-tenant recovery restores directory data from a source tenant to a separate, distinct target tenant to ensure business continuity during catastrophic events.

        • Microsoft Intune protection

          RSC protects critical Microsoft Intune objects to maintain a resilient identity and endpoint posture.

        • Microsoft Entra ID user object attribute recovery

          Attribute recovery for a Microsoft Entra ID user object selectively restores specific attributes that were maliciously or accidentally altered without performing a full object restore. You can compare user object attributes from a snapshot against live data and recover only the changed attributes.

        • Microsoft Entra ID object recovery across snapshots

          With RSC, you can search Microsoft Entra ID user, groups, and roles objects for recovery across different snapshots. This functionality enables recovery of objects and relationships from a specific date and time.

        • Microsoft Entra ID Conditional Access recovery

Recovering Conditional Access policies and relationships

Recover Microsoft Entra ID Conditional Access policy objects and relationships from a specific point in time usingRSC.

Before you begin

  • Ensure that your Microsoft Entra ID subscription has a license for Conditional Access. The Microsoft Entra ID documentation provides information on Conditional Access licensing.
  • If you have added your Microsoft Entra ID directory without using OAuth, be sure you have set the necessary permissions.

    Policy.Read.All

    Policy.ReadWrite.ConditionalAccess

  • Apply these permissions to the Microsoft Entra ID application before taking a snapshot.

To recover Conditional Access objects and relationships for other Conditional Access object types, including named locations, authentication contexts, authentication strengths, and terms of use, see Recovering Conditional Access objects and relationships.

About this task

This topic describes a feature or features that Rubrik is gradually rolling out to customers.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, select Inventory > Identity > Microsoft Entra ID .
    The inventory list for Microsoft Entra ID appears.
  4. In the Domain column, click a Microsoft Entra ID domain.
    Alternatively, you can enter a name in the search field or use the filters at the top of the list.
    The local page for the domain appears, where the Snapshots card displays the month view.
  5. On the Snapshots card, select a day marked with a green dot.
    The green dot indicates that RSC successfully took a snapshot on that day.
    The Snapshots card displays the Day view with a list of snapshots.
  6. Hover over a snapshot, click , and then select Recover.
    The Microsoft Entra ID Recovery wizard appears, with the Select Recovery Type page selected.
  7. Select Granular Recovery.
  8. Click Next.
    The Select Objects to Recover page appears.
  9. Click Conditional Access.
  10. Click Authentication Contexts and select an authentication context from the list.
    You can select multiple authentication contexts. You can also search for an authentication context by name. Clicking the number in the Linked Policies column displays the policy linked to an authentication context. RSC does not recover the linked policy unless it is selected.
  11. Click Policies and select a policy from the list.
    You can select multiple policies. You can also search for a policy by name. Clicking the number in the Linked Objects or the Assignments column displays the Conditional Access objects related to a selected policy. RSC does not recover the related object unless it is selected.
  12. Click Next.
    The Recovery Options page appears.
  13. Select Recover to source directory.
    RSC only supports recovery to the source directory for In-Place Recovery.
  14. Select a relationship recovery mode.
    ModeDescription
    Roll back relationships Recovers all selected objects and object relationships to their snapshot state.
    Update relationships Recovers all selected objects and object relationships in the snapshot, while preserving the relationships of the other objects in the current directory.
  15. Click Next.
  16. Select a policy recovery method.
    For each recovered policy, you can choose to overwrite the existing policy or to create a copy of the existing policy. When you choose to create a copy of the policy, RSC prompts you to enter a new policy name.

    When RSC cannot find a selected policy for recovery on Microsoft Entra ID, RSC recreates that policy from the settings described in the snapshot and from the information gathered on the Microsoft Entra ID portal.

  17. Select a policy state and click Next.
    These options set the state of the policy after recovery. You can choose from On, Report Only, or Off. The Microsoft Entra ID documentation describes these modes.
    The Recovery Confirmation page appears.
  18. Review and apply required permissions for domains using OAuth and just-in-time (JIT) access.
    RSC displays a notification if additional permissions are required. Clicking the notification lists those permissions. Clicking Assign Permissions temporarily assigns those permissions for the recovery.
    Note:
    Because Microsoft 365 depends on Microsoft Entra ID for metadata recovery, assign Full Access to the Rubrik service principal before performing a full Teams or SharePoint relic recovery to a domain using OAuth and JIT access. This prevents potential data loss.
  19. Click Recover.

Results

RSC recovers the Conditional Access policy objects and relationships from a specific point in time.
