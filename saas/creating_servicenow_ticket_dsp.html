Creating a ServiceNow ticket in Data Security Posture

Create a ServiceNow ticket to address sensitive data access issues identified by Insights in Data Security Posture.

About this task

ServiceNow ticket creation is available only for addressing data access issues identified by Insights in Data Security Posture. The ServiceNow integration for Data Security Posture is available as part of the Rubrik Enterprise Proactive Edition.

Procedure

  1. Log in to RSC.
  2. Click and select Data Security Posture.
    The Data Security Posture Dashboard appears.
  3. Click a link in the Insights widget.
    Insights and Identity Inventory are available as part of the Rubrik Enterprise Proactive Edition.

    If a ticket has already been created, a View Ticket or a View Resolved ticket option displays.

    According to the link clicked, the Identity Inventory page or the Sensitive Data page appears with a message about the selected insight.
  4. Click Create ticket.
    RSC displays Create ticket when no ticket exists for this insight. RSC displays View ticket when a ticket has already been created based on the insight. The links are inactive when ServiceNow has not been configured or a ticket is in the process of being created for the insight.
    The Create a ticket dialog box appears.
  5. In Title, enter a title for the ticket.
  6. In Reason for reviewing access, provide a reason for reviewing the access issue identified by Insights.
  7. In Attachments, select the check boxes to attach additional contextual data.
    • A list of user names
    • A list of file paths to sensitive files

      RSC attaches the contextual data as a file accessible though a protected link in the ServiceNow ticket. Attached files can be downloaded through the protected link for 90 days before the link expires.

  8. Click Create.

Results

RSC creates a ServiceNow ticket for the selected issue.
