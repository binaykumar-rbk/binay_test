Limitations on upgrading disks for Azure CCES

Azure CCES PremiumV2_LRS functions as a staging disk. A staging disk has certain restrictions involving OS disk use, region support, supported software versions and zones, and node limits.

The following list describes limitations that could affect a PremiumV1_LRS to PremiumV2_LRS for Azure CCES upgrade.

Any limitation on this list blocks the upgrade and triggers a notification:

  • A PremiumV2_LRS functions only as a staging disk; it cannot be an OS disk. The upgrade applies only to the data disks for each node in the cluster.
  • Disk upgrades are supported only in specific Azure regions, as described in the Microsoft documentation on deploying PremiumV2_LRS.
  • Azure CCES supports disk upgrades only on Rubrik CDM version 9.0.3-p4 or later and on version 9.1.1-p1 or later. The cluster must be running on a supported version prior to the disk upgrade.
  • PremiumV2_LRS supports only zonal virtual machines. When upgrading the cluster, new nodes are placed in the same zone as in the original cluster. When the cluster lacks an availability zone, new nodes default to availability zone 1.
  • PremiumV2_LRS is supported only for clusters with more than 3 nodes. Single node clusters are not supported.
