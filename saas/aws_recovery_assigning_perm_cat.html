Assigning a permission category for AWS recovery

When you are performing recovery operations such as exporting, restoring, or recovering files and the target AWS account does not have the required permission category, you must assign the permission category to ensure recovery.

About this task

In the following scenarios, the relevant recovery wizard displays a warning message if the target AWS account lacks the necessary permission categories.
  • Exporting or Restoring: When exporting a snapshot to a target AWS account or restoring an EC2 instance to the source account, RSC checks if the target account has the Export or Restore permission categories.
  • Recovering files: When downloading files or folders using the Download option under Recover files, RSC checks if the target AWS account has the Recover Files permission category.

Procedure

  1. (Export) In the EC2 Instance Recovery wizard, on the Virtual machine details page, verify the target AWS account and click Assign Permissions.
  2. (Restore) In the EC2 Instance Recovery wizard, click Assign Permissions.
  3. (Recover files) In the EC2 Instance Recovery wizard, on the Select Folder or File Recovery Method page, click Assign Permissions.
    The Assign Permissions page appears, showing the status for the target AWS account.
  4. Select the method to update the target account.
    Method Description
    Authorize using CloudFormation (CFT) RSC generates an updated CFT containing the elevated permissions. Use this method to perform a stack update in the target AWS account console.

    This is recommended for accounts onboarded using the Add using CFT method.
    Authorize manually RSC provides the required JSON policy statements. Use this method to manually update the IAM role policy in the target AWS account.

    This is recommended for accounts onboarded using the Add manually method.
  5. Follow the on-screen instructions to update the IAM role in your AWS account.
    After the AWS resources are updated, return to the Assign Permissions page in RSC.
  6. Click Submit.

Results

RSC validates that the required permission category is available on the IAM role associated with the target AWS account.

What to do next

You can now perform the recovery operation using RSC. After the recovery operation completes, the elevated permissions do not expire automatically. To restore the least privilege model, an administrator must manually revoke the permissions:
  • CFT method: Use the Manage Granular Permissions option within RSC to clear the recovery category and run the updated CFT stack update in the AWS Management Console.
  • Manual method: Use the Manage Granular Permissions option within RSC to clear the recovery category and manually remove the corresponding JSON policy statements from the IAM role in the AWS environment.
