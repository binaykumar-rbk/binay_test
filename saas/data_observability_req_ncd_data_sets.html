NAS CD Data Observability requirements and limitations

NAS CD data sets have specific requirements for enabling backups and configuring Data Observability.

Requirements

Enabling Data Observability for NAS CD data sets requires fulfilling the following tasks:

  • For backup to the Rubrik Cloud Vault (RCV) Backup tier, creating a NAS CD policy with a retention period that equals or exceeds 30 days.
  • For backup to the RCV Archive tier, creating a NAS CD policy with a retention period that equals or exceeds 180 days.
  • Applying the Data Security Posture policy before NAS CD takes the first full backup as a recommended practice.
    Important:
    After RSC takes the first backup, you have a grace period of 14 days to apply the Data Security Posture policy. If you do not apply the Data Security Posture policy within the grace period, RSC does not scan the backup data.
  • Enabling the Anomaly Detection feature before RSC sends the first full backup to RCV to establish a baseline for Anomaly Detection when using RCV.

    When using a custom storage provider, enabling the Anomaly Detection and Threat Monitoring features before the first full backup completes is not required. In this case, RSC does not scan preexisting backups for Anomaly Detection. However, Threat Monitoring runs a full backup scan of the preexisting snapshots when you enable Threat Monitoring on a NAS CD cluster.

Limitations

Limitations of enabling Data Observability for NAS CD data sets include the following:

  • Restricted threat analytics scope: Only Anomaly Detection, Threat Monitoring, and Turbo Threat Hunt in Data Threat Analytics are available for NAS CD protected data sets. Advanced Threat Hunt and Quarantine capabilities are not available.
    Note:
    Threat Monitoring scans executable files that are less than 15 MB in size within snapshots.
  • Specific workflow for custom analyzers: When using custom analyzers for a NAS CD data set, RSC requires that you add all the required custom analyzers to a Data Security Posture policy before assigning the policy to your NAS CD object for the first time. To add a new custom analyzer later, you are required to delete the snapshot and create a backup to apply the new custom analyzer.
  • Impact of editing custom analyzers: If you edit a custom analyzer after assigning it to a NAS CD object through a policy, RSC scans only the newly backed up data using the edited values for the custom analyzer. The On Demand page in Data Security Posture retains all the classification results generated using the old custom analyzer for the previously backed-up data.
