Using RSC API proxy for NAS CD REST API

Use the RSC API proxy to send NAS CD REST API for managing your NAS CD cluster from an RSC instance.

Before you begin

Ensure the following before you use RSC API proxy to manage your NAS CD clusters:
  • Verify from the RSC Cluster page that the NAS CD cluster is connected with the RSC instance you are using.
  • Add a service account with VIEW_CLUSTER and MANAGE_DATA_SOURCE permissions or use an administrator account to operate the RSC API proxy.
  • Obtain the client ID and client secret for the service account by copying the account details or downloading the JSON file after adding the service account.
  • Contact Rubrik Support to enable the RSC API proxy.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click API Playground.
    The API Playground page appears.
  4. Click Open API Playground.
    API Playground opens in a new tab.
  5. In the query editor, enter the following GraphQL API command to retrieve the NAS CD cluster ID:allClusterConnection
    Use the RSC GraphQL API with productType set to "CLOUD_DIRECT".
    The GraphQL API lists the managed NAS CD clusters with their cluster IDs.
  6. Retrieve the JSON Web Token (JWT) for authenticating the session requests.
    At the prompt, enter:
    curl -X POST --header "Content-Type: application/json" rsc_domain_url/api/ncd-compatibility/service_account/session -d '{"client_id": "service account client ID", "client_secret": "service account client secret", "cluster_id": "cluster ID"}'
    Where:
    • client_id and client_secret refer to the client ID and client secret respectively for the service account you added.
    • The cluster_id refers to the NAS CD cluster UIID.

    If the JWT for the session expires, run the command again to retrieve another token.

    The JWT for the session appears in the following format:
    {"id":"CloudDirectSessionId_98qq2312-50a9-000a-b89d-3a13fa773642","token":"eyJhbGc ... rav-A","expiration":86400}
  7. Run the NAS CD REST API command with the bearer token authorization header to issue a request.
    At the prompt, enter:
    https://rsc_account_endpoint/api/ncd-compatibility/cluster_uuid/actual_REST_path
    Where:
    • cluster_uuid is the NAS CD cluster ID.
    • actual_REST_path is the NAS CD REST API you want to use.

    Use the retrieved JWT in the authorization header for authenticating the session.

    For example: 
    curl 'https://example.com/api/ncd-compatibility/33qq4563-1234-1234-abcd-5s4ds5f5f5ds/x/igneous/v2/cloud-buckets/usage?system=00.0.00.0&export=/ifs/data/abce' -H 'Authorization: Bearer eyJhbGc ... rav-A"
    The Rubrik NAS Cloud Direct API Guide provides more information about NAS CD REST APIs for performing various API requests.

Results

RSC processes the NAS CD REST API request and returns the requested data from the NAS CD cluster.

The following is a sample response for a NAS CD API request using the RSC API proxy:

[{"Protocol":"nfs","SourceSize":0,"RemoteSizes":[{"Location":"s3","CloudBucket":"sri-test-buck","Usage":0}]}]
