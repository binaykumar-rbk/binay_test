Prerequisites for Data Security Posture ServiceNow integration

The Data Security Posture ServiceNow integration from RSC has specific prerequisites.

Specific prerequisites for the ServiceNow integration in RSC are:
  • Enabling the ServiceNow feature by contacting Rubrik Support
  • Enabling the Rubrik Enterprise Proactive Edition license
  • Logging in with an owner or administrator account

Enabling Data Security Posture ServiceNow integration in RSC requires the following actions in ServiceNow:

  • Creating a local user account on ServiceNow
  • Creating a username and password that are local on the instance

    ServiceNow web services cannot authenticate against an LDAP or SSO identity provider. This is unlike the authentication flow that ServiceNow supports for typical administrative users who access the service using a web browser.

  • Assigning the sn_incident_write role to the user account

    This role grants web service access and permissions for creating, reading, and writing tickets.

  • Configuring an OAuth token

    This ensures that the OAuth client ID and client secret required for configuring the ServiceNow instance are in RSC.

  • Selecting the Short Description and Description fields in the ServiceNow incident form to appear in the ServiceNow Incident view

    RSC uses these fields to provide details regarding an incident. The settings for the Short Description and Description fields are accessible in the Selected tab in the Form layout option. The form layout option is accessible in Additional actions for the form. If the fields do not appear after being enabled on the Selected tab, you can enable them from the Personalize Form option.

The ServiceNow documentation provides more information about completing the prerequisites for ServiceNow integration.
