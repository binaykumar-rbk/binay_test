Performing a failback for RCV archival location

To restore your Rubrik Cloud Vault to its original configuration, perform a failback from the secondary region to the primary region for archival locations with Multi-region redundancy. This step re-establishes the primary region as the active one, ensuring that normal data operations resume after a failover.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Infrastructure > Rubrik Cloud Vault.
    The Rubrik Cloud Vault dashboard appears.
  4. Click RCV Locations.
    A list of RCV locations appears.
  5. Click on the Rubrik Cloud Vault (RCV) with Multi-region redundancy and select Prepare for Failback.
    The Ready for failback text appears next to the name of the RCV.
    The Failback wizard appears.
  6. Enter FAILBACK in the description box.

    FAILBACK must be in uppercase.

    Ensure you read the failover process implications before proceeding.

  7. Click Failback.
    If quorum authorization is configured for RCV, RSC initiates an approval.
    The Request Quorum Authorization dialog box appears.
  8. Click Submit Request.
    The RCV fails back to the primary region only when the quorum authorization request is approved.
    A message appears confirming that the failback to the primary region has been initiated for the RCV.

Results

RSC prepares the RCV for the failback.

You can monitor the progress of the failback task on the Rubrik Cloud Vaults page. Additionally, under , in the Notifications tab, you can view the status of the failback process.
