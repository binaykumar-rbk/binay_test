          Using RSC, you can take storage snapshots of SAP HANA systems running on an Azure virtual machine and use those snapshots to recover your SAP HANA databases.         • Legal hold for snapshot retention

          Legal hold ensures that snapshots are retained indefinitely until the hold is removed, overriding the scheduled expiry of an SLA Domain policy. This ensures that snapshots are available as long as necessary to meet legal obligations.

        • Manage retention of SAP HANA snapshots

          To meet data governance and regulatory compliance, you can increase the retention period of individual SAP HANA snapshots, as required.

        • SAP HANA replication

        • SAP HANA recovery

Configuring post-failover backup behavior

Define how RSC handles the first backup after an SAP HANA failover occurs at the system level.

Before you begin

Ensure that the following prerequisites are met:
  • Log in to RSC with an account that has data protection permissions.
  • Verify that the SAP HANA systems are connected to a Rubrik cluster running version 9.5 or later.
  • Verify that all selected systems belong to the same Rubrik cluster if you are performing a bulk configuration.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click on the SAP HANA inventory card.
    The SAP HANA page appears.
  5. Select Systems.
  6. Perform one of the following to configure the failover backup:
    • To configure a single system, locate the target SAP HANA system, click , and select Configure Post-Failover Backup.
    • To configure multiple systems, select the checkboxes for the SAP HANA systems you want to configure, click , and select Configure Post-Failover Backup.
      Note:
      You can select multiple systems in bulk only if they are connected to the same Rubrik cluster. The Configure Post-Failover Backup option is unavailable if you select systems from different Rubrik clusters.
    The Configure Post-Failover Backups screen appears.
  7. Select a backup behavior:
    • Trigger backup as per SLA Domain: This is the default behavior. Rubrik cluster triggers the first backup after failover based on the frequency assigned in the SLA Domain.
    • Trigger a full backup: Rubrik cluster automatically converts the next scheduled delta backup to a full backup upon detecting a failover.
  8. Click Configure.

Results

You have successfully configured the automatic full backup feature. The first backup triggered after a failover follows the selected behavior, ensuring backups remain consistent.

What to do next

You can verify the configuration status in the Post-Failover Backup Type column on the SAP HANA Systems page.
