SQL Server database requirements

Data Discovery and Classification for SQL Server databases has specific requirements, depending on authentication method.

Authenticate with SQL Server credentials

The following table lists requirements for authenticating with SQL Server credentials.
Requirement Description
Production database credentials You must provide the user account credentials for the production database through the Edit Host wizard in the SQL Server Host page in RSC to authenticate the user account to the host and perform the sampling. The user account requires read privileges to query the tables in the databases on the relevant host.

Rubrik and its users operate under a shared responsibility model, where each one has obligations and responsibilities related to security and the Rubrik Service. For security purposes, Rubrik recommends:

  • Using a specific set of credentials that are unique to this workload, storing the credentials in a secure location, and only providing the credentials to an authenticated and limited group of users.
  • Taking additional security measures such as frequently rotating the password and auditing credentials access according to the policies or best practices of your organization.
SQL Server authentication You must enable SQL Server authentication for the instances for which sensitive data classification needs to be done, in addition to Windows authentication.

The Change server authentication mode topic in the Microsoft SQL Server documentation provides more information about configuring SQL Server authentication.
TCP port 443 access You must enable port 1433 to accept connections so that RSC can connect remotely to the SQL Server instance using SQL server authentication using this port.

You can configure port 1433 as the network protocol in the SQL Server configuration manager. For Rubrik clusters running CDM 9.2.1, if there are multiple instances on the same host, only instances using port 1433 are supported for Data Discovery and Classification.
Network protocol You must configure the SQL Server instance to use the public key certificate of the Certificate Authority (CA) that provided the certificate for the SQL Server host. Rubrik uses the TLS network protocol to encrypt the communication between the Rubrik cluster and the SQL Server host. RSCverifies the identity of the SQL Server host using the certificate as input, along with the user account details provided on the host registration page.

The Certificate Management (SQL Server Configuration Manager) topic in the Microsoft SQL Server documentation provides more information about generating a certificate. To authenticate the SQL Server host, you must add the generated certificate to the Certificate Management page in RSC and then add the certificate through the Edit Host wizard in the SQL Server database page.
Policy assignment You must assign the required policies in Data Security Posture to enable Data Discovery and Classification for SQL Server databases.

Authenticate with RBA

For Rubrik clusters running CDM 9.4.1 and later, you can authenticate with an existing Rubrik Backup Agent (RBA) service account user.

The following table lists requirements for authenticating with RBA.
Requirement Description
SLA Domain Ensure that the database is actively backed up by an SLA Domain.
Policy assignment You must assign the required policies in Data Security Posture to enable Data Discovery and Classification for SQL Server databases.
RBA service account user You must configure the RBA service account user with sysadmin or db_datareader privileges for the databases.
