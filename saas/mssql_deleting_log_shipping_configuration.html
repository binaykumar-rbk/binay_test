          For a database that uses the full recovery model or the bulk-logged recovery model, the Rubrik cluster uses a combination of a snapshot of the database and the database transaction log backups to recover a database.

Deleting the log shipping configuration

Delete a log shipping configuration from the Rubrik cluster.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click the SQL Server Databases inventory card.
    The SQL Server Databases page appears.
  5. Click Log Shipping Targets.
    The list of log shipping configurations appears.
  6. Open the ellipsis menu for the log shipping configuration to delete and select Remove.
    The Remove secondary database wizard appears.
  7. Click Delete from Windows host to also remove the secondary database from the Windows host.
    If this option is not selected, the Rubrik cluster leaves the secondary database in place, but it removes the log shipping configuration.
  8. Click Remove.

Results

The Rubrik cluster deletes the log shipping configuration and the secondary database, if that check box was selected.
