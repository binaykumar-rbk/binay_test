      • Microsoft Entra ID

        RSC supports backup and recovery of Microsoft Entra ID.

        • Known limitations for Microsoft Entra ID on RSC

          Microsoft Entra ID on RSC includes some known limitations for recovering groups, users, relationships, and Microsoft Entra ID properties.

        • Non-recovery of soft-deleted objects by RSC

          RSC does not recover higher-privileged, soft-deleted Microsoft Entra ID user or group objects. Instead, RSC creates new objects, leaving the soft-deleted ones in the Microsoft Recycle Bin.

        • Adding Microsoft Entra ID with OAuth

          Add Microsoft Entra ID as a protected workload that uses OAuth for authentication.

        • Adding Microsoft Entra ID without OAuth

          Add a Microsoft Entra ID directory as a protected workload without OAuth, and instead use your existing Microsoft Entra ID application registration.

        • Protecting Microsoft Entra ID

          Choose a data protection policy for a Microsoft Entra ID directory.

        • Taking on-demand snapshots of Microsoft Entra ID

          Take on-demand snapshots of Microsoft Entra ID protectable objects in addition to the policy-based snapshots provided by the SLA Domain.

        • Deleting a Microsoft Entra ID snapshot

          Delete a Microsoft Entra ID snapshot that has the Forever retention policy assigned.

        • Viewing Microsoft Entra ID directories

          View details for the Microsoft Entra ID directories.

        • Microsoft Entra ID object and relationship recovery

          RSC recovers various Microsoft Entra ID objects and object relationships, including users, groups, roles, enterprise apps, and app registrations.

        • Microsoft Entra ID user object attribute recovery

          Attribute recovery for a Microsoft Entra ID user object selectively restores specific attributes that were maliciously or accidentally altered without performing a full object restore. You can compare user object attributes from a snapshot against live data and recover only the changed attributes.

        • Microsoft Entra ID object recovery across snapshots

          With RSC, you can search Microsoft Entra ID user, groups, and roles objects for recovery across different snapshots. This functionality enables recovery of objects and relationships from a specific date and time.

        • Microsoft Entra ID Conditional Access recovery

          For Microsoft Entra ID directories that support Conditional Access, RSC can protect Microsoft Entra ID Conditional Access objects and relationships.

        • Recovery in a hybrid identity environment

          In a hybrid Microsoft Entra ID and Active Directory environment on RSC, you can view Active Directory group and user assignments synced from the Active Directory domain. You can additionally recover relationships for synced Active Directory user and group objects and protect the Active Direct domain.

          • Recovering Microsoft Entra ID user objects from a hybrid environment

            Recover Microsoft Entra ID user objects and relationships and synced Active Directory user relationships in RSC.

          • Recovering Microsoft Entra ID user objects from a hybrid environment using manual passwords

            You can recover up to 50 Microsoft Entra ID user objects and relationships with manual passwords, as well as recover synced Active Directory user relationships.

          • Recovering Microsoft Entra ID group objects from a hybrid environment

            Recover Microsoft Entra ID group object properties and relationships, and synced Active Directory group objects relationships through RSC.

        • Microsoft Entra ID full domain recovery

          Using Microsoft Entra ID full domain recovery, administrators can restore all objects and relationships in a domain from a previous state. You can specify objects to exclude from recovery.

        • Downloading Microsoft Entra ID recovery details

          Download a CSV file containing information about a completed Microsoft Entra ID snapshot recovery. The information includes details about object IDs, object display names, object types, and relationship recovery.

        • Editing Microsoft Entra ID directories with OAuth

          Edit the login accounts that access Microsoft Entra ID as a protected workload that uses OAuth.

        • Editing Microsoft Entra ID directories without OAuth

          Edit login accounts that access Microsoft Entra ID as a protected workload that does not use OAuth.

        • Deleting Microsoft Entra ID domains from directories

          Delete a Microsoft Entra ID domain from a directory when the domain no longer requires protection.

Recovering Microsoft Entra ID user objects from a hybrid environment

Recover Microsoft Entra ID user objects and relationships and synced Active Directory user relationships in RSC.

About this task

RSC limits recovery of users synced from an Active Directory (AD) domain to relationship recovery only. To recover all user properties in RSC, first recover the users in AD, then use Microsoft Entra Connect to sync the recovered objects to Microsoft Entra ID. The Microsoft Entra documentation describes how to sync objects and credentials from an on-premises AD domain to Microsoft Entra ID.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
  3. On the navigation bar, click Inventory.
  4. Click the Microsoft Entra ID inventory card.
    The inventory list for Microsoft Entra ID appears.
  5. In the Domain column, select the name of a Microsoft Entra ID domain.

    Alternatively, enter a name in the search field or use the filters at the top of the list.

    Domain entries with a yellow dot indicate that AD domains are synced to that Microsoft Entra ID domain.

    The local page for the domain appears, with the Snapshots card displaying the month view.
  6. On the Snapshots card, select a day that has a green dot.

    The green dot indicates that RSC successfully took a snapshot on that day.

    When at least one synced on-premises AD domain is not protected by RSC, a dialog box appears. Clicking the Go to Active Directory option directs you to the Active Directory domains page, where you can perform AD domain protection operations.

    The Snapshots card displays the Day view with a list of snapshots.
  7. Hover over a snapshot and click Recover.
    The Select Recovery Type wizard appears.
  8. Select Granular Recovery and click Next.
    The Select Objects to Recover page appears.
  9. Click Users.
    The list of users available at the time the snapshot was taken appears.
  10. Select the user to recover.
    RSC recovers the selected user from the time the snapshot was taken. You can filter the results to select from All Users, Entra ID users, or On-Premises Synced users.
  11. Click Next.
    The Recovery Options page appears.
  12. Select Recover to source directory.
    RSC only supports recovering to the source directory for In-Place Recovery.
  13. Select a relationship recovery mode.
    ModeDescription
    Roll back relationships Recovers all selected objects and object relationships to their snapshot state.
    Update relationships Recovers all selected objects and object relationships in the snapshot, while preserving the relationships of the other objects in the current directory.
  14. Click Next.
    The User Recovery page appears.
  15. For recovering Microsoft Entra ID users only, in Temporary Password, select Automatically generate.

    RSC automatically generates a random and complex password for Microsoft Entra ID user accounts only. Depending on the Microsoft Entra ID directory settings, RSC prompts Microsoft Entra ID user accounts to reset the automatically generated password when logging into Microsoft Entra ID after recovery. RSC does not record the login operation and does not save the password.

    New passwords are applicable only to Microsoft Entra ID user accounts that exist exclusively within the snapshot or are located in the recycle bin at the time of recovery.

  16. For recovering Microsoft Entra ID users only, select an option for multi-factor authentication.

    This option manages enabling multi-factor authentication (MFA) for the selected Microsoft Entra ID user accounts. When you do not enable MFA , a warning message appears. RSC requires you to acknowledge the warning message to continue.

    The MFA option applies only for Microsoft Entra ID user accounts that are present in the snapshot or are in the Recycle Bin at the time of recovery.

  17. Click Next.
    The Recovery Confirmation page appears.
  18. Accept the agreement under Included Objects.
  19. Click Recover.

Results

RSC recovers the user objects to a specific point in time. User objects recovered from Microsoft Entra ID include property and relationship recovery. User objects synced from an AD domain include relationship recovery only.
