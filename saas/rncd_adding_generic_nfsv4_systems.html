Adding Generic NFSv4 systems

Add a Generic NFSv4 system as a data source using RSC. After adding the system, you can discover and protect the data in RSC.

Before you begin

Ensure that you obtain the IP address or the hostname of the Generic NFSv4 storage account that you want to add.

About this task

RSC has the following limitations for Generic NFSv4 data sets:
  • RSC reads sparse files as normal files.
  • RSC does not support named attributes.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Data Sources and select NAS Cloud Direct Systems.
    The NAS Cloud Direct Systems page appears.
  4. Click Add System.
    The Add a System dialog box appears.
  5. From the NAS Cloud Direct Site dropdown list, select the Rubrik cluster for adding the system.
  6. From the System Type dropdown list, select Generic NFSv4.
  7. In IP/Hostname, enter the FQDN (recommended) or IP address of the Generic NFSv4 management interface.
    The IP address is the management endpoint IP address for the file system that you want to add from your Generic NFSv4 console.
  8. Click ADD.

Results

RSC initiates the task to add the Generic NFSv4 system and displays a message indicating that the process has started. RSC imports the system asynchronously in the background. You can monitor the progress and verify the successful addition of the system on the Event Logs page.
