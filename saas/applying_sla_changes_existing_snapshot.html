Applying SLA Domain changes to existing snapshots

Apply the changes made to an SLA Domain to existing snapshots, including on-demand and downloaded snapshots.

About this task

This topic describes a feature or features that Rubrik is gradually rolling out to customers.

When you edit an SLA Domain, the changes you make do not automatically apply to snapshots that were previously created under that SLA Domain. However, if your data protection strategy requires it, you can apply these SLA Domain changes to the existing snapshots.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click SLA Domains.
    The SLA Domains page appears.
  4. Click the SLA Domain that you want to modify.
    The SLA Domains Details page appears.
  5. Click Edit.
    The Edit SLA Domain wizard appears.
  6. Modify the SLA Domain as required, then navigate to the Confirmation page.
  7. Turn on Apply changes to existing snapshots.
    Note:
    • For SLA Domains protecting data center workloads, RSC applies these changes to snapshots stored in primary and archival locations.
    • For SLA Domains protecting cloud-native workloads, RSC applies these changes to snapshots stored in primary, archival, and replicated locations.
    RSC applies the SLA Domain changes to all snapshots previously generated using this SLA Domain.
  8. Optional: Select Include on-demand and downloaded snapshots.
    If the SLA Domain is exclusively for cloud-native workloads, the option will be labeled Include on-demand snapshots, as downloaded snapshots are not applicable in this context.
    RSC applies the SLA Domain changes to the on-demand and downloaded snapshots.
  9. Click Save.

Results

RSC applies the changes to existing snapshots based on your selection, including on-demand and downloaded snapshots.
