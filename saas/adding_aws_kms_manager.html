Adding an AWS KMS key manager

To manage encryption for archives stored at AWS archival locations, add an AWS KMS key support to securely manage and store encryption keys, ensuring secure access and management of sensitive data.

Before you begin

Create a new IAM role and corresponding IAM user in your Amazon Web Services (AWS) account with the required permissions to access the keys you want to use in Rubrik for data encryption.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Security and select Encryption Management.
    The Encryption Management page appears.
  4. Click KMS Settings.
    A list of the KMS instances available to the Rubrik cluster appears.
  5. Click Add KMS.
  6. In Select Key Management Service Type, select AWS KMS.
    The Add Key Manager page appears.
  7. In IAM Role ARN, enter the role Amazon Resource Name (ARN).

    This IAM role must have access to the encryption keys you intend to use for data encryption using this AWS KMS key manager.

  8. Optional: In External ID, enter the external identifier of the IAM user.
  9. In IAM User Access Key, enter the unique access key ID of the IAM user.
  10. In IAM User Secret Key, enter the secret key associated with the IAM user for authentication.
  11. In Region, specify the region where you want the Rubrik cluster performing data backup to connect to the AWS KMS service to retrieve the encryption keys.
  12. Optional: Specify STS and KMS endpoint overrides.

    To use private connectivity between your Rubrik cluster performing data backup and AWS KMS service, endpoints must be in the same region as the Rubrik cluster.

  13. Click Next.
  14. In Key Manager Name, enter a unique name that helps identify the AWS KMS key manager.
  15. Optional: In Description, enter a brief description to provide additional context or details about the AWS KMS key manager.
  16. Click Add.
    A message appears indicating that the AWS KMS has been added.

Results

RSC verifies the connectivity to the AWS KMS with the provided credentials, and if the verification is successful, it adds the AWS KMS key manager.

The new AWS KMS appears on the KMS Settings tab with specific details. You can expand the key manager to view the storage locations, key ID, and version information of the encryption keys accessible using the AWS KMS key manager.

Related concepts
  • Encryption management
    With the encryption management feature in RSC, you can view information about data-at-rest encryption for the Rubrik clusters and archives, access to your Key Management Services, and enhance data security through automated key rotation every 30 days.
