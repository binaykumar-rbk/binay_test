Recovering a batch of AHV virtual machines using Live Mounts

Create live mounts for recovering a batch of AHV virtual machines in a single operation using Anomaly Detection.

Procedure

  1. Log in to RSC.
  2. Click and select Data Threat Analytics.
    The Data Threat Analytics dashboard appears.
  3. On the navigation bar, click Anomaly Detection.
    The Anomaly Detection page appears.
  4. Select the AHV virtual machines to recover.
    • The selected machines must belong to a single cluster for batch recovery.
    • Batch recovery is not supported when you select virtual machines along with Recovery Plans.
    • You can select a maximum of 100 virtual machines at once.
  5. Click Recover.
    The Virtual Machine Recovery wizard opens and displays the Select a recovery approach page.

    If you selected only a single virtual machine for recovery, the Anomaly Recovery wizard opens and displays the Select a recovery approach page.

  6. Enter a date and time.
    The Restore to closest snapshot in time option is selected by default and uses the closest snapshot before or after a specified date and time. This option will apply to all the selected virtual machines.
    When you specify a Before date but do not specify the time, the closest snapshot used is the snapshot taken before 00:00:00 on the specified date. When you specify an After date but do not specify the time, the closest snapshot used is the snapshot taken after 23:59:59 on the specified date.
  7. Click Next.
    The wizard advances to the Confirm snapshot selection page.
  8. To change the snapshot of a virtual machine, click on the respective row.
    A graph appears on this page and displays the suspicious, added, modified, and deleted files.
  9. Click Change Selected Snapshot.
    The Select a snapshot dialog box appears.
  10. Select a snapshot to recover and click Close.
  11. Click Next.
    The wizard advances to the Select a recovery option page.
  12. Select a data migration method for the Live Mount recovery operation.
    OptionDescription
    Live Mount with Optional Migration Involves immediate or delayed migration of the data source.
    Live Mount without Migration Eliminates the ability to migrate the Live Mounted virtual machine to a Nutanix AOS cluster after the recovery.
  13. Click Next.
    The wizard advances to the Select cluster page.
  14. Select a Nutanix cluster to Live Mount the virtual machine.
  15. Click the chevron for the Advanced mount options.
    A few additional fields appear.
  16. In VM Name Prefix, enter a prefix for the live mounted virtual machines.
  17. Select a configuration you want for the live mounted virtual machines.
    OptionDescription
    Remove all network devices Removes all network devices when networking changes or issues prevent the virtual machines from starting.
    Power on mounted virtual machine Starts the virtual machines when the mount task completes.
    You can select multiple options.
  18. (Live Mount with Optional Migration) Click Next.
    The wizard advances to the Select Container page.
  19. (Live Mount with Optional Migration) From the list, select a Nutanix storage container.
  20. (Live Mount with Optional Migration) To automatically initiate the data source migration immediately after the mounting operation is complete, select the Migrate immediately check box.
    You can clear the check box to manually initiate the migration of the data source at a later time from the Live Mounts page.
  21. Click Mount.

Results

Anomaly Detection mounts the selected virtual machines on the selected compute cluster.
