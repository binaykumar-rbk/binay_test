Supported keys and permissions in AWS KMS

Permission requirements to add AWS KMS keys for data encryption on locations created using Rubrik CDM cluster 9.3.3 and later.

RSC supports customer-managed AWS KMS keys. This is applicable to AWS KMS keys for data encryption on locations created using CDM version 9.3.3 and later.
Important:
AWS KMS supports only symmetric encryption keys with the usage of the key set for encryption or decryption.

When using AWS KMS to encrypt data, Rubrik requires specific permissions. These permissions must be granted to the IAM role that is specified in the AWS KMS key manager for accessing the required keys for data encryption. The following table lists the permissions required.

Permission type Permission Description
Actions View Key Properties To view the properties of a specific KMS key.
List Keys To list and retrieve details about KMS keys and their aliases.
DataActions Encrypt with Key To encrypt data using the specified key.
Decrypt with Key To decrypt data with the specified key.
Re-encrypt with Key To re-encrypt data with a different key.
Generate Data Keys To generate data encryption keys from the Customer Master Key (CMK) in the AWS KMS console.
