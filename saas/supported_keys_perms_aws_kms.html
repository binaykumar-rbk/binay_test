  • Security

    Configure and manage RSC security features, including user access, SSO, Quorum Authorization, and data encryption.

    • Users and Access

      RSC provides role-based access control, and several methods for authenticating a user account.

    • Quorum authorization

      Quorum authorization is a security feature that requires additional approvals before a requester can perform data-modifying actions in RSC.

    • Single sign-on

      Rubrik Security Cloud supports SSO using the SAML 2.0 standard. With SSO, you can log in to RSC using credentials associated with an identity provider.

    • Security settings

      Rubrik Security Cloud provides enhanced security authentication services and features.

      • IP allowlist

        RSC administrators can specify IPv4 addresses for the IP allowlist. RSC blocks unauthorized login attempts and unauthenticated API calls that originate from addresses that are not on the allowlist.

      • Authentication

        Rubrik Security Cloud authentication restricts access to a specified set of users.

      • Certificate management

        RSC Certificate Management provides a centralized dashboard to manage certificates across all connected Rubrik clusters and for RSC workflows.

      • Encryption management

        With the encryption management feature in RSC, you can view information about data-at-rest encryption for the Rubrik clusters and archives, access to your Key Management Services, and enhance data security through automated key rotation every 30 days.

        • Data-at-rest encryption

          Depending on the specific model of the Rubrik Brik, Rubrik clusters encrypt data at rest using software or hardware encryption.

        • Key management for data-at-rest encryption

          With advanced key management, including key rotation, rekeying, and KMS integration, Rubrik enhances data-at-rest encryption by providing secure and flexible management of encryption keys to ensure ongoing protection of stored data.

        • Encryption management details

          The Encryption Management page provides centralized access to encryption settings, Key Management Systems, and encryption status, enabling effective management and monitoring of encryption.

        • Key Management Services

          Rubrik supports Key Management Services integration so that you can use your own encryption keys for data at rest. This Bring Your Own Key approach ensures Rubrik never accesses your master keys.

          • Adding an AWS KMS key manager

            To manage encryption for archives stored at AWS archival locations, add an AWS KMS key support to securely manage and store encryption keys, ensuring secure access and management of sensitive data.

            • Supported keys and permissions in AWS KMS

              Permission requirements to add AWS KMS keys for data encryption on locations created using Rubrik CDM cluster 9.3.3 and later.

          • Adding Azure Key Vaults

            Add an Azure Key Vault to manage encryption for Azure based archives or Rubrik Cloud Vault. The key vault securely stores your encryption keys and controls access to sensitive data.

          • Adding KMIP servers

            To manage your Key Encryption Key and self-encrypting drive passwords on a Rubrik cluster using a KMIP-compliant external key manager, you must add the details of the KMIP server in RSC.

          • Editing KMS

            Edit the KMS credentials to update configuration details for an existing KMS instance as per the KMS type.

          • Deleting KMS

            Delete an AWS KMS key manager, Azure Key Vault, or KMIP server when it is no longer needed and no particular KMS is currently encrypting any data locations.

        • Performing a rekey

          Perform a rekey of the master key and/or root key to update your encryption key to an alternative encryption key. This ensures that your data location uses up-to-date encryption.

        • Updating the master key for a reader location

          Update the master key to ensure reader locations can continue accessing archived data after a rekey operation on the owner location.

        • Rotating encryption keys

          To provide enhanced encryption key security, you must rotate the encryption key or switch to another root of trust for your encryption key. This functionality does not apply to encryption keys for data center archival locations.

      • User account lockout

        The user account lockout feature enhances system security by providing global administrators the ability to lock and unlock user accounts. This feature also prevents unauthorized access from brute force attacks and safeguards local user accounts by automatically locking them after a configurable period of inactivity.

      • Session management

        The session management feature enhances system security in the following ways: ending inactive user sessions, ending active user sessions after a specific duration, and limiting the number of concurrent sessions allowed for each user.

      • Password policy

        RSC administrators can enhance the security of their RSC domain by using the password policy feature to configure password complexity rules for local users.

      • New device login notification

        RSC notifies you when a new device is used to log in to your RSC account, providing details such as device name, IP address, and location.

Supported keys and permissions in AWS KMS

Permission requirements to add AWS KMS keys for data encryption on locations created using Rubrik CDM cluster 9.3.3 and later.

RSC supports customer-managed AWS KMS keys. This is applicable to AWS KMS keys for data encryption on locations created using CDM version 9.3.3 and later.
Important:
AWS KMS supports only symmetric encryption keys with the usage of the key set for encryption or decryption.

When using AWS KMS to encrypt data, Rubrik requires specific permissions. These permissions must be granted to the IAM role that is specified in the AWS KMS key manager for accessing the required keys for data encryption. The following table lists the permissions required.

Permission type Permission Description
Actions View Key Properties To view the properties of a specific KMS key.
List Keys To list and retrieve details about KMS keys and their aliases.
DataActions Encrypt with Key To encrypt data using the specified key.
Decrypt with Key To decrypt data with the specified key.
Re-encrypt with Key To re-encrypt data with a different key.
Generate Data Keys To generate data encryption keys from the Customer Master Key (CMK) in the AWS KMS console.
