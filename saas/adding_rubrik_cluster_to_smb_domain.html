Adding a Rubrik cluster to an SMB Domain

To enforce user authentication through Active Directory, add a Rubrik cluster that hosts SMB shares to an SMB domain.

Before you begin

You must have the following information:
  • Fully qualified domain name of the Active Directory (AD) server
  • Username and password of a user with domain access privileges
  • List of fully qualified hostnames of domain controllers
  • AD service account name of the Rubrik cluster
  • AD organization unit

About this task

A Rubrik cluster securely protects data on Server Message Block (SMB) shares by requiring that SMB clients authenticate through Active Directory (AD) before gaining access to SMB shares. To begin enforcing the requirement of AD authentication for access to SMB shares, a Rubrik cluster must be included in the AD domain using a computer account.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Data Sources and select Access Credentials.
    The Access Credentials page appears.
  4. Click SMB Security.
    The SMB Security tab appears.

    The Authentication Status column displays whether an SMB domain is configured for a Rubrik cluster.

  5. Select a method to configure SMB security for a Rubrik cluster.
    • Open the ellipsis menu for a Rubrik cluster with the authentication status as Not Configured and select Authenticate.
    • Click the plus icon.
    The Join the Cluster to the SMB Domain dialog box appears.
  6. Add the SMB domain information.
  7. To complete the configuration, click Add.

Results

The selected Rubrik cluster is added to the SMB domain and enforces the requirement of AD authentication for access to SMB shares.
