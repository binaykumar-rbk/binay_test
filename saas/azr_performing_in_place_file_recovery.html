    • Cloud Workloads
      • Amazon Web Services workloads

        RSC protects the following types of Amazon Web Services workloads: EC2 instances, EBS volumes, Amazon S3, Amazon RDS and Aurora databases, and Amazon DynamoDB.

      • Google Cloud Platform workloads

      • Microsoft Azure workloads

        RSC protects the data and resources of Azure virtual machines, Azure managed disks, Azure Blob Storage, and Azure SQL workloads in Azure subscriptions.

        • Introduction to Azure protection
        • Azure protection implementation workflow

          The Azure protection implementation workflow involves a structured approach to secure data management, starting with planning and preparation, followed by cloud account setup, infrastructure configuration, workload protection, continuous management and optimization, and workload recovery.

        • Quick links for Azure workloads

          Immediate access to important tasks and information, enabling quick setup, daily operations, and critical recovery.

        • Plan your Azure protection

          Before setting up an Azure cloud account in RSC, you are required to plan the user and access management for designated RSC users. This planning also involves reviewing the permissions that RSC requires in your Azure environment to protect your Azure workloads.

        • Set up Azure cloud accounts

          You can set up an Azure cloud account in RSC to start protecting Azure workloads.

        • Set up Exocompute on Azure

          With the Exocompute on Azure feature, RSC launches an Azure Kubernetes Service cluster to perform various operations to protect Azure workloads. RSC uses the Azure Kubernetes Service cluster to serve as a compute resource in your Azure subscription.

        • Set up Azure storage tiering

          You can use Azure storage tiering to archive Azure virtual machine or managed disk snapshots, which are older than a specified threshold, to Azure storage access tiers. By archiving snapshots to these access tiers, you can optimize cost.

        • Set up Azure workloads protection

          You can set up SLA Domains to apply data protection policies, ensuring automatic and hierarchical protection of Azure subscriptions, resource groups, and individual Azure workloads, with granular control overriding higher-level assignments.

        • Manage Azure virtual machines

          After completing the protection setup for Azure virtual machines and managed disks, RSC takes automated snapshots of those virtual machines and disks based on the configured policies in SLA Domains. You can also take on-demand snapshots, refresh Azure subscriptions, and add Azure custom tags for better resource management.

        • Manage Azure Blob Storage

          By using the protection services offered by RSC, you can protect and recover your objects in Azure Blob Storage.

        • Manage Azure SQL Database

          RSC enhances Azure native backup capabilities for SQL Databases by offering advanced, Rubrik-managed protection. This solution overcomes native limitations with customizable SLA Domain policies, on-demand snapshots, and flexible cross-subscription recovery, ensuring a consistent and automated approach to data security.

        • Manage Azure SQL Managed Instance

          RSC enhances Azure native backup capabilities for SQL Managed Instances by offering advanced, Rubrik-managed protection. This solution overcomes native limitations with customizable SLA Domain policies, on-demand snapshots, and flexible cross-subscription recovery, ensuring a consistent and automated approach to data security.

        • Recover Azure workloads

          After RSC takes snapshots of your Azure workloads, you can recover the required workloads.

          • Privilege elevation for Azure recovery

            RSC provides you an option to secure Azure cloud accounts by using a least permission model, granting only necessary read permissions at the subscription level. For recovery operations, you can temporarily elevate permissions either by authorizing a service principal role or by using a security token.

          • Azure virtual machine recovery

            You can recover Azure virtual machines and managed disks using the various recovery options that RSC provides.

            • File indexing for Azure virtual machines

              RSC uses Exocompute instances to extract file system index metadata from the managed disk snapshots created while protecting Azure virtual machines or managed disks.

            • Recovery options for Azure virtual machines

              A comparative overview of the various recovery options for Azure virtual machines and managed disks.

            • Recover Azure virtual machines

              You can recover your Azure virtual machines and managed disks using options such as recovering files or folders, exporting, restoring, mounting, and Azure in-place file recovery.

              • Recovering files or folders to an Azure storage account

                Recover files or folders from Azure virtual machines or managed disks to an Azure storage account if the files that you want to access are deleted, corrupted, or encrypted.

              • Cross-region export of ADE virtual machines

                Cross-region export of Azure Disk Encryption-enabled virtual machines involves securely migrating encryption keys and secrets to a new key vault in the target Azure region.

              • Exporting Azure virtual machine snapshots

                Export a snapshot of an Azure virtual machine to create a virtual machine from a point-in-time copy of the source virtual machine.

              • Exporting an Azure managed disk snapshot

                Export a snapshot of an Azure managed disk to create a new disk from a point-in-time copy of the source managed disk.

              • Exporting relic snapshots

                Export a snapshot of a relic Azure virtual machine.

              • Restoring Azure virtual machine using a snapshot

                Use an Azure virtual machine snapshot to replace the original virtual machine with a fully functional point-in-time copy.

              • Mounting Azure virtual disks for data recovery

                To recover critical data in a disaster or outage that can pose a significant business problem, mount the Azure virtual disks to the source virtual machine or a different virtual machine running in another region or subscription.

              • Viewing Live Mounts page for Azure virtual machines

                To review detailed information about the target virtual machines in Azure, such as the number of mounted virtual disks, the time when the disks were mounted, and the status of the mount task, open the Live Mounts page for Azure virtual machines. Additionally, you can unmount a disk from the target virtual machine.

              • Unmounting an Azure virtual disk from Azure virtual machine

                When you no longer need an Azure virtual disk attached to an Azure virtual machine, from the Live Mounts page, unmount the corresponding virtual disk to free up storage space.

              • Azure In-place File Recovery

                With the In-place File Recovery feature, you can recover Azure files from an indexed snapshot either to a source virtual machine or a different virtual machine or export the files to a different virtual machine.

Performing an Azure In-place File Recovery

To recover the files of an Azure virtual machine from an indexed snapshot to the source virtual machine or export the files to a different virtual machine, perform an In-place File Recovery.

Before you begin

Ensure that you fulfill the prerequisites, as described in Prerequisites for Azure In-place File Recovery.

About this task

If Rubrik Backup Service (RBS) is already installed on Windows and Linux machines, you must uninstall and then reinstall the RBS on these machines before performing In-place File Recovery. This is a one-time upgrade process.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click the Azure - VMs & Applications inventory card.
    The Azure Virtual Machines page appears.
  5. From the list, select a virtual machine.
    The virtual machine details page appears and displays all the snapshots of the selected virtual machine.
  6. In the calendar view, click the date of the on-demand snapshot.
    The status of the snapshot must be Indexed for performing an In-place file recovery. You can view the status under the Status column.
  7. Open the ellipsis menu for the snapshot you want to recover and select Recover.
    The Azure VM Recovery wizard appears.
  8. Select Recover files and click Next.
    The Select files page appears.
  9. Select the files and folders you want to recover.
  10. Click Next.
    The Select a recovery source page appears.
  11. Select the snapshot you want to use as the recovery source for the selected files and folders.
    A snapshot can be present on the source, archival, or replication target location. You can select one of these snapshots as the recovery source.
  12. Click Next.
    The Select recovery method page appears.
  13. Select a method to recover the selected files.
    MethodDescription
    Overwrite existing Overwrites the existing files and folders with the selected files and folders from the snapshot.
    Recover Recovers the selected files and folders to a different location in the source virtual machine.
    Export Exports the selected files and folders to a different virtual machine.
  14. (Recover) In Folder Path, enter the path of the folder within the source virtual machine to which you want to recover the selected files or folders.
  15. (Export) In the Virtual Machine drop-down list, select a different virtual machine to which you want to export the selected files and folders.
  16. (Export) In Folder Path, enter the path of the folder within the selected virtual machine to which you want to recover the selected files or folders.
  17. Click Recover.

Results

RSC recovers the selected files and folders to the location that you selected.
