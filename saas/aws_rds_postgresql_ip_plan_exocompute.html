IP address planning for Exocompute on RDS PostgreSQL

For the Exocompute on AWS feature that uses Elastic Kubernetes Service clusters as the underlying compute resources, depending on your Amazon RDS PostgreSQL databases, you need to plan the number of IP addresses required for your cluster nodes and use a subnet size accordingly.

Based on the number of Amazon RDS PostgreSQL database instances you want to protect, the Elastic Kubernetes Service (EKS) cluster requires additional IP address planning. A temporary PostgreSQL database instance is created from the manual snapshot as part of the archival process.

The EKS cluster is initially launched with one node, however, can be scaled up to 64 nodes depending on the workload.

Each node requires a minimum of 10 pods to perform backup and recovery operations for PostgreSQL database instances. Each node and pod requires a separate IP address.

To calculate the number of IP addresses required, add the number of nodes, number of PostgreSQL database instances to the output of the number of nodes and the number of pods. For example, if the number of nodes is 32 and the number of PostgreSQL database instances is 32, then add 32 to the output of 32 multiplied by 10 and 32 for the number of PostgreSQL database instances. Hence the number of IP addresses required will be 384.

The following table describes various scenarios and their IP address and subnet range requirements.

Number of disks IP address requirement Subnet range
Less than or between the range of 10 to 20 19 + (1910) + 19 = 228 /24
Less than or between the range of 30 to 40 39 + (3910) + 39 = 468 /23
Less than or between the range of 50 to 60 59 + (5910) + 59 = 708 /22
