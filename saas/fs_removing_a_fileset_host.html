Removing a fileset host

You can delete the host from the Rubrik cluster to stop protecting the fileset data on a host.

Before you begin

The Rubrik cluster must be in a connected state to delete a host from that Rubrik cluster.

About this task

Note:
Deleting hosts from a Rubrik cluster disables protection for all protected objects, such as filesets, Managed Volumes, SQL Server instances, and Hyper-V virtual machines associated with that host.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click a fileset card based on the OS type of the host.
    • Windows filesets
    • Linux & Unix filesets
    The corresponding filesets page appears.
  5. From the list of hosts, select a host to delete.
    Multiple hosts connected to the same Rubrik cluster may be selected.

    Filter the hosts by Rubrik cluster, fileset names, SLA Domains assigned to the filesets, RBS status, and protection status of the filesets. Search for a host by hostname or IP address.

  6. Open the top-level ellipsis menu and select Delete hosts.
    The Delete host dialog box appears.
  7. Click Delete.

Results

RSC removes the host from the Linux & Unix Hosts tab or the Windows Hosts tab, and converts all the existing snapshots for filesets associated with the host to relics.
