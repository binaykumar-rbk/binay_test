Immutability support for Cloud Cluster ES

Rubrik Security Cloud supports enabling immutability when provisioning an AWS or Azure Cloud Cluster Elastic Storage.

Rubrik Security Cloud supports enabling immutability when provisioning an AWS or Azure Cloud Cluster Elastic Storage (ES). When enabled, the Cloud Cluster ES stores the backed-up data in a Write Once Read Many (WORM) locked storage. This preserves the backups even if Rubrik clusters running CDM version 8.1 or later have been deleted. The immutable lock to the backed-up data applies for seven days after the deletion of the Rubrik cluster. You can recover the Rubrik cluster within the seven days.
