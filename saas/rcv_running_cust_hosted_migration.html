          • Prerequisites for migrating archival locations to RCV

            Before migrating your data center archival location to Rubrik Cloud Vault, ensure your source location, Rubrik cluster, and SLA Domain configurations meet all requirements.

          • RCV migration workflow

            Migration of archival locations to Rubrik Cloud Vault involves specific responsibilities for both the customer and Rubrik Professional Services, depending on whether the workflow is customer-hosted or Rubrik-hosted.

          • Customer-hosted migration process

            You can migrate an archival location to RCV by running the migration tool in your own AWS environment.

            • Preparing for customer-hosted migration

              Prepare the source location for migration by validating source location eligibility, disabling archival locations, and updating encryption keys.

            • Configuring customer-hosted migration

              Configure the security permissions required for AWS DataSync to move data, including fetching bucket names, creating IAM roles, and registering destination roles with RSC.

            • Running the customer-hosted migration task

              Create and start the AWS DataSync transfer task to migrate data from the source location to the destination RCV bucket.

            • Finalizing customer-hosted migration

              After the data transfer is complete, perform the final configuration and validation steps to finalize the transition to the RCV location.

          • Rubrik-hosted migration process

            You can migrate an archival location to RCV by authorizing Rubrik to run the migration tool within the Rubrik AWS environment.

        • Immutability conversion for Azure RCV locations

          Using RSC, you can convert non-immutable data center RCV locations on Azure to immutable locations.

        • RCV license replacement

          You can replace an existing Rubrik Cloud Vault license with a new license mid-term, ensuring backup and archiving continuity during the transition.

    • Cascading archival

      Using cascading archival, you can increase replicated data redundancy and decrease data storage costs by archiving replicated snapshots from a replication target Rubrik cluster to a remote archival location.

    • Replication

      The replication feature directs the Rubrik cluster to send replicas of source snapshots or backups to a target Rubrik cluster and defines the maximum time to keep the replica on each cluster.

Running the customer-hosted migration task

Create and start the AWS DataSync transfer task to migrate data from the source location to the destination RCV bucket.

Procedure

  1. In your AWS Console, navigate to AWS DataSync and create a task.
    Use the following naming convention for the task: migration-customer-<rsc account name>-<source-location-id>-<any random 2 digit integer>.
  2. Optional: Alternatively, create the DataSync task using the AWS CLI.
    Run the following command to create the task:
    aws datasync create-task \
  --source-location-arn "<source datasync location-arn>" \
  --destination-location-arn "<destination-datasync-location-arn>" \
  --name "<task-name>" \
  --options VerifyMode=ONLY_FILES_TRANSFERRED,OverwriteMode=ALWAYS,PreserveDevices=PRESERVE,PosixPermissions=PRESERVE,TaskQueueing=ENABLED,LogLevel=BASIC --region <region> \
// For Basic mode add cloud-watch-log-group-arn, this is not needed for enhanced mode 
--cloud-watch-log-group-arn 'arn:aws:logs:<region of rcv bucket>:<account>:log-group:/aws/datasync' \
// For S3 compatible locations, please remove the option PosixPermissions=PRESERVE
// Add the below this option when its S3->S3 transfer
--task-mode <task-mode> \ // example ENHANCED or BASIC

* sample region - us-east-2 <this is the same region in which you are migrating>

    For more information on creating a DataSync task, refer to the AWS documentation for creating a task for transferring your data.

  3. Start the AWS DataSync transfer task, as described in the AWS documentation for starting a task to transfer your data.
  4. Monitor the progress of the task.
  5. Inform the Rubrik Professional Services team after the transfer completes.

Results

After the transfer task is complete, the data resides in the target RCV location and is ready for final verification by the Rubrik Professional Services team.
