      RSC administrators can specify IPv4 addresses for the IP allowlist. RSC blocks unauthorized login attempts and unauthenticated API calls that originate from addresses that are not on the allowlist.

      With the encryption management feature in RSC, you can view information about data-at-rest encryption for the Rubrik clusters and archives, access to your key management services, and enhance data security through automated key rotation every 30 days.

    • User account lockout

      The user account lockout feature enhances system security by providing global administrators the ability to lock and unlock user accounts. This feature also prevents unauthorized access from brute force attacks and safeguards local user accounts by automatically locking them after a configurable period of inactivity.

      The session management feature enhances system security in the following ways: ending inactive user sessions, ending active user sessions after a specific duration, and limiting the number of concurrent sessions allowed for each user.

Common permission errors and solutions

Information about common permission-related errors, their causes, and recommended solutions.

The following table describes the common permission-related errors, the common causes, and the recommended action.

Error Common cause Recommended action
Permission denied The role assigned to the user or service account does not include the necessary privilege for the action being attempted. You can review the privileges of the assigned role and add the required permission.
Not found for an object The role assigned to the user does not have View Inventory permissions for that object type, making the object invisible to the user. You can ensure the role includes the View Inventory permission for the object type in question.
Unauthorized The user session has expired, or the service account client secret is incorrect, has been rotated, or revoked. A user can re-authenticate to establish a new session. For a service account, you can verify the client credentials or rotate the client secret if it is lost.
