    Rubrik Security Cloud - Data Protection securely manages and protects data from a wide variety of workload types.

    • Rubrik Security Cloud workloads

      RSC securely manages and protects data from a wide variety of workload types.

    • Cloud workloads
      • Amazon Web Services workloads

        RSC protects the following types of Amazon Web Services workloads: EC2 instances, EBS volumes, Amazon S3, Amazon RDS and Aurora databases, and Amazon DynamoDB.

      • Google Cloud Platform workloads

        RSC protects the following types of Google Cloud Platform workloads: Google Compute Engine instances and persistent disks.

      • Microsoft Azure workloads

        RSC protects the data and resources of Azure virtual machines, Azure managed disks, Azure Blob Storage, and Azure SQL workloads in Azure subscriptions.

        • Introduction to Azure protection
        • Get started with Azure workloads protection

          RSC protects Azure workloads through a sequence of dependent configuration steps. The implementation workflow provides a complete phase-by-phase reference. The quickstart covers the minimum required steps to protect your first Azure subscription.

        • Plan your Azure protection

          Before adding an Azure subscription to RSC, establish RSC user roles and review the permissions that RSC requires in your Azure environment.

        • Set up Azure cloud accounts

          You can add your Azure subscriptions to RSC as cloud accounts to enable workload discovery, protection, and recovery.

        • Set up Exocompute on Azure

          Exocompute on Azure uses an Azure Kubernetes Service (AKS) cluster in your Azure subscription as compute infrastructure for data protection operations such as file indexing, file recovery, and storage tiering.

        • Set up Azure storage tiering

          You can use Azure storage tiering to archive Azure virtual machine or managed disk snapshots, which are older than a specified threshold, to Azure storage access tiers. By archiving snapshots to these access tiers, you can optimize cost.

        • Set up Azure workloads protection

          You can set up SLA Domains to apply data protection policies, ensuring automatic and hierarchical protection of Azure subscriptions, resource groups, and individual Azure workloads, with granular control overriding higher-level assignments.

        • Manage Azure virtual machines

          After completing the protection setup for Azure virtual machines and managed disks, RSC takes automated snapshots of those virtual machines and disks based on the configured policies in SLA Domains. You can also take on-demand snapshots, refresh Azure subscriptions, and add Azure custom tags for better resource management.

        • Manage Azure Blob Storage

          By using the protection services offered by RSC, you can protect and recover your objects in Azure Blob Storage.

        • Manage Azure SQL Database

          RSC enhances Azure native backup capabilities for SQL Databases by offering advanced, Rubrik-managed protection. This solution overcomes native limitations with customizable SLA Domain policies, on-demand snapshots, and flexible cross-subscription recovery, ensuring a consistent and automated approach to data security.

        • Manage Azure SQL Managed Instance

          RSC enhances Azure native backup capabilities for SQL Managed Instances by offering advanced, Rubrik-managed protection. This solution overcomes native limitations with customizable SLA Domain policies, on-demand snapshots, and flexible cross-subscription recovery, ensuring a consistent and automated approach to data security.

        • Recover Azure workloads

          After RSC takes snapshots of your Azure workloads, you can recover the required workloads.

          • Privilege elevation for Azure recovery

            RSC provides you an option to secure Azure cloud accounts by using a least permission model, granting only necessary read permissions at the subscription level. For recovery operations, you can temporarily elevate permissions either by authorizing a service principal role or by using a security token.

          • Azure virtual machine recovery

            You can recover Azure virtual machines and managed disks using the various recovery options that RSC provides.

          • Azure Blob Storage recovery

            You can recover Azure Blob Storage using the various recovery options that RSC provides.

          • Azure SQL workload recovery

            RSC supports recovering Azure SQL workloads at the individual database level.

            • Azure SQL recovery from Rubrik backups

              There are three options for recovering Azure SQL workloads from Rubrik backups.

            • Restoring Azure SQL workloads to a point in time

              Restore an Azure SQL workload to a point in time using the short-term retention backups.

            • Exporting a long-term retention backup

              Recover Azure SQL workloads by exporting a long-term retention backup. The long-term retention backup can be immutable or non-immutable.

            • Downloading the BACPAC file for Azure SQL MI

              Use this method to recover Azure SQL Managed Instances by downloading the BACPAC file from a Rubrik backup. The file is stored in a storage account where RSC uploads it.

Azure SQL Managed Instance recovery settings

Recovering Azure SQL Server databases by exporting snapshots requires specific configuration.

RSC requires the following settings for recovering Azure SQL Server databases from backups.

Setting Description
Database Name The name of the exported database.

RSC populates this field with a randomly generated database name.
(Rubrik backups) Username and Password The credentials of the database admin account for authentication during recovery.
(Rubrik backups) Subscription The target Azure subscription to export the workload.
(Rubrik and long-term retention backups) Region The geographical region within the subscription.
Managed Instance The target managed instance.
Resource group The resource group of the exported database.

The resource group is the same as that of the source database and cannot be edited.
(Rubrik backups) Azure Active Directory Domain The name of the Azure Active Directory Domain for authentication using OAuth during recovery.
