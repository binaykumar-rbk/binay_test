Adding an AWS account using a template for storage

To use the archiving feature, add the AWS account in RSC using the AWS CloudFormation template for storage.

Before you begin

Before adding an AWS account in RSC:
  • Use an account that is assigned to the CloudFormation Service Role to launch and run the CloudFormation stack provided by RSC. The AWS CloudFormation service role topic in the AWS documentation provides information about this service role.
  • Provide a minimum set of permissions for running AWS CloudFormation in the structure contained in the Rubrik GitHub JSON file: Minimum permissions for AWS.
  • Review and adjust Service Control Policies (SCPs) to ensure they do not restrict necessary permissions, as SCPs can impact the RSC operations. The AWS SCP restrictions topic provides more information about the SCPs affecting RSC operations and how to mitigate them.

When you create an AWS cloud account and assign roles, contact an AWS administrator to assign specific policies to you so that you can run a stack on AWS. The AWS documentation provides information about creating a policy and attaching it to a user.

Procedure

  1. Log in to an AWS account.
    The AWS Management Console page appears.
  2. Using the same browser session, log in to RSC.
  3. Click and select Settings.
    The Settings menu appears.
  4. Click Cloud Accounts and select AWS.
    The AWS page appears.
  5. Click Add AWS Account.
    The Add Cloud Account wizard opens.
  6. On the Select use case page, select Storage > Cloud-native Location.
  7. Click Next.
  8. Select Add using CFT.
  9. Click Next.
  10. In Account ID, enter an ID for the account or select an account from the dropdown list.
    The account ID must be the 12-digit numeric ID.
  11. In Account name, enter a name for the account.
  12. Click Next.
  13. Optional: Click Review Permissions to view the permissions that will be updated after adding the AWS account.
  14. Click Download the CloudFormation template.
    This option downloads the list of permissions for offline analysis. Rubrik does not recommend this method to launch the CloudFormation template.
  15. Click Launch CloudFormation Stack to launch the CloudFormation template.
    RSC launches the Create Stack flow page on the AWS CloudFormation console. The template URL is already filled in.

    The redirection URL is a link to the CloudFormation template and is a combination of the template URL and stack name that the redirection URL uses as query parameters. The stack name is already filled in from the redirection URL that RSC initiates.

  16. On the Specify stack details page, in the ResourcePath field, either use the default path or enter the path where you want to create the roles and resources.
    The default path is /. You can override the default path with a custom path.
  17. Review the information and select I acknowledge that AWS CloudFormation might create IAM resources.
  18. Click Create.
    AWS CloudFormation creates the stack and displays it in the CloudFormation > Stacks page.
  19. Click Submit.

Results

RSC creates the AWS cloud account for the specified AWS account with the IAM user and displays the account on the AWS page.
