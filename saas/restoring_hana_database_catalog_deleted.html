        • SAP HANA snapshots

          RSC protects and manages data through snapshots of the source object and enables you to take full, incremental, and differential snapshots.

        • SAP HANA storage snapshots

          Using RSC, you can take storage snapshots of SAP HANA systems running on an Azure virtual machine and use those snapshots to recover your SAP HANA databases. Storage snapshots are fast, encrypted, have negligible network load, and capture the entire data area at a particular point in time.

        • Legal hold for snapshot retention

          Legal hold ensures that snapshots are retained indefinitely until the hold is removed, overriding the scheduled expiry of an SLA Domain policy. This ensures that snapshots are available as long as necessary to meet legal obligations.

        • Manage retention of SAP HANA snapshots

        • SAP HANA replication

Restoring a SAP HANA database without the backup catalog

Restore a SAP HANA database to the same or a different system using SAP HANA Studio when the backup is deleted from the catalog.

Before you begin

Obtain the restore snapshot Backup ID and Backup Prefix from RSC.

About this task

When the backup for a database is deleted from the backup catalog, a SAP HANA database can be restored using SAP HANA Studio.

The ability to restore a SAP HANA database after the database backup has been deleted from the backup catalog is supported for Rubrik clusters running Rubrik CDM version 6.0.2 and newer.

Procedure

  1. In SAP HANA Studio, open the context menu for a database.
    To recover a tenant database, open the context menu from its system database.
  2. Right-click on the system database that is the restore target.
  3. Click Backup and Recovery > Recover Tenant Database.
    The Recovery wizard appears.
  4. Select the tenant database to restore.
  5. Click Next.
  6. Click Recover the database to a specific data backup.
  7. Click Next.
    The Specify Backup Location page appears.
  8. Click Recover without the backup catalog.
  9. Click Next.
    The Specify the Backup to Recover page appears.
  10. In Destination Type, select Backint.
  11. In Backup Prefix, enter the Backup ID and Backup Prefix of the snapshot with the format Backup ID:::Backup Prefix.
  12. Click Next.
    Make no changes on the Other Settings page when it appears.
    The Other Settings page appears.
  13. Click Next.
    The SQL statement to be used for the restore can be viewed by clicking Show SQL Statement.
    A summary of the selected options appears.
  14. Click Finish.
    The restore begins and the progress appears in the dialog box

Results

RSC restores the database.
