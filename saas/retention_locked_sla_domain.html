        • Creating a custom SLA Domain

          Create a custom SLA Domain to streamline and customize workload protection and management policies.

        • Creating SLA Domains for many-to-one replication

          Create an SLA Domain for many-to-one replication, the traditional replication mode supported by RSC SLA Domains. This method consolidates data protection from multiple source clusters onto a single target cluster, ensuring centralized recovery and protection.

        • Creating SLA Domains for one-to-one replication

          Create an SLA Domain for one-to-one replication when there are more than two replication targets and the source and replication clusters are in a dedicated replication configuration. With one-to-one replication, you can configure multiple replication pairs, but a source cluster can replicate to a maximum of two target clusters.

        • Archival policy

          A Rubrik cluster archives data according to an archival policy that instructs the Rubrik cluster to copy protected data to archival locations.

        • Replication settings for specific workloads

          When replication is enabled for specific workloads when creating or editing an SLA Domain, it requires specific settings to be configured.

        • Replication target changes

          Editing the replication targets in an SLA Domain does not impact existing snapshots.

        • Replication configuration types in SLA Domain

          You can configure replication types, such as many-to-one and one-to-one, and configure replication between multiple source and target clusters using a single SLA Domain to maintain consistent backup and disaster recovery.

        • Continuous Data Protection-only SLA Domains for Amazon RDS

          RSC workloads, such as Amazon RDS, support both continuous data backup and scheduled backups. You can enable them as needed. With Continuous Data Protection-only SLA Domains, you can exclude snapshot schedules for supported workloads by turning off the frequency and retention when creating or editing the Continuous Data Protection-only SLA Domains.

          • Creating a retention-locked SLA Domain

            Create a retention-locked SLA Domain to prevent the retention period and frequency assigned to snapshots protected by the SLA Domain from being reduced.

          • Modes of retention-locked SLA Domains

            Retention-locked SLA Domains support two modes, governance mode and compliance mode, each offering distinct methods for managing data security and compliance requirements within SLA Domains.

          • Retention-locked SLA Domain for Microsoft 365

            A retention lock on an RSC SLA Domain prevents premature deletion of snapshots belonging to Microsoft 365 workloads.

          • Retention-locked SLA Domain for cloud workloads

            A retention lock on an RSC SLA Domain is a security feature that prevents premature deletion of snapshots.

          • Behavioral differences between governance and compliance retention-locked SLA Domains

            Governance and compliance retention-locked SLA Domains have specific differences based on the SLA Domain modifications. Administrators make these modifications to adjust backup frequency, retention duration, archival locations, and so on. These changes ensure that data protection strategies comply with regulatory requirements and organizational policies, optimizing storage costs and data accessibility.

          • Retention-locked SLA Domain modes and authorization

            Retention-locked SLA Domains offer modes for regulating data management and ensuring compliance. Each mode has a different set of configurations and authorization requirements.

          • Changing an SLA Domain to a retention-locked SLA Domain

            To ensure enhanced control and authorization for retention policies, change an SLA Domain to a retention-locked SLA Domain.

          • Changing to compliance mode or disabling the retention-locked SLA Domain

            To comply with regulatory requirements or remove specific data security measures, change the mode of a retention-locked SLA Domain from governance to compliance mode or disable the retention lock.

Retention-locked SLA Domains

The retention lock feature on RSC SLA Domains ensures the security and integrity of your data for a specified duration by preventing the premature deletion of snapshots for various workloads.

The retention lock feature is available only for Rubrik clusters running Rubrik CDM version 7.0.2 or later.

With the retention lock feature, you can achieve the following results:
  • Control and secure the retention settings for data.
  • Ensure that data remains unaltered and protected against accidental deletion.
  • Maintain data integrity and adhere to data retention policies.

You can enable the retention-locked SLA Domain feature by contacting Rubrik Support. After Rubrik Support enables the retention lock feature on a Rubrik cluster, when you create or edit an SLA Domain, a toggle to apply the retention lock appears, providing you with the option to use the retention lock feature.

When you are creating or editing an SLA Domain, you can assign settings that affect the level of security for your workloads, including the following parameters:
  • Snapshot frequency: How frequently snapshots are taken
  • Data retention: How long data is retained before deletion
To understand the security levels of SLA Domains, consider the following definitions:
  • Less secure SLA Domain: Retention and frequency settings that provide reduced protection or a shorter data retention period compared to the current SLA Domain settings for the snapshots. This implies weaker data protection.
  • More secure SLA Domain: Retention and frequency settings that offer enhanced security and an extended data retention period compared to the current SLA Domain settings for the snapshots.
When you enable Apply to existing snapshots:
  • If a more secure SLA Domain retains a snapshot, the compliance mode remains unchanged.
  • If the current SLA Domain has a lesser retention period or lower snapshot frequency than the new compliance mode SLA Domain, the compliance mode SLA Domain policies are applied to the snapshot. This results in stronger data protection due to more frequent snapshots and longer retention periods.
  • If a retention-locked SLA Domain retains existing snapshots, whether in governance mode or a less secure compliance mode, the more secure retention lock of the new SLA Domain protects them.
  • If snapshots are in compliance mode, the current SLA Domain continues to retain them.

When you disable the retention lock, existing snapshots remain retention-locked unless you enable Apply changes to existing snapshots. If you enable the Apply changes to existing snapshots option, the existing snapshots or the new snapshots are not retention locked.

A retention lock prevents the retention period assigned to an SLA Domain from being decreased. Before you can set a retention lock, you must define a minimum retention period for the SLA Domain. Setting the retention lock enforces this minimum retention period. After the retention lock is enabled for an SLA Domain, the following actions cannot be performed:
  • Removing the retention lock from that SLA Domain
  • Deleting the snapshots protected by the SLA Domain during the retention period
  • Decreasing the minimum retention period until the snapshot has expired
Note:
Rubrik cannot guarantee that no data is lost if an attacker gains access to a non-immutable or cascading archival location.
