Creating RCV locations for data center workloads on AWS

Create a Rubrik Cloud Vault location, assign it to a Rubrik cluster, and configure encryption with an RSA Key for archiving backups of data center workloads.

Before you begin

Perform the following tasks:
  • Generate an RSA key for encryption, as described in Generating an RSA key.

    Store the RSA key in a safe location. Rubrik cannot recover any lost RSA keys.

  • Allow outbound access through port 443 on each Rubrik cluster to access the Rubrik Cloud Vault (RCV) location, as described in Outbound connections for RCV on AWS.
  • Review the best practices, as described in Rubrik Cloud Vault best practices.

About this task

You can create an RCV location in AWS on clusters running Rubrik CDM version 9.3.1 or later.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Infrastructure > Rubrik Cloud Vault.
    The Rubrik Cloud Vault dashboard appears.
  4. Click RCV Locations.
    A list of RCV locations appears.
  5. Click Create RCV Location.
    The Create an RCV Location wizard appears, displaying the supported workload types.
  6. Select Data Center and click Next.
    The Select clusters page appears.
  7. Select the cluster you want to protect using RCV as the archival location and click Next.
    You can also select multiple clusters. If you choose multiple Rubrik clusters, RSC creates a unique RCV location for each cluster.
  8. In Location Name, enter a display name for the archival location.
  9. In Cloud Provider, select AWS.
  10. From Tier, select a storage tier.
    TierDescription
    Backup The Backup tier in RCV offers a minimum retention period of 30 days, providing fast access to data for immediate recovery needs.
    Archive You can use the archive tier for long-term data retention that does not require instant access. The recovery from the archive tier can take from several hours up to a few days. The minimum retention period is 180 days.
    The Rubrik Cloud Vault storage tiers topic provides more information and a comparison between the storage tiers to help you make an informed decision based on your data storage requirements.
  11. From Redundancy, select a redundancy type.
    Redundancy typeDescription
    Single zone Stores your data redundantly across multiple devices within a single Availability Zone. This option is available only for the backup tier.
    Multi-zone Stores your data on multiple devices across a minimum of three Availability Zones in a region. This option is available only for both backup and archive tiers.
  12. From Region, select a geographical region for the RCV location.
    The list of regions appears based on the storage tier.
  13. Optional: Turn on Bypass proxy.
    Selecting this option bypasses the system proxy from routing the archival requests and data.
  14. From Encryption Type, select RSA Key and enter the RSA key value.
    The RSA key must include the following lines in the key value.
    -----BEGIN PRIVATE RSA KEY----- 
-----END PRIVATE RSA KEY-----
  15. Select I’ve stored this key in accordance with my organization’s security policies to confirm that you have securely stored the RSA key in compliance with the security policies of your organization.
  16. Select I understand that Rubrik is unable to retrieve this key for any reason to acknowledge that Rubrik is not responsible for retrieving the RSA key.
    Important:
    Without this key, you cannot retrieve your data in a disaster recovery scenario.
  17. Click Next.
  18. Review the summary information and click Create.

Results

RSC creates the RCV location for the selected Rubrik cluster on AWS.

The new RCV location appears on the Rubrik Cloud Vaults page and on the Data Center Archival Locations page.

What to do next

Configure an SLA domain with archival enabled for the RCV location, as described in Configuring an archival policy.
