Creating roles by object type for RSC-based protection

Create a custom role in RSC with Role-Based Access Control privileges for RSC-based protection of the NAS CD objects with permissions configured by object type. These privileges provide granularity for determining the permissions to perform actions, and on which objects.

About this task

Creating custom roles by object type enables configuring access permissions for data sources depending on the object type you select.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Users & Accessand then select Roles.
    The list of roles appears.
  4. Click Create Role.
    The Create Role wizard appears.
  5. Select Custom role.
  6. In Data Management Privileges, click Configure to set the data management privileges for the role.

    The wizard displays the Data Management Privileges page.

  7. Select By object type and click Next.
  8. Select NAS Cloud Direct and click and then click Done for each of the options.
    OptionDescription
    All protectable objects
    For selected object types, permissions apply to one of the following options:
    • All existing and future protected NAS CD objects
    • Specific NAS CD objects

    You can add permission to view, protect, and restore from specific NAS CD systems in RSC.
    All recovery targets
    For selected object types, permissions apply to one of the following options:
    • All existing and future protected NAS CD recovery targets
    • Specific NAS CD recovery targets

    You can use the selected NAS CD objects as recovery targets from another share. You can recover to only the target locations that you have access to, as defined by the RSC role assigned to you by the RSC administrator.

    If the administrator does not specify a recovery target during role creation, you cannot recover to any target location.
    Full privileges
    For the selected objects, this option provides the following View permissions:
    • View protectable objects
    For the selected object, this option provides the following Recovery permissions:
    • Export files
    • Restore over original
    For the selected objects, this option provides the following protection permissions:
    • Manage protection
    • Delete snapshots
    For selected object types, use this option to add and delete resources that contain protectable objects and recovery targets:
    • Data source management
  9. Click Next.
    The Assign Datacenter Archival Locations page appears.
  10. Select an option for the target and click Next.
    OptionDescription
    Select all existing and future archival locations Provides permission to create or manage targets-based on the following options you select:
    • Add archival location: Selecting this provides permission to add targets.
    • Edit archival location (pause/resume, enable/disable, and promote cluster as reader): Selecting this provides permission to edit targets by pausing, resuming, enabling and disabling targets.
    • Delete archival location: Selecting this provides permission to delete a target.
    Select specific archival locations Provides permission to manage only selected targets.

    Selecting this option requires that you specify the targets for the role to manage.
    The Select SLA Domains page appears.
  11. Select an option for the SLA Domain management and then click Done.
    OptionDescription
    Self-Service Permissions for all SLA Domain creation and management actions.
    Select all existing and future SLA Domains Permissions for SLA Domain creation or management.
    The following specific permissions are available with this option:
    • Assign manage permission to specific SLA Domains: Selecting this option requires that you select an SLA Domain that the role can manage.
    • Assign manage permission to all existing and future SLA Domains: Selecting this option provides manage permissions for all SLA Domains.
    • Don't assign manage permission to any SLA Domains: Selecting this restrict the role from managing any SLA Domain.
    Select specific SLA Domains Permissions for specified SLA Domain creation and management to specific.

    Selecting this option requires that you select the SLA Domain for creation and management.

    You can select the following options for granular control of the role permissions:
    • Include Don't Protect: Selecting this includes SLA Domains with the Don't Protect setting in the list of SLA Domains that the role can manage and create.
    • Clear current SLA Domain: Selecting this includes SLA Domains with Clear current SLA Domain setting in the list of SLA Domains that the role can manage and create.
  12. In Assign System Configuration Privileges, click Configure to set the system configuration privileges for the role.
    Select the required permissions for the role.
  13. In Rubrik Application Privileges, click Configure.
  14. Optional: Select the necessary permissions for managing Rubrik application features.
    The Data Observability for NAS CD data sets section provides more information on the supported Rubrik application features.
  15. Click Done, and then click Next.
  16. In Role name, enter a name for the role.
  17. Optional: In Description, enter a description of the role.
  18. Click Create.
    A message appears indicating that the role is created.

Results

RSC creates a new role based on your selections and refreshes the Roles page.
