        • Planning for AWS protection

          Before setting up an AWS cloud account in RSC, it is necessary to plan the user and access management for designated RSC users. This planning also involves reviewing the permissions that RSC requires in your AWS environment to protect your AWS workloads.

          You can set up an AWS cloud account in RSC and start protecting your AWS workloads.

          With the Exocompute on AWS feature, RSC launches an Elastic Kubernetes Service cluster to perform various operations to protect AWS workloads. RSC uses the Elastic Kubernetes Service cluster as a compute resource in your AWS account.

        • AWS storage tiering setup

          You can use AWS storage tiering to archive Amazon EC2 instance, EBS volume, or DynamoDB table snapshots that exceed a specified threshold to Amazon S3 storage classes. By archiving snapshots to these storage classes, you can optimize costs.

        • AWS workloads protection setup

          You can set up SLA Domains to apply data protection policies, ensuring automatic and hierarchical protection of AWS accounts and individual AWS workloads, with granular control overriding higher-level assignments.

              You can recover your Amazon EC2 instances and EBS volumes using options such as recovering files or folders, exporting, and restoring.

              • Restoring an EC2 instance snapshot

                Restore an EC2 instance snapshot to replace the original instance with a fully functional point-in-time copy.

              • Exporting an EC2 instance snapshot

                Export an EC2 instance snapshot to create an instance. The new instance can be exported to the original region or to a new region.

              • Exporting an EBS volume snapshot

                Export an EBS volume snapshot to create a volume. The new volume can be exported to the original region or to a new region.

              • Recovering files or folders from an EC2 snapshot

                Recover files or folders from EC2 instances or EBS volumes to an S3 bucket if the files that you want to access are deleted, corrupted, or encrypted.

              • Recovering files from relic EC2 instance or EBS volume snapshots

                Recover relic EC2 instances or EBS volumes from the selected snapshots.

Restoring an EC2 instance snapshot

Restore an EC2 instance snapshot to replace the original instance with a fully functional point-in-time copy.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click the AWS - EC2 & Applications inventory card.
    The AWS - EC2 page appears.
  5. Select an EC2 instance.
    The EC2 instance details page appears.
  6. Click a date on the calendar to view the list of snapshots taken on that day.
  7. Select a snapshot and click Recover.
    The EC2 Recovery wizard appears.
  8. Select Restore and click Next.
    On the Recovery option page, if the original copy of the snapshot has expired, the Recover from list displays only the replicated and archived copies. When multiple archival locations are configured in the associated SLA Domain, multiple archival copies are available for selection.
  9. In Recover from, select the snapshot to restore if multiple copies of the backup exist.
    • Original copy
    • Replicated copy (Replication enabled)
    • Archived copy (Archiving enabled)
  10. Optional: Clear Restore tags.

    If selected, the tags on the EC2 instance is reverted to the time of the backup.

  11. Click Recover.

Results

RSC restores the instance to a fully functional point-in-time copy.
