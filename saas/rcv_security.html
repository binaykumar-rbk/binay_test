        • RCV security for data center workloads

          Rubrik Cloud Vault adheres to security standards for the protection of archived backups from data center workloads.

        • Immutable Rubrik Cloud Vault locations

          Rubrik Cloud Vault locations provide immutability for cloud-native, NAS CD, and data center workloads, depending on the Rubrik CDM version and the cloud provider. Immutability ensures retention policies are enforced and prevents alteration or deletion of snapshots for the specified period, safeguarding data integrity.

        • RCV encryption options

          RSC supports and enforces either client-side or server-side data encryption for all RCV locations.

        • Quorum authorization for Rubrik Cloud Vault management actions

          You can protect the management actions on your Rubrik Cloud Vault locations using quorum authorization. When enabled, quorum authorization requires the requesters of the protected actions to obtain additional approvals to perform the requested actions.

        • Secure connectivity with Azure private endpoints

          A connection between Rubrik Cloud Vault and a private endpoint ensures that your data is transferred to Rubrik Cloud Vault through a dedicated private connection.

        • Allowlist for RCV on Azure

          To optimize network traffic restrictions to trusted Rubrik Cloud Vault location endpoints, you can allowlist a customer-specific endpoint for your Rubrik Cloud Vault locations.

        • Outbound connections for RCV on AWS

          For Rubrik Cloud Vault locations on AWS, the Rubrik cluster automatically opens an outbound connection through port 443 to access specific sites.

Rubrik Cloud Vault security

Rubrik Cloud Vault adheres to security standards for the protection of archived backups.

Rubrik Cloud Vault (RCV) provides robust security measures to ensure data protection across different environments.

Data center workloads

The Rubrik cluster encrypts the data for your data center workloads with AES-256 GCM before sending it to RCV. You can use an RSA key or a key from a customer-managed Azure Key Vault. If you use an RSA key, you need to store it securely for potential future recovery, as Rubrik does not store this key.

Cloud-native workloads

In cloud-native environments, RCV uses Server Side Encryption (SSE) with platform-managed keys to encrypt your data. This method simplifies the key management as the cloud service provider manages the encryption keys.

NAS CD workloads

NAS CD supports SSE with platform-managed keys to secure your data, ensuring data protection with minimal user intervention in key management.

Related concepts
  • Blob-level immutability for RCV locations on Azure
    RSC applies blob-level immutability, by default, to the Rubrik Cloud Vault locations created for data center workloads on Rubrik clusters running Rubrik CDM versions 9.1 and later. Rubrik Cloud Vault locations with blob-level immutability do not require manual configuration of immutability settings in RSC.
