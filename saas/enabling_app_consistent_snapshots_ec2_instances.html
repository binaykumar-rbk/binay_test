  3. Cloud Workloads
  4. Amazon Web Services workloads

    RSC protects the following types of Amazon Web Services workloads: EC2 instances, EBS volumes, Amazon S3, Amazon RDS and Aurora databases, and Amazon DynamoDB.

  5. Amazon EC2 instances and EBS volumes

    After you set up protection for Amazon EC2 instances and EBS volumes, RSC takes automated snapshots of those instances and volumes based on the configured policies in SLA Domains. You can also take on-demand snapshots, refresh AWS accounts, and add AWS custom tags for better resource management.

  6. Application-consistent snapshot for AWS EC2 instances

    Application-consistent snapshot is a feature that you can use to take snapshots that include data from pending I/O operations. You can use this feature with AWS to take snapshots that include data from pending I/O operations between the applications running on the AWS EC2 instance and the disk attached to the AWS EC2 instance.

  7. Enabling application-consistent snapshots for an AWS EC2 instance

    After you have installed the Rubrik Backup Service on an AWS EC2 instance, registered that AWS EC2 instance with RSC, and configured the pre-snapshot and post-snapshot scripts, to take application-consistent snapshots, enable the application consistency feature for the corresponding AWS EC2 instance.

Enabling application-consistent snapshots for an AWS EC2 instance

After you have installed the Rubrik Backup Service on an AWS EC2 instance, registered that AWS EC2 instance with RSC, and configured the pre-snapshot and post-snapshot scripts, to take application-consistent snapshots, enable the application consistency feature for the corresponding AWS EC2 instance.

Before you begin

Ensure that the AWS EC2 instances are running on an AWS account that has a configured Exocompute instance.

Procedure

  1. Log in to RSC.
  2. Click and selectÂ Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click on the AWS - EC2 & Applications inventory card.
    The AWS - EC2 page appears.
  5. Select an AWS EC2 instance.
    You can select multiple AWS EC2 instances.
  6. Open the ellipsis menu and select Manage app consistency.
    The Manage app consistency dialog box appears.
  7. Select Enable app consistency and click Confirm.
    A notification appears at the bottom of the screen, informing that app consistency is enabled for the selected AWS EC2 instances.

Results

Snapshots for the selected AWS EC2 instances are application-consistent.
