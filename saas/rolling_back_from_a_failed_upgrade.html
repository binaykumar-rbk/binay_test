Rolling back from a failed upgrade

To restore a Rubrik CDM cluster to the pre-upgrade state, use the command line to rollback a failed upgrade.

About this task

Note:
The rolling back from a failed upgrade feature is only available for the fast upgrade and not for the rolling upgrade.

Procedure

  1. Log in to the Rubrik CLI using the administrator account by opening an SSH session on a node of the Rubrik cluster.
    Enter ssh admin@node_ip
    where node_ip is the IPv4 address of the node that has the upgrade files.
    A field to enter the administrator password appears.
  2. At the prompt, enter the password for the administrator account.
    The CLI prompt appears.
  3. At the Rubrik CLI prompt, enter cluster upgrade rollback.

Results

The Rubrik CDM cluster rolls back the failed upgrade to the pre-upgrade state.
