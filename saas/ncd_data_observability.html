Data Observability for NAS CD data sets

To provide Data Observability, you can enable Data Threat Analytics and configure Data Security Posture for NAS CD data sets. Data Threat Analytics detects ransomware attacks and Data Security Posture identifies sensitive data in your NAS CD data sets.

Overview and components

By using the Data Observability features, you can monitor your data for ransomware and reduce sensitive data exposure by discovering the types of sensitive data you have.

Data Observability for NAS CD data sets has the following components:
  • Data Threat Analytics: Scans backups for anomalies and security threats.
  • Data Security Posture: Identifies sensitive data that must be secured.

You must back up your data to use the Data Observability features in RSC. You can use Rubrik Cloud Vault (RCV) or other storage provider to provide simplified archiving of backups, including archiving backups to locations that provide immutable storage. RCV provides a secure storage target. For NAS CD backups and archives, you can use RCV in any newly defined NAS CD policy or RSC SLA Domain. You can also add it to an existing protection policy to maintain a second copy of the data for Data Observability and cyber resiliency.

Data Threat Analytics

Data Threat Analytics for NAS CD data sets comprises Anomaly Detection, Threat Monitoring, and Turbo Threat Hunting to detect and investigate ransomware attacks.

The following table lists the Data Threat Analytics features available for NAS CD.
Feature Description Enablement
Anomaly Detection Monitors for anomalous file system changes to detect and automate recovery from encryption or mass deletion events.

You can automate detection of anomalous files and folders and their recovery by enabling Anomaly Detection.

 You can enable Anomaly Detection for the following:
  • RCV storage targets
  • NAS CD clusters or sites
Threat Monitoring Processes threat intelligence against backup data to identify malware infections.
You can:
  • Configure and enable Threat Monitoring for your backups to automate Threat Monitoring that identifies malware infections.
  • Add custom intel feeds for NAS CD Threat Monitoring to add more indicators of compromise that RSC uses to scan your data.
Note:
NAS CD does not support Threat Monitoring using IOCs with YARA rules.
 You can enable Threat Monitoring only for the NAS CD clusters.
Turbo Threat Hunt Performs an on-demand scan of Rubrik backups for user-defined indicators of compromise.

You can perform an on-demand scan to identify systems potentially affected by a ransomware attack.

 You can perform a Turbo Threat Hunt scan only on NAS CD clusters.

Data Threat Analytics works with RCV storage or any custom storage that your NAS CD cluster might be using.

The RSC Data Threat Analytics topic provides more details on Data Threat Analytics.

Data Security Posture

Data Security Posture for NAS CD data sets includes configuring Data Security Posture to discover types of sensitive data in your data sets. Data Security Posture searches indexed snapshots for specific types of sensitive data that must be secured.

Data Security Posture works only with an RCV storage that is allocated to the NAS CD cluster. After NAS CD backups have been uploaded to RCV, you can enable Data Security Posture on backups sent to RCV storage targets. Data Security Posture identifies sensitive data within backups that match a specified data policy.

Data Observability configuration

To configure Data Observability, you are required to complete the following tasks:

  • Creating an RCV or custom storage target allocated to the NAS CD cluster. For more information about RCV storage location, see Rubrik Cloud Vault.
  • Enabling Anomaly Detection, Threat Monitoring, and performing Threat Hunt for the appropriate NAS CD clusters. For more information about enabling Data Observability features, see Enabling Anomaly Detection for NAS CD clusters, Enabling Anomaly Detection for RCV archival locations, and Enabling Threat Monitoring for NAS CD clusters.
  • Creating a NAS CD policy to send backups to the storage target. For more information about creating policy in NAS CD UI, seeCreating a NAS CD policy for Data Observability.
  • Enabling Data Security Posture by assigning required policies to scan NAS CD data sets. For more information about Data Security Posture, see Data Security Posture.
  • Assigning the created NAS CD policy to the relevant NAS CD data sets
