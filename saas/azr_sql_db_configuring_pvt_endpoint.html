Configuring a private endpoint for Azure SQL DB

RSC uses your Exocompute instance to connect to the Rubrik-managed Azure SQL server, take Rubrik backups, and upload those backups to Azure storage. For enhanced security and improved performance, configure a private endpoint for this connection.

About this task

This task assumes that, for the hostname resolution, all of your Azure Virtual Networks (VNets) use the default integrated DNS server setting provided by Azure rather than a custom DNS. If a VNet is using a custom DNS, configure additional settings for private endpoint name resolution to the correct FQDN of the Rubrik-managed SQL server within the custom DNS. The Azure Private Endpoint DNS integration topic provides more information about the custom DNS integration.
Note:
Setting up a private endpoint is a one-time task within each subscription and each source region for backing up your Azure SQL Databases. You must create the private endpoint after RSC creates the Rubrik-managed SQL Server, which happens during the first backup attempt in a new subscription or region.

Procedure

  1. In RSC, from the Exocompute for Azure details page, retrieve the VNet and subnet configured for the Exocompute instance.
    You will use this information when you create a private endpoint on the Azure portal.
  2. In the Azure portal, log in to the Azure subscription in which your Azure SQL Databases are running.
  3. Search for and select SQL servers, and then select the Rubrik-managed SQL server.
  4. Navigate to the Networking page and open the Private access tab.
  5. Create a private endpoint using the details you retrieved from RSC.
    The new private endpoint connects the Rubrik-managed Azure SQL server to the VNet used for the Exocompute instance. The Azure Private Link for Azure SQL Database and Azure Synapse Analytics topic provides more information about creating a private endpoint.
    Note:
    Rubrik recommends that the region and VNet for the private endpoint match the region and VNet for the Exocompute instance. This is because the Exocompute VNet has default access to the private DNS zone. However, you can use any VNet for the private endpoint if it has peer connectivity to the VNet for the Exocompute instance.
  6. Optional: When you use a VNet other than the Exocompute VNet for the private endpoint, link the VNet to the private endpoint by using the private DNS zone.
    You can search for Private DNS zones in the Azure portal and select the privatelink.blob.core.windows.net DNS zone. When adding a virtual network link, select the VNet used for the private endpoint. The Link the virtual network topic provides more information about adding a virtual network link.

Results

You have configured the Azure private endpoint for the Rubrik-managed SQL server.
