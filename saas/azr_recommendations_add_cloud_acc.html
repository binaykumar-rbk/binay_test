        • Introduction to Azure protection
        • Get started with Azure workloads protection

          RSC protects Azure workloads through a sequence of dependent configuration steps. The implementation workflow provides a complete phase-by-phase reference. The quickstart covers the minimum required steps to protect your first Azure subscription.

        • Plan your Azure protection

          Before adding an Azure subscription to RSC, establish RSC user roles and review the permissions that RSC requires in your Azure environment.

        • Set up Azure cloud accounts

          You can add your Azure subscriptions to RSC as cloud accounts to enable workload discovery, protection, and recovery.

          • Recommendations for adding Azure cloud accounts

            Rubrik recommends specific actions and best practices for adding Azure cloud accounts to RSC.

          • Permission categories for Azure workloads

            For tasks that operate on Azure workloads, such as indexing, recovery, and storage tiering, RSC requires specific permissions. RSC groups these permissions into categories for granular management so that you can include or exclude these categories as required.

          • Evaluate Azure cloud account setup options

            Review the available methods for adding an Azure cloud account to RSC — Configure with OAuth, Configure without OAuth, and bulk or individual addition — to select the approach that fits your environment.

          • Azure policy restrictions

            Azure policies can obstruct RSC operations such as storage account creation. Knowing how to adjust these policies is crucial for maintaining both security and compliance.

          • Configure Azure cloud accounts

            Add your Azure subscription to RSC as a cloud account using Configure with OAuth, Configure without OAuth, or Terraform.

          • Manage Azure cloud accounts

            Edit, upgrade, and delete Azure cloud accounts after initial setup, and migrate existing accounts to the cloud discovery model.

        • Set up Exocompute on Azure

          Exocompute on Azure uses an Azure Kubernetes Service (AKS) cluster in your Azure subscription as compute infrastructure for data protection operations such as file indexing, file recovery, and storage tiering.

        • Set up Azure storage tiering

          You can use Azure storage tiering to archive Azure virtual machine or managed disk snapshots, which are older than a specified threshold, to Azure storage access tiers. By archiving snapshots to these access tiers, you can optimize cost.

        • Set up Azure workloads protection

          You can set up SLA Domains to apply data protection policies, ensuring automatic and hierarchical protection of Azure subscriptions, resource groups, and individual Azure workloads, with granular control overriding higher-level assignments.

        • Manage Azure virtual machines

          After completing the protection setup for Azure virtual machines and managed disks, RSC takes automated snapshots of those virtual machines and disks based on the configured policies in SLA Domains. You can also take on-demand snapshots, refresh Azure subscriptions, and add Azure custom tags for better resource management.

        • Manage Azure Blob Storage

          By using the protection services offered by RSC, you can protect and recover your objects in Azure Blob Storage.

        • Manage Azure SQL Database

          RSC enhances Azure native backup capabilities for SQL Databases by offering advanced, Rubrik-managed protection. This solution overcomes native limitations with customizable SLA Domain policies, on-demand snapshots, and flexible cross-subscription recovery, ensuring a consistent and automated approach to data security.

        • Manage Azure SQL Managed Instance

          RSC enhances Azure native backup capabilities for SQL Managed Instances by offering advanced, Rubrik-managed protection. This solution overcomes native limitations with customizable SLA Domain policies, on-demand snapshots, and flexible cross-subscription recovery, ensuring a consistent and automated approach to data security.

        • Recover Azure workloads

          After RSC takes snapshots of your Azure workloads, you can recover the required workloads.

Recommendations for adding Azure cloud accounts

Rubrik recommends specific actions and best practices for adding Azure cloud accounts to RSC.

Cloud discovery is the essential first step for any Azure subscription in RSC. Cloud discovery is enabled by default to give you immediate and full visibility into your cloud environment. Rubrik recommends selecting the discovered objects associated with all Azure workloads when adding an Azure cloud account to RSC.

Selecting all discovered Azure workloads for protection

Adopting this onboard once approach ensures operational efficiency and scalability. Even if you only plan to protect one workload, Rubrik recommends selecting all workload types during setup.
  • Automatic identification: RSC automatically identifies all workloads across your selected regions, including Azure virtual machines and managed disks, Azure SQL databases, Azure SQL managed instances, and Azure Blob .
  • Operational efficiency: Selecting all workload types during setup prevents the need to reconfigure permissions or workflows later. You can instantly scale protection to new workload types without restarting the onboarding process.

To maintain the principle of least privilege, Rubrik recommends using the Manage Granular Permissions feature during setup. By selecting the Basic permission category, you grant RSC only the essential access needed to function, ensuring your environment remains secure.

The key aspects of selecting all workloads are as follows:
Feature Description
Discovery of cloud objects Discovery of all the following cloud workloads and the associated objects across all selected regions is the baseline capability for each onboarded Azure subscription:
  • Azure virtual machines and managed disks
  • Azure SQL databases
  • Azure SQL managed instances
  • Azure Blob Storage
Separation of permissions RSC separates permissions into read-only discovery permissions for broad visibility and optional backup, that is, backup permissions. You can discover all workloads while choosing to provide backup permissions only for the specific workloads you intend to protect.
Hierarchy expansion The discovery and protection framework includes Azure virtual machines and managed disks, Azure SQL databases, Azure SQL Managed Instances, and Azure Blob Storage. This approach ensures full visibility for security features such as Optimize Backup Management, which require broad metadata access to identify risks across all regions without requiring backup overhead for every object.
Region-level management You can select a region while onboarding the Azure subscription to automatically enable discovery for all workloads in that region.

Assigning SLA Domain at the region level

After you add your Azure cloud account to RSC, Rubrik recommends assigning SLA Domains at the region level. This ensures that any new workload created in that region inherits the correct protection policy automatically, eliminating the risk of leaving new resources unprotected. For more information, see Region-level SLA Domain assignment for Azure.
