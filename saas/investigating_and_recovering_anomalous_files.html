Investigating and recovering anomalous files for filesets

View and recover suspicious, deleted, or modified files at the protected object level or at the level of individual files or filesets.

About this task

You can view and recover suspicious, deleted, or modified files at the protected object level or at the level of individual files or filesets. The view and recovery options for selected files and folders are downloading, restoring to the original path, recovering to a separate folder, recovering to a different host, and exporting to a different cluster service.

Procedure

  1. Log in to RSC.
  2. Click and select Data Threat Analytics.
    The Data Threat Analytics dashboard appears.
  3. On the navigation bar, click Anomaly Detection.
    The Anomaly Detection page appears.
  4. Click an object name to view details.
    The Anomaly Detection details page appears.
  5. Select an option to initiate recovery.
    • Select the check box for the protected object and click Recover.
    • Click within the protected object to view files, select files to recover, and click Recover.
    The Fileset Recovery wizard opens, displaying the Select recovery option page.
  6. Select a recovery option.
    OptionDescription
    Download Creates a link to download the selected file or folder.
    Overwrite original Restores the selected files and folders to the original path, overwriting the existing file or folder.
    Recover to a separate folder Restores the selected files and folders to another location.
    Export to a different host Recovers the selected files and folders to a different host.
    Export to a clustered service Recovers the selected files and folders to a different clustered service.
  7. (Recover to a separate folder, Export to a different host, and Export to a clustered service) Enter the full path of the restore location in Folder path.
  8. (Export to a different host) Select the export host from the dropdown options.
  9. (Export to a clustered service) Select the export service from the dropdown options.
  10. Optional: (Overwrite original, Restore to separate folder, Export to a different host, and Export to a clustered service) Select the Continue on errors option to instruct the Rubrik cluster to continue the restore job after encountering a restore error.
  11. Click Recover.

Results

Anomaly Detection recovers the files according to the selected option.
