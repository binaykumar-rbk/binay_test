    • Rubrik clusters

      A Rubrik cluster is an instance of Rubrik CDM running on any virtual or physical platform. You can add the Rubrik cluster to RSC to enable extended features for the cluster.

    • Security

      Rubrik Security Cloud provides end-to-end security for data management information.

    • Outbound connection through ports

      RSC requires you to open an outbound connection through port 443 for a Rubrik cluster to access specific sites.

    • SMB security

      With the SMB security feature in RSC, a Rubrik cluster provides security for SMB shares by requiring Active Directory authentication and enforcing SMB security.

    • Rubrik cluster deployment

      Deploying a Rubrik cluster running on Rubrik appliance nodes involves installing the CDM software on the nodes and setting up that software for your environment.

    • Cluster registration

      Rubrik offers two modes for registering clusters with Rubrik Security Cloud, online and offline.

    • Rubrik MFA with TOTP

      The Rubrik two-step verification feature uses time-based one-time passwords in addition to the user login credentials to enforce multi-factor authentication on user accounts.

    • CLI administrator role

      As a CLI administrator, a Rubrik cluster user can connect to nodes in Rubrik clusters through the CLI Secure Shell (SSH) protocol.

    • Jumbo frame support

      For Rubrik CDM versions 9.4 and later, you can set the value for the maximum transmission unit to a maximum value of 10000, instead of the default value of 1500. This larger unit size is called jumbo frames.

    • System shutdown

      A system shutdown can be done from the Rubrik CLI.

Downloading an upgrade package

Download a Rubrik CDM software upgrade package to a Rubrik cluster using RSC before upgrading the Rubrik CDM software.

Before you begin

Complete the following tasks before you begin downloading the Rubrik CDM software upgrade package to a Rubrik cluster using RSC:
  • Log in to RSC using an account that has privileges to view and upgrade Rubrik clusters.
  • Verify that the Rubrik clusters have access to the Support Portal or to an HTTP, NFS, or FTP server that hosts the software upgrade package and the associated MD5 checksum. Currently, NFS only works with generic Linux NFS servers.
  • Use an NFS server with insecure access enabled when using NFS to transfer the upgrade package to a Rubrik cluster running Rubrik CDM version 5.3.0 or later.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Clusters and select CDM Software Upgrades.
    The Rubrik CDM Upgrades page appears.

    You can view the list of Rubrik clusters added to RSC and the respective installed version.

  4. Select a Rubrik cluster to upgrade and click Download.
    You can locate a specific cluster by using the available filters or by entering a cluster name in the search field.
    Note:
    If you select multiple Rubrik clusters to download the upgrade package, ensure that all the selected clusters are running the same Rubrik CDM software version.
    The Download Package wizard appears.
  5. Select the source of the upgrade package.
    OptionDescription
    Rubrik release repository Downloads the upgrade package from the release repository on the Rubrik Support portal.

    To download the package from the Rubrik release repository over HTTP proxy, your firewall settings need a configuration to allow inbound and outbound traffic for https://rubrik-support-download.s3.amazonaws.com.
    Local Requires you to enter the URL and MD5 hash of the upgrade package.

    You can use this option if you prefer not to have your cluster communicating outside the local network over the internet. To use this option, providing a local URL and MD5 hash of the upgrade package is required. The first step is to download the ZIP file of the upgrade package from the Rubrik Support Portal and transfer the ZIP file to a machine within your local network. The next step is to generate an MD5 hash for the ZIP file. With this done, you can now provide the local URL and hash value in RSC.
    The following is an example of an HTTP entry for the URL when using the Local option as the source of the Download Package:
    http://10.0.189.142:8080/rubrik-image-9.0.1-p1-25067.zip
    The following is an example of an NFS entry for the URL when using the Local option as the source of the Download Package:
    nfs://test-logs.rubrik-lab.com/bug_logs_repo_00/trident/zipfiles/rubrik-image-8.1.3-p5-25014.zip
  6. For Rubrik release repository, under Package versions, select the Rubrik CDM software version of the upgrade package.
    RSC displays the latest version of the Rubrik CDM software running on the selected Rubrik cluster and the latest version of the Rubrik CDM software for the next major release.
  7. Optional: For Rubrik release repository, click Add Other Packages and select the required version from the list.
    You can use this option to select older versions of the Rubrik CDM software running on the selected Rubrik cluster and the Rubrik CDM software for the next major release.
    The selected Rubrik CDM software version appears under the Package versions area, and you can select the added software version from the list.
  8. Click Next.
    The confirmation page appears.
  9. Verify the selected clusters and click Confirm.
  10. Click Done.

Results

RSC initiates the download of the upgrade package. The status displayed for the Rubrik cluster changes to Downloading.

When the download process completes, RSC runs a series of precheck tasks. The status of the Rubrik cluster changes to Ready for upgrade when the precheck tasks are successful.

What to do next

Upgrade the Rubrik CDM version running on any connected Rubrik cluster to the version you downloaded using RSC.
