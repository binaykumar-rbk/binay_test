      • Failover for RCV on Azure

        The failover process for Rubrik Cloud Vault ensures continuous data availability during regional outages and disasters by switching between primary and secondary regions configured with Multi-region redundancy.

        • Performing a failover for data center archival locations

          To ensure data availability during a disaster, perform a failover from the primary region to the secondary region for data center archival locations with Multi-region redundancy. The failover process shifts data operations to the secondary region, protecting against regional outages and minimizing downtime.

        • Performing a failover for cloud native archival location

          To ensure data availability during a disaster, perform a failover from the primary region to the secondary region for cloud native archival locations with Multi-region redundancy. The failover process shifts data operations to the secondary region, protecting against regional outages and minimizing downtime.

        • Failback for Rubrik Cloud Vault

          The failback process for Rubrik Cloud Vault involves switching to the new secondary region after geo-redundant storage has been re-established. This resumes the backup operations in the designated region.

Multi-region redundancy statuses

Information about the various redundancy statuses and scenarios in Rubrik Cloud Vault, detailing the transition process during failover and failback to former primary region and re-establishing redundancy to Multi-region.

Scenario Redundancy Status Description
Failover initiated or in progress Single zone In progress Rubrik or Microsoft has initiated the failover. This process might take a few hours as the system transitions to the secondary region.
Failed over to secondary region Multi-region Failed over from Multi-region Failover process is complete, with the primary region now set to the secondary region. The redundancy has been updated to Single zone.
Re-establishing redundancy to Multi-region Single zone Re-establishing to Multi-region Rubrik is re-establishing redundancy to Multi-region. This process completes after data services in the original primary region become available again.
Multi-region redundancy re-established Multi-region Failover completed Failover is complete. The primary region has been updated to the original region, and redundancy is restored to Multi-region.

This step re-establishes the primary region as the active one, ensuring that normal data operations resume after a failover.
