Enabling version-level immutability when creating a container in Azure

When you are creating a container in Azure, to use immutability to back up and preserve data on the Rubrik Cloud Cluster ES, enable version-level immutability on the container.

About this task

When you configure version-level immutability on an Azure container, blob data cannot be modified or deleted for a specified period of time. Microsoft documentation provides more information about configuring immutability for blob versions.

Procedure

  1. Log in to the Azure portal.
  2. Navigate to the Containers page for your storage account.
  3. Click + Container.
    The New container dialog box appears.
  4. Expand the Advanced section.
  5. Select Enable version-level immutability support.
  6. Click Create.

Results

A container is created in Azure, with version-level immutability support enabled.
