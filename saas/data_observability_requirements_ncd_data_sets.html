Data Observability requirements for NAS CD data sets

NAS CD data sets have specific requirements for enabling backups and configuring Data Observability.

This topic describes a feature or features that Rubrik is gradually rolling out to customers.

Enabling Data Observability for NAS CD data sets requires fulfilling the following tasks:

  • Creating a NAS CD policy for backup to the Rubrik Cloud Vault Backup tier, with a retention period that equals or exceeds 30 days.
  • Creating a NAS CD policy for backup to the Rubrik Cloud Vault Archive tier, with a retention period that equals or exceeds 180 days.
  • Applying the Data Security Posture policy before NAS CD takes the first full backup as a recommended practice. After RSC takes the first backup, you have a grace period of 14 days to apply the Data Security Posture policy. If you do not apply the Data Security Posture policy within the grace period, Data Security Posture does not scan the backup data.
  • Enabling the Data Threat Analytics feature before the first full backup is sent to Rubrik Cloud Vault to establish a baseline for Anomaly Detection when using Rubrik Cloud Vault.

    Enabling the Data Threat Analytics feature before the first full backup completes is not required when using a custom storage provider. In this case, RSC does not scan preexisting backups for Data Threat Analytics.

Limitations of enabling Data Observability for NAS CD data sets include the following:

  • Only Anomaly Detection in Data Threat Analytics is available for NAS CD protected data sets. Threat hunting and quarantine capabilities are not available.
  • When using custom analyzers for a NAS CD data set, RSC requires that you add all the required custom analyzers to a Data Security Posture policy before assigning the policy to your NAS CD object for the first time. To add a new custom analyzer at a later point in time, you are required to delete the snapshot and create a backup to apply the new custom analyzer.
  • If you edit a custom analyzer after assigning it to a NAS CD object through a policy, RSC scans only the newly backed up data using the edited values for the custom analyzer. The On Demand page in Data Security Posture retains all the classification results generated using the old custom analyzer for the previously backed-up data.
