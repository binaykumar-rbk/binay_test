Deleting the Exocompute on Azure host subscription

After you have unmapped an Exocompute on Azure host subscription from its associated application subscriptions, deactivate and delete the Exocompute host subscription from the Azure page.

Before you begin

Ensure that the Exocompute host subscription you want to delete is not mapped to any application subscriptions. You can manage the mapping, as described in Mapping Azure accounts to centralized Exocompute.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Cloud Accounts and select Azure.
    The Azure page appears.
  4. From the dropdown list, select Rubrik Compute.
    A list of Azure Active Directory (AD) domains appears.
  5. Select an Azure AD domain.
    The list of Exocompute host subscriptions appears.
  6. Click for the Exocompute host subscription you want to delete and select Disable.
    The Disable protection confirmation dialog box appears.
  7. Click Disable.
    Once deactivated, the Status column is updated to Disabled for the Exocompute host subscription.
  8. Click for the deactivated Exocompute host subscription and select Delete.
    The Delete Azure subscriptions dialog box appears.
  9. (Subscription without OAuth) Click Delete Account Without OAuth if you manually deleted the associated custom roles in the Azure portal.
    If you manually deleted the associated custom roles in the Azure portal, RSC displays the subscription status as Disconnected in the Cloud Accounts page for Azure.
  10. Optional: (Subscription with OAuth) Click Review Permissions to view the list of permissions assigned to the subscriptions.
  11. (Subscription with OAuth) Click Sign In With Azure to grant permissions for deleting the subscription.
    The Authorization successful message appears, and the Delete button becomes active.
  12. In the Delete Azure subscriptions dialog box on RSC, click Delete.

Results

RSC deletes the Exocompute on Azure host subscription.
