Key management for data-at-rest encryption

With advanced key management, including key rotation, rekeying, and KMS integration, Rubrik enhances data-at-rest encryption by providing secure and flexible management of encryption keys to ensure ongoing protection of stored data.

Rubrik clusters offer advanced key management functionalities, including key rotation and rekeying, to strengthen data-at-rest encryption and safeguard encryption keys over time. These functionalities are integral to managing the encryption keys that protect data stored on disk, ensuring that data remains secure throughout its lifecycle.

Key management components

The following table describes the key components involved in data encryption so that you can maintain data security.
Component Function Protection
Data Encryption Key (DEK) Encrypts the actual data at rest. Secured by Key Encryption Keys.
Key Encryption Key (KEK) Protects the DEKs. Managed through key rotation and rekeying.
Root KEK Protects the KEKs. Secured by the master key.

The root KEK is unique for each archival location.

Four-level key hierarchy

A four-level key hierarchy provides a structured framework for managing and organizing cryptographic keys. This hierarchy incorporates various KEKs and DEKs to ensure robust data protection and efficient key management.
Component Description
KMS, Passphrase, or TPM Serves as the root source to wrap and secure the key hierarchy.

These root sources belong to the customer environment, and Rubrik has no control over this component.
Root KEK Represents the top layer in the key hierarchy.

Root sources protect the root KEK. Each root KEK hierarchy corresponds to different archival locations.
Level 2 KEKs Intermediate KEKs derived from the root KEK.
DEKs Derived from Level 2 KEKs.

These keys are used to encrypt data blobs. The number of data blobs encrypted by a single DEK depends on the configuration of the component. For example, CDM archival uses one DEK for each data blob.

Key management functionalities

Rubrik provides two primary functionalities for effective encryption key management: key rotation and rekeying.

Key rotation is a background process designed to enhance the security of encryption keys by regularly updating them. The essential elements of key rotation are as follows:
  • Purpose: Regularly updates the KEK to reduce the risk of key compromise.
  • Frequency: Updates the keys every 30 days.
  • Process: Runs as a background job without requiring customer input or interference.
Rekeying ensures that encryption standards are maintained and vulnerabilities are mitigated by changing existing KEKs without affecting data access. The essential elements of rekeying are as follows:
  • Purpose: Changes existing KEKs to enhance security, often necessary for upgrading encryption standards or addressing vulnerabilities.
  • Impact: No impact on data accessibility during the process.

Key Management Service (KMS)

Rubrik KMS offers robust control over encryption key management, supporting a range of environments from data centers to cloud archives.

Rubrik supports the key management of data center workload storage on Rubrik clusters through standardized protocols.
  • Supported protocol: Key Management Interoperability Protocol (KMIP)
  • Benefit: Standardized and interoperable key management across different environments.
For cloud archives, Rubrik integrates with native cloud key management services to ensure compliance and enhance security.
  • Integration: Native key management services for various cloud providers.
  • Benefits: Enhances security and management of encryption keys, ensuring compliance with cloud security requirements.
