      Using the data management and protection services offered by Rubrik Security Cloud, you can protect Microsoft 365 data operations.

      • Microsoft 365 dashboard

        The Microsoft 365 dashboard provides important insights into your protected Microsoft 365 environment. By using the dashboard, you can view and monitor information such as deployment locations and settings, backup and recovery status, events, and compliance information.

      • Microsoft 365 immutability

        RSC uses immutable storage for Azure Blob Storage to provide immutability protection for Microsoft 365 when creating an Azure storage account for Microsoft 365 deployments. Immutability protection ensures that your data remains unaltered, which is critical for disaster recovery, protection from ransomware attacks, and data compliance.

      • Microsoft 365 protection

      • Backup and recovery with Microsoft 365 snapshots

        RSC protects and manages Microsoft 365 data using snapshots of source Microsoft 365 objects. You can take snapshots of backup data for Microsoft Exchange, OneDrive, SharePoint, and Teams to recover lost or compromised information.

      • Relic Microsoft 365 management

        Microsoft 365 does not support user accounts once the user license is revoked. RSC archives inactive user accounts without snapshots and manages the ones with snapshots as relics.

Adding Teams groups

Add a wildcard group in Teams to assign specialized data protection policies.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
  3. On the navigation bar, click Inventory.
  4. On the Inventory page, click the Microsoft 365 inventory card.
  5. From the Selected subscription dropdown list, select the Microsoft 365 subscription.
  6. Click Enter Subscription.
  7. Click Groups.
  8. Click Add Group.
    The Add Group wizard appears.
  9. If you have a partially authenticated or unauthenticated management app, perform one of the following steps.
    ScenarioDescription
    Partially authenticated app
    1. On the Reauthenticate M365 Management Tool page, log in to your Microsoft 365 account with administrator credentials.
    2. Click Next.
    Unauthenticated app
    1. On the Grant Access page, log in to your Microsoft 365 account.

      RSC creates a new management app.

    2. Click Next.
  10. Select Teams Group and then click Next.
    The Create Teams Group page appears.
  11. Add the group information.
    SelectionDescription
    Group Name The name of the group.
    Expression A wildcard expression used to locate group members and objects.
    PDL The Preferred Data Location (PDL) for the group. A PDL can also restrict Teams objects to a specified region. You can select more than one PDL.
    A list of objects that matches the wildcard expression appears.
  12. Optional: Click Add Attribute and then select one or both options.
    Selecting this option adds a custom attribute to a list of Teams users. RSC then uses this custom attribute to assign these users to specific groups to manage user assignments efficiently.
    OptionDescription
    Administrative unit membership Filters users based on administrative unit membership.
    1. Select Administrative unit membership.
    2. From the Administrative Unit dropdown list, select the administrative unit to which the user belongs.
    3. Click Save.
    Entra ID attribute Filters group members based on one or more Entra ID attributes.
    1. Select Entra ID attribute.
    2. Add one or more conditions.
    3. Click Save.
    The list of objects appears. This list is filtered using the specified attributes.

    You can add additional attributes for Teams by clicking Add Attribute. The Microsoft-provided attributes help filter only those groups that have the specified values.

  13. Click Next.
    The Confirmation page appears.
  14. Click Add.

Results

RSC adds the Teams group according to the settings.

What to do next

Assign an SLA Domain to the group, as described in Protecting Microsoft 365 workloads using groups.
