Create Amazon S3 target

You can create an Amazon S3 target for the secure storage of backup snapshots and archived data. Additionally, you can customize proxy settings and choose an S3 endpoint for managing data transfers to the target.

You can use RSC to create an Amazon S3 target to protect your data by backing up or archiving them in Amazon S3 target destination. Using Amazon S3 as a backup target for your data involves configuring it as a target in RSC to securely store the backup snapshots and archived data.

Prerequisites

Before creating an Amazon S3 target, ensure you have completed the following:
  • Adding a cloud account: You must add an Amazon S3 cloud account with access keys to RSC for data center workloads. For more information, see Adding AWS cloud accounts with access keys for data center archival locations.
  • Configuring IAM permissions: The Amazon S3 cloud account must have the required IAM permissions for immutability of the NAS CD data sets. For more information about the required IAM permissions, see Amazon S3 immutability prerequisites for NAS CD data sets.
  • Accessing bucket details: RSC uses the cloud account to access the Amazon S3 bucket for creating backup snapshots and archival data. You also need the bucket details to create Amazon S3 as your target destination.

Customization and data management

You can customize how RSC manages and transfers your data to the target:
  • Proxy settings: You can create an Amazon S3 target destination with proxy settings of your choice to protect and manage your data transfers.
  • S3 endpoints: You can choose an S3 endpoint for managing data transfers to the target to ensure secure storage of backup snapshots and archived data.
