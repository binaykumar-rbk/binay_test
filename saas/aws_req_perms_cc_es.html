        A Rubrik Cloud Cluster Elastic Storage protects applications and data by running Rubrik CDM software on Amazon EC2 instances within a virtual private cloud.

        • Prerequisites for Rubrik Cloud Cluster ES for AWS

          Setting up a Rubrik Cloud Cluster Elastic Storage for AWS requires that you have an AWS account, a Rubrik license, a compatible web browser, and a VPC endpoint with internet connectivity. Additional requirements include an SSH key pair, an S3 bucket with specific settings, an instance profile, an IAM role for S3 access, and a security group with configured rules.

        • Security group rules

          The security group for the Rubrik Cloud Cluster requires specific inbound and outbound rules.

          Rubrik Cloud Cluster has specific node requirements for AWS.

        • AWS Cloud Cluster ES provisioning tasks

          Provisioning an AWS Cloud Cluster Elastic Storage on RSC consists of several tasks.

        • Upgrading AWS CCES clusters to use GP3 disks

          To use GP3 disk features in an existing AWS CCES cluster with GP2 disks, upgrade GP2 disks to GP3 using the upgrading disks workflow.

      • Rubrik Cloud Cluster recoverability

        To restore your data, you can recover a Rubrik Cloud Cluster Elastic Storage that has immutability enabled. You can do this only when the cluster nodes cannot be recovered.

      • Adding a node to Cloud Cluster ES

        Add a node to a Cloud Cluster ES to increase capacity and improve performance.

      • Adding a pre-deployed node to Rubrik Cloud Cluster ES

        Add a node that you have manually deployed in AWS or Azure to the Rubrik Cloud Cluster ES in RSC. Additional nodes increase capacity and improve performance.

      • Node removal in RSC

        If you no longer require a node in a Rubrik cluster, a supported third-party hardware cluster, or a Rubrik Cloud Cluster, you can remove the node by using RSC. To remove a node, you must meet the requirements for node removal.

      • Extra dense node migration for CCES clusters

        Extra dense nodes offer greater storage and performance, without a loss of storage capacity. Using extra dense nodes, organizations can reduce node counts by half during migration. Customization options are available through the GraphQL API.

Required permissions for Rubrik Cloud Cluster ES for AWS

Information about specific permissions AWS automatically applies to the CCES instance when you add the AWS cloud account to RSC. These permissions correspond to the roles and resources that RSC creates for cluster operation when you add the cloud account.

The following table describes the AWS permissions required for Rubrik Cloud Cluster ES functionality on RSC.
Permission Access level Use case
ec2:AttachVolume Write
  • To restore EC2 instances and attach exported EBS volumes.
  • To export crash-consistent snapshots of an EC2 instance.
  • To export and replace EBS volumes.
  • To attach a temporary EBS volume to Amazon Elastic Kubernetes Service (EKS) cluster for indexing and storage tiering.
ec2:CopyImage Write
  • To perform cross-region export of EC2 instances.
  • To perform cross-region replication of EC2 instance snapshots.
ec2:CopySnapshot Write
  • To perform cross-region export and replication of EBS volumes.
  • To perform cross-region export and replication of crash-consistent snapshots of EC2 instances.
ec2:CreateImage Write To create an image of an EC2 instance for backup.
ec2:CreateSecurityGroup Write To create a no-connectivity security group for exporting a virtual machine that is in a powered-off state.
ec2:CreateSnapshot Write To create a snapshot of an EBS volume for backup.
ec2:CreateSnapshots Write To create a crash-consistent snapshot of an EC2 instance.
ec2:CreateTags Tagging To assign signing metadata tags to created resources such as EBS volumes, EC2 instances, images, and snapshots.
ec2:CreateVolume Write
  • To export EBS volumes.
  • To restore EC2 instances.
  • To export crash-consistent snapshot of an EC2 instance.
  • To create a temporary EBS indexing volume for file search, file recovery, and storage tiering.
ec2:DeleteSnapshot Write

Requires the resource to have the rk_component tag with the value set to Cloud Native Protection.
  • To delete EBS volume snapshots associated with expired backups.
  • To delete associated snapshots when deleting an Amazon Machine Image (AMI). Deleting the associated snapshots makes it possible to revert a Create EC2 instance snapshot task or to export an EC2 instance snapshot to a different region.
  • To delete snapshots during the process of reversing the Create EBS volume snapshot task.
  • To delete snapshots when exporting the EBS volume snapshot to a different region.
  • To delete crash-consistent snapshots of excluded EBS volumes.
  • To delete snapshots when deleting an AWS account.
  • To delete expired snapshots.
  • To delete temporary snapshots created during replication and exporting.
ec2:DeleteVolume Write

Requires the resource to have the rk_component tag with the value set to Cloud Native Protection.
  • To delete the EBS volumes launched when the backup of a restore or export EC2 instance task fails during the process.
  • To delete a temporary EBS volume launched for file indexing and storage tiering.
ec2:DeleteTags Tagging
  • To overwrite existing tags when restoring tags in the EC2 instance restore job.
  • To overwrite Rubrik metadata tags when the tag limit is exceeded.
  • To delete Garbage Collection (GC) tags when the GC task is no longer required for the resource.
ec2:DeregisterImage Write
  • To delete the AMI associated with an expired backup.
  • To delete an image to revert an EC2 instance snapshot task.
  • To delete an image after copying it to a new region during the cross-region exporting.
  • To delete AMIs after a snapshot expiration.
  • To delete any temporary AMI created during replication and export.
ec2:DescribeAvailabilityZones Read To retrieve the list of availability zones while exporting EBS volume and EC2 instance snapshots.
ec2:DescribeImages Read
  • To track the image creation status in the EC2 backup task.
  • To retrieve image details for the recovery tasks.
  • To ensure periodically that the AMI associated with RSC backups is available and not deleted by an external actor.
ec2:DescribeInstances Read
  • To retrieve the list of EC2 instances during the refresh task.
  • To retrieve instance details during snapshot and recovery tasks.
ec2:DescribeKeyPairs Read
  • To retrieve the list of SSH key pairs during the virtual machine recovery.
  • To retrieve the associated key pairs for EC2 instances when a backup is taken.
ec2:DescribeSecurityGroups Read
  • To retrieve the list of security groups during the account refresh task.
  • To retrieve the list of security groups while exporting EC2 instance snapshots.
ec2:DescribeSnapshots Read
  • To track the snapshot creation status during the EBS volume snapshot task.
  • To retrieve snapshot details for the recovery tasks.
  • To ensure periodically that the snapshot associated with RSC backups is available and not deleted by a user-initiated script.
ec2:DescribeSnapshotAttribute Read To perform a check to determine whether an EBS volume snapshot is from the AWS marketplace.
ec2:DescribeSubnets Read
  • To retrieve the list of subnets during the account refresh task.
  • To retrieve the list of subnets while exporting EC2 instance snapshots.
ec2:DescribeTags Read, Tagging
  • To back up the tags attached to an EC2 instance during backup.
  • To avoid the tag limit while assigning tags to new resources.
ec2:DescribeVolumeAttribute Read To allow RSC to identify EBS volumes launched from the AWS marketplace.
ec2:DescribeVolumes Read To describe the properties of a given EBS volume or all EBS volumes.
ec2:DescribeVolumesModifications Read To describe the most recent volume modification request for the specified EBS volumes.
ec2:DescribeVpcs Read
  • To retrieve the list of all Virtual Private Clouds (VPCs) while exporting EC2 instance snapshots.
  • To retrieve the list of VPCs during the account refresh task.
ec2:DetachVolume Write
  • To detach the existing volumes and replace them with recovered copies during the restore EC2 instance.
  • To detach an existing EBS volume when the Replace where attached option is selected at the time of exporting an EBS volume.
  • To detach the EBS volume from the temporary EKS cluster when the file search is enabled for an EC2 instance or EBS volume.
ec2:GetEbsEncryptionByDefault Read To check whether default EBS encryption is enabled in the AWS account of a user for the given region.
ec2:ModifyVolume Write To check whether parameters of a EBS volume can be modified.
ec2:ModifyImageAttribute Write To modify the description of EC2 instance snapshots created by RSC. The image description is used to identify all GC leaked images.
ec2:ModifyInstanceAttribute Write

Requires the resource to have the rk_component tag with the value set to "Cloud Native Protection" or "Cloud Cluster".

 To change security groups of an EC2 instance from the no-connectivity security group to a user-specified security group for exporting an EC2 instance in a powered-off state.
ec2:RevokeSecurityGroupEgress Write

Requires the resource to have the rk_component tag with the value set to Cloud Native Protection.

 To remove the default egress rule from a no-connectivity security group for exporting an EC2 instance in a powered-off state.
ec2:RunInstances Write To launch a virtual machine from a backup.
ec2:StartInstances Write To start a virtual machine that was restored with new disks.
ec2:StopInstances Write To stop an EC2 instance.

When restoring a virtual machine, disks can be replaced only when the EC2 instance is stopped.
ec2:TerminateInstances Write

Requires the resource to have the rk_component tag with the value set to "Cloud Native Protection" or "Cloud Cluster".

 To terminate EC2 instances created during failed exports.
kms:CreateGrant Permissions management To enable EC2 service to use the Customer Managed Key (CMK) on behalf of the user.

Example: If an EBS volume snapshot is copied from key K1 to key K2, the EC2 service decrypts the K1 encrypted snapshot so that it can be encrypted to K2. To enable the EC2 service, it must be allowed to create grants for K1 and K2.
kms:Decrypt Write To perform the custom CMK flow while exporting EC2 instance or EBS volumean .
kms:DescribeKey Read To perform the custom CMK flow while exporting an EC2 instance or EBS volume.
kms:Encrypt Write To perform the custom CMK flow while exporting an EC2 instance or EBS volume.
kms:GenerateDataKey Write To perform the custom CMK flow while exporting an EC2 instance or EBS volume.
kms:GenerateDataKeyWithoutPlaintext Write To perform the custom CMK flow while exporting an EC2 instance or EBS volume.
kms:ListKeys Read To display Key Management Service (KMS) keys for the AWS account in a region.
kms:ListAliases Read To display KMS aliases for the AWS account in a region.
kms:ReEncryptFrom Write To perform the custom CMK flow while exporting an EC2 instance or EBS volume.
kms:ReEncryptTo Write To perform the custom CMK flow while exporting an EC2 instance or EBS volume.
s3:ListAllMyBuckets Read To list buckets.

It is used when listing S3 bucket endpoints.
s3:ListBucket Read To list bucket objects and check whether the bucket exists.

It is used in the download job.
s3:AbortMultipartUpload Write To perform the multipart upload.

It is used in the file recovery job.
s3:CreateBucket Write To create a bucket.

It is used in the file recovery job.
s3:PutObject Write To upload objects.

It is used in the file recovery job.
s3:PutBucketTagging Write To add tags to S3 buckets.

It is used in the file recovery job.
ec2:ModifySnapshotAttribute Write To share EBS snapshots with other AWS accounts for cross-account replication.
ebs:ListSnapshotBlocks

ebs:ListChangedBlocks

 Read

Requires the resource to have the rk_component tag with the value set to "Cloud Native Protection".

 To get the indexes of blocks with data or the changed block indexes that are used for optimal archival.
s3:PutObjectTagging Write To add tags to s3 objects.
ec2:GetEbsEncryptionByDefault Read To check whether EBS encryption by default is enabled in the AWS customer account and region.
ec2:CreateSecurityGroup Write To create a no-connectivity security group for exporting AWS virtual machines that are powered off.
iam:ListInstanceProfiles Read To list the IAM instance profiles to associate with EC2 instances for s3 bucket access.
ec2:AssociateIamInstanceProfile Write To associate an IAM instance profile with an EC2 instance during an EC2 export.
ec2:ReplaceIamInstanceProfileAssociation Write To replace an IAM instance profile for the EC2 instance during an EC2 export.
