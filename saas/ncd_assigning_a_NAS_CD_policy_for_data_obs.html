Assigning a NAS CD policy for Data Observability

Assign a NAS CD protection policy to a NAS data set for enabling Data Observability.

Before you begin

Create a NAS CD policy to send backups to a storage target, as described in Creating a NAS CD policy for Data Observability. For Data Threat Analytics, you may use Rubrik Cloud Vault or any other storage provider of your choice, but for Data Security Posture, you can only use Rubrik Cloud Vault.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
  3. On the navigation bar, click Infrastructure > Clusters.
  4. Click a NAS CD cluster.
    The NAS CD Systems page appears in a new tab in the web browser.
  5. Click a storage system.
    The storage system displays the list of SMB shares and NFS exports.
  6. Click an SMB share or NFS export.
    A details window appears.
  7. In Protection, in the dropdown menu, select the policy that sends backups to the appropriate storage target and then click Apply.

Results

NAS CD assigns the policy to the selected share and the policy runs during the next backup window defined as the Enqueue Time in the policy.
