Adding Azure subscriptions for configuring Data Security Posture

Add a new Azure account to RSC and configure Data Security Posture. You can use the insights to measure and manage your data security risk.

Procedure

  1. Log in to the Azure subscription that contains the workloads to be protected.
    The Azure portal page appears.
  2. Click and select Settings.
  3. Click Cloud Accounts and select Azure.
    The Azure page appears.
  4. Click Add Azure Subscription.
    The Add Azure Subscription wizard appears and displays the Select use case page.
  5. Select Cloud Management and click Next.
    This account is used for the storage, protection, and Data Security Posture use cases.
    The Cloud Management Use Cases page appears.
  6. Select Protection and compute resources and the cloud objects that you want to protect across the selected regions.
    Also ensure to select only the workloads not already installed for Protection and compute resources.
  7. Select Data security posture management (DSPM) to scan the data within the selected cloud objects and generate insights.
    Workloads selected under Protection and compute resources are the only ones to be secured by both Data Protection and Data Security Posture.
  8. Click Next.
  9. Select Configure with OAuth.
  10. In Azure AD Domain, select the Azure domain you want to configure Data Security Posture on.
  11. Click Log in with Azure.
    The Microsoft Azure portal login page appears.
  12. Log in to complete the steps and select Consent on behalf of your organization.
  13. Review the displayed information and click Accept.
    The list of subscriptions for the newly added Microsoft Entra ID tenant appears.
  14. From the list, select the subscriptions and click Next.
    You might not be able to locate subscriptions that already have some permissions added. Permissions are already installed on Azure subscription that do not display. You can remove the installed permission follow the steps to add them again or add the subscription ID directly.
    The Configure subscription page appears.
  15. In Regions, select the region or regions for the subscription.
    This option is available only for the regions selected for Protection and compute resources and Data Security Posture.
  16. Optional: Click Review Permissions to view the change in permissions after adding the subscriptions.
    Hovering over the icon adjacent to the account use case name displays the subscriptions affected as a result of differences in permission levels. The difference between the oldest and the latest version of a permission is also highlighted.
  17. Click Next.
    The Resource group details page appears.
  18. In Resource group name, enter the name of the resource group that is already created within the selected subscriptions.
    When you enter a new resource group, RSC creates that resource group.
  19. In Resource group region, select a region.
    If the resource group already exists, you must choose the existing resource group region.
  20. Optional: Click Add Resource Group Tag to add key value pairs for the resource group.
  21. Click Next.
    The Review subscriptions confirmation window appears.
  22. Review the resource group details and click Next.
    When the specified resource group name does not exist in the selected subscription, RSC creates a resource group for that subscription using the Azure API.
  23. Select the Azure subscription to designate as the outpost account and click Next.
    RSC uses this outpost subscription to perform data discovery and run classification scans.

    If you have already designated and configured an outpost account, the outpost account ID appears, and you can go to the next page in the wizard.

  24. Review the subscription details and click Submit.

Results

You have added an Azure subscription with Data Security Posture enabled. RSC provides data classification and threat information is now available for your data in the Azure subscription.
