  • Security

    Configure and manage RSC security features, including user access, SSO, Quorum Authorization, and data encryption.

    • Users and Access

      RSC provides role-based access control, and several methods for authenticating a user account.

    • Quorum authorization

      Quorum authorization is a security feature that enforces the requirement of getting additional approvals before a requester can perform data-modifying actions in RSC.

    • Single sign-on

      Rubrik Security Cloud supports SSO using the SAML 2.0 standard. With SSO, you can log in to RSC using credentials associated with an identity provider.

    • SAML 2.0 metadata files

      SAML 2.0 uses information from metadata files to exchange information between an identity provider and a service provider, such as RSC, to establish a trust relationship.

    • RSC SSO workflow

      RSC supports configuring single sign-on with an identity provider for secure authentication.

    • Okta integration workflow

      The integration workflow provides a high-level view of the requirements for configuration of SSO with Okta.

    • ADFS integration workflow

      The workflow provides a high-level view of the tasks involved in configuring SSO with ADFS.

    • Azure SSO

      Configuring single sign-on with Azure involves several individual tasks, which must be performed in the correct order.

    • OneLogin integration workflow

      The OneLogin integration workflow provides a high-level view of the tasks involved in configuring single sign-on with OneLogin.

    • IP allowlist

      RSC administrators can specify IPv4 addresses for the IP allowlist. RSC blocks unauthorized login attempts and unauthenticated API calls that originate from addresses that are not on the allowlist.

    • Authentication

      Rubrik Security Cloud authentication restricts access to a specified set of users.

      • Rubrik MFA with TOTP

        The Rubrik two-step verification feature uses time-based one-time passwords in addition to the user login credentials to enforce multi-factor authentication on user accounts.

        • Mandatory multi-factor authentication

          Multi-factor authentication enforcement is mandatory for all user accounts in your RSC environment.

        • Configuring MFA for your account

          Increase the security of your account by enabling multifactor authentication.

        • Managing Rubrik two-step verification enforcement

          An administrator or RSC account owner can enable or not enable enforcement of multifactor authentication through Rubrik two-step verification.

        • Enforcing Rubrik Two-step verification for specific users

          A user with the required privileges can enforce multifactor authentication for selected local user accounts.

        • Changing the TOTP device

          Rubrik Two-Step Verification uses time-based one-time passwords generated by a user device as an additional authentication factor. A user can change which device is used to generate this authentication factor from RSC.

        • Resetting Rubrik Two-step verification

          Reset the multifactor authentication configuration for a user to reconfigure Rubrik Two-step verification from the beginning.

        • Passkeys for MFA

          Passkeys are available as an alternate method of MFA in RSC. Passkeys use biometrics or hardware security keys to verify user identity. After configuration, passkeys become the primary method of MFA for users to log in to RSC.

          • Creating passkeys

            Create a platform or roaming security passkey to enable web authentication for your RSC user account.

          • Removing passkeys

            Remove a passkey when it no longer meets your requirements or you are no longer using it.

          • Editing passkey names

            Edit a passkey name to match with your requirements and make it easily identifiable.

          • Turning off passkeys

            Turning off one or both types of passkeys when you no longer want to allow them for web authentication.

          • Resetting passkeys for a user

            When a user is locked out and is unable to authenticate access using any of the preconfigured passkeys or TOTP, the administrator can reset passkeys to enable user authentication.

    • Certificate management

      RSC Certificate Management provides a centralized dashboard to manage certificates across all connected Rubrik clusters and for RSC workflows.

    • Encryption management

      With the encryption management feature in RSC, you can view information about data-at-rest encryption for the Rubrik clusters and archives, access to your key management services, and enhance data security through automated key rotation every 30 days.

    • User account lockout

      The user account lockout feature enhances system security by providing global administrators the ability to lock and unlock user accounts. This feature also prevents unauthorized access from brute force attacks and safeguards local user accounts by automatically locking them after a configurable period of inactivity.

    • Session management

      The session management feature enhances system security in the following ways: ending inactive user sessions, ending active user sessions after a specific duration, and limiting the number of concurrent sessions allowed for each user.

    • Password policy

      RSC administrators can enhance the security of their RSC domain by using the password policy feature to configure password complexity rules for local users.

    • New device login notification

      RSC notifies you when a new device is used to log in to your RSC account, providing details such as device name, IP address, and location.

Turning off passkeys

Turning off one or both types of passkeys when you no longer want to allow them for web authentication.

About this task

Only a user with the global administrator account can turn off passkeys.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Security and select MultiFactor Authentication.
    The MultiFactor Authentication page appears.

    An information appears for the global organization if global enforcement is enabled and warns that the tenant organizations must set a stricter configuration than the global settings.

  4. Clear the selection of one or more of the passkeys that are enabled.
    Clearing selection of the platform or roaming passkeys option prevents users who have lost their time-based one-time password (TOTP) device from logging in.
    The Disable Passkeys dialog box appears.
  5. Optional: Select the checkbox to retain all user-configured passkeys.
    If you do not select this option, all user-configured passkeys for the account will be turned off. The account administrator needs to re-enable the passkeys to make the passkeys available to the user for logging in.
