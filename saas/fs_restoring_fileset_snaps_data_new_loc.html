Restoring fileset snapshot data to a new location

Restore files or folders from the fileset snapshot to a location that is different from the original location of the data on the source host.

Before you begin

Do the following in the order specified:
  • Confirm the Rubrik Backup Service on the host is in a connected state.
  • Locate a fileset snapshot to recover, as described in Locating a fileset snapshot for recovery, or search for a file or folder to recover, as described in Searching for a file or folder for recovery.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click a fileset card based on the OS type of the host.
    • Windows filesets
    • Linux & Unix filesets
    The corresponding filesets page appears.
  5. In the Name column, click a host name.
    The local host page appears.
  6. In the calendar view of the Snapshots card, click a date.
    The Day view of the Snapshots card lists all the backups created on the selected date.
  7. In the Day view, click Recover files for the fileset backup to recover.
    The Fileset Recovery wizard opens the Select folders and files page, displaying the list of folders and files in the fileset.
  8. On the Select Recovery Type page of the recovery wizard, select Recover to separate folder.
  9. In Folder Path, enter the full path for a folder on the new source host location.
    Note:
    Do not enter the original path of the source folder or file. When Restore to separate folder is selected, the object cannot be restored to a folder that contains an object of the same name.
  10. Optional: Select Continue on restore errors.
    Selecting this option will ensure the Rubrik cluster proceeds with the restore job even if it encounters an error with a file, folder, or symlink. If you clear this option, the restore job will stop as soon as an error occurs, but any files already restored will remain on the restore target.
  11. Click Recover.

Results

RSC initiates a task to restore the fileset data to the specified location on the source host. The Events tab on the local host page tracks the status of the task.
