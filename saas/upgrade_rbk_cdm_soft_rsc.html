      RSC requires you to open an outbound connection through port 443 for a Rubrik cluster to access specific sites.

    • SMB security

      With the SMB security feature in RSC, a Rubrik cluster provides security for SMB shares by requiring Active Directory authentication and enforcing SMB security.

    • Rubrik cluster deployment

      Deploying a Rubrik cluster running on Rubrik appliance nodes involves installing the CDM software on the nodes and setting up that software for your environment.

    • Cluster registration

      Rubrik offers two modes for registering clusters with Rubrik Security Cloud, online and offline.

    • Rubrik MFA with TOTP

      The Rubrik two-step verification feature uses time-based one-time passwords in addition to the user login credentials to enforce multi-factor authentication on user accounts.

    • CLI administrator role

      As a CLI administrator, a Rubrik cluster user can connect to nodes in Rubrik clusters through the CLI Secure Shell (SSH) protocol.

    • Jumbo frame support

      For Rubrik CDM versions 9.4 and later, you can set the value for the maximum transmission unit to a maximum value of 10000, instead of the default value of 1500. This larger unit size is called jumbo frames.

Rubrik CDM software upgrade using RSC

After you have downloaded the Rubrik CDM software upgrade package, you can upgrade the Rubrik CDM version running on any connected Rubrik cluster to the latest version using RSC.

When you have successfully completed downloading the Rubrik CDM software upgrade package, the status under the Upgrade Status column on the Rubrik CDM Upgrades page changes to Ready for upgrade. At this point, you can initiate the upgrade task to upgrade the Rubrik CDM version running on any connected Rubrik cluster. Precheck task failure or upgrade failure can occur during the upgrade.

The following table categorizes the expected outcomes.
Outcome Description
The download and precheck operations were successfully completed with Ready for upgrade status, but the status changed to Precheck: action needed before you initiated the upgrade

RSC automatically runs a series of precheck tasks every 24 hours. Suppose you do not initiate the upgrade task immediately after a successful download operation; there is a possibility that a precheck failure may occur during the precheck tasks that RSC runs every 24 hours.

If a precheck failure occurs, the Precheck: action needed status appears under the Upgrade Status column on the Rubrik CDM Upgrades page.

The View precheck logs option in the ellipsis menu next to the cluster provides more information. The Precheck Logs dialog box displays the failed prechecks and corresponding causes and remedies. After you have resolved the precheck errors, it is mandatory to rerun the prechecks to continue the download and precheck operations.
One or more precheck tasks failed during the upgrade Rubrik runs the precheck tasks during the upgrade. If a precheck failure occurs, the upgrade task fails, and the Precheck: action needed status appears under the Upgrade Status column on the Rubrik CDM Upgrades page.

To proceed with the upgrade, it is mandatory to resolve the issues and rerun the prechecks from the Precheck Logs dialog box. Once the prechecks have been rerun successfully, you can initiate the upgrade task.
The upgrade operation failed An upgrade failure may occur for different reasons, including network issues or one or more cluster nodes not functioning correctly.

Identifying the root cause and taking necessary action will fix the error. After you fix the error, you can resume the upgrade from the ellipsis menu next to the cluster. If you are still unable to fix the error, you can create a Rubrik Support ticket for further assistance.

Suppose you want to revert to the previous Rubrik CDM software version, running before you downloaded an upgrade package for the latest version. In that case, you can use the Rollback option in the Upgrade Log dialog box. However, the Rollback option is available only for the fast upgrade.
