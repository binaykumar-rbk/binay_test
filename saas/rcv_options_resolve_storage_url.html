Options to resolve the storage URL

To use Rubrik Cloud Vault with a private endpoint, the Rubrik cluster must be able to resolve the storage URL to the private IP address of the endpoint created.

There are two options for enabling the Rubrik cluster to resolve the storage URL.

Option Description
When DNS entries are available You can update the DNS servers as described in the Azure documentation for using private endpoints for Azure storage and DNS for on-premises and Azure resources. Rubrik recommends this option for custom DNS entries.
When DNS entries are not available You can configure the Rubrik cluster host file to map the private endpoint IP address to the public storage URL. Rubrik documentation on enabling the Rubrik cluster to resolve the storage URL provides more information on this process.
