Private Container Registry for AWS

For security purposes, when you do not want to retrieve Docker images from a registry hosted by Rubrik, you can configure a Private Container Registry to retrieve these Docker images from a registry hosted by you.

A Private Container Registry (PCR) securely stores and manages Docker images within an organization, restricting access to authorized users and controlling the distribution of these images. When using Exocompute, RSC pulls the Docker images from an Amazon Elastic Container Registry (ECR) hosted by Rubrik. However, for security purposes, when you do not want Exocompute to retrieve Docker images into your environment from a registry hosted by Rubrik, you can launch an Elastic Kubernetes Service (EKS) cluster using images hosted in your PCR.
Important:
The PCR feature does not support Private Exocompute. If you enable PCR, the Exocompute workflow continues to use public endpoint access. For more information about the private Exocompute on AWS feature, see Exocompute on AWS.

Important aspects of PCR

The important aspects of PCR are as follows:
  • Security: Ensures secure storage and access control through authentication.
  • Control: Provides more control over the management and retention of Docker images.
  • Customization: Adapts to specific organizational needs.
  • Compliance: Meets regulatory and governance requirements with compliance checks.
  • Performance: Enhances performance by reducing latency and improving availability when hosted close to the deployment environment.
  • Scalability: Scales according to organizational demands to maintain reliable performance.

Workflow for using PCR

For Rubrik to retrieve the images from your PCR, the Docker images must be available in PCR. For the Docker images to be available in the PCR, it is important to complete certain tasks in a sequential manner:
  • Downloading the Docker images from an image bundle associated with a RSC release from a registry hosted by Rubrik.

    You can download the Docker images using the Rubrik GraphQL APIs exposed on RSC. The Rubrik API documentation provides more information about downloading the Docker images using the Rubrik GraphQL APIs.

  • Scanning the Docker images for Common Vulnerabilities and Exposures (CVEs) to meet your security requirements. This is optional.
  • Adding PCR while configuring an AWS Exocompute host account for a Rubrik-managed cluster:

    You can add the PCR when setting up an AWS account. This allows the retrieval of Docker images from the Rubrik-hosted container registry to the PCR. The specified AWS account must have the necessary permissions to authenticate with the Rubrik registry to download these Docker images.

  • Uploading the Docker images to your PCR.

    You must ensure that the images are tagged with the bundle version that you receive while downloading the Docker images using the Rubrik GraphQL API.

  • Notifying Rubrik about the approval or rejection of the image bundle.

    You can approve or reject the image bundle using the GraphQL APIs exposed on RSC. The Rubrik API documentation provides more information about approving or rejecting an image bundle using the Rubrik GraphQL APIs.

Versioning and support of image bundles

A new Exocompute image bundle with the Docker images is available after an RSC release. Each image bundle corresponds to an RSC release and has a semantic version.

The image bundle follows the <major>.<minor> semantic version scheme. Rubrik releases a new major version every month. For example, if Rubrik releases version 10.0, that is the main update for the month. Any updates Rubrik releases later that month are minor versions, so you might see 10.1 or 10.2 as Rubrik adds small fixes or improvements.

Rubrik supports each major version for two months, meaning Rubrik supports the current major version and the version from the previous month. For example, if the current major version is 12.0, Rubrik supports 12.0 (the version from this month), and Rubrik also supports 11.0 (the version from last month). However, Rubrik does not support older versions such as 10.0 and earlier.

Automation recommendation

Rubrik releases updated Exocompute image bundles with each RSC release. To ensure the latest Docker images are available in the PCR, Rubrik recommends automating the following PCR workflow tasks on a monthly schedule:
  • Downloading the Docker images from an image bundle
  • Scanning the Docker images for CVEs
  • Uploading the Docker images to your PCR
  • Notifying Rubrik about the approval or rejection of the image bundle

You can use tools such as Jenkins to automate this entire workflow.

Important:
  • If you do not update your Exocompute image bundle version monthly, your Exocompute jobs might fail.
  • Rubrik supports the current major version (n) and the preceding major version (n-1). For example, if the latest RSC release is version 10.x and your RSC version is 9.x, your Exocompute jobs will succeed. However, if your RSC version is 8.x or earlier, then your Exocompute jobs may fail.
