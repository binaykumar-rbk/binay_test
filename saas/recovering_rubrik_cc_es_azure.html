Recovering Rubrik Cloud Cluster Elastic Storage for Azure

Recover a Rubrik Cloud Cluster Elastic Storage to restore data even when you cannot retrieve the nodes. You cannot recover Live Mounts.

Before you begin

Before you recover a Rubrik Cloud Cluster Elastic Storage, ensure that the following conditions are met:
  • The cluster is registered with RSC.
  • Immutability is enabled.
  • None of the nodes are alive.
  • The cluster is disconnected and cannot be reconnected.

Procedure

  1. Contact Support to enable recovery of Rubrik Cloud Cluster Elastic Storage.
  2. Log in to RSC.
  3. Click and select Data Protection.
  4. On the navigation bar, click Infrastructure > Clusters.
  5. Next to the cluster you want to recover, click Rebuild.
    The Recover Rubrik Cluster dialog box appears.
  6. Click Azure Subscription and select the same Azure subscription.
    This must be the same subscription as the subscription used for the old cluster.
  7. Click Region and select a region for the cluster.
    This must be the same region as the region used for the old cluster.
  8. In Virtual Machine Resource Group , select the virtual machine resource group.
    You can modify this field as long as you have access to the storage, but it is recommended that you only modify subnet and NTP server information.
  9. In NIC Resource Group, select the NIC resource group.
    You can modify this field as long as you have access to the storage account, but it is recommended that you only modify subnet and NTP server information.
  10. In Subnet, select the subnet of the cluster.
    You can modify this field as long as you have access to the storage account, but it is recommended that you only modify subnet and NTP server information.
  11. In NTP servers, enter the address of an NTP server or accept the default server, pool.ntp.org.
    You can modify this field as long as you have access to the storage account, but it is recommended that you only modify subnet and NTP server information.
  12. In Email Address, enter the same email address for the administrator account.
    This must be the same administrator account used for the old cluster.
  13. In Admin Password, enter the password of the administrator account.
    This must be the same password as the administrator password for the old cluster.
  14. In Confirm Admin Password, re-enter the password to confirm it.
  15. Select I confirm that I want to rebuild the Rubrik cluster and understand that it's a disruptive operation.
  16. Click Rebuild.
    Recovery of the cluster starts, and the cluster downloads metadata from the storage account.

Results

RSC creates new nodes by transferring this metadata to new nodes. The backups continue as scheduled, and you can access all the snapshots on the cluster.
