    Detect and govern risk using Threat Analytics and Data Security Posture. Prepare for automated Orchestrated Application Recovery.

    • RSC Data Threat Analytics

      RSC Data Threat Analytics provides Anomaly Detection and data recovery services on protected resources.

      • Anomaly Detection

        Anomaly Detection examines a range of file system statistics to check for anomalies in file system usage patterns.

        • Enabling Anomaly Detection

          Enable Anomaly Detection to perform behavioral and content analysis on the file system and show details about potentially anomalous incidents.

        • Managing LLM analysis for Anomaly Detection

          Manage the use of LLM analysis for Anomaly Detection in RSC by disabling it when it is not required. To enhance the accuracy of Anomaly Detection, enable LLM analysis for Anomaly Detection.

        • Anomaly Detection features

          Anomaly Detection protects data by performing file system behavioral and content analysis.

        • Anomaly Detection behavioral model

          Anomaly Detection uses machine learning techniques and behavioral heuristics to build models that define normal baseline behavior and identify malicious data encryption patterns.

        • VM Encryption Detection

          VM Encryption Detection identifies encryption on the host, allowing you to receive anomaly alerts for encryption activity on virtual machines. You can use this information to quickly assess the blast radius and recover from such attacks.

        • Data Threat Analytics dashboard

          The Data Threat Analytics dashboard provides a comprehensive analysis of data threats including a summary of anomalies, threat monitoring matches, matched threat hunts for protected objects, and object timeline analysis.

        • Viewing Anomaly Detection

          View information about anomalous incidents detected by Data Threat Analytics.

        • Anomaly Detection filters

          The Anomaly Detection page provides information about objects that contain detected anomalies. You can use the filters to sort the data on the page.

        • RBAC for Anomaly Detection

          You can define specific privileges to enable RBAC in Anomaly Detection.

          • Creating a custom role for Anomaly Detection

            To grant specific privileges for Anomaly Detection and other security features, you can create a custom role and manually assign permissions.

        • Anomaly Detection details

          The Anomaly Detection details page provides detailed information about the anomaly and the object it was indicated on, files changed within the snapshot, and the event timeline.

        • Resolving an anomaly

          You can resolve anomalies that are not indicative of abnormal behavior after you complete your investigation.

        • Reporting a false positive anomaly

          You can report an anomaly as a false positive when it is not an indicator of any malicious activity and is triggered by an internal system process.

        • Investigating and recovering anomalous files for filesets

          View and recover suspicious, deleted, or modified files at the protected object level or at the level of individual files or filesets.

        • Investigating and recovering anomalous files for virtual machines

          View and recover suspicious, deleted, or modified files at the protected object level or at the level of individual files for virtual machines.

        • Full snapshot recovery of a virtual machine

          A full snapshot of a virtual machine can be recovered from the Anomaly Detection page.

        • Creating a Live Mount of a virtual machine for recovery

          Create a virtual machine from the snapshot.

        • Creating a Live Mount of a virtual disk snapshot for recovery

          Create new virtual disks from a snapshot using the Rubrik cluster as the datastore.

        • Performing an Instant Recovery of a vSphere virtual machine

          Replace the source virtual machine with a new virtual machine created from the selected snapshot.

        • Performing an Export from a snapshot

          Create a virtual machine from a snapshot for recovery.

        • Recovery of a batch of virtual machines

          When you want to recover multiple vSphere, AHV, and Hyper-V virtual machines from a snapshot in a single operation for Rubrik clusters, you can perform a batch recovery of the virtual machines.

        • Performing bulk recovery for Recovery Plans

          Recover virtual machines in multiple Recovery Plans.

        • Performing cyber recovery of virtual machines using Live Mount

          Select and recover virtual machines using Live Mount, based on threat hunting results, to a local, isolated recovery environment.

        • Performing cyber recovery of virtual machines using Export

          Select and recover virtual machines using export, based on threat hunting results, to a local, isolated recovery environment.

      • Data Threat Analytics events

        Data Threat Analytics provides filters and real-time search capabilities that can be used to find specific events.

      • Anomaly Detection reports

        Anomaly Detection reports provide information about the changes in snapshots for anomaly events. You can also download, save, or delete the reports.

      • Threat Monitoring

        Threat Monitoring automatically and continuously scans snapshots for new and existing indicators of compromise, and provides visibility into potential threats.

      • Threat hunts

        On-demand threat hunts scan Rubrik backups for user-defined indicators of compromise and identify the systems potentially affected by an attack.

      • Quarantine overview

        The Quarantine feature in Data Threat Analytics isolates potentially malicious files or objects to allow seamless data recovery and to avoid reinfection.

      • RBAC privileges for quarantine

        RBAC privileges are required to remove files or objects from quarantine, bypass quarantine, and download files for forensic analysis.

      • Quarantining matched files or objects

        Quarantine a matched file or object indicated in a threat hunt to separate potentially malicious objects or files before proceeding with recovery.

      • Viewing quarantined files or objects in threat hunts

        You can view threat hunts that have quarantined files or objects as well as the number of impacted snapshots.

      • Removing a quarantined file or object

        Remove a quarantined file or object to make it available for recovery.

      • Downloading quarantined files for forensic analysis

        Download quarantined files to examine and analyze them for malicious content.

Creating a custom role for Anomaly Detection

To grant specific privileges for Anomaly Detection and other security features, you can create a custom role and manually assign permissions.

About this task

The steps in this workflow ensure that you grant only the minimum required access while maintaining full functionality.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Navigate to Users and Access > Roles.
  4. Click Create Role.
    The Create Role wizard appears.
  5. Click Custom role.
  6. In Rubrik Application Privileges, click Configure.
  7. Under Anomaly Detection, select these privileges and then click Done.
    • View anomaly detection results
    • View file details
    These privileges enable comprehensive viewing of anomaly events and affected file details.
  8. In Assign System Configuration Privileges, click Configure.
  9. Select these privileges and then click Done.
    Privilege group nameRequired privilege and purpose
    Cluster configuration View clusters

    This is required for viewing detailed anomaly events and their corresponding cluster in the Events timeline.
    Events
    • View user activity
    • View audit logs

    These permissions are required for viewing the Anomaly Detection events.
    Reports All Reports

    This ensures that all Anomaly Detection reports are correctly assigned.
  10. In Data Management Privileges, click Configure.
    Ensure that the permissions are set to only View for the clusters or object types you want to monitor. Avoid assigning any protection or recovery privileges to security-focused roles.
  11. Assign the custom role to the appropriate users or groups.
  12. Click Done.
  13. Click Next.
  14. Enter a role name.
  15. Optional: Enter a description.
  16. Click Create.

Results

RSC creates and assigns the new custom role to users, granting them the necessary privileges to view Anomaly Detection events and reports while adhering to the principle of least privilege.
