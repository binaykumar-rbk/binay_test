Creating custom roles for NAS CD data sets by object type

Create a custom role in RSC with Role Based Access Control privileges for NAS CD objects with permissions configured by object type. These privileges provide granularity for determining the permissions to perform actions, and on which objects.

About this task

Creating custom roles by object type allows configuring access permissions for data sources depending on the object type you select.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Users & Access, then select Roles.
    The list of roles appears.
  4. Click Create Role.
    The Create Role wizard appears.
  5. Select Custom role.
  6. In Role name, enter the name of the role.
  7. Optional: Click Add Description, to enter a description of the role.
  8. In Data Management Privileges, click Configure to set the data management privileges for the role.

    The wizard displays the Data Management Privileges page.

  9. Select By object type and click Next.
  10. Select NAS Cloud Direct and click for one of the following options.
    OptionDescription
    All protectable objects
    For selected object types, permissions apply to one of the following options:
    • All existing and future protected NAS CD objects
    • Specific NAS CD objects

    You can add permission to view and restore from specific NAS CD systems in RSC.
    All recovery targets
    For selected object types, permissions apply to one of the following options:
    • All existing and future protected NAS CD recovery targets
    • Specific NAS CD recovery targets

    You can use the selected NAS CD objects as recovery targets from another share. You can recover to only the target locations that you have access to, as defined by the RSC role assigned to you by the RSC administrator.

    If the administrator does not specify a recovery target during role creation, you cannot recover to any target location.
    Full privileges
    For the selected objects, this option provides the following View permissions:
    • View protectable objects
    For the selected object, this option provides the following Recovery permissions:
    • Export files
    • Restore over original
  11. Click Next and then click Done.
    You can click Next to skip the following pages, which are not applicable for NAS CD objects:
    • Assign Datacenter Archival Locations
    • Select SLA Domains
  12. In Assign System Configuration Privileges, click Configure to set the system configuration privileges for the role.
  13. Optional: Select Federated access, View clusters, and then click Done.
    Note:
    The Federated access and View clusters are required permissions for accessing the NAS CD UI. If you do not want to provide access permissions to the NAS CD UI, select only View clusters.
  14. In Rubrik Application Privileges, click Configure.
  15. Optional: Select the necessary permissions for managing Rubrik application features.
    The Data Observability for NAS CD data sets section provides more information on the supported Rubrik application features.
  16. Click Create.
    A message appears indicating that the role is created.
  17. Optional: Click Go to Users Page.
    The Users page appears, displaying a list of available users.
  18. Click Done.

Results

RSC creates a new role based on your selections and refreshes the Roles page.
