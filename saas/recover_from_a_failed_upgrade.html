        The Rubrik two-step verification feature uses time-based one-time passwords in addition to the user login credentials to enforce multi-factor authentication on user accounts.

Recover from a failed upgrade

The Rubrik CDM upgrade process allows a paused upgrade to resume or a failed upgrade to roll back.

While performing the upgrade of a Rubrik CDM cluster, the upgrade operation may fail due to some issues.

Rubrik provides you with the following two options to recover from a failed upgrade:
  • Resuming an upgrade: You can resume the upgrade operation from the point at which the upgrade failed after resolving all issues.
  • Rolling back from a failed upgrade: You can restore a Rubrik CDM cluster to the pre-upgrade state.

The Rubrik CDM CLI Reference provides information about using Rubrik CLI commands that are associated with the CDM upgrade process.
