Adding API permissions to Microsoft Entra application

After you have created a Microsoft Entra application for Rubrik, you must add API permissions to that application.

About this task

After creating a Microsoft Entra application for Rubrik, you must specify the scopes of a web API in that application. This allows the application to obtain an access token containing those scopes from the Microsoft identity platform. After the web API receives the access token, it can provide permission-based access to its resources based on the scopes included in the token. This is a secure way to ensure that only authorized users can access the web API resources.

Procedure

  1. On the Microsoft Entra admin center, click Identity > Applications > App registrations.
  2. Click the Microsoft Entra application you created for Rubrik.
  3. Under Manage, click API permissions.
  4. Click Add a permission.
  5. Under Microsoft APIs, search for the following APIs and select the required permission.
    API namePermission to add
    Azure Service Management user_impersonation
    Azure Storage user_impersonation
    Azure SQL database user_impersonation
    Microsoft Graph User.Read under Delegated permissions
    You need to add the required permission for each API one by one.
  6. Click Add permissions after you have selected the required permission for the corresponding API.
  7. Click All APIs to go back to the Request API permissions page.
  8. Repeat the same steps to add the required permission for remaining APIs.

Results

You have added the required API permissions. On the API permissions page, under Configured permissions, you can view the added permissions.

What to do next

Add a client secret to the Microsoft Entra application for Rubrik, as described in Adding a client secret.
