          • RCV migration workflow

            Migration of archival locations to Rubrik Cloud Vault involves specific responsibilities for both the customer and Rubrik Professional Services, depending on whether the workflow is customer-hosted or Rubrik-hosted.

          • Customer-hosted migration process

            You can migrate an archival location to RCV by running the migration tool in your own AWS environment.

          • Rubrik-hosted migration process

            You can migrate an archival location to RCV by authorizing Rubrik to run the migration tool within the Rubrik AWS environment.

            • Preparing for Rubrik-hosted migration

              Prepare for a Rubrik-hosted migration by validating source location eligibility, disabling archival activities, and configuring encryption settings.

            • Configuring Rubrik-hosted migration

              Configure your environment to permit the Rubrik-hosted DataSync service to read data from AWS S3 or on-premises S3-compatible stores.

            • Running Rubrik-hosted migration task

              After the configuration is finalized, the Rubrik Professional Services team runs the migration task while you monitor its progress to completion.

            • Finalizing Rubrik-hosted migration

              Finalize the migration by coordinating with the Rubrik Professional Services team to convert metadata, run conversion commands, and re-enable the archival location.

        • Immutability conversion for Azure RCV locations

          Using RSC, you can convert non-immutable data center RCV locations on Azure to immutable locations.

        • RCV license replacement

          You can replace an existing Rubrik Cloud Vault license with a new license mid-term, ensuring backup and archiving continuity during the transition.

    • Cascading archival

    • Replication

      The replication feature directs the Rubrik cluster to send replicas of source snapshots or backups to a target Rubrik cluster and defines the maximum time to keep the replica on each cluster.

Preparing for Rubrik-hosted migration

Prepare for a Rubrik-hosted migration by validating source location eligibility, disabling archival activities, and configuring encryption settings.

About this task

Before you start the migration process, follow these steps to prepare the source location.

Procedure

  1. Work with Rubrik Professional Services to validate the source location eligibility.
    Note:
    If the RCV location region is different from the source location region, additional charges for cross-region transfer apply.
    The Rubrik Professional Services team validates the integrity of the data in the source location.
  2. In RSC, disable the source archival location and inform the Rubrik Professional Services team.
    The Rubrik Professional Services team waits for two hours to permit any ongoing jobs to complete. This prevents new backups from being written to the location during migration.
  3. Optional: Coordinate with the Rubrik Professional Services team to generate a list of files in the source location.
    The Rubrik Professional Services team can use this list to compare the files in the new bucket after the migration.
  4. Confirm that the Rubrik Professional Services team has run the backend tooling.
    The target RCV bucket and resources are created.
  5. Log in to the RSC API Playground.
  6. Update the encryption key by running the UpdateEncryptionKeyForRCVMigration mutation.

    You can use the following GraphQL to update the encryption key:

    // Mutation
mutation updateEncryptionKey($updateRcvTargetInput: UpdateEncryptionKeyForRcvMigrationInput!)
{
  updateEncryptionKeyForRcvMigration(input:$updateRcvTargetInput)
  {
    status
  }
}

// Inputs
{
  "updateRcvTargetInput": {
    "locationId": "<location-id>",
    "rsaKey": "<encryption-key>"
  }
}

    As an input, provide the locationId and the rsaKey for the source location. Providing these inputs allows RSC to manage encryption in the new RCV destination.

Results

The source location is prepared and validated, and the encryption configuration is finalized for the Rubrik-hosted migration.
