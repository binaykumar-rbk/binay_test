        • Key features for protecting AWS workloads

          RSC provides unified data management, automated global data protection, various recovery options for AWS workloads, and advanced security features to protect your workloads.

        • Common AWS protection terms

          It is important to understand some common terms when protecting AWS workloads with RSC.

        • Planning for AWS protection

          Before setting up an AWS cloud account in RSC, it is necessary to plan the user and access management for designated RSC users. This planning also involves reviewing the permissions that RSC requires in your AWS environment to protect your AWS workloads.

        • AWS cloud account setup

          You can set up an AWS cloud account in RSC and start protecting your AWS workloads.

        • Exocompute on AWS

          With the Exocompute on AWS feature, RSC launches an Elastic Kubernetes Service cluster to perform various operations to protect AWS workloads. RSC uses the Elastic Kubernetes Service cluster as a compute resource in your AWS account.

        • AWS storage tiering setup

          You can use AWS storage tiering to archive Amazon EC2 instance, EBS volume, or DynamoDB table snapshots that exceed a specified threshold to Amazon S3 storage classes. By archiving snapshots to these storage classes, you can optimize costs.

        • AWS workloads protection setup

          You can set up SLA Domains to apply data protection policies, ensuring automatic and hierarchical protection of AWS accounts and individual AWS workloads, with granular control overriding higher-level assignments.

        • AWS workloads recovery

          After RSC takes snapshots of your AWS workloads, you can recover the required workloads.

        • Amazon EC2 instances and EBS volumes

          After you set up protection for Amazon EC2 instances and EBS volumes, RSC takes automated snapshots of those instances and volumes based on the configured policies in SLA Domains. You can also take on-demand snapshots, refresh AWS accounts, and add AWS custom tags for better resource management.

Amazon DynamoDB details

Viewing and analyzing detailed information about Amazon DynamoDB tables helps in making informed decisions. Key aspects include protection status, detailed attributes, snapshots calendar, and events log for audit and compliance.

Analyzing detailed information about Amazon DynamoDB is vital for making informed decisions about your cloud infrastructure.

The following table describes various tabs available on the Amazon DynamoDB details page, providing information on protection status, events, and snapshot details:
Tab Description
Overview
  • Protection status: You can view protection statuses, such as the assigned SLA Domain, Exocompute configuration status, and whether file indexing is enabled. This ensures your Amazon DynamoDB tables are adequately protected.
  • Detailed attributes: You can find information about essential attributes, such as the AWS region, the associated AWS account, the table type, and size. This data is important for resource management and optimization.
  • Snapshots calendar: The calendar view displays snapshot information at different levels of granularity for the entire year, month, and day. Green dot indicators on specific dates signify that a snapshot was taken on that date, providing a clear visual representation of your backup history.
Events The Events tab displays recovery and snapshot events, providing a historical log that assists in audit and compliance processes. This information is essential for maintaining a secure and compliant cloud environment.
Snapshots The Snapshots tab provides a list of all the snapshots taken for the workload, specifying whether the snapshot is policy-based or on-demand, and the SLA Domain for policy-based snapshots. This information is important for tracking and managing backup operations because it allows you to verify that snapshots are being taken according to the SLA Domain and facilitates efficient recovery of the workload when needed.
