Amazon EC2 instances and EBS volumes details

Viewing and analyzing detailed information about Amazon EC2 instances and EBS volumes helps in making informed decisions. Key aspects include protection status, detailed attributes, snapshots calendar, and events log for audit and compliance.

Viewing and analyzing detailed information about Amazon EC2 instances and EBS volumes is essential for making informed decisions regarding your cloud infrastructure.
The following table describes the various tabs available on the details page of the EC2 instance and EBS volume, providing detailed information on protection status, Data Security Posture, events, and snapshot details:
Tab Description
Overview
  • Protection status: You can view protection statuses, such as the assigned SLA Domain, the configuration status of Exocompute, and whether file indexing is enabled. This helps ensure your EC2 instances and EBS volumes are adequately protected.
  • Data Security Posture: When Data Security Posture is enabled, you can view the sensitivity and sensitive hits for the specific EC2 instance or EBS volume. This helps in identifying and managing sensitive data to ensure security and compliance, providing key insights without affecting production systems.
  • Detailed attributes: You can find information about essential attributes, such as the AWS region, the associated Amazon VPC and AWS account, the type of EC2 instance and EBS volume, the capacity of the EBS volume, and attached tags. This data is important for resource management and optimization.
  • Snapshots calendar: The calendar view displays snapshot information at different levels of granularity for the entire year, month, and day. Green dot indicators on specific dates signify that a snapshot was taken on that date, providing a clear visual representation of your backup history.
Events The Events tab displays recovery and snapshot events, providing a historical log that assists in audit and compliance processes. This information is essential for maintaining a secure and compliant cloud environment.
Snapshots The Snapshots tab provides a list of all the snapshots taken for the workload, specifying whether the snapshot is policy-based or on-demand, and the SLA Domain for policy-based snapshots. This information is important for tracking and managing backup operations because it allows you to verify that snapshots are being taken according to the SLA Domain and facilitates efficient recovery of the workload when needed.

By leveraging these insights, you can optimize your EC2 instances and EBS volumes, ensuring efficient and secure cloud operations.
