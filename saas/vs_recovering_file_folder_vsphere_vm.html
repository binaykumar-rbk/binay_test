    • Data Center Workloads
      • AHV virtual machines

        RSC provides data management and protection for virtual machines deployed in a Nutanix Acropolis environment.

      • Hyper-V virtual machines

        A Rubrik cluster provides data management and protection for virtual machines that are deployed in a Microsoft Hyper-V environment. The Rubrik cluster can manage and protect virtual machines in an environment with multiple Hyper-V servers and virtual machines.

      • vSphere virtual machines

        Rubrik Security Cloud provides data management and SLA Domain protection for vSphere virtual machines, including vSphere VM templates that are managed either through a VMware vSphere environment or a content library.

        • vSphere VM templates

          Rubrik Security Cloud support selected backup and recovery operations for vSphere VM templates and content library items. These operations include file recovery, export, live mount, mounting virtual disks, and recovering virtual machine images as a template.

        • Application consistency

          You can set application consistency for vSphere virtual machines individually or in batch at different levels in the vSphere hierarchy.

        • vSphere virtual machine protection

          RSC provides protection for virtual machines through individual assignment of the virtual machine to an SLA Domain or through automatic protection.

        • vSphere management

          The Rubrik cluster accesses virtual machine data through a connection with the VMware vCenter Server or ESXi host that manages the hypervisor running the virtual machine.

        • vSphere diagnostics

          Rubrik Security Cloud vSphere diagnostics confirm connectivity and access permissions for a vCenter server connected to a Rubrik cluster. You can use vSphere diagnostics to verify and troubleshoot connectivity, backup, and recovery of a vSphere object or component.

        • VMware Cloud on AWS

          A Rubrik Cloud Cluster can protect virtual machines deployed on VMware Cloud on AWS.

        • Microsoft Azure VMware Solution

          A Rubrik cluster can protect virtual machines deployed on the Microsoft Azure VMware Solution.

        • Google Cloud VMware Engine

          A Rubrik cluster can protect virtual machines deployed on the Google Cloud VMware Engine.

        • VMDK files

          Virtual machines can include VMDK files that do not need to be protected.

        • vSphere virtual machine scripts

          RSC allows you to configure the Rubrik cluster to run scripts on a guest OS before a snapshot, after a snapshot, and after the Rubrik cluster completes the backup process.

        • Rubrik Backup Service

          The Rubrik Backup Service is software installed on a host that coordinates activity between the Rubrik cluster and the host.

        • On-demand snapshots of vSphere virtual machines

          In addition to policy-based snapshots, create virtual machine snapshots by using the on-demand snapshot process.

        • vSphere virtual machine details page

          The vSphere virtual machine details page provides detailed information about the virtual machine, protection of the virtual machine, available snapshots, and tasks related to the virtual machine and snapshots.

        • Recovery of vSphere virtual machines

          To recover a vSphere virtual machine, a Rubrik cluster mounts a point-in-time copy of that virtual machine.

          • Large file recovery with VMware tools and disk mount

            Large file recovery uses VMware tools and disk mount to restore large files efficiently, enhancing the speed for faster data recovery operations.

          • Recovering a file or a folder from a vSphere virtual machine

            When files are deleted, corrupted, or encrypted, recover files or folders from a vSphere virtual machine using snapshots, replicas, or archival snapshots for data restoration. RSC provides multiple recovery options, enabling seamless data retrieval and minimizing downtime.

          • Downloading virtual machine files

            Download virtual machine files from Rubrik clusters to perform data analysis.

          • Creating a Live Mount of a vSphere virtual machine

            When you are unable to access a vSphere virtual machine or when a virtual machine is deleted, create a virtual machine from a point-in-time copy of the source virtual machine on the vSphere VM page.

          • Creating live mounts in a batch

            When you are unable to access several vSphere virtual machines, or when multiple virtual machines are deleted and are required to be live-mounted together, create live mounts in a batch.

          • Creating a Live Mount of a virtual disk snapshot of a vSphere virtual machine

            A Live Mount creates a virtual disk or disks from point-in-time copies of the disks on a source vSphere virtual machine. The recovered virtual disks use the Rubrik cluster as the datastore.

          • Performing instant recovery for vSphere virtual machines

            Perform an instant recovery to replace the source vSphere virtual machine with a fully functional point-in-time copy.

          • Exporting a vSphere virtual machine on CDM clusters 8.0 and earlier

            To create a new vSphere virtual machine from a point-in-time copy of the source virtual machine, export a vSphere virtual machine on Rubrik CDM version 8.0 or earlier.

          • vSphere VM template export

            You can export a vSphere VM template snapshot as a virtual machine or as a VM template.

          • Exporting a vSphere virtual machine

            Export a vSphere virtual machine snapshot or a VM template managed through a vCenter folder or content library to create a point-in-time copy for backup or migration.

          • Exporting a vSphere VM template as a VM template

            You can export a vSphere VM template that is managed either through a vCenter folder or a content library to a new VM template. Exporting a VM template allows you to create a copy of a virtual machine that you can use to create and provision new virtual machines.

          • Powering off after Instant Recovery or Live Mount

            Power off a mounted virtual machine from the Live Mounts page of the RSC web UI.

          • Unmounting after Instant Recovery or Live Mount

            After performing instant recovery or creating a live mount of a virtual machine, remove the virtual machine from the Live Mounts page.

          • Removing a virtual machine entry after live migration

            After live migration of a recovered virtual machine, for example through the VMware Storage vMotion process, the Rubrik cluster maintains an entry for the migrated virtual machine on the Live Mounts page.

          • Migrating to a vCenter datastore

            Migrate the VMDKs for a live mounted virtual machine to a vCenter Server datastore. Migration moves the data and control of the VMDKs to the vCenter Server. Migration also frees up the Rubrik cluster storage that was used for the datastore.

          • In-Place Recovery of virtual machines

            VMware In-Place Recovery is a faster and more efficient alternative for virtual machine recovery, compared with other VMware recovery options.

          • Bulk recovery of vSphere virtual machines

            If you want to recover multiple vSphere virtual machines at the same time, you can use the vSphere VMs page to perform a bulk recovery operation.

        • Storage array integration

          Rubrik Security Cloud provides a user interface for integrating Rubrik clusters with storage arrays, reducing the time a virtual machine is quiescent during snapshot operations.

        • Continuous Data Protection

          Continuous Data Protection uses VMware vSphere APIs for I/O Filtering to continuously capture changes to data on a vSphere virtual machine and sends them to a Rubrik cluster.

        • Standalone ESXi host protection

          Rubrik supports protecting and restoring virtual machines on standalone ESXi hosts, providing continuous data security in environments that do not include a vCenter server.

Recovering a file or a folder from a vSphere virtual machine

When files are deleted, corrupted, or encrypted, recover files or folders from a vSphere virtual machine using snapshots, replicas, or archival snapshots for data restoration. RSC provides multiple recovery options, enabling seamless data retrieval and minimizing downtime.

Procedure

  1. Log in to RSC.
  2. Click and selectData Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click the vSphere VMs inventory card.
    The vSphere VMs page appears, displaying all the virtual machines on the Rubrik cluster.
  5. Optional: Filter the vSphere virtual machine type using the VM Type filter.
    Filter nameDescription
    Virtual Machine Filters vSphere virtual machines
    Template Filters vSphere VM templates managed using a vCenter folder
    Content Library Item Filters vSphere VM template managed using a vSphere content library
    You can also select multiple VM types.
    The list of VM types matching the filter selection appears.
  6. Click a virtual machine that has snapshots.
    The details page for the selected VM type appears.
  7. Click a date on the calendar to view the list of snapshots taken on that date.
    Alternatively, you can search for a specific file using the filename in the search bar. From the search results, you can select a version of the file to recover.
  8. Hover over a snapshot and click Recover.
    The Virtual Machine Recovery wizard starts, and the Select recovery option page appears.
  9. Select Recover files and click Next.
  10. Select folders or individual files to recover and click Next.
  11. Select one of the following recovery types.
    OptionDescription
    Download Downloads the selected files.

    For Download, enter a note to attach to the object type for this task in Add a note.

    Adding a note is optional. You can enter notes up to 1024 characters.
    Overwrite original Restores the selected file or folder to the original path. This option overwrites the existing file or folder at the original path.
    Recover to separate folder Restores the file or folder to a different location.

    For Recover to separate folder, in Folder Path, enter the full path of the restore location

    .
    Export Exports the files from the current virtual machine to a different recovery target.
    For Export, perform the following steps:
    • From Recovery Target Type dropdown, select a target type.
      Note:
      You can recover to a target type other than VMware VMs if you use a Rubrik cluster running Rubrik CDM version 9.3 or later.
    • For Export, from Recovery Target, select a target.
      Note:
      You can only recover to a target with the same operating system as the source virtual machine. RSC does not support recovery to a different operating system.
    • In Folder Path, enter the full path of the restore location.
  12. For Overwrite original, Recover to separate folder, and Export, select a recovery option.
    OptionDescription
    Rubrik Backup Service Downloads the Rubrik Backup Service software. You can use register to install the Rubrik Backup Service software on the virtual machine.
    VMware Tools Facilitates the use of VMware Tools for restoring data and Access Control List (ACLs), which requires guest OS credentials with sufficient privileges to perform the restore.
    Following are the options in VMware Tools:
    • Live mount snapshot disks temporarily to the target virtual machine for faster recovery of large files.
    • Use existing guest OS credentials for restoration.
    • Add new guest OS credentials. To do so, provide the domain details associated with the guest operating system, if applicable, along with a description to help identify the credentials or their intended use. You must also enter the username and password for the guest operating system. Optionally, you can use the Store credentials option for future use to save these credentials for future recoveries on all virtual machines. This option requires the Manage guest OS credentials permission.

      You can find the description only for Rubrik clusters running Rubrik CDM version 9.2 or later.

    • Select Continue on restore errors to proceed with the restore process even if a file, directory, or symbolic link fails to restore.
    Note:
    For Export recovery type, if you select the target type other than VMware VMs, you can only use Rubrik Backup Service (RBS) as the recovery option.
  13. Click Recover.
    A message appears that the request to download the file from the cluster has been successfully initiated.
  14. Click Done.

Results

RSC successfully recovers the selected files or folders using the selected recovery method.
