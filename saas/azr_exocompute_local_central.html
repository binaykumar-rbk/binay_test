Localized versus centralized Exocompute on Azure

Overview of deployment models for Exocompute on Azure.

When deciding on the deployment model for Exocompute, you can choose between localized Exocompute and centralized Exocompute. Localized Exocompute is suited for environments that require independent operation and tailored configurations for each application subscription. In contrast, centralized Exocompute is optimal when centralized management and efficient resource sharing are priorities.

The following table provides a comparative overview of the choices, highlighting their use cases, advantages, and considerations:
Aspect Localized Exocompute Centralized Exocompute
Overview Directly configures an Exocompute in an Azure application subscription that has resources you want to protect with RSC. Creates a central Exocompute host subscription in RSC and then maps application subscriptions to the Exocompute host subscription.
Use cases
  • Isolated environments: Each application subscription operates independently.
  • Custom requirements: Unique configurations for each application subscription.
  • Scalability: Independent scaling based on the workloads running in each application subscription.
  • Centralized management: Unified management of multiple application subscriptions.
  • Resource sharing: Efficient resource use across application subscriptions.
  • Streamlined configuration: Reduces repetitive configuration tasks.
Advantages
  • Autonomy: Complete control for each Azure application subscription.
  • Isolation: Minimizes risk of resource conflicts or dependencies.
  • Customization: Allows tailored configurations for specific needs.
  • Efficiency: Better resource utilization and reduced costs.
  • Simplified management: Easier to manage a single, shared setup.
  • Consistency: Uniform configurations across all application subscriptions mapped to the Exocompute host subscription.
Considerations
  • Management overhead: Requires maintaining separate configurations.
  • Resource utilization: Potential underutilization if subscriptions do not fully use capacities.
  • Shared dependency: Issues in the Exocompute host subscription could impact multiple application subscriptions.
  • Initial setup complexity: Requires thorough planning for accurate mapping and configuration.
