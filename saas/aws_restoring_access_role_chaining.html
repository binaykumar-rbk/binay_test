            You can manage the AWS cloud accounts available in RSC.

            • Viewing the AWS accounts page

              Access the AWS accounts page to view information about the cloud accounts created on RSC.

            • Updating an individual AWS account permissions by template

              Update permissions for an individual AWS cloud account to ensure that the required permissions for the new features are included in RSC. You can use the AWS CloudFormation template to update the permissions. You can also remove permissions that are no longer needed.

            • Updating permissions for AWS Organization accounts

              Update permissions for all the AWS cloud accounts that are added using an AWS Organization to ensure that the required permissions for the new features are included in RSC. You can use the AWS CloudFormation template to update the permissions. You can also remove the permissions that are no longer needed.

            • Updating AWS account permissions manually

              Update permissions for an AWS cloud account to ensure that the required permissions for the new features are included in RSC. You can update the permissions manually. You can also remove unneeded permissions.

            • Upgrading an AWS cloud account

              Upgrade an AWS cloud account in the "Update permissions" state to retrieve the current set of permissions.

            • Edit an AWS account

              You can edit an existing AWS account in RSC to change the data protection preferences for your AWS account.

            • Migrating existing AWS accounts to AWS Organization

              Migrate existing AWS accounts to an AWS Organization. When you choose to migrate an AWS account to an AWS Organization, RSC selects all the AWS accounts eligible for migration instead of providing an option to select which accounts to migrate.

Restoring access to a revoked role chaining account

After revoking access, you can restore RSC access to your AWS accounts by reverting the IAM trust policy modification on the AWS role chaining account.

Procedure

  1. Log in to the AWS Management Console for the account ID <Role chaining account ID>.
  2. From the AWS Services menu, open the IAM console.
  3. In the Roles section, select the role <Role name> that RSC uses.
  4. In the Trust relations tab, click Edit trust policy.
  5. In the Effect field, update the value to Allow.
  6. Save the policy.

Results

You have successfully reestablished RSC access to your AWS accounts. Role chaining is immediately restored, enabling RSC to resume secure interactions with linked AWS accounts. RSC reflects the updated connection status after the next automated check.
