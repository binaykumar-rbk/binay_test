ServiceNow ticket creation for Data Security Posture

You can create ServiceNow tickets directly from RSC to address data access issues in Data Security Posture. You can also provide additional contextual information through a protected link and view the status of tickets through this integration.

Note:
The ServiceNow integration for Data Security Posture is available as part of the Rubrik Enterprise Proactive Edition.

Users who observe sensitive data access issues in RSC might not be those who can take action to resolve the issues. The ServiceNow ticketing integration provides a seamless interface between RSC and the external ticketing platform. This integration allows ServiceNow ticket creation directly from RSC to address sensitive data access issues. You can also add relevant information the data administrator can use to fix the issue without accessing RSC.

The status of ServiceNow tickets is available in RSC for tracking, with an approximately 15-minute time lag. The status of a ServiceNow ticket can be one of the following:
  • Open
  • Closed
  • Pending, which shows that the ticket is being created
  • Creation in progress, which displays when RSC is waiting for the status from ServiceNow
  • Creation failed

Accessing this feature requires a login account with owner or administrator permission. This feature is available only for creating tickets to address data access issues identified by Insights in Data Security Posture.

The ServiceNow ticket creation workflow for data access issues identified by Data Security Posture involves the following:
  • Connecting a ServiceNow instance
  • Ensuring the prequisites are met
  • Creating a ServiceNow ticket in Data Security Posture
  • Tracking the progress of tickets in the Events page
