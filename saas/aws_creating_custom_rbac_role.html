Creating a custom role for AWS

Create a custom role with Role-based Access Control privileges for managing your AWS workloads. These privileges provide finer granularity for determining which permissions an RSC user has for performing specific actions on specific objects.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Users & Access and select Roles.
    The Roles page appears.
  4. Click Create Role.
    The Create Role wizard appears.
  5. Click Custom role.
  6. In Role name, enter a name for the role.
  7. Optional: Click Add Description.
    Enter a description for the custom role.
  8. In Data Management Privileges, click Configure to set the data management privileges for the role.
    The wizard displays the Data Management Privileges page.
  9. Select By object type and click Next.
  10. Select the AWS workloads for which the custom role requires permission.
    • AWS native - EC2
    • AWS native - RDS
    • AWS native - S3
    • AWS native - DynamoDB
    The default permissions grant the custom role full access to all the selected AWS workloads on RSC. For example, selecting AWS native - EC2 grants full access to all EC2 instances on RSC.
    Selected workloads display the All protectable objects, All recovery targets, and Full privileges buttons. By default, all the permissions are selected within the Full privileges button.
  11. Determine the objects, targets, and privileges for which you are granting permission.
    OptionDescription
    All protectable objects Use this option to select the source objects you want to protect using this custom role. When you want to select specific objects such as EC2 instances, EBS volumes, RDS databases, S3 buckets, DynamoDB tables, and AWS accounts, select them using the Select specific AWS objects option.

    To assign specific permissions such as view, protection, and recovery to this custom role, use the Full Privileges option.

    Using the Search by name field, you can narrow the search by specifying the object name.
    All recovery targets Use this option to select the recovery targets where you want to recover the snapshots. Users with this custom role can only recover to the selected recovery targets.

    To assign specific permissions to the selected recovery targets, use the Full privileges option. During recovery, you may need to perform operations on existing resources such as an S3 bucket, DynamoDB objects, or an EC2 instance in the target AWS account. For example, if you are recovering a snapshot to an S3 bucket that already exists in the target account, users with the assigned custom role must have the necessary recovery permissions for the target account to access and operate on that S3 bucket.

    Using the Search by name field, you can narrow the search by specifying the target name.
    Full privileges Use this option to set up data management privileges for the selected source objects and recovery targets.

    For example, you can select the View, Protection, and Recovery headings to grant all the privileges in these categories.
  12. Click Done.
  13. Click Next.
  14. Determine which archival location you are selecting.
    OptionDescription
    Select specific archival locations Use this option to select specific archival locations where you want the user with this custom role to archive the snapshots of the protected AWS objects.

    Filter the search by selecting from the Storage type, Archival Location Type, Rubrik Cluster, and Status lists.
    Select all existing and future archival locations Use this option to select all existing and future archive locations. By doing so, users with this custom role can use all current and future archival locations in RSC to archive snapshots of the protected AWS objects.

    Additionally, you can add permissions for this custom role to add, edit, delete, or manage archival locations.
  15. Click Next.
  16. Optional: Turn on Self-Service.
    Enabling self-service turns off the ability to select specific SLA Domains and specifies the manage permission for all existing and future SLA Domains.
  17. Determine the SLA Domains for which you want to grant permissions for this custom role.
    OptionDescription
    Select all existing and future SLA Domains Use this option if you want users with this custom role to have permissions on all existing and future SLA Domains. By doing so, the users with this custom role can manage these SLA Domains and assign them to the protectable AWS objects.

    Additionally, you can use the dropdown list to select specific permissions related to the management of SLA Domains. Evaluate the available options and select the one that aligns with your requirements.
    Select specific SLA Domains Choose this to select specific SLA Domains. Select the SLA Domains from the list you want this custom role to have access to, and click the Edit icon to assign the Manage permission, if required.

    Filter the search by selecting from the Assigned Orgs, Creator Org, and Managed By lists.
  18. Optional: For specific SLA Domains, select Don't protect if you want this protection setting to appear when managing protection for an AWS workload.
  19. Optional: For specific SLA Domains, select Clear current SLA Domain if you want this protection setting to appear when managing protection for an AWS workload.
  20. Click Done.
  21. In Assign System Configuration Privileges, click Configure to set the system configuration privileges for the role.
  22. In Cloud account management, under AWS, select View AWS accounts and Add AWS accounts.
    You can also select Edit AWS accounts and Delete AWS accounts, if required.
  23. Click Done.
    The Create Role page reappears.
  24. Click Create.
    RSC creates an AWS custom role with RBAC privileges and refreshes the Roles page.

What to do next

Assign the AWS custom role to the required RSC user.
