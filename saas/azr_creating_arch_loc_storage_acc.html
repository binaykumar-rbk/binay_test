Creating archival locations for Azure storage account

Create an archival location for the Azure storage account created for archiving the Azure virtual machine or managed disk snapshots and for storing Azure SQL backups.

About this task

Note:
An archival location created for a specific Azure workload is dedicated and cannot be used for other Azure workloads. For example, if you create an archival location to store Azure virtual machine snapshots and link it to an SLA Domain, you can then assign this SLA Domain to the Azure virtual machine. However, you cannot use this same combination of SLA Domain and archival location for Azure SQL workloads or Azure Blobs, as archival locations cannot be shared across different Azure workloads.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Archival & Storage and select Cloud Archival Locations.
    The Cloud Archival Locations page appears.
  4. Click Create Archival Location.
    The Create Archival Location wizard appears.
  5. Select Customer-controlled Archival Location and click Next.
    The Archival Location Region page appears.
  6. In Name, enter a name for the archival location.
  7. In Cloud Provider, select Azure.
  8. In Cloud Account, select the Azure cloud account that you have created for the storage use case.
    If the cloud account is not available, you can add a cloud account using the icon.
  9. Select a region type.
    OptionDescription
    Source region Archives the snapshots to the Azure storage located in the same region as the virtual machines or managed disks. This is a cost-effective option as it involves less data transfer cost.
    Specific region Archives the snapshots to an Azure storage located in a different region.

    Using the Specific region option is more expensive due to the egress charges. Select the Specific region option only if there is a need to maintain the snapshot copy in a different region.

    Selecting Specific region enables the Region field.

    Select a region.
  10. Click Next.
    The Storage account details page appears.
  11. For Source region, in Storage Account Prefix, enter a unique prefix string.
    Storage account prefix must contain numbers and lowercase letters only. RSC creates the storage account with the archived snapshots with the prefix entered in Storage Account Prefix.
  12. For specific region, in Storage Account Name, enter a name for the storage account.
    Storage account name must contain numbers and lowercase letters only. RSC creates the storage account for archiving snapshots with the Storage Account Name.
  13. In Container Name, enter a name for the container.
    Container name must start with a letter or number and must contain only lowercase letters, numbers, and the dash (-) character. The Container Name field does not appear if you are using Azure SQL version 2 or later.
  14. Select a storage tier from the following options:
    • Cool
    • Hot
    • Archive
    • Cold
    You cannot use Cold and Archive tier archival locations for Azure SQL backups.
    Note:
    RSC displays a warning if you select a snapshot retention period shorter than the minimum retention period required for a specific storage tier. However, when you select a shorter retention, you are still charged for the minimum retention period of the selected storage tier.
  15. In Redundancy, select the required redundancy option.
    Azure storage always stores multiple copies of the data to protect it from planned or unplanned events. The Azure Storage Redundancy topic in the Azure documentation provides more information about the redundancy options.
  16. In Network Access, select the network access configuration.
    OptionDescription
    All Networks Data flows over the public internet. This option is less secure but requires no additional configuration.
    Specific Virtual Networks (Exocompute's subnet) Data flows over the Azure backbone network through the Exocompute subnet. This option is more secure as you can allowlist only specific VNet and IP addresses, and restrict other VNet and IP addresses. Rubrik recommends using this option as it reduces data transfer costs and keeps traffic within the Azure environment.
    If you select this option, configure one of the following service endpoints for the subnet used by your Exocompute instance:
    • Microsoft.Storage.Global: Required for non-paired cross-region scenarios and also works for same or paired region scenarios.
    • Microsoft.Storage: Required for same or paired region scenarios.
  17. Optional: Click Add Storage Account Tag to add key-value pairs for the storage account.
  18. Click Next.
    The Custom storage encryption page appears.
  19. Optional: Turn on Custom storage encryption to use your custom keys to encrypt the snapshots.
    If you do not use the Custom storage encryption option, the default Microsoft-managed encryption key encrypts the snapshots.

    If you turn on Custom storage encryption and the Encryption permission category is not assigned to the Azure cloud account, the wizard displays a message specifying that this category is missing. You can modify the permission categories for an Azure cloud account to include the Encryption category.

    The Region field appears.
  20. In Region, select a region to assign an encryption model.
    You can select multiple regions. If an encryption type is not specified for a region, RSC assigns the platform-managed keys to that region by default.
    The Key Vault and Key fields appear.
  21. In Key vault, enter or select a key vault.
    If the user-assigned managed identity created by RSC does not have permission to access the selected key vault, then the key vault name is disabled. This can happen if the key vault has the vault-based permission model. You need to edit the permission model of a specific key vault to grant access to the user-assigned managed identity created by RSC. The minimum required permissions to access a customer-managed key in Azure Key Vault are wrapkey, unwrapkey, and get.
  22. In Key, select a key for encrypting the data.
  23. Click Create.

Results

RSC creates an archival location for the storage account and stores the keys and connection information.
