    • New device login notification

      RSC notifies you when a new device is used to log in to your RSC account, providing details such as device name, IP address, and location.

Performing a rekey

Perform a rekey of the master key and/or root key to update your encryption key to an alternative encryption key. This ensures that your data location uses up-to-date encryption.

About this task

A master key is a customer-owned key (KMS, passphrase, or TPM) that secures the key hierarchy. It is the topmost key that encrypts and secures the other keys within the hierarchy, consisting of only one root, one data, and several other Key Encryption Keys (KeKs). While the user provides the master key, Rubrik generates the root, intermediate, and data keys.

Rekeying updates these encryption keys to enhance security and prevent old keys from accessing protected data. This process is available only for master and root keys, not for other KEKs.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Security and select Encryption Management.
    The Encryption Management page appears.
  4. Click Archives.
  5. Click on the data location and select Rekey.
    You can select the Azure archival locations created on Rubrik cluster running CDM version 9.1.2 or later and AWS archival locations created on Rubrik cluster running CDM version 9.3.3 or later, respectively, for encryption.
  6. In Current Encryption Key Details, RSC displays the selected location and encryption key type, which cannot be edited.
  7. In Select KEK to Rekey, select one of the following options.
    OptionDescription
    Rekey master key Select this option to replace the master key with a new key to protect the root key. When you perform a rekey of the master key, you can enter a new RSA key or select a new KMS key as the rekey target.

    When you select KMS key as the rekey target, you can rekey either the master key or the root key. The master key is customer-owned, and the root key is Rubrik-owned. If you want to protect your data against compromise at Rubrik, you need to rekey the root key. Similarly, if you want to protect your data against any compromise in your environment, you need to rekey the master key.
    Rekey root key Select this option to replace the root KEK with a new key to protect the level 2 KEKs. When you perform a rekey of the root key, RSC initiates a Root KEK rekey job. You can view the status of the job on the Events page.
  8. Click Next.
    The Configure Rekey Master Key page appears. This page appears only if you have selected the Rekey master key option.
  9. In Current RSA Key, enter the RSA key to enable the rekey options.
    This option is available only if the location is currently encrypted with an RSA key.
  10. In Select a Rekey Option, select one of the following options.
    OptionDescription
    Enter a new RSA key In Enter a new RSA key, enter the RSA key in plaintext.

    Select the I've escrowed this key accordingly to my organization's security policies check box. This setting ensures that the RSA key is securely stored in compliance with security policies of your organization.

    Select the I understand that Rubrik isn't capable of retrieving this key for any purpose check box. This acknowledgment confirms that Rubrik is not responsible for retrieving the RSA key.

    Selecting both check boxes are required to proceed with a rekey.
    Select a new KMS key as the rekey target Select a new KMS key as the rekey target to enhance security with a more secure encryption key.
    • Azure Key Vault

      You must select values for Key Vault, Key ID, and Key Version for the new KMS key as the rekey target.

      When selecting a value for key version, you can choose a different key version or a different key ID if the same key ID is currently in use with the current key version.

    • AWS KMS

      You must select values for Key Manager and Key ID as the rekey target.
  11. Click Next.
    The Rekey Confirmation page appears.
  12. Review the summary information and click Rekey.
    A message appears confirming that the master key rekey has been initiated for the selected data location.
  13. (Rekey root key) In Select KEK to Rekey, choose Rekey root key and click Rekey.
    A message appears confirming that the root KEK rekey has been initiated for the selected data location.
  14. Click Okay.

Results

RSC initiates the rekey of the master Key Encryption Key for the selected data location.

You can monitor the status of the master Key Encryption Key process on the Event Logs page, which displays the status of succeeded or failed data locations. For a failed status, you can expand to view the detailed message to identify the issue and take corrective action.
