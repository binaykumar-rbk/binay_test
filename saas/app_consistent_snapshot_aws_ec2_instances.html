Application-consistent snapshot for AWS EC2 instances

Application-consistent snapshot is a feature that you can use to take snapshots that include data from pending I/O operations. You can use this feature with AWS to take snapshots that include data from pending I/O operations between the applications running on the AWS EC2 instance and the disk attached to the AWS EC2 instance.

A snapshot of an AWS EC2 instance is considered application-consistent if, in addition to capturing all of the data at the instant of the crash, the running applications have completed all their operations and flushed their buffers to the disk attached to the AWS EC2 instance. Rubrik recommends using application-consistent snapshot for database operating systems and applications such as SQL, Oracle, and Microsoft Exchange.

The application-consistent snapshot feature requires advance notice to the application so that the application can complete all of its operations and flush the buffers to the disk. Before a snapshot is taken, RSC provides advance notice to the application by running a pre-snapshot script. The application then ensures that all I/O operations are completed, buffers are flushed, and files are closed. After taking the snapshot, RSC runs a post-snapshot script to notify the application that the snapshot has been taken.

Important:
RSC does not support the archival of application-consistent snapshots of EC2 instances when using the centralized Exocompute configuration.
To take application-consistent snapshots for AWS EC2 instances, you must perform the following sequence of tasks:
  • Creating an Exocompute configuration to enable communications between RSC and the AWS EC2 instances

    RSC launches a temporary Exocompute instance based on the configuration to coordinate backup activity with the Rubrik Backup Service (RBS) agent installed on the AWS EC2 instances.

  • Downloading the RBS software package
  • Installing the RBS on the required AWS EC2 instance

    The Rubrik Backup Service section in the Rubrik CDM User Guide provides more information on how to install the RBS software package.

  • Registering the AWS EC2 instance running the RBS
  • Configuring pre-snapshot and post-snapshot scripts for the AWS EC2 instance
  • Enabling the application-consistent snapshot feature for the AWS EC2 instance
