Creating a fileset

Create a fileset to define a set of data in a file system. You can assign a fileset to a host to protect the data set specified by the fileset on that host.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click a fileset card based on the OS type of the host.
    • Windows filesets
    • Linux & Unix filesets
    The corresponding filesets page appears.
  5. Click Filesets.
  6. Click Add Fileset.
    The Add Fileset dialog box appears.
  7. In Cluster, select a Rubrik cluster.
    To assign a fileset to a host for protection, associate the fileset with the Rubrik cluster to which the host is connected.
  8. In Name, enter a unique name for the fileset.
  9. In Includes, enter a comma-separated list of values.
  10. Optional: In Excludes, enter a comma-separated list of values.
    The Rubrik cluster uses these values to determine which folders and files to remove from the fileset defined by the values specified in Includes.
  11. Optional: In Do not exclude, enter a comma-separated list of values.
    The Rubrik cluster uses these values to determine which folders and files to include back into the fileset from the folders and files removed based on the values specified in Excludes.
  12. Optional: For Linux and Unix hosts, select Follow Network Shares.
    The fileset includes the network shares that are mounted on the Linux or Unix host.

    The Backup hidden folders options appear.

  13. Optional: For Linux and Unix hosts, select Backup hidden folders.
    The fileset includes hidden folders.
  14. Optional: For Linux, Unix, and Windows hosts, click Enable pre/post scripts.
  15. In Pre-backup script path, enter a path to a script.
  16. Select Don't execute pre-backup script if the first backup attempt fails.
    Selecting this option prevents the pre-backup script from running again if the initial backup attempt fails. This option can be useful to avoid unnecessary script execution in scenarios where the first failure is due to issues external to the script, such as network connectivity problems or insufficient disk space, which the script itself cannot fix.
  17. Select Cancel backup if pre-backup script fails.
    Selecting this option automatically stops the backup process if the pre-backup script fails. This prevents the creation of inconsistent or corrupted backups, saving resources and ensuring data integrity.
  18. In Post-backup script path, enter a path to a script.
  19. Click Add.

Results

RSC creates and stores the fileset on the Rubrik cluster.

What to do next

Assign the fileset to a host to manage the protection of the data set included in the fileset.
