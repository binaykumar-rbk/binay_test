Creating a VNet

Create a VNet on the Azure portal to configure localized or centralized Exocompute with Rubrik-hosted ACR and Rubrik-managed AKS clusters.

About this task

An Azure Virtual Network (VNet) is the fundamental building block for private networks in Azure. Azure VNet enables Azure resources such as virtual machines to communicate securely with each other and over the internet.

Using the Azure portal is necessary to create a VNet for configuring localized or centralized Exocompute with Rubrik-hosted ACR and Rubrik-managed AKS clusters.

Procedure

  1. Log in to the Azure portal using your Azure account.
  2. Select Create a resource.
  3. In the Search the Marketplace field, enter the virtual network details and select the value from the search results.
  4. Click Create to start the process.
  5. In Subscription, select the appropriate subscription.
  6. In Resource group, select an existing resource group or create a resource group.
  7. In Name, enter the name of your VNet.
  8. In Network, select the region where you want to deploy the VNet.
  9. In IPv4 Address space, enter the IP address of your VNet.
  10. Click Add subnet and enter the subnet name in the Subnet name field and subnet address range in the Subnet address range field to configure subnets.
  11. Click Add to create the subnets.
  12. Optional: Add tags to organize your resources.
  13. Review your configuration settings.
  14. Click Create to create the virtual network.

Results

You have successfully created a VNet.

What to do next

Ensure that all networking requirements for Exocompute are met as described in Set up network for Exocompute on Azure and create an Azure cloud account for the Exocompute as described in Set up Azure cloud accounts.
