          • Manage Azure cloud accounts
        • Set up Exocompute on Azure

          With the Exocompute on Azure feature, RSC launches an Azure Kubernetes Service cluster to perform various operations to protect Azure workloads. RSC uses the Azure Kubernetes Service cluster to serve as a compute resource in your Azure subscription.

        • Set up Azure storage tiering

          You can use Azure storage tiering to archive Azure virtual machine or managed disk snapshots, which are older than a specified threshold, to Azure storage access tiers. By archiving snapshots to these access tiers, you can optimize cost.

        • Set up Azure workloads protection

          You can set up SLA Domains to apply data protection policies, ensuring automatic and hierarchical protection of Azure subscriptions, resource groups, and individual Azure workloads, with granular control overriding higher-level assignments.

        • Manage Azure virtual machines

          After completing the protection setup for Azure virtual machines and managed disks, RSC takes automated snapshots of those virtual machines and disks based on the configured policies in SLA Domains. You can also take on-demand snapshots, refresh Azure subscriptions, and add Azure custom tags for better resource management.

        • Manage Azure Blob Storage

          By using the protection services offered by RSC, you can protect and recover your objects in Azure Blob Storage.

        • Manage Azure SQL Database

          RSC enhances Azure native backup capabilities for SQL Databases by offering advanced, Rubrik-managed protection. This solution overcomes native limitations with customizable SLA Domain policies, on-demand snapshots, and flexible cross-subscription recovery, ensuring a consistent and automated approach to data security.

Cloud workload dashboard for Azure

The cloud workload dashboard for Azure provides a high-level view of the objects within your Microsoft Azure accounts, classified by sensitivity, region, and object type.

The following table describes the information panes on the Cloud workload dashboard.
Information pane Description
Object Protection Summary A doughnut graph showing the number of Azure objects classified by protection status and sensitivity:
  • Unprotected Objects: Number of Azure objects that do not have an SLA Domain assigned for protection.
  • RSC Protected Objects: Number of Azure objects that RSC protects.
  • Do Not Protect: Number of Azure objects that are excluded from all further SLA Domain assignments and protection.
The number of objects in each of these protection categories are further classified based on their sensitivity and Azure services. You can hover over a protection category to view the classification in a bar chart. Each bar represents an Azure service. You can click a service name to view the objects within the service on the Inventory page. Each bar shows the total number of objects within the Azure service broken down into the following statuses:
  • Sensitive objects: The number of objects that contain personal or business data that might carry high risk if compromised.
  • Non-sensitive objects: The number of objects that do not contain any personal or business data that might carry high risk if compromised. Exposure of such data does not damage the company, the clients, or the employees.
  • Pending sensitivity scan: The number of objects for which sensitivity scan is still in progress.
  • Scan not enabled: The number of objects for which you have not enabled sensitivity scan. You can enable data scanning for your Azure accounts to enable sensitivity scan.

You can click a category in the bar to view the list of objects on the Inventory page.
Unaccessed Objects A bar showing the number of objects that did not have any activity in the last 90 days. This bar includes only the objects in the Azure accounts where you have enabled data scanning. The bar is broken down into the following categories:
  • Sensitive objects: The number of objects that contain personal or business data that might carry high risk if compromised.
  • Non-sensitive objects: The number of objects containing no personal or business data that might carry high risk if compromised. Exposure of such data does not damage the company, the clients, or the employees.
  • Objects pending scan: The number of objects for which sensitivity scan is still in progress.

You can hover over a category in the bar to view the classification of objects based on the Azure services. You can click a service name to view the unaccessed objects within that service.
Relic Objects A bar showing the number of objects that were deleted in the production environment but still available in RSC snapshots. This bar includes only the objects in the Azure accounts where you have enabled data scanning. RSC breaks down the bar into the following categories:
  • Sensitive objects: The number of objects that contain personal or business data that might carry high risk if compromised.
  • Non-sensitive objects: The number of objects that do not contain any personal or business data that might carry high risk if compromised. Exposure of such data does not damage the company, the clients, or the employees.
  • Objects pending scan: The number of objects for which sensitivity scan is still in progress.

You can hover over a category in the bar to view the classification of objects based on the Azure services. You can click a service name to view the unaccessed objects within that service.
Cloud Objects by Region The global map displays the number of Azure objects based on regions. You can hover over the number in a region to view the breakdown of objects based on Azure services. You can click a service name to view the list of objects on the Inventory page.
Cloud Objects by Type A doughnut graph showing the number of Azure objects classified by the Azure services. You can click an area in the chart to view the objects within that Azure service on the Inventory page.
