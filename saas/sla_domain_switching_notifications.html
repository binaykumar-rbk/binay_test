      • SLA Domains switching

        • Switching SLA Domains

          To manage and protect multiple workloads across Rubrik clusters, switch SLA Domains from a Rubrik cluster to RSC SLA Domains.

        • Extending the SLA Domain switching period

          To have more time to plan switching your SLA Domain management from a Rubrik cluster to RSC, you can request a one-time extension of the switching period.

        • Switching all SLA Domains

          As a user with administrator privileges, switch all Rubrik cluster SLA Domain to Rubrik Security Cloud to manage all SLA Domains through a single platform.

        • Switching SLA Domains manually

          As a user with administrator privileges, manually switch Rubrik cluster SLA Domains to Rubrik Security Cloud to manage all SLA Domains through a single platform.

SLA Domain switching notifications

Information about notifications during SLA Domain switching in RSC.

The following table describes various notifications that occur during the SLA Domain switching in RSC.
Notification Description
Failure Indicates the number of failed SLA Domains when switching occurs for each cluster. Users can view details on the Switch SLA Domains page.
Success (individual clusters) Appears when all SLA Domains assigned to a workload are switched successfully for each cluster.
Success (across clusters) Appears when all SLA Domains assigned to a workload are switched successfully across clusters.
One-time extension approval notification after submitting request Confirms the approval of a one-time extension requested through the Need Help? option.
