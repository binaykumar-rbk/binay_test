      Quorum authorization is a security feature that enforces the requirement of getting additional approvals before a requester can perform data-modifying actions in RSC.

      • Key actors and responsibilities in quorum authorization

        Understanding the roles and responsibilities of key actors in quorum authorization management is important to ensure a secure and effective system.

      • Quorum authorization policy

        A quorum authorization policy is the means by which quorum authorization is enforced. To ensure that a single user does not run data-modifying actions, a quorum authorization policy specifies such actions that require additional approval.

        • Quorum authorization management actions

          Actions related to the management of quorum authorization policies and roles require additional approvals. For such actions, RSC enforces the requirement of additional approvals through policies. Only a user with the QAuth administrator role can approve or deny requests to perform these actions.

        • System configuration quorum authorization policies

          A quorum authorization policy for system configuration actions is a user-defined policy that prevents a user from running actions related to system configuration without the required approvals. The policy applies to actions related to system configuration for the connected Rubrik clusters and to global configuration actions for RSC.

        • Data management quorum authorization policies

          A quorum authorization policy for data management actions is a user-defined policy that prevents a user from running data management actions on specific objects without the required approvals.

        • Quorum authorization and multitenancy

          In a multitenant environment, when quorum authorization is enabled in the global organization, the quorum authorization policies also apply to the actions initiated in the tenant organizations.

      • Quorum authorization roles

        Quorum authorization roles in RSC provide structured management of approvals and permissions, ensuring appropriate reviews for system changes. The QAuth administrator role oversees quorum authorization activities. The QAuth approver role, which can be added to custom roles, allows users to approve specific policy changes.

      • Quorum authorization process

        The Quorum Authorization (QAuth) process secures actions related to data management and system configuration. This involves enabling QAuth, creating and associating QAuth policies and roles, requesting and reviewing actions, and optionally excluding service accounts to ensure automation scripts are not blocked.

      • Initial setup of quorum authorization

        The initial setup of Quorum Authorization (QAuth) involves identifying and configuring primary and backup QAuth administrators, ensuring they do not have conflicting privileges, and enabling the QAuth feature to manage secure user authentication and administrative requests.

      • Configuration of quorum authorization policies and roles

        Configuring quorum authorization policies and roles enforces comprehensive data governance, enabling RSC administrators to centrally control, customize, and secure data management and system configuration actions.

      • Quorum authorization requests and approvals

        When a requester attempts an action protected by a policy, RSC triggers a quorum authorization request. The requester cannot perform the action until the request is approved by a user with the QAuth approver role.

      • Retention lock and quorum authorization

        The retention lock feature helps organizations meet legal and compliance standards by providing Compliance and Governance modes, both requiring quorum authorization approvals for specific actions.

Quorum authorization management actions

Actions related to the management of quorum authorization policies and roles require additional approvals. For such actions, RSC enforces the requirement of additional approvals through policies. Only a user with the QAuth administrator role can approve or deny requests to perform these actions.

If Quorum Authorization (QAuth) is enabled, actions related to the management of QAuth policies and roles require additional approval. Users with the QAuth administrator role can approve or deny requests to perform these actions.
Management action Description
Disable QAuth A user with the global RSC administrator role requires QAuth approval from a QAuth administrator to disable QAuth for the RSC domain.
Assign QAuth role

A user with the global RSC administrator role requires QAuth to assign the QAuth approver role to user accounts.

Revoking a QAuth role does not require additional approval.
Edit QAuth role When the QAuth approver role is assigned to user accounts, a user with the global RSC administrator role requires additional approval to make the following edits to the QAuth approver role:
  • Adding permission to approve or deny QAuth requests
  • Adding or removing policies whose actions the reviewer can approve or deny

When the QAuth approver role is not assigned to any user account, edits to the role do not require QAuth.
Reset MFA configuration for QAuth administrator or QAuth approver user accounts A user with the global RSC administrator role or a role with the Manage User Access privilege requires QAuth to reset MFA for a user with the QAuth administrator and QAuth approver roles.

To reset their own MFA configuration, users do not require additional approval.
Edit QAuth policy A user with the global RSC administrator role requires QAuth to make the following edits to a QAuth policy:
  • Removing objects
  • Removing actions
  • Adding exempt service accounts

Adding objects and actions or removing exempt service accounts does not require additional approval.
Delete QAuth policy A user with the global RSC administrator role requires QAuth to delete a QAuth policy.
Unlock QAuth administrator or QAuth approver user accounts A user with the global RSC administrator role requires QAuth to unlock the locked user accounts with the QAuth administrator and QAuth approver roles.

Related tasks
  • Unlocking user accounts administratively
    Unlock a local user account or an SSO user account, regardless of whether the account was locked out due to administrative actions, brute force, or user account inactivity, if you have administrator or RSC account owner privileges.
  • Disabling quorum authorization
    Disable quorum authorization for your RSC domain to relax the requirement of obtaining additional approvals before allowing critical, data-modifying actions. Disabling quorum authorization cancels pending quorum authorization requests, deletes the configured policies and the QAuth Admin role, and removes the permission to approve or deny requests from the QAuth reviewer role.
  • Deleting a quorum authorization policy
    Delete a quorum authorization policy if you no longer need additional approval for the actions protected by the policy. Deleting a quorum authorization policy requires approval from a user account with the QAuth Admin role.
