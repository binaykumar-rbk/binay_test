  • Security

    Configure and manage RSC security features, including user access, SSO, Quorum Authorization, and data encryption.

    • Users and Access

      RSC provides role-based access control, and several methods for authenticating a user account.

    • Quorum authorization

      Quorum authorization is a security feature that enforces the requirement of getting additional approvals before a requester can perform data-modifying actions in RSC.

      • Key actors and responsibilities in quorum authorization

        Understanding the roles and responsibilities of key actors in quorum authorization management is important to ensure a secure and effective system.

      • Quorum authorization policy

        A quorum authorization policy is the means by which quorum authorization is enforced. To ensure that a single user does not run data-modifying actions, a quorum authorization policy specifies such actions that require additional approval.

      • Quorum authorization roles

        Quorum authorization roles in RSC provide structured management of approvals and permissions, ensuring appropriate reviews for system changes. The QAuth administrator role oversees quorum authorization activities. The QAuth approver role, which can be added to custom roles, allows users to approve specific policy changes.

      • Quorum authorization process

        The Quorum Authorization (QAuth) process secures actions related to data management and system configuration. This involves enabling QAuth, creating and associating QAuth policies and roles, requesting and reviewing actions, and optionally excluding service accounts to ensure automation scripts are not blocked.

      • Initial setup of quorum authorization

        The initial setup of Quorum Authorization (QAuth) involves identifying and configuring primary and backup QAuth administrators, ensuring they do not have conflicting privileges, and enabling the QAuth feature to manage secure user authentication and administrative requests.

      • Configuration of quorum authorization policies and roles

        Configuring quorum authorization policies and roles enforces comprehensive data governance, enabling RSC administrators to centrally control, customize, and secure data management and system configuration actions.

      • Quorum authorization requests and approvals

        When a requester attempts an action protected by a policy, RSC triggers a quorum authorization request. The requester cannot perform the action until the request is approved by a user with the QAuth approver role.

      • Retention lock and quorum authorization

        The retention lock feature helps organizations meet legal and compliance standards by providing Compliance and Governance modes, both requiring quorum authorization approvals for specific actions.

        • Governance mode QAuth policy setup

          Maintaining the integrity of retention-locked SLA Domains in Governance mode requires setting up quorum authorization with specific administrators and approvers. This involves configuring quorum authorization, creating policies, assigning custom roles, and ensuring proper approval processes for any actions on retention-locked SLA Domains.

          • Creating an Objects policy for retention-locked SLA Domains

            Create an Objects policy for retention-locked SLA Domains by specifying the actions you want to protect and the objects whose data you want to safeguard.

          • Creating an SLA Domains policy for retention-locked SLA Domains

            Create an SLA Domains policy for retention-locked SLA Domains by including the actions and SLA Domains you want to protect.

          • Creating a retention-lock custom role for QAuth approver

            Create a retention-lock custom role for QAuth approvers by configuring system privileges and selecting the Objects and SLA Domains policies you created for the retention-locked SLA Domains.

          • Assigning the retention-lock custom role to local users

            Assign the retention-lock custom role to local users by selecting the local users and editing their role assignments. This ensures that the designated users have the necessary permissions to approve quorum authorization requests for retention-locked SLA Domains.

Creating a retention-lock custom role for QAuth approver

Create a retention-lock custom role for QAuth approvers by configuring system privileges and selecting the Objects and SLA Domains policies you created for the retention-locked SLA Domains.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Users & Access > Roles.
    The list of roles appears.
  4. Click Create Role.
    The Create Role wizard appears.
  5. Click Custom role.
    The page to select the type of privileges for the role appears.
  6. In Assign System Configuration Privileges, click Configure.
    The Assign System Configuration Privileges page appears.
  7. Click Clear All to remove all permissions.
  8. Select Quorum authorization and then select all permissions under it.
  9. Click All policies next to Quorum authorization.
    The Select Policies dialog box appears.
  10. Select Select specific policies.
    A list of quorum authorization policies appears.
  11. Select the Objects and SLA Domains policies you created for retention-locked SLA Domains.
    For example, the Any RL object Objects policy and the All RL SLA SLA Domains policy.
  12. Click Done.
    The Assign System Configuration Privileges page reappears.
  13. Click Done.
    The Create Role wizard reappears.
  14. Click Next.
  15. In Role name and Description, enter a name for the new role and an appropriate description.
    For example, the role name can be Approve all RL actions.
  16. Click Create.

Results

RSC creates the retention-lock custom role for QAuth approvers.

What to do next

Assign the retention-lock custom role to the identified local users responsible for approving actions on retention-locked SLA Domains, as described in Assigning the retention-lock custom role to local users.
