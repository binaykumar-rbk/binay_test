Recovering point-in-time Informix backups

Recover an IBM Informix instance to the state of a specific transactional moment in time.

About this task

A point-in-time recovery restores the instance to a consistent state corresponding to a specific point in time.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click IBM Informix.
    The inventory page for IBM Informix appears.
  5. Click the instance to recover.
    The instance details page appears.
  6. In the Snapshots card, click a date with a green dot in the calendar view.
    Dates with snapshots are indicated by a green dot.
    The Snapshots card displays a timeline of available snapshots for that date.
  7. Slide the green bar to the time that you want to recover the Informix instance to, and click Recover.
    The Informix Recovery wizard appears.
  8. Specify the mount destination settings and click Next.
    OptionDescription
    Target IP Address or Host Name IP address or host name of the target mount destination.
    Directory Path Path to the mount directory.
    Mount Name in RSC (optional) Name of the mount as specified in RSC. If you do not provide a mount name, RSC uses a default naming format of Instance_Name + Snapshot_Time or Instance_Name + time specified in the recoverable range slider.
  9. Review the recovery configuration on the Confirmation page and click Recover.

Results

RSC initiates the point-in-time recovery job by preparing the backed up files on the Rubrik cluster, copying critical files to the input directory, and preparing setup files for the ON-Bar restore. Monitor the job status on the Events page.

What to do next

Run ON-Bar recovery commands from the Informix host system to finish the recovery.
