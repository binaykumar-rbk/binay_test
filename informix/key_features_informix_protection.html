Key features for Informix protection

RSC provides unified data protection, flexible recovery options, and advanced security for IBM Informix instances.

RSC offers a comprehensive suite of features to protect and manage your Informix instances for Rubrik CDM versions 9.5 and later.

Unified data management

Feature Supported CDM versions Description
RSC data protection 9.5.0 Provides a SaaS platform for managing and protecting Informix instances.
Consolidated monitoring and reporting 9.5.0 Offers unified reporting to track SLA Domain assignment, protection and recovery activities, and SLA Domain policy compliance for your Informix instances.
Comprehensive dashboard 9.5.0 Provides access for managing Informix instances, backups, and recovery operations from a single interface to reduce administrative complexity.
Cloud Cluster Elastic Storage support 9.5.0 Extends data protection capabilities for Informix instances deployed on cloud environments, such as AWS and Azure.

Automated global data protection through SLA Domains

Feature Supported CDM versions Description
Protection policies 9.5.0 Assigns and manages protection policies, known as SLA Domains, for Informix instances to ensure that backup and recovery operations meet defined business needs, RTO, or RPO requirements.
Automated backups 9.5.0 Uses SLA Domains to automate backups of your Informix instances, setting schedules to regularly protect your Informix data without manual intervention.

Backup and recovery

Feature Supported CDM versions Description
On-demand backups 9.5.0 Take a snapshot of an Informix instance when you need an immediate recovery point outside of the scheduled backup policy.
Instance-level backups 9.5.0 Perform instance-level backups to capture all dbspaces and associated data.
Archival storage 9.5.0 Add an archive policy to manage the long-term retention of snapshots by moving them to lower-cost archival storage, such as cloud object storage, for compliance and cost efficiency.
Data replication 9.5.0 Add a replication policy to replicate backup snapshots to other Rubrik clusters for disaster recovery purposes, ensuring offsite data availability.
Point-in-time recovery 9.5.0 Restore an Informix instance to the state it was in at a specific date and time.

Requires a combination of steps from the Data Protection dashboard in RSC and ON-Bar commands from the Informix host to complete the recovery.

Security and access control

Feature Supported CDM versions Description
Role-based Access Control (RBAC) 9.5.0 Offers granular permissions to administrators to manage data protection operations for Informix instances, ensuring security and compliance with organizational access policies.
Secure connectivity 9.5.0 Establishes secure communication channels for data transfer and management operations between Informix instances and RSC.

The methods for secure communication and data transfer between Informix instances and RSC are as follows:

  • Management and control operations using either socket-based or TCP-based authentication, and configuring SSL for secure connectivity.
  • Transferring backup data using secure thrift.
  • Restoring logs using secure thrift.
Quorum Authorization 9.5.0 Enforces authorized approvals to modify data protection decisions, such as policy changes and deletion requests, to prevent unauthorized actions and protect data integrity.
Retention lock 9.5.0 Enforces immutable retention periods on backups, preventing unauthorized deletion or modification of data for compliance and legal requirements. You can configure a retention lock while configuring the SLA Domain. You cannot reduce the retention period in a retention-locked SLA Domain, as it may lead to premature deletion of the snapshots.
