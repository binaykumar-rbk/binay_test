Manually performing an Informix recovery

Manually recover an IBM Informix instance to a specific snapshot or point in time.

Before you begin

Use RSC to make the backup images available to the Informix instance before starting this procedure.

To review the files and environment variables that will be accessed during this procedure, review the considerations for manual data recovery.

Procedure

  1. Log in to the Informix host as the root user.
  2. Use one of the following ON-Bar commands to verify that backups that were created by the ON-Bar utility are complete and can be restored.

      This abbreviated example shows that the last whole-system backup successfully completed.

  3. Verify that the Informix instance server is offline.
  4. Back up your current ixbar file.
    This example backs up the ixbar.0 file to a copy named ixbar.0_copy.
    cp /opt/Informix/etc/ixbar.0 /opt/Informix/etc/ixbar.0_copy
  5. Find the latest ixbar file.
    This example uses the Unix find command to search for the latest ixbar file in the restore-09-29-2025-at-08-45-45-pdt folder. 
    cp $(find /tmp/restore-09-29-2025-at-08-45-45-pdt/ -name "*ixbar" | sort | tail -n 1) /opt/Informix/etc/ixbar.0
  6. Find and delete the corrupted data from the original ixbar file.
    The corrupted data typically appears as a line of @ symbols at the beginning or end of the file, and will prevent ON-Bar from parsing the file correctly if they remain.
  7. Verify the availability of all files required for the restore operation.
    This example uses the onbar -v -t "time" command to find the files.
    RISM_PARAMS_FILE_PATH=/root/testmount/ae59c444-83ff-4022-b6bd-f6b9178551e6/rism_param_file RISM_LOG_LEVEL=4 RISM_LOG_FILE_PATH=/opt/Informix/tmp/rismp.log onbar -v -t "2025-09-19 15:39:13"
  8. Use one of the following ON-Bar commands to perform the recovery.
    You can use either of these methods to perform a point-in-time recovery:
    • Recover to a specific point in time. This method requires that you write down the current value of the LTAPEDEV configuration parameter for future reference, and then temporarily set the value of LTAPEDEV to dev/null in your ONCONFIG file to avoid log salvaging. Use the onbar -r -t command to start the restore operation. The -r parameter starts the restore operation and the -t parameter specifies the point in time to which you are restoring the Informix instance.
      This example recovers the instance using the snapshot from 2025-09-19 15:39:13.
      RISM_PARAMS_FILE_PATH=/root/testmount/ae59c444-83ff-4022-b6bd-f6b9178551e6/rism_param_file RISM_LOG_LEVEL=4 RISM_LOG_FILE_PATH=/opt/Informix/tmp/rismp.log onbar -r -t "2025-09-19 15:39:13"

      When the recovery is complete, you must revert the LTAPEDEV configuration parameter back to its original value.

    • Apply the logs to recover the instance to a specific point in time. For this method, first restore the physical backup by using the onbar -r -p -t command, then apply the logical logs with the onbar -r -l -t command.
      This example first backs up the instance to the snapshot for 2025-09-25 11:55:15 and then applies the logs to complete the recovery.
      RISM_PARAMS_FILE_PATH=/tmp/restore_11_55_15/0fce9c5e-a2d5-4939-8de0-855321f7ede8/rism_param_file onbar -r -p -t "2025-09-25 11:55:15"

RISM_PARAMS_FILE_PATH=/tmp/restore_11_55_15/0fce9c5e-a2d5-4939-8de0-855321f7ede8/rism_param_file onbar -r -l -t "2025-09-25 11:55:15"
    Use the following recovery method for a snapshot recovery:
    • This method requires that you write down the current value of the LTAPEDEV configuration parameter for future reference, and then temporarily set the value of LTAPEDEV to dev/null in your ONCONFIG file to avoid log salvaging. Use the onbar -r -w -t command to start the restore operation. The -r parameter starts the restore operation. The -w parameter indicates a whole system recovery, and the -t parameter specifies the date and time at which the snapshot was created.

    After a successful restore, the Informix instance is in a quiescent mode.

  9. To determine the status of the Informix instance, enter the onstat - command.
    This example displays a status of Quiescent for the Informix instance.
    onstat -
IBM Informix Dynamic Server Version 12.10.FC12 -- Quiescent -- Up 00:04:58 -- 256648 Kbytes
  10. To bring the Informix instance online, enter the onmode -m command.
