Workload configuration prerequisites when using Envoy with RSC

The hosts of the workloads must be assigned to the corresponding organization network to provide a secure connection to the Rubrik cluster.

Workload host configuration for use with the corresponding organization network has specific prerequisites:

  • Creating a tenant organization and organization network
  • Registering the Rubrik cluster to RSC and assigning it to one or more organization networks
  • Registering at least one Rubrik Envoy virtual appliance to each organization network.
  • If applicable, migrating existing workloads from the Rubrik cluster organization to RSC or removing the Rubrik cluster organization before adding workloads to RSC. A workload should not be in a Rubrik cluster organization and RSC at the same time.
  • To connect a host to an organization network, a user with appropriate permissions must log in to the tenant organization and use the Add host option from within the tenant organization. RSC then uses the organization network and corresponding Envoy virtual appliance. An administrator cannot use the Add host option to connect a host to an organization network when logged in to the global organization.
