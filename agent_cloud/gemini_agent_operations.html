          Connect AI platforms that have built-in agent capabilities to Agent Cloud. These platforms use cloud-to-cloud integrations that do not require deploying a model gateway.

          • Microsoft Copilot Studio deployed in Agent Cloud

            Connect Microsoft Copilot Studio to Agent Cloud to monitor Copilot agent activity, detect policy violations, and optionally enable real-time tool blocking that prevents unauthorized tool execution before it occurs.

          • Google Cloud Gemini Enterprise in Agent Cloud

            Connect Google Cloud Gemini Enterprise to Agent Cloud to monitor Gemini agent and reasoning engine activity and detect policy violations.

            • Google Cloud Gemini Enterprise requirements for Agent Cloud

              Connecting Google Cloud Gemini Enterprise to Agent Cloud requires specific Google Cloud and RSC administrative accounts, plus Google Cloud Identity and Access Management (IAM) settings.

            • Determining Domain Restricted Sharing policy enforcement

              Determine whether your organization enforces a Domain Restricted Sharing policy, and add an exception for the Rubrik service account or the Rubrik organization domain before connecting Google Cloud Gemini Enterprise to Agent Cloud.

            • Connecting Google Cloud Gemini Enterprise to Agent Cloud

              Connect Google Cloud Gemini Enterprise to Agent Cloud to discover agents and receive conversation history, giving you visibility into AI usage and potential security violations.

          • Claude Enterprise deployed in Agent Cloud

            Connect Claude AI Enterprise to Agent Cloud to monitor Claude agent activity and detect policy violations.

          • Microsoft 365 Copilot deployed in Agent Cloud

            Connect Microsoft 365 Copilot to Agent Cloud to monitor Copilot activity across Office applications and Microsoft Teams.

          • ChatGPT Enterprise deployed in Agent Cloud

            Connect ChatGPT Enterprise to Agent Cloud to monitor ChatGPT agent activity and detect policy violations.

          • Microsoft Foundry (new) deployed in Agent Cloud

            Connect Microsoft Foundry to Agent Cloud to monitor AI applications built and managed on the Foundry platform.

          • Microsoft Foundry (classic) deployed in Agent Cloud

            Connect Microsoft Foundry Classic to Agent Cloud to monitor AI applications on the previous version of the Foundry platform.

          • AWS Bedrock Agents deployed in Agent Cloud

            Connect AWS Bedrock to Agent Cloud to monitor Bedrock agent activity and detect policy violations.

        • Endpoint detection in Agent Cloud

          Endpoint detection connects Agent Cloud to AI agents running on managed devices — desktops and laptops enrolled in an MDM system — so Agent Cloud can monitor their activity.

        • Custom agents managed by Agent Cloud

          If your organization has built AI agents that call LLM APIs directly, connect them to Agent Cloud through a model gateway. A model gateway routes and monitors API traffic between your agents and the LLM, enabling Agent Cloud to enforce policies on custom agent behavior.

      • Onboard Agent Cloud

        After setting up your data plane and configuring your AI platform connections, complete onboarding in RSC to activate Agent Cloud in your environment. Onboarding creates the model router that connects your agents to the LLM providers Agent Cloud monitors.

      • Manage Agent Cloud policies

        Agent Cloud policies define the governance rules for AI agent behavior. Enable system policies for broad protection, then create custom policies for requirements specific to your organization.

      • Monitor AI agent activity

        Use the Agent Cloud UI to maintain continuous visibility into what your AI agents are doing, which tools they are using, and whether they are violating your policies.

      • Rewind AI agent violations

        When an AI agent action causes an unintended or unauthorized data change, use Agent Cloud's Rewind feature to reverse the specific change without restoring the entire system.

Google Cloud Gemini Enterprise in Agent Cloud

Connect Google Cloud Gemini Enterprise to Agent Cloud to monitor Gemini agent and reasoning engine activity and detect policy violations.

You are connecting Gemini Enterprise because your organization uses Google Cloud's Gemini models and reasoning engines. This connection gives Agent Cloud visibility into Gemini agent sessions and the actions they take. Work through the Requirements topic first, then follow the connection task.

Observability and logging

By connecting to Google Cloud Gemini Enterprise, Agent Cloud provides granular observability into AI agent behavior. Agent Cloud retrieves conversation history to display timelines of AI agent actions and correlate them with user identities. This visibility is essential for identifying policy violations or destructive actions that require remediation.
