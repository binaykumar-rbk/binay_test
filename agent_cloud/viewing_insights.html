    • Agent Cloud with RSC

      Agent Cloud is the RSC solution for governing autonomous AI agents — discovering them, monitoring their actions, enforcing policies, and reversing unauthorized data changes without taking systems offline.

      • Introduction to Agent Cloud
      • High-level implementation workflow

        Follow this end-to-end workflow to set up Agent Cloud, connect your AI platforms, configure governance, and begin monitoring AI agent activity in your environment.

      • Quick links

        Quick access to the most common setup, governance, monitoring, and rewind tasks in Agent Cloud.

      • Plan your Agent Cloud deployment

        Before connecting AI platforms or configuring policies, complete three planning steps: confirm your environment meets the requirements, review best practices for your AI agents, and decide where Agent Cloud's data plane will run.

      • Connect your AI platforms

        Agent Cloud connects to the AI platforms, frameworks, and agent infrastructure in your environment. The connection method depends on whether your AI agents use a native platform, endpoint detection, or a custom model gateway.

      • Onboard Agent Cloud

        After setting up your data plane and configuring your AI platform connections, complete onboarding in RSC to activate Agent Cloud in your environment. Onboarding creates the model router that connects your agents to the LLM providers Agent Cloud monitors.

      • Manage Agent Cloud policies

        Agent Cloud policies define the governance rules for AI agent behavior. Enable system policies for broad protection, then create custom policies for requirements specific to your organization.

      • Monitor AI agent activity

        Use the Agent Cloud UI to maintain continuous visibility into what your AI agents are doing, which tools they are using, and whether they are violating your policies.

        • Dashboards page

          The Agent Cloud dashboard gives you an immediate summary of your AI agent environment — agents discovered, high-risk agents, active violations, and active alerts. Start here when you open Agent Cloud.

        • Inventory page

          The Inventory section shows every AI agent, tool, and model that Agent Cloud has discovered across all your connections. Use it to understand what is running in your environment, review tool authorization status, and identify agents or tools that need attention.

        • Posture page

          The Posture section is where you triage and resolve policy violations. The Violations page shows what Agent Cloud flagged as non-compliant. The Insights page provides AI-generated analysis of agent behavior patterns with remediation recommendations.

          • Viewing violation details

            View details regarding policy violations, including the affected application, the tool used, and the underlying LLM model.

          • Marking a violation as safe

            Mark a policy violation as safe to create a policy exception that suppresses future alerts, violations, and blocks for a specific tool or agent.

          • Viewing Insights

            View the Insights page to review security recommendations and posture improvements identified by Agent Cloud.

        • Runtime page

          The Runtime section provides real-time visibility into agent sessions, individual tool calls, and alerts. Use it to investigate a specific incident, trace the sequence of actions in a session, or triage open alerts.

        • Connectors page

          The Connectors page shows all AI platforms and gateways connected to Agent Cloud — their status, tool blocking configuration, and the number of agents discovered through each connection.

        • Events page

          The Events page lists platform-level activity in Agent Cloud, such as policy updates, connection changes, and gateway configuration updates.

      • Rewind AI agent violations

        When an AI agent action causes an unintended or unauthorized data change, use Agent Cloud's Rewind feature to reverse the specific change without restoring the entire system.

Viewing Insights

View the Insights page to review security recommendations and posture improvements identified by Agent Cloud.

Procedure

  1. Log in to RSC.
  2. Click and select Agent Cloud.
    The Agent Cloud page appears.
  3. Click Posture > Insights.
    Agent Cloud displays the Insights page.
  4. Optional: Select an insight to view its details.
    Agent Cloud opens the insight details tray, which includes a description of the finding and recommended actions.

Results

You have viewed the Insights page.
