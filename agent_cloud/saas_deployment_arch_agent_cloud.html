SaaS deployment architecture for Agent Cloud

The SaaS deployment of Agent Cloud uses a fully Rubrik-managed architecture: AI agent traffic is brokered through a dedicated tenant in the RSC data plane. You can monitor and govern agents without operating any model gateway infrastructure in your own cloud account.

Architecture overview

The SaaS architecture consists of a Rubrik-managed control plane and a Rubrik-managed data plane. Both components run in RSC:

  • A Agent Cloud control plane that handles orchestration, business logic, policy management, and the user interface (UI).
  • A Rubrik-managed data plane that provisions per-account tenants, AI gateways, and log processors. The data plane intercepts AI agent traffic for connections that route through Rubrik, enriches session logs, evaluates policies, and stores the results in Rubrik-managed object storage.

You do not deploy or operate any infrastructure for the SaaS architecture. RSC provisions the tenant, gateways, and log processors the first time you use Agent Cloud. For details about the components that RSC provisions and the inputs that you supply, see Provision SaaS infrastructure.

Data flow

The data flow depends on the connection type. Agent Cloud supports two integration patterns for SaaS-tier connections:

This table describes the integration patterns that Agent Cloud uses to collect activity from SaaS-tier connections.
Integration pattern Description
Gateway proxy For connections to providers such as ChatGPT Enterprise, Claude AI Enterprise, Azure OpenAI, AWS Bedrock, AWS Bedrock AgentCore, and Google Cloud Gemini Enterprise, a Rubrik-managed AI gateway routes the AI agent traffic. The gateway records prompts and responses, enforces policies in real time, and forwards approved traffic to the provider endpoint.
API or webhook ingestion For vendor-managed SaaS platforms such as Microsoft Copilot Studio, Microsoft 365 Copilot, and Microsoft Foundry, Agent Cloud uses the OAuth credentials that you supply to read agent activity from the provider API. For LiteLLM deployments, the LiteLLM gateway sends webhook callbacks to Agent Cloud.

Regardless of the integration pattern, a managed log processor processes the raw activity, enriches the session, evaluates default and custom policies, and writes the enriched session log to Rubrik-managed object storage. The control plane ingests the enriched session log and makes it available in Agent Cloud for inventory, dashboards, alerts, and reporting.

Data privacy and security mechanisms

The SaaS architecture is designed for security and data privacy.

This table describes how the Agent Cloud SaaS architecture implements specific mechanisms to ensure data security and tenant isolation within the Rubrik-managed environment.
Mechanism Description
Tenant isolation Each RSC account is provisioned with a dedicated tenant that has a unique identifier and short code. Session logs, gateways, and log processors are scoped to the tenant and isolated from other RSC customers.
Encryption at rest Session logs in Rubrik-managed object storage are encrypted with tenant-specific data encryption keys. Tenant credentials and data plane authentication tokens are encrypted with the RSC key management service before they are stored in account settings.
Encryption in transit All traffic between AI agents, the data plane, and the Agent Cloud control plane is encrypted with Transport Layer Security (TLS). The control plane uses bearer tokens to authenticate to the data plane, which RSC rotates periodically.
Credential storage OAuth client secrets, API keys, and service account keys that you provide for AI connections are stored in the Rubrik-managed secrets store. Credentials are never returned to the UI or to API responses after they are saved.
Log retention Session logs are retained in Rubrik-managed object storage for a default retention period (for example, 60 days) and then deleted by a lifecycle policy. Contact your Rubrik account team to adjust the retention period for your tenant.
Real-time policy enforcement For connections that use the gateway proxy pattern, policies are evaluated inline with the request. Tool calls and responses that violate a policy are blocked before they reach the provider or the calling agent.

Regional considerations

Tenants and the resources they contain — AI gateways, log processors, and object storage buckets — are provisioned in the RSC region that matches your RSC account's location. Cross-region replication is not used for SaaS-tier session logs. See Region support for Agent Cloud for the list of supported regions.

Responsibility boundary

The following table summarizes which components Rubrik operates and which components you operate in the SaaS architecture.

This table describes the responsibility boundary between Rubrik and the customer for the SaaS architecture.
Operated by Components
Rubrik The Agent Cloud control plane, the Rubrik-managed data plane (tenants, AI gateways, log processors), Rubrik-managed object storage for session logs, and the secrets store for connection credentials.
You Your AI agents and the infrastructure on which they run, the provider accounts and subscriptions that host the AI services, and the application registrations, service accounts, and API keys that grant Agent Cloud access to your providers.
