Configuring AWS PrivateLink using the AWS CLI

Configure AWS PrivateLink to create a private endpoint in your VPC using the AWS CLI.

About this task

AWS PrivateLink creates a VPC endpoint in your VPC that tunnels traffic to the internal NLB of the data plane using the AWS backbone. The VPC endpoint provides the strongest network isolation.
When configuring the AWS Private Link, consider that:
  • The VPC Endpoint Service must exist before you can create a VPC endpoint.
  • The VPC endpoint consumes one ENI and IP address per subnet you specify.
  • Data transfer charges apply.

Procedure

  1. Ask your Rubrik representative for your VPC Endpoint Service Name.
  2. On the AWS CLI, enter the aws ec2 create-vpc-endpoint command to create the VPC Endpoint.
    This output shows the details required for the creating the VPC Endpoint.
    # Your workload VPC details
WORKLOAD_VPC_ID="<your-workload-vpc-id>"
WORKLOAD_SUBNET_IDS="<subnet-id-1>,<subnet-id-2>"
WORKLOAD_SG_ID="<security-group-allowing-443-outbound>"


# VPC Endpoint Service Name (from Rubrik or AWS Console)
SERVICE_NAME="<vpc-endpoint-service-name>"


aws ec2 create-vpc-endpoint \
  --vpc-id $WORKLOAD_VPC_ID \
  --vpc-endpoint-type Interface \
  --service-name $SERVICE_NAME \
  --subnet-ids $WORKLOAD_SUBNET_IDS \
  --security-group-ids $WORKLOAD_SG_ID
  3. Enter the aws ec2 create-vpc-endpoint command to create the VPC Endpoint.
  4. Notify Rubrik that you have created the VPC endpoint.
    If you created the VPC Endpoint in the same AWS account as the dataplane, AWS automatically approves the connection.
  5. Enter the aws ec2 describe-vpc-endpoint-connections command to view the pending endpoint connections.
    Run this command in the dataplane account.
    aws ec2 describe-vpc-endpoint-connections \
  --filters Name=service-id,Values=<endpoint-service-id>
  6. Enter the command to accept the endpoint connection.
    The dataplane account must accept the VPC endpoint connections from the service side.
    Run this command to accept the pending VPC endpoint.
    aws ec2 accept-vpc-endpoint-connections \
  --service-id <endpoint-service-id> \
  --vpc-endpoint-ids <vpce-id-from-above>
  7. Retrieve the VPC endpoint details and provide them to Rubrik.
    Enter this command to retrieve the VPC endpoint details.
    aws ec2 describe-vpc-endpoints \
  --vpc-endpoint-ids <vpce-id> \
  --query 'VpcEndpoints[0].DnsEntries[0].DnsName' \
  --output txt

Results

You have configured the VPC endpoint using the AWS CLI.

Rubrik uses the VPC Endpoint DNS or IP address to configure the CNAME record and TLS certificate for your HTTPS endpoint. Once configured, Rubrik provides your direct ingress hostname. Your applications use this HTTPS hostname. They should never use the raw DNS name or IP.
