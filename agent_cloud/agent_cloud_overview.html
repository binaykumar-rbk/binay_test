Agent Cloud with RSC

Agent Cloud with RSC provides a comprehensive solution for discovering AI agents in your environment, understanding which tools they have access to, detecting risky actions they perform on your data and configurations, and rewinding unauthorized changes without downtime.

Agent Cloud detects risky actions caused on your data by autonomous AI agents, helps you find their root causes, and rewinds the actions to repair changes to the data and configurations without downtime.

The challenges of agentic AI

Organizations using agentic AI, which are autonomous agents capable of executing tasks, face significant risks when these agents accidentally alter or delete critical data, leading to extended downtime and potential data loss. An unfortunate reality of agentic AI is a lack of visibility into agent actions, the data they affect, and clear path to recover the affected data.

For example, a support AI agent closes a Salesforce ticket and, as an enhancement, changes a date field from a string to a date, rendering Salesforce unusable for most users.

In another example, a nightly garbage collection agent, designed to optimize SharePoint storage, erroneously deletes recent data. Days later, a user reports missing files, prompting a help desk ticket. The IT team confirms the data loss but lacks crucial information: they do not know the extent of the missing data or that an agent caused the deletion. Furthermore, they have no assurance of a backup or a clear recovery path.

In both these situations, Agent Cloud can follow the AI agent changes back to the root cause, enabling you to revert to the changes, rendering the data correct.

How Agent Cloud solves these problems

RSC addresses these visibility and recovery gaps by providing a centralized pane of glass for monitoring AI agents across multiple providers and frameworks.

  • Root cause analysis: Agent Cloud tracks every action an agent takes, enabling you to trace an error, such as the SharePoint deletion, back to the root cause: the specific agent, the prompt, and tool that caused the error.
  • Monitoring: Agent Cloud enables you to monitor activity on your system. This can prompt critical questions about your AI environment, such as which AI agents are most active, which tools they are utilizing, and whether any specific interactions are violating your security policies.
  • Transaction rewind: Instead of rolling back an entire system, Agent Cloud enables you to rewind specific AI agent transactions. This selectively undoes the erroneous changes to data and configurations, restoring the system to a correct state while keeping other operations intact.
