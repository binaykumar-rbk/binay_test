Plan your Agent Cloud deployment

Before connecting AI platforms or configuring policies, complete three planning steps: confirm your environment meets the requirements, review best practices for your AI agents, and decide where Agent Cloud's data plane will run.

Planning is a short phase with three steps. Work through them in order — they set up everything that follows in the Connect and Onboard phases.

Step 1 — Confirm prerequisites

Agent Cloud requires an RSC account with the Agent Cloud feature enabled. If you plan to use the Rewind feature to reverse AI agent data changes, the target system — the application or database the agent modifies — must be actively protected by Rubrik Data Protection.

See Agent Cloud prerequisites.

Step 2 — Review best practices for your AI agents

Before you connect any AI agent to Agent Cloud, review the best practices for agent naming, tool design, and identity configuration. Applying these practices before deployment — rather than after — ensures Agent Cloud can accurately monitor, govern, and trace the actions of your agents. Some practices apply to all connection types; others apply specifically to custom agents.

See Best practices for AI agents managed by Agent Cloud.

Step 3 — Choose where your data plane runs

Agent Cloud's data plane processes agent traffic — prompts, responses, tool calls — and enforces the policies you configure. You choose one of two locations:

  • SaaS: Rubrik manages the data plane in a dedicated tenant. No infrastructure work required. Best for native integrations or evaluating Agent Cloud.
  • VPC (AWS or Azure): You deploy the data plane in your own cloud account. Agent traffic stays within your boundary. Required when data residency policy prohibits sending prompt and response data to Rubrik-managed infrastructure.

See Choose your data plane.
