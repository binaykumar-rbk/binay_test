Deploying AWS VPC on Agent Cloud

Deploy AWS VPC by provisioning the AWS VPC infrastructure in your AWS account. Rubrik Support completes the deployment by performing the final deployment steps

Procedure

  1. Connect to the playground on your RSC instance.
    Go to:https://your_tenant_name.my.rubrik.com/playground
  2. Use the mutation command to enable VPC support.
    Enter this in the edit field:
    mutation createVpcTenant($input: AirCreateVpcTenantInput!) {
  airCreateVpcTenant(input: $input) {
    id
    name
    shortcode
  }
  3. Enter this text in the Variables field: 
    {
  "input": {}
}
  4. Click the red button to execute the query.
    The text in the results field start with this structure:
    {
  "data": {
    "airCreateVpcTenant": null
  },
.
.
.
  5. Request your PredibaseEnvironmentID from your Rubrik representative.
  6. Sign in to your dedicated AWS Management Console.
    You should verify you are operating in the AWS region you intend to use for the VPC deployment.
  7. Navigate to CloudFormation.
  8. Click Create stack > With new resources (standard).
  9. From Specify the template, select Amazon S3 URL.
  10. Paste the CloudFormation URL in the text field.
    The CloudFormation URL is https://s3-us-west-2.amazonaws.com/predibase-public-us-west-2/infrastructure/rbrk-dataplane-preprovision.json.
  11. Click Next.
  12. Specify the stack details.
    Your Rubrik contact will provide you with the pre-configured values for the parameters on this page. Using incorrect values causes the deployment to fail.
  13. Click Next.
  14. Select the box titled I acknowledge that AWS CloudFormation might create IAM resources with custom names.
  15. Click Create Stack.
    The AWS Management Console creates the stack. This typically takes between four and five minutes to complete. When the stack is created, the AWS Management Console displays a status of CREATE_COMPLETE.
  16. In the AWS Management Console, select the stack you just created and navigate to the Outputs tab.
  17. Copy the value for the ARN key named CustomerEnvironmentRoleArn.
  18. Send the value for the CustomerEnvironmentRoleArn ARN key to your Rubrik contact.
    You can send the ARN value using your shared Slack channel or by using email.

Results

You have deployed AWS VPC on RSC by provisioning the VPC infrastructure in your AWS account.
When Rubrik Support receives the CustomerEnvironmentRoleArn, they complete the final provisioning steps to connect your AWS VPC infrastructure to RSC. These steps include:
  • Provisioning the infrastructure. The automated provisioning process takes approximately 20 to 30 minutes.
  • Verifying the infrastructure. Rubrik Support validates that the infrastructure was provisioned successfully and inform you that it is ready to use. This can take up to 48 hours.
  • Completing the deployment: Rubrik Support notifies you once the environment is connected and Agent Cloud is enabled and visible on RSC.
