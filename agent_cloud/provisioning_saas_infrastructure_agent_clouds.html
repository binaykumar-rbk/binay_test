Provision SaaS infrastructure

You chose SaaS deployment, which means Rubrik manages the data plane in a dedicated tenant. No deployment steps are required. This topic describes what Rubrik creates automatically and what credentials you provide for each connection type.

You chose SaaS deployment, which means Rubrik manages the data plane in a dedicated tenant. RSC automatically creates this infrastructure the first time you onboard an AI provider connection. Your only responsibility is to provide the credentials and access permissions that allow Agent Cloud to reach your AI platforms — the specifics depend on which platforms you are connecting.

The SaaS deployment option is intended for AI providers and agent platforms that you consume as a service, such as Microsoft Copilot Studio, Microsoft 365 Copilot, Microsoft Foundry, ChatGPT Enterprise, Claude AI Enterprise, Google Cloud Gemini Enterprise, and AWS Bedrock AgentCore.

With SaaS deployment, you do not host a model gateway or virtual private cloud (VPC) data plane in your own cloud account. Instead, RSC provisions and operates the data plane on your behalf in a dedicated tenant.

Components that Rubrik provisions

The following table describes the components that Agent Cloud provisions automatically when you first onboard a SaaS-tier account. You do not need to request or manage these components.

This table describes the components that Rubrik provisions for a SaaS-tier Agent Cloud account.
Component Description
Dedicated tenant Agent Cloud creates a tenant with a unique identifier and a short code. The tenant scopes all of your Agent Cloud data and and isolates it from other Agent Cloud customers.
Data plane authentication token An encrypted token that authorizes Agent Cloud to communicate with the tenant. Rubrik stores your account settings and rotated by Rubrik.
AI gateways Agent Cloud provisions one or more managed gateways to route requests to your configured model endpoints, such as ChatGPT Enterprise, Claude AI Enterprise, Azure OpenAI, and AWS Bedrock. Each gateway records prompts, responses, and metadata for policy evaluation.
Log processors Managed log processors:
  • Collect session logs from the gateways and connected agent platforms.
  • Encrypt them with tenant-specific data encryption keys.
  • Store them in Rubrik-managed object storage for analytics and reporting.
Regional resources Rubrik allocates compute and storage in the RSC region that matches your tenant's location. See Region support for Agent Cloud for supported regions.

What you provide

SaaS deployment requires only the credentials and access scopes that allow Agent Cloud to reach your AI providers and agent platforms. The specific inputs depend on the connection type.

This table describes the customer-provided inputs required for each connection type when you onboard Agent Cloud on a SaaS-tier account.
Connection Customer-provided inputs
Microsoft Copilot Studio, Microsoft 365 Copilot, Microsoft Foundry A Microsoft Entra ID application registration with an application (client) ID, client secret, tenant ID, and the required Graph and Copilot API permissions. For Microsoft Foundry, you also supply the subscription ID and the Application Insights resource URI.
Google Cloud Gemini Enterprise (Gemini Enterprise Agent Engine) A Google Cloud project identifier and a service account key (JSON) that grants the minimum IAM roles needed to list and read Gemini Enterprise agents and reasoning engines.
AWS Bedrock and AWS Bedrock AgentCore AWS credentials with read access to Bedrock, the target region, and the model or agent identifier.
Azure OpenAI The Azure resource endpoint, API key, and deployment name for each model you want to monitor.
Anthropic Claude, Claude AI Enterprise, ChatGPT Enterprise, and other OpenAI-compatible providers An API key, and a base URL for OpenAI-compatible endpoints.
Rubrik-managed gateways and LiteLLM For LiteLLM, the webhook callback URL and shared secret. For Rubrik-managed gateways, no credentials are required because the gateway is provisioned and operated by Agent Cloud.

When to use SaaS deployment

The SaaS deployment option fits when one or more of the following conditions apply:

  • Your AI agents run on vendor-managed SaaS platforms, such as Microsoft Copilot Studio, Microsoft 365 Copilot, or ChatGPT Enterprise, where you do not control the underlying infrastructure.
  • You consume foundation models directly from a provider API, such as Anthropic Claude or OpenAI, and do not require traffic to remain inside your own cloud boundary.
  • You want to minimize the operational overhead of deploying and maintaining a model gateway in your own AWS account or Azure subscription.
  • You are evaluating Agent Cloud and want the fastest path to onboarding without provisioning customer-managed infrastructure.

If you require sensitive prompt and response data to remain inside your own cloud boundary, use the VPC deployment option instead. See Choose your data plane for a comparison of deployment options.
