Key capabilities

Deploying Azure VPC includes these key capabilities when deployed inAgent Cloud
  • Data sovereignty: The VPC processes all sensitive agent traffic, such as prompts and responses, locally within your Azure cloud. This ensures that private data remains under your control and is never exposed outside your cloud environment.
  • Hybrid management: You utilize a hybrid operational model where the data plane resides in your Azure subscription, but all orchestration, business logic, and policy management are handled through RSC.
  • Automated provisioning: You provide access to Agent Cloud so it can deploy the required resources directly into your subscription. This automated setup streamlines the creation of the gateway infrastructure while ensuring it meets security standards.

When to use Azure VPC

You can deploy the Azure VPC when your custom AI agents and associated infrastructure are hosted in Azure. This minimizes latency by keeping the gateway close to your AI agents and enables you to leverage your existing Azure security configurations and compliance controls.

Deploying Azure VPC on Agent Cloud

Deploying Azure VPC in Agent Cloud creates the following:
  • A resource group: This is the security boundary for all VPC data plane resources. This resource group is dedicated to RSC and should not contain other workloads.
  • A custom RBAC role: The Rubrik VPC Dataplane Provisioner role includes service-level wild card permissions scoped to the resource group. This role can manage Kubernetes, container registries, key vaults, networking, storage, and identity resources within the resource group, but cannot access any other location in your subscription.
  • A role assignment: Agent Cloud assigns this role to the Rubrik service principal with an RBAC condition that restricts role assignment write and delete permissions to pre-approved built-in roles. Agent Cloud cannot assign Owner, Contributor, or anything outside this allowlist.
