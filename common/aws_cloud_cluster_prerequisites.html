Prerequisites for Rubrik Cloud Cluster ES for AWS

Setting up a Rubrik Cloud Cluster Elastic Storage for AWS requires that you have an AWS account, a Rubrik license, a compatible web browser, and a VPC endpoint with internet connectivity. Additional requirements include an SSH key pair, an S3 bucket with specific settings, an instance profile, an IAM role for S3 access, and a security group with configured rules.

The Amazon documentation provides more information about setting up AWS prerequisites.
The following table describes the settings and components required to provision Rubrik Cloud Cluster ES on AWS.
Requirement Description
AWS account The configured AWS account accesses the data sources that the Rubrik Cloud Cluster ES manages and protects.
Rubrik Cloud Cluster ES license You can purchase a license from a Rubrik reseller or from the AWS Marketplace.
Web browser Rubrik supports the latest version of Google Chrome and the previous two versions.
VPC endpoint for Rubrik Cloud Cluster ES The VPC endpoint creates a network link between the VPC and S3 so that the Rubrik Cloud Cluster ES can access the S3 bucket used for data. The configured VPC endpoint has access as the policy.
VPC The subnet must have internet connectivity to communicate with the Rubrik public RSC endpoints. This subnet must have connectivity to the wider internet to be able to communicate with the Rubrik public RSC endpoints.
S3 bucket The S3 bucket stores the data for the Rubrik Cloud Cluster ES. The S3 bucket must have the following settings:
  • The S3 bucket and the Rubrik Cloud Cluster ES are assigned to the same region.
  • The default encryption type used is server-side encryption.

    If you use AWS Key Management Service Keys (SSE-KMS), specify the encryption ARN key and update the JSON policy template. Replace <bucket name> with the actual name of the bucket, and replace <kms key arn> with the KMS key ARM.

  • Optionally, object locking can be enabled for immutability.
Instance profile An instance profile passes the IAM role to an EC2 instance when the instance starts. If you created an IAM role using EC2, AWS automatically generates an instance profile with the same name.

Otherwise, use the CLI or API to create an instance profile.
IAM role for S3 access RSC authenticates the Rubrik Cloud Cluster ES nodes with the S3 bucket. The IAM role is used only for Rubrik Cloud Cluster ES and has the following settings:
  • The name of the IAM role uses the following prefix: rubrik-cces-
  • AWS is the trusted entity.
  • EC2 is the use case.
The IAM role requires the following permissions to access the S3 bucket:
  • s3:AbortMultipartUpload
  • s3:PutObject*
  • s3:GetObject*
  • s3:DeleteObject*
  • s3:BypassGovernanceRetention
  • s3:ListMultipartUploadParts
  • s3:ListBucket*
  • s3:GetBucket*
  • s3:PutBucketObjectLockConfiguration
The example includes the JSON policy template, which describes the necessary permissions for accessing the S3 bucket.

After copying and pasting the JSON template in the JSON text editor, replace <bucket name> with the actual name of the bucket.

When you create an IAM role using EC2, AWS generates an instance profile with the same name as the IAM role.
Security group The configured security group rules allow all outbound traffic from the Cloud Cluster ES nodes and inbound traffic on ports 22 and 443.

JSON policy template

In the following JSON policy template, replace bucket-name with the name of the S3 bucket.


{
     "Version": "2012-10-17",
      "Statement": [
            {
                 "Effect": "Allow",
                  "Action": [
                        "s3:AbortMultipartUpload",
                        "s3:PutObject*",
                        "s3:GetObject*",
                        "s3:DeleteObject*",
                        "s3:BypassGovernanceRetention",
                        "s3:ListMultipartUploadParts"
                     ],
                     "Resource": "arn:aws:s3:::<bucket-name>/*"
            },
            {
                 "Effect": "Allow",
                  "Action": [
                        "s3:ListBucket*",
                         "s3:GetBucket*",
                         "s3:PutBucketObjectLockConfiguration"
                   ],
                   "Resource": "arn:aws:s3:::<bucket-name>"
             }
       ]
}

JSON policy template when using SSE-KMS keys

If you are using SSE-KMS keys, in the following JSON policy template, replace bucket-name with the name of the S3 bucket, and replace <kms key arn> with the SSE-KMS key ARN.


         
{
     "Version": "2012-10-17",
      "Statement": [
            {
                 "Effect": "Allow",
                  "Action": [
                        "s3:AbortMultipartUpload",
                        "s3:PutObject*",
                        "s3:GetObject*",
                        "s3:DeleteObject*",
                        "s3:BypassGovernanceRetention",
                        "s3:ListMultipartUploadParts"
                     ],
                     "Resource": "arn:aws:s3:::<bucket-name>/*"
            },
            {
                 "Effect": "Allow",
                  "Action": [
                        "s3:ListBucket*",
                         "s3:GetBucket*",
                         "s3:PutBucketObjectLockConfiguration"
                   ],
                   "Resource": "arn:aws:s3:::<bucket-name>"
             },
             {
                 "Effect": "Allow",
                  "Action": [
                        "kms:Encrypt",
                        "kms:Decrypt",
                        "kms:ReEncrypt*",
                        "kms:GenerateDataKey*",
                        "kms:DescribeKey"
                   ],
                   "Resource": "<kms key arn>"
             }
       ]
}
