Prerequisites for Rubrik Cloud Cluster on GCP

Information about requirements for configuring Rubrik Cloud Cluster on GCP.

This topic describes a feature or features that Rubrik is gradually rolling out to customers.

The following table describes the requirements for running the Google Cloud Platform (GCP) deployment script to configure the Rubrik Cloud Cluster deployment on GCP.
Requirement Description
Google cloud account Requires access to the data sources managed and protected by the Rubrik Cloud Cluster on GCP.
Google Cloud Storage (GCS) bucket Stores the data for the Rubrik cluster. The GCS bucket must have the following settings:
  • GCS bucket and the cluster assigned to the same region
  • Storage class: Standard
  • Object versioning enabled
  • Object retention enabled
  • Soft delete policy disabled
  • Default encryption type is Google-managed encryption key, but for Cloud Key Management Service (KMS) key, you must use your customer-managed key.
Service account for RSC When added to RSC, automatically configures the permissions and the roles required for GCP access.
Virtual Private Cloud (VPC) role Uses network access on the shared VPC network for the cluster. This role requires:
  • The minimum permissions for network access: compute.subnetworks.get and compute.subnetworks.use
  • A configured Cloud Cluster service account
Web browser Rubrik supports the latest version of Google Chrome and the two previous versions.
