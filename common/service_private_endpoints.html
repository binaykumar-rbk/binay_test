Service and private endpoints

Azure endpoints permit locally routed communication between Rubrik Cloud Cluster Elastic Storage and the Azure Blob storage.

By using Azure endpoints, the traffic between Rubrik Cloud Cluster Elastic Storage (ES) and the Azure Blob storage is transmitted locally on the Azure network. Otherwise, the data is sent to the internet before returning to the Azure Blob storage.

Rubrik Cloud Cluster ES requires using an endpoint so that Rubrik Cloud Cluster ES establishes the lowest lattency connection to the Azure Blob storage.

Rubrik recommends configuring a Virtual Network (vNet) service endpoint. Azure vNet service endpoints provide private linking to the public IP addresses of the Azure Blob storage account. The Azure Blob storage account stores the data locally on the Azure network. vNet service endpoints require minimal configuration and are free to use.

Rubrik also supports using private endpoints, which bypass the public IP address of the Azure Blob storage account. Private endpoints allow you to more closely restrict the traffic between the Rubrik Cloud Cluster ES and the Azure Blob storage. You should be aware that additional costs are associated with private endpoints.
