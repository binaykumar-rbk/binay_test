        RSC provides management and protection of file system data on supported Linux, Unix, and Windows hosts through filesets.

        • Host management for filesets

          RSC supports protection of file system data on Windows, Linux, and Unix hosts.

        • Filesets

          RSC manages the protection of folders and files on host computers through filesets.

          • Fileset fields, rules, and value types

            The Rubrik cluster interprets a fileset based on the values provided in the Include, Exclude, and Do Not Exclude fields. The Rubrik cluster applies a set of rules to the values provided in these fields and permits several types of values to be added to the fields.

          • Host-specific fileset description rules

            Rules for fileset descriptions differ for host types, ensuring accurate file path configurations, preventing backup errors, and optimizing backup processes for different systems.

          • Accepted values for fileset descriptions

            Rubrik clusters accept a specific set of values in fileset descriptions.

          • Fileset description rules

            Fileset descriptions must meet specific requirements.

          • Regular expression handling

            Fileset rules convert to regular expressions that follow the Boost regular expression syntax guidelines.

          • Fileset regular expression conversions

            The Rubrik cluster converts regular expressions used during the specification of a fileset rule.

          • Fileset details

            For every fileset, RSC displays details including the fileset rules and the number of hosts being protected through the fileset.

          • Creating a fileset

            Create a fileset to define a set of data in a file system. You can assign a fileset to a host to protect the data set specified by the fileset on that host.

          • Deleting a fileset from a host

            Delete the association between a fileset and a host to stop SLA Domain protection of the selected host fileset.

          • Deleting a fileset from the Rubrik cluster

            Delete a fileset from the Rubrik cluster to remove all existing associations of that fileset with the hosts that use the fileset to protect files and folders on the hosts.

          • Fileset management actions

            For every fileset, RSC provides the ability to edit, clone, and delete the fileset.

        • Fileset protection for hosts

          The combination of a fileset with a host creates a protectable object referred to as a host fileset.

        • Fileset protection for failover clusters

          RSC supports backup of hosts grouped into failover clusters through filesets.

        • Local pages for fileset objects

          The local pages of a fileset host and a failover cluster service provide detailed information about tasks related to protecting filesets and managing available snapshots.

        • Data recovery from a host fileset

          RSC allows recovering data from a host fileset.

Host-specific fileset description rules

Rules for fileset descriptions differ for host types, ensuring accurate file path configurations, preventing backup errors, and optimizing backup processes for different systems.

Rule Linux, Unix, and NAS (NFS) Windows and NAS (SMB)
Case-sensitivity

Case-sensitive.

A file name extension indicates the file type but does not determine the file type with certainty. The Rubrik cluster does not check file signatures (magic numbers) to determine the file type.

 Case-insensitive
Path delimiter Slash character: / Backslash character: \
Start of a file path File paths can be full or partial. A full path starts with a forward slash. Paths cannot include a dot (.) or double dot (..) character.

File paths can be full or partial.

Windows host: Full paths start with a drive letter, a colon, and a backslash, for example, C:\ for a path on the C drive.

NAS share (SMB): Full paths start with a backslash.

Paths cannot include the single dot (.) or double dot (..) character.
End of a file path Paths that do not end with an asterisk (indicating all the contents of the last named folder) are modified to add /** to the end of the path. This includes all the files and folders in the last specified folder. Paths that do not end with an asterisk (indicating all the contents of the last named folder) are modified to add \** to the end of the path. This includes all the files and folders in the last specified folder.
Network mounts

Linux and Unix hosts: Selecting Follow Network Shares and specifying the full path to the mount point is required.

NAS share (NFS): Does not apply.

Windows host: Specifying the UNC path for a network share. For example, the path \networkshare\folder or \192.168.1.64\folder might be the UNC path to a specific folder. Alternatively, you could get all the shares of a host by specifying the host directly, for example, \hostname\** and using wildcard characters.

NAS share (SMB): Does not apply.

The mount or mount.cifs command can include the ‘nocase’ option. This option causes case-insensitive path name matching for the paths on the network share. Fileset rules applicable to a network share with the ‘nocase’ option should account for case-insensitivity.
Blocked file path
The backup system blocks the following files to prevent backing up system directories that might interfere with the Linux and Unix backup process:
  • /dev
  • /proc
  • /run
  • /sys
  • /var/lib/ntp/proc
  • /etc/rubrik
  • /usr/bin/rubrik
  • /var/log/rubrik
