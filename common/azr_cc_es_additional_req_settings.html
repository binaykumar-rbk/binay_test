    • Rubrik Cloud Cluster Elastic Storage

      You can use Rubrik Security Cloud to provision a Rubrik Cloud Cluster Elastic Storage on AWS or a Rubrik Cloud Cluster Elastic Storage on Azure. Provisioning a Rubrik Cloud Cluster Elastic Storage creates a virtual Rubrik Cloud Cluster running in a virtual private cloud, providing data protection for hosts and applications.

      • Rubrik Cloud Cluster ES for Azure

        You can use Rubrik Cloud Cluster Elastic Storage for Azure to protect applications and data within a Microsoft Azure virtual network.

        • Prerequisites for Rubrik Cloud Cluster ES on Azure

          Information about the prerequisites for provisioning Rubrik Cloud Cluster Elastic Storage.

          • Additional required CCES storage account settings on Azure

            Information about additional required settings for provisioning Rubrik Cloud Cluster Elastic Storage storage accounts on Azure.

          • Service and private endpoints

            Azure endpoints permit locally routed communication between Rubrik Cloud Cluster Elastic Storage and the Azure Blob storage.

        • Node requirements for Rubrik Cloud Cluster on Azure

          Rubrik Cloud Cluster has specific node requirements for Azure.

        • Required permissions for Azure Cloud Cluster provisioning

          RSC uses a cloud account that requires certain permissions or actions to provision Rubrik Cloud Cluster Elastic Storage for Azure.

        • Transitioning to using Azure managed identities

          For enhanced security, transition your cluster from using connection strings to using managed identities as the authentication type to your storage container.

        • Workflow for Rubrik Cloud Cluster ES for Azure

          Provisioning an Azure Cloud Cluster Elastic Storage on RSC consists of several tasks.

      • Rubrik Cloud Cluster ES for AWS

        A Rubrik Cloud Cluster Elastic Storage protects applications and data by running Rubrik CDM software on Amazon EC2 instances within a virtual private cloud.

      • Rubrik Cloud Cluster recoverability

        To restore your data, you can recover a Rubrik Cloud Cluster Elastic Storage that has immutability enabled. You can do this only when the cluster nodes cannot be recovered.

      • Adding a node to Cloud Cluster ES

        Add a node to a Cloud Cluster ES to increase capacity and improve performance.

      • Adding a pre-deployed node to Rubrik Cloud Cluster ES

        Add a node that you have manually deployed in AWS or Azure to the Rubrik Cloud Cluster ES in RSC. Additional nodes increase capacity and improve performance.

      • Node removal in RSC

        If you no longer require a node in a Rubrik cluster, a supported third-party hardware cluster, or a Rubrik Cloud Cluster, you can remove the node by using RSC. To remove a node, you must meet the requirements for node removal.

      • Extra dense node migration for CCES clusters

        Extra dense nodes offer greater storage and performance, without a loss of storage capacity. Using extra dense nodes, organizations can reduce node counts by half during migration. Customization options are available through the GraphQL API.

    • Multitenant organizations

      Multitenancy in RSC refers to the logical isolation of shared compute, storage, and network resources in such a way that each organization can only see and access data that belongs to that organization, especially for shared infrastructure platforms.

    • Multitenancy and Rubrik Envoy

      With Rubrik Envoy, you can enable secure data movement between Rubrik clusters and a tenant organization network.

Information about additional required settings for provisioning Rubrik Cloud Cluster Elastic Storage storage accounts on Azure.

The following table describes the required Advanced settings for storage accounts.
Parameter Value
Require secure transfer for REST API operations Enabled
Enable storage account key access Enabled
Minimum TLS version 1.2
Permitted scope for copy operations (preview) From any Storage Account
Enable SFTP Not enabled
Enable network file system v3 Not enabled
Allow cross-tenant replication Not enabled
Access tier Hot
The following table describes the required Networking settings for storage accounts.
Parameter Value
Network access

Rubrik supports both Service endpoints and private endpoints.

For cost effectiveness, Rubrik recommends setting the value to Enable Public Access from selected virtual networks and IP addresses.
Virtual network subscription The virtual network for Cloud Cluster Elastic Storage (CCES)
Virtual network The virtual network for CCES
Subnets The subnet for CCES
Network routing Microsoft network routing
