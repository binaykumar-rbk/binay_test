Best practices and limitations for Ruby

Guidelines for optimizing your experience with Ruby and an overview of technical boundaries.

This topic describes a feature or features that Rubrik is gradually rolling out to customers.

Using Ruby effectively

Using Ruby effectively requires an understanding of how generative AI interacts with your specific Rubrik Security Cloud environment. While the AI agent streamlines complex workflows and provides deep environmental insights, its performance depends on the quality of user input and adherence to security protocols. Review the following best practices and limitations to ensure that Ruby provides accurate and secure assistance for your data management tasks.

Best practices

To ensure security and accuracy when using Ruby, follow these guidelines:
  • Verify AI-generated content: Always treat Ruby output as a recommendation. Before authorizing a data-modifying action, such as a mass recovery or snapshot quarantine, manually verify the object list, timestamps, and target locations.
  • Precision prompting: Ruby performs best with specific context. Include Rubrik cluster names, object types, or specific time ranges to reduce ambiguity in the response.
  • Role management: Periodically audit RBAC settings. Because Ruby honors existing permissions, ensure that the Owner and Administrator roles are strictly reserved for users who require the authority to run recovery and containment tasks.
  • Contextual awareness: Launch Ruby from the specific dashboard page under investigation. The AI agent can read the current page metadata to provide a more relevant summary of active alerts or capacity risks.

Known limitations

Ruby operates within specific technical boundaries designed to maintain data privacy and system performance. The AI agent is currently optimized for the English language and using other languages might result in inaccurate analysis or a failure to recognize user intent. To protect tenant privacy, chat sessions are strictly isolated. Ruby does not retain context between different users or browser sessions. Active sessions automatically time out after 30 minutes of inactivity. The chat history is purged upon timeout.

The data provided by Ruby is based on the metadata synced from Rubrik clusters to the RSC metadata lake. There might be a slight latency between an event occurring on a local cluster and its availability for AI analysis. Ruby provides broad environmental visibility, but orchestrated remediation and recovery workflows are currently limited to specific object types, including virtual machines, cloud instances, and Microsoft 365 objects.
