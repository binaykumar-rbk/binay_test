Inventory page

The Inventory page displays a list of AI agents monitored by Agent Operations, a list of tools found running on agents, and a list connections to AI agents from Agent Operations.

Agents Inventory page

The Agent Inventory page includes an overview of all the agents Agent Operations has discovered using the connections you have configured, the number of these agents that are high risk, and the number of AI agents that are inactive.

This table describes the filters available on the Agent Inventory page.
Filter Description
Applications Accessible Applications accessible by the AI agent.
Agent Risk Level of risk the action performed by the AI agent poses for your data or configuration.
This table describes the columns displayed on the Agent Inventory page.
Column Description
Name Name of the discovered AI agent.
Agent Risk Current risk level Agent Operations assigns to the AI agent.
Active Violations Number of active violations for each AI agent.
Active Alerts Number of active alerts for each AI agent.
Apps Accessible Applications the AI agent can access.
Connection Type Connection from which the agents originate. For example, Micosoft Copilot Studio or LiteLLM or Rubrik Gateway.
First Discovered Initial date and time Agent Operations discovered the AI agent.

Tools Inventory page

The following table describes the columns displayed for the Tools Inventory page.
Filter Description
Tools Name of the tool Agent Operations found running on the workload.
Application Application issuing the tool.
Agents with Access Names of the agents with permission to access the tools.
Active Violations Number of times this tool has caused a violation in the workload.
Active Alerts Number of active alerts that resulted from this violation.
Approval Status Tool status.
Last Used Date and time the tool was last issued.
First Discovered Date and time when the tool was first discovered.

Agent Operations limits AI agents to read-only, planning, and agent-to-agent tools. Agent Operations does not permit using write and delete tools.

Agent Operations flags tool violations for tools that have a capability to perform at least one of the following actions:
  • Create
  • Update
  • Delete
Agent Operations flags alerts for actions that use tools to perform at least one of the following actions:
  • Create
  • Update
  • Delete

The Connections page

The Connections page displays all connections to Agent Operations and the API keys used to authenticate these gateways. The Connections page includes information about model gateways and native integrations such as Microsoft Copilot. You can click on the Connection Name to view details about the connection.

This table describes the information included in the Connected Inventory tab.
Heading Description
Connection Name Name of the connection specified in Agent Operations.
Agents Discovered Number of AI model gateways included with this connection.
Last Sync Date and time of the last sync for this connection.
Added by User account that added the connection.
Time added Date and time this connection was added.
This table describes the information included in the Connected Models tab.
Heading Description
Model Name Name of the AI router model specified by the Provider.
Provider Name of the API router model provider.
