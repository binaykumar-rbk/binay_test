  • Agent Operations

    Detect risky actions caused on your data by autonomous AI agents, and rewind the actions to repair changes to the data and configurations without downtime.

    • Agent Operations with RSC

      Agent Operations with RSC provides a comprehensive solution for discovering agents, detecting risky actions on your data caused by autonomous AI agents, finding their root causes, and rewinding the actions to repair changes to the data and configurations without downtime.

    • Key concepts for Agent Operations

      Information about the core components of Agent Operations and how these components work together to provide observability, security, and remediation for your AI infrastructure.

    • Common Agent Operations terms

      It is important to understand some common terms when protecting AI agents with Agent Operations.

    • Configure AI connections

      Agent Operations connections provide governance and monitoring for AI agent interactions.

      • Understand how model gateways work

        Model gateways are the fundamental infrastructure components that enable Agent Operations to govern, monitor, and secure your AI agent interactions.

      • Microsoft Copilot Studio deployed in Agent Operations

        Integrating Agent Operations with Microsoft Copilot Studio threat detection for AI agent activities and an optional real-time tool blocking.

      • Google Vertex AI deployed in Agent Operations

        Google Vertex AI integration enables Agent Operations to discover AI agents and ingest conversation history by establishing a secure link through a Rubrik service account and Google Cloud IAM roles.

      • Microsoft Foundry (new) deployed in Agent Operations

        Deploying Microsoft Foundry (new) enables Agent Operations to monitor AI agents by ingesting activity logs from Azure Application Insights through a secure service principal connection.

      • LiteLLM model gateways deployed in Agent Operations

        LiteLLM serves as a model gateway that ingests AI agent activity into Agent Operations to provide granular visibility and policy enforcement for custom agents.

      • JAMF deployed in Agent Operations

        JAMF integrates endpoint management data into Agent Operations to provide granular visibility into agent identities and the devices they assume.

      • Agent Operations deployment or provisioning options

        Agent Operations provides flexible deployment options using cloud-based Virtual Private Clouds and SaaS platforms to connect to AI agents through model gateways.

        • AWS and Azure region support for Agent Operations

          Agent Operations supports a variety of AWS and Azure regions.

        • Provisioning AWS VPC infrastructure in Agent Operations

          Deploying AWS VPC in Agent Operations involves provisioning the necessary infrastructure directly within your AWS account to establish a dedicated data plane.

        • Deploy Azure VPC in Agent Operations

          You can deploy the Azure VPC infrastructure in Agent Operations.

    • Onboard Agent Operations

      After deploying the model gateways or native integrations for your custom AI agents, you must onboard Agent Operations within RSC to monitor AI agent actions and rewind the actions when required.

    • Manage Agent Operations policies

      Agent Operations policies are the governance rules that define the acceptable behavior for your AI agents to ensure they operate within safe boundaries.

    • Monitor and analyze AI agent activity

      You can view information about AI agents, violations, tools, policies, and AI gateways on the Agent Operations UI.

    • Rewind AI agent violations using Agent Operations

      You can use Agent Operations to rewind the AI agent violations, undoing high-risk alterations to data and configurations, or create tickets to resolve the errors at a later date.
