Common Agent Operations terms

It is important to understand some common terms when protecting AI agents with Agent Operations.

Category Term Definition
Core entities AI agent A software entity that performs autonomous actions, such as closing support tickets or optimizing storage, often by interacting with a large language model (LLM).
Tools Specific functions or APIs, such as reading, writing, or deleting data that AI agents use to perform actions on a workload.
LLM The external AI engine, such as OpenAI, Azure OpenAI, or Anthropic, that processes prompts sent by the AI agent.
Infrastructure Model gateway The infrastructure component deployed in your environment that acts as a secure proxy for AI agent traffic, providing governance and monitoring.
Model router A component configured in Agent Operations that connects to AI model providers and routes AI agent requests to the correct model. For example, routing a request to gpt-4.
Hybrid architecture The deployment model where the control plane is managed by Agent Operations while the data plane, that is processing of sensitive traffic, resides in the cloud account of the customer.
Native integration A software application that has AI built in instead of added as a plugin. Copilot is an example of native integration.
BYO gateways Model gateways that allow you to use your own API keys and infrastructure to manage, monitor, and secure AI traffic.
Customer data plane The set of infrastructure resources, such as EKS or AKS clusters and storage buckets, deployed within your own cloud account to process and store agent data locally.
Connectivity and security Endpoint The specific URL where API requests are sent. The model gateway generates a unique endpoint that replaces the standard public URL of the LLM provider in your agent configuration.
Upstream API key The API key provided by the LLM vendor, such as OpenAI, that enables the model gateway to authenticate itself with the LLM provider.
Gateway API key A unique key generated by Agent Operations that enables your custom AI agents to authenticate with the model gateway. This replaces the upstream API key in your agent code.
Governance Policy A set of rules governing agent behavior, such as restricting access to specific tools or data. Violating these rules triggers a violation alert.
AI judge An internal mechanism that evaluates custom policies written in natural language, rating their strength, such as Weak, Moderate, Strong, and providing recommendations for improvement.
Authorized tool A tool that is explicitly approved for use by AI agents. Tools not marked as authorized may trigger policy violations if the No unauthorized tools policy is enabled.
Monitoring Violation Agent Operations determines a violation occurs when it encounters a configuration that would violate a policy, and an alert occurs when an agent performs an action that violates a policy.

There are policies that, depending on their definition, may trigger violations, alerts, or both.
Explorer A visual interface in Agent Operations that provides an immutable audit trail, displaying a roadmap of actions from their execution back to the root cause.
Activity log A detailed list of AI agent actions identified as erroneous or risky, including the tool invoked and the impact level.
Remediation Root cause The initial action or prompt in a chain of events that resulted in a specific outcome or violation.
Rewind The capability to safely undo specific data or configuration changes caused by the error of an AI agent without rolling back the entire system.
Cost and resources Base cost The fixed cost for idle cloud resources required to run the model gateway infrastructure, such as EKS or AKS clusters and storage buckets.
Compute usage cost The variable cost incurred only when active workloads, such as CPU or GPU engines, are processing data.
