Key concepts for Agent Operations

Information about the core components of Agent Operations and how these components work together to provide observability, security, and remediation for your AI infrastructure.

Understanding the core components of Agent Operations is essential for successfully deploying and managing your AI infrastructure. Agent Operations provides visibility into AI agent behavior, enabling you to monitor actions, enforce governance, and recover from data errors.

Agent Operations

Agent Operations is a governance and observability platform within RSC. Unlike standard logging tools, Agent Operations is specifically designed for agentic AI, which are AI systems that can autonomously execute tasks and modify data. Agent Operations captures the full context of the activity of an agent, including input prompts, memory context, prompting chains, and specific tool usage. This visibility enables you to detect risky behaviors and trace data errors back to their root causes.

Tools

In the context of Agent Operations, a tool refers to the specific function, API, or capability that an AI agent invokes to perform a task. A delete_file API or a query_database function is considered a tool. Agent Operations discovers these tools and enables you to authorize or restrict their usage to reduce risk.

Model gateways

A model gateway is the infrastructure component that sits between your custom AI agents and the large language models (LLMs) that these agents utilize. This gateway functions as a secure proxy, intercepting all API requests to providers such as OpenAI, Azure OpenAI, or Anthropic.

Agent Operations also supports native integrations like Microsoft Copilot, which has AI built in instead of added as a plugin, and BYO gateways, which allow you to use your own API keys and infrastructure to manage, monitor, and secure AI traffic.

  • Role: The model gateway processes request data locally for monitoring and policy enforcement before forwarding it to the LLM.
  • Requirement: Deploying a model gateway is mandatory for Custom AI Agents to ensure their traffic is monitored. Native integrations, such as Microsoft Copilot, connect directly to Agent Operations and do not require a customer-deployed model gateway.
  • Authentication: The model gateway manages authentication with the LLM provider using upstream API keys, while issuing separate Gateway API keys to your developers. This ensures that your sensitive provider credentials are never exposed directly to agent code.

Hybrid architecture

Agent Operations uses a hybrid deployment model designed to maintain strict data privacy and security. This architecture splits responsibilities between the Rubrik control plane and your local cloud environment.
  • Agent Operations: This control plane manages the user interface, orchestration, and policy definitions. It handles metadata but does not process sensitive payload data.
  • Customer dataplane: Deployed within your own cloud account, such as AWS VPC or Azure subscription, this layer contains the model gateways and private storage buckets. All sensitive agent traffic, such as prompts and responses, is processed and logged locally within this boundary, ensuring private data never leaves your environment.

Policies

Policies are the rules that govern AI agent actions. Agent Operations actively monitors agent traffic against these policies to prevent data loss and security violations. Agent Operations includes default policies and custom policies:
  • Default policies: These are policies that are included with Agent Operations. You do not need to add these policies, but you can configure them.
  • Custom policies: You can define custom policies using natural language. For example, Do not allow PII to be sent to the LLM. An AI Judge interprets these descriptions to enforce complex governance rules.

Rewind errors

Rewind is a remediation capability of Agent Operations that enables you to reverse specific data changes made by an AI agent without rolling back the entire system. Agent Operations maps every tool use, such as an API call to delete a file, to specific timestamped events. Therefore, Agent Operations can identify the precise moment an error occurred. You can use the Rewind feature to undo granular transactions caused by agent hallucinations or logic errors, restoring data to its previous valid state.
