Microsoft Copilot Studio deployed in Agent Operations

Integrating Agent Operations with Microsoft Copilot Studio threat detection for AI agent activities and an optional real-time tool blocking.

The onboarding process establishes a secure communication channel between Microsoft Copilot Studio and Agent Operations, leveraging the Microsoft Copilot Studio external threat detection provider to monitor AI agent activity and, if configured, block tool execution based on established security policies.

A central component of this integration is the creation of a trust relationship using Federated Identity Credentials (FIC). The FIC allows the Microsoft identity platform to trust tokens issued by the external identity provider (IdP). Once this trust is established, Copilot Studio can securely exchange tokens for access to Rubrik API endpoints.

To ensure data integrity, every request is validated through signature and claims validation. Agent Operations fetches public signing keys from Microsoft to confirm that tokens have not been altered. The system then verifies that the issuer and audience claims match the expected configuration before allowing agent actions to proceed.
