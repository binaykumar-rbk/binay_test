Deploying Microsoft Copilot Studio in Agent Operations

Onboard a Microsoft Copilot Studio to Agent Operations to enable agent discovery, activity logging, and real-time tool blocking.

Before you begin

Onboarding a Microsoft Copilot Studio to Agent Operations requires:
  • Adding the RSC application user
  • Power Platform Admin permissions to create application users and manage security roles.
  • Azure AD Global Administrator permissions to grant admin consent for threat detection.
  • Access to the Environment URL and Tenant ID for the target Microsoft Copilot Studio environment.

About this task

Onboarding Microsoft Copilot Studio to Agent Operations provides comprehensive visibility into AI agent activities and allows Rubrik to intercept and block unauthorized tool usage in real-time.

Attention:
The onboarding process requires you to copy information from the Microsoft Power Platform Admin Center (MPPAC), environment and paste it into the Agent Operations environment. This is easiest done by running both UIs and copying and pasting the data between them.

Procedure

  1. Log in to RSC.
  2. Click and select Agent Operations.
    The Agent Operations page appears.
  3. Click Inventory > Connections.
  4. Click Cloud Platforms > Microsoft Copilot Studio.
  5. Click Add a Connection.
    Agent Operations displays the Connect to Microsoft Copilot Studio wizard.
  6. In Microsoft Power Platform Admin Center (MPPAC), select the environment in which your agents are deployed.
  7. Locate the Environment URL and Tenant ID in the Details pane.
    You will copy and paste this information to Agent Operations
  8. In Agent Operations, enter the information for connecting to the MPPAC environment.
    FieldDescription
    Environment URL URL to the environment in which your agents are deployed. Copied from MPPAC Details page.
    Tenant ID Tenant ID for the environment in which your agents are deployed. Copied from MPPAC Details page.
    Name Name for this identity in Agent Operations.
  9. Click Next.
  10. In Microsoft Power Platform Admin Center (MPPAC), select the Settings > Users + Permissions > Security Roles.
  11. Select Create Role and then Create.
    You will need to name the this security role. Often, a name like Rubrik Reader is used. This role must have read access to these tables:
    • bot
    • botcomponent
    • botcomponentcollection
    • conversationtranscript
    • systemuser
  12. In MPPAC, select the Settings > Users + Permissions > Application users.
  13. Click + New App User.
  14. Enter information in the fields for the new App user and click Create.
    FieldAction
    Add an app Select the RSC app.
    Business Unit Select a business unit.
    Security roles Optional. Click to view the security roles.
  15. In Agent Operations, copy the app ID from the Use this App ID for the Application User field in the Connect to Microsoft Copilot Studio wizard.
  16. In MPPAC, click Add an app.
  17. In MPPAC, paste the app ID and then select the RSC app.
  18. In MPPAC, assign permissions for the application user and click Create.
    FieldAction
    Business Unit Select your root business unit.
    Security Roles Select the role, possibly Rubrik Reader, you created earlier.
  19. In Agent Operations, select I have created the Application User with the required roles and click Next.
  20. Optional: Click Enable Tool Blocking.
    Enabling tool blocking for Microsoft Copilot Studio allows Agent Operations to intercept and block unauthorized tool usage and policy violations in real-time.
  21. If you did not enable tool blocking, click Done.

Results

You have connected Microsoft Copilot Studio environment to Agent Operations.

What to do next

If you enabled tool blocking, continue the MPPAC configuration in Enabling tool blocking on Agent Operations.
