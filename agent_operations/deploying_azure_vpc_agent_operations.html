Deploying Azure VPC on Agent Operations

Deploy Azure VPC on Agent Operations by provisioning the Azure infrastructure in your Azure account. Rubrik Support completes the deployment by performing the final deployment steps

Before you begin

This task requires that you know the Client ID and resource group name for your Azure Cloud subscription. Your Rubrik representative provides these.

About this task

You deploy the resources in these steps to allow Agent Operations to provision the Resource Group and Service Principal resources required to run the VPC.

Procedure

  1. Open Azure Cloud Shell.
    The Azure Cloud Shell is located at https://shell.azure.com.

    Alternatively, you can select the carrot icon from the Azure Portal top bar.

  2. Select your shell.
    The Azure Cloud Shell supports Bash or PowerShell.
  3. Select a Azure Cloud subscription.
    To view your current session, you can enter this command in a Bash or PowerShell shell:
    az account show --query '{Name:name, Id:id}' -o table
  4. If required, change to a different Azure Cloud subscription.
    you can enter this command in a Bash or PowerShell shell:
    az account set --subscription "<subscription-id>"
  5. Set the variables for your Azure Cloud subscription.

    Your Rubrik representative can provide your Client_ID. Log the Client_ID in a location you can access later.

    You can use Bash or PowerShell to set your variables.
    • Bash:
      CLIENT_ID="<client-id>"
RG_NAME="<resource-group-name>"
REGION="<azure-region>"
    • PowerShell:
      $CLIENT_ID = "<client-id>"
$RG_NAME = "<resource-group-name>"
$REGION = "<azure-region>"

    Where azure-region is one of westus2, eastus, westeurope, and so on.

  6. Verify that your resource providers are registered.
    You can use Bash or PowerShell to check that your resource providers are registered.
    • Bash:
      for ns in Microsoft.ContainerService Microsoft.Network Microsoft.Storage \
          Microsoft.KeyVault Microsoft.ContainerRegistry Microsoft.Compute \
          Microsoft.ManagedIdentity Microsoft.Authorization; do
  state=$(az provider show -n $ns --query registrationState -o tsv 2>/dev/null)
  if [ "$state" != "Registered" ]; then
    echo "Registering $ns..."
    az provider register --namespace $ns
  else
    echo "OK: $ns"
  fi
done
    • PowerShell:
      $providers = @("Microsoft.ContainerService","Microsoft.Network","Microsoft.Storage",
  "Microsoft.KeyVault","Microsoft.ContainerRegistry","Microsoft.Compute",
  "Microsoft.ManagedIdentity","Microsoft.Authorization")
foreach ($ns in $providers) {
  $state = az provider show -n $ns --query registrationState -o tsv 2>$null
  if ($state -ne "Registered") {
    Write-Host "Registering $ns..."
    az provider register --namespace $ns
  } else {
    Write-Host "OK: $ns"
  }
}
    If any providers were not registered, wait a few minutes and re-run the command to confirm.
  7. Create the Rubrik service principal.
    You can use Bash or PowerShell to create the Rubrik service principal.
    • Bash:
      SP_OBJECT_ID=$(az ad sp create --id $<CLIENT_ID> --query id -o tsv)
echo "Service Principal Object ID: $SP_OBJECT_ID"
    • PowerShell:
      $SP_OBJECT_ID = az ad sp create --id $<CLIENT_ID> --query id -o tsv
Write-Host "Service Principal Object ID: $SP_OBJECT_ID"
    If Azure Cloud Shell reports already exists, the service principal was created previously. In this case, use Bash or PowerShell to retrieve the Object ID instead.
    • Bash:
      SP_OBJECT_ID=$(az ad sp show --id $CLIENT_ID --query id -o tsv)
    • PowerShell:
      $SP_OBJECT_ID = az ad sp show --id $CLIENT_ID --query id -o tsv
  8. Use the Azure UI or CLI to deploy the Agent Operations Azure VPC data plane permissions to your Azure subscription.
    Deploying the permissions creates the resource group, a custom RBAC role, and a role assignment with Attribute-Based Access Control (ABAC) conditions. However, deploying the permissions requires Owner or User Access Administrator on the subscription because it creates a custom role definition and role assignment.

    You can open this URL to use the Azure UI to deploy permissions (recommended method).

    https://portal.azure.com/#create/Microsoft.Template/uri/https://predibase-public-us-west-2.s3.us-west-2.amazonaws.com/infrastructure/azure-dataplane-preprovision.json/createUIDefinitionUri/https://predibase-public-us-west-2.s3.us-west-2.amazonaws.com/infrastructure/azure-dataplane-preprovision-ui.json

    Agent Operations deploys the Bicep template described here to deploy permissions.

    https://predibase-public-us-west-2.s3.us-west-2.amazonaws.com/infrastructure/azure-dataplane-preprovision.json

    Alternatively, you can issue Bash or PowerShell commands to deploy the permissions from the Azure CLI:
    • Bash:
      az deployment sub create \
  --location $REGION \
  --name rubrik-onboarding \
  --template-uri "https://predibase-public-us-west-2.s3.us-west-2.amazonaws.com/infrastructure/azure-dataplane-preprovision.json" \
  --parameters \
    rubrikAppId="$CLIENT_ID" \
    servicePrincipalObjectId="$SP_OBJECT_ID" \
    resourceGroupName="$RG_NAME" \
    region="$REGION"
    • PowerShell:
      az deployment sub create `
  --location $REGION `
  --name rubrik-onboarding `
  --template-uri "https://predibase-public-us-west-2.s3.us-west-2.amazonaws.com/infrastructure/azure-dataplane-preprovision.json" `
  --parameters `
    rubrikAppId="$CLIENT_ID" `
    servicePrincipalObjectId="$SP_OBJECT_ID" `
    resourceGroupName="$RG_NAME" `
    region="$REGION"

    If you are using your own infrastructure, you can append byoVnet=true or byoStorage=true to the --parameters line to reduce the permission surface.

  9. If you are using the Azure UI, select your subscription and region and click Next.
  10. Enter the Object ID and the resource group name.
    You retrieved the Object ID in an earlier step. The resource group name is provided by Rubrik.
  11. If you are providing your own infrastructure, enable BYO VNET.
  12. Click Review+Create > Create.
  13. Send the deployment output to Rubrik.
    You can view the deployment output as a table using Bash or PowerShell.
    • Bash:
      az deployment sub show \
  --name rubrik-onboarding \
  --query 'properties.outputs.{SubscriptionId:subscriptionId.value, TenantId:tenantId.value, ResourceGroup:resourceGroupName.value, Region:region.value}' \
  -o table
    • PowerShell:
      az deployment sub show `
  --name rubrik-onboarding `
  --query 'properties.outputs.{SubscriptionId:subscriptionId.value, TenantId:tenantId.value, ResourceGroup:resourceGroupName.value, Region:region.value}' `
  -o table
  14. Send the table to your Rubrik representative.
    If you are bringing your own infrastructure, include the applicable resource IDs as well.
    Resource ID value When to include
    Subscription ID Always
    Tenant ID Always
    Resource group name Always
    Region Always
    VNET ID If you are using your own VNET
    Subnet IDs If you are using your own subnets
    AKS identity resource ID If you are using your own VNET
    Storage account ID If you are using your own storage account
    Storage identity resource ID If you are using your own storage account

Results

You have deployed Azure VPC on Agent Operations by provisioning the Azure infrastructure in your Azure account.

When Rubrik Support receives the information, they complete the final provisioning steps to connect your Azure VPC infrastructure to Agent Operations.

What to do next

You can establish private network connectivity and set up the HTTPS endpoint for API access.
