Deploying JAMF in Agent Operations

Deploy JAMF on Agent Operations to monitor and manage AI agent actions on managed endpoints.

About this task

Deploying JAMF enables Agent Operations to correlate AI agent activity with specific managed devices and identities, providing granular visibility into endpoint-based AI usage.

Procedure

  1. Log in to RSC.
  2. Click and select Agent Operations.
    The Agent Operations page appears.
  3. Select Inventory > Connections.
  4. Click Endpoint Detection > JAMF.
  5. Click Add a Connection.
    Agent Operations displays the Connect JAMF wizard.
  6. In Connection Name, enter the name.
    This is the connection from JAMF to Agent Operations.
  7. Click Generate Credentials.
    You use these to authenticate your connection from JAMF to Agent Operations.
    The Connect JAMF wizard generates a Webhook URL and an API key.
    Attention:
    It is a best practice to save the Webhook URL and an API key values to a safe location. Agent Operations does not display them again after this.
  8. Click View Configuration Instructions.
    Agent Operations displays the Configuration Guide, which describes the actions require to connect JAMF to Agent Operations.
  9. Connect to your AWS system running JAMF and perform these steps.
    1. Click Rubrik.MDM.py to download the monitoring script.
    2. Upload the rubrik_mdm.py script to Settings > JAMF.
    3. Create a new policy using the rubrik_mdm.py script.
    4. Edit the script, pasting your generated credentials in the Script Parameters fields.
    5. Assign the policy to the target computers listed in the Scope tab.
    6. Save the changes.
      The updated agent runs after the next check-in.
  10. Click Done to exit the Configuration Guide window.
  11. Click Finish.

Results

You have connected JAMF and it appears on the Connections dashboard, and Agent Operations ingests endpoint metadata and device-to-user mappings into the resilience engine.
