ProtectionSet CRD spec and status fields

The ProtectionSet custom resource definition spec and status fields define an application on the Kubernetes cluster that can be protected and the status of the protection. The spec field can also be used when assigning an SLA Domain to protect the application.

ProtectionSet.spec field

The following table describes the fields in the ProtectionSet.spec field.
Field Description
filter

Fields that defines filtering for the protection set. This is a required field.
hookConfig

Fields that define the configuration of a hook for the protection set.
name

String value that is the name of the protection set. This is a required field.

This value may include characters not permitted by Kubernetes.
slaDomainUuid

String value that is the UUID of an SLA Domain that will manage protection of the protection set. This is an optional field.

When you provide a valid UUID of an SLA Domain, that SLA Domain manages the protection of the protection set.

ProtectionSet.spec.filter fields

The following table describes the fields in the ProtectionSet.spec.filter field.
Field Object description
include

Fields that specify the list of resources to include in the protection set. This is an optional field.
exclude

Fields that specify the list of resources to exclude from the protection set. This is an optional field.
includeClusterScopedResources Boolean flag that indicates whether to include resources that are scoped at the cluster level. The default value is 'true'.

ProtectionSet.spec.filter.include and ProtectionSet.spec.filter.exclude fields

The fields in the ProtectionSet.spec.filter.include and ProtectionSet.spec.filter.exclude fields define which resources are included and excluded from the protection set.

The following table describes the optional fields in the ProtectionSet.spec.filter.include field and ProtectionSet.spec.filter.exclude field.
Field Description
group

String value that is a group to filter. This is an optional field.

Example: 'apps'.
LabelSelectors

Fields that identify labels for picking the pod to run the commands in the hook. This is an optional field.

Example: 'app=deployment1'.
name

String value that is the name of the resource to filter. This is an optional field.

Example: 'deployment1'.
resource

String value that is the resource to filter. This is an optional field.

Example: 'Deployment'.
version

String value that is the version of the resource to filter. This is an optional field.

Example: 'v1'.

ProtectionSet.spec.hookConfig fields

The ProtectionSet.spec.hookConfig field specifies the configuration of a hook.

The following table describes the fields in the ProtectionSet.spec.hookConfig field.
Field Description
containerName

String value that is the name of the container that is the target of the command. This is a required field.

If you do not specify ContainerName, RSC runs the hook in all the containers in the pod.
name

String value that is the name of the Hook custom resource in that namespace. This is a required field.
podSelectors

Fields that generate a list of labels to identify the pods that are the target of the hook commands. This is a required field.

ProtectionSet.spec.hookConfig.selectors fields

The fields in the ProtectionSet.spec.hookConfig.selectors field specifies labels used to identify the pods that are the targets of the hook commands.

The following table describes the fields in the spec.hookConfig.selectors field.
Field Description
matchLabels

A map of key-value pairs. Each entry in this map represents a label key and its corresponding value. For example, key1: value1.

A single key-value pair in matchLabels is equivalent to an element in the matchExpressions field, where the key field is the key, the operator is In, and the values array contains only the specified value.

All the key-value pairs in the matchLabels map are combined using a logical AND operation, meaning all pairs must be satisfied for the selector to match the resource.

For example:

matchLabels:
environment: production
app: frontend

This example matches resources that have both the labels environment: production and app: frontend.

The matchLabels field is optional.
matchExpressions

A list of label selector requirements. These requirements are combined using a logical AND operation, meaning all must be satisfied for the selector to match.

Each item in the matchExpressions list is an object that includes the following properties:

  • A key which is a string representing the label key to which the selector applies. For example, "key1".

  • An operator which is a string that specifies the condition to be applied on the key. The operator can be one of the following:

    • "In": The value of the key must be one of the values listed.
    • "NotIn": The value of the key must not be any of the values listed.
    • "Exists": The key must exist. The validity of the value is not checked.
    • "DoesNotExist": The key must not exist.

  • A list of string values associated with the key.

    • If the operator is In or NotIn, this list must contain at least one string value.
    • If the operator is Exists or DoesNotExist, this list should be empty.

For example, if a matchExpressions list contains:

  • A requirement with key: "environment", operator: "In", and values: ["production"].
  • Another requirement with key: "tier", operator: "NotIn", and values: ["frontend"].

It would match resources that have the label environment: production and do not have the label tier set to frontend.

The matchExpressions field is optional.

ProtectionSet.status fields

The fields in the ProtectionSet.status field specifies the most recent known status of the ProtectionSet custom resource.
The following table describes the fields in the ProtectionSet.status field. Each field is required.
Field Description
assignedSLADomainUuid

String value that is the UUID of the currently assigned SLA Domain.
effectiveSLADomainUuid

String value that is the UUID of the effective SLA Domain on the protection set.
observedGeneration

int64 value that is the last Generation the controller worked on.
uuid

String value that is the UUID assigned by the Rubrik cluster to the protection set.
valid

Boolean flag that indicates whether the protection set filter is a valid definition that is currently possible.
