      • Kubernetes protection

        You can use Kubernetes protection to protect your on-premise or cloud-native Kubernetes clusters using a protection set.

        • Deployment architecture for Kubernetes protection

          For the data protection deployment architecture in Kubernetes protection, RSC serves as the control plane and stores the Persistent Volume backups on a Rubrik cluster.

        • Kubernetes protection prerequisites

          There are common and specific prerequisites for Kubernetes protection.

        • Kubernetes protection requirements

          You can use Rubrik Security Cloud to secure your on-premise or public cloud Kubernetes clusters by protecting either entire Kubernetes namespaces or specific subsets within a namespace using the protection set filtering feature.

        • Kubernetes cluster onboarding

          To onboard your Kubernetes cluster, add it to your RSC domain and provide RSC with access to the cluster and sufficient permissions.

          • Default protection sets

            RSC includes default protection sets that are automatically created for Kubernetes clusters. These protection sets include all namespaces, and relieve you of having to create protection sets manually.

          • Adding a Kubernetes cluster to RSC

            To protect the applications and data in your distribution cluster, add the Kubernetes cluster to RSC by uploading a kubeconfig or applying a generated manifest.

          • Adding an Amazon EKS cluster to RSC

            To protect the applications and data in an Amazon EKS cluster, add the EKS cluster to RSC by configuring an AWS cloud account and an ARN for the EKS cluster.

          • Editing the Kubernetes cluster

            You can edit the Kubernetes cluster for manifest and kubeconfig onboarding.

          • Manifest-based onboarding resources

            When you onboard a Kubernetes cluster using the manifest-based method, some resources are deployed in the Kubernetes cluster. After the connection is established, RSC deploys additional resources during the first refresh job.

          • Deleting a Kubernetes cluster

            Delete a Kubernetes cluster from RSC to stop protecting the data and applications associated with the protection sets of the cluster.

          • Viewing the manifest file

            For Kubernetes clusters that were added to RSC using an manifest file, you can view that file to determine whether it is producing the appropriate state for Kubernetes objects within the cluster.

          • Refreshing a Kubernetes cluster

            Start a refresh of a Kubernetes cluster to update the information for the protected namespaces on that cluster.

          • Regenerating the manifest file

            Regenerate the manifest file to update the Kubernetes cluster configuration.

        • Least privilege configuration for Kubernetes

          You can set up your Kubernetes kubeconfig with the least privileges so that only minimal RBAC permissions are assigned to users and service accounts.

        • Kubernetes labels

          RSC integrates with Kubernetes native labels to manage data protection, enabling you to use Kubernetes labels to identify and group resources, and then associate those labels with SLA Domains.

        • Protection sets for Kubernetes cluster workloads

          A Kubernetes cluster protection set is a protectable workload that you create and protect using RSC. A protection set might include an entire Kubernetes namespace or a subset of resources within the namespace.

        • Browse Kubernetes snapshots

          You can use RSC to browse the resources in a Kubernetes snapshot for protection sets and virtual machines.

        • OpenShift Virtual Machine workload

          You can use RSC to protect containers and virtual machines managed on Kubernetes OpenShift clusters.

        • Kubernetes workload protection policies

          In RSC, you can take both policy-based and on-demand snapshots of Kubernetes protection sets.

        • Kubernetes snapshot retention

          RSC allows changing the retention period of Kubernetes snapshots with the help of SLA Domains. Kubernetes snapshot retention works only with Rubrik clusters running Rubrik CDM version 7.0.2-p2 and later.

        • RSC Custom resource definitions

          You can use the RSC custom resource definitions to create custom resources for backup and recovery workflows using APIs native to Kubernetes.

        • Anomaly Detection for Kubernetes

          You can enable Anomaly Detection for Kubernetes virtual machines and protection sets to identify and analyze anomalous events that can indicate a ransomware attack.

Adding a Kubernetes cluster to RSC

To protect the applications and data in your distribution cluster, add the Kubernetes cluster to RSC by uploading a kubeconfig or applying a generated manifest.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, hover over Inventory and click Data Center.
    The Inventory page for Data Center workloads appears.
  4. Click the Kubernetes Protection card.
    The Kubernetes page appears.
  5. Click Manage Kubernetes Clusters.
    The Kubernetes Protection data source settings page appears.
  6. Click Add Kubernetes Cluster.
    If an overview page appears, you can choose Next to proceed to the first page of the wizard.
    The Add Kubernetes Cluster wizard appears.
  7. In Kubernetes Cluster, enter a name to identify the Kubernetes cluster in RSC.
  8. In Kubernetes Distribution, select the Kubernetes distribution to which to add this Kubernetes cluster.
    The available Kubernetes distributions are:
    • AWS EKS
    • AWS ROSA
    • Azure AKS
    • Azure ARO
    • RedHat OpenShift
    • SuSE Rancher
    • VMware Tanzu Kubernetes Grid
    • Upstream Kubernetes
    • Mirantis
  9. In Cluster, select a Rubrik cluster.
    The selected Rubrik cluster provides protection for the Kubernetes cluster.
  10. In Kubernetes Cluster Transport Method, select the transport method RSC uses to connect with the Kubernetes cluster.
    The available transport methods are Load Balancer and Node Port.
  11. In Container Image, select a method for accessing the container image.
    OptionDescription
    Rubrik Agent Container Image Downloads the Rubrik agent container images from the Docker hub. To use this method, the Kubernetes cluster must have internet access.
    Mirrored Private Registry Obtains the Rubrik agent container images from a private registry. Select this when the Kubernetes cluster does not have access to the Docker Hub. The registry location and possibly also the Kubernetes service account secret are needed for this.
  12. Click Next.
    The Setup Options page appears.
  13. Determine how the Rubrik cluster accesses the Kubernetes cluster setup options.
    OptionDescription
    Upload Kubeconfig file Allows Rubrik to access your Kubernetes environment to complete the setup. You can click Upload Kubeconfig File to upload a kubeconfig file in YAML format.

    You can upload the kubeconfig file with credentials only to the cluster being added. It is important to ensure that the current context is set to this cluster.

    The Rubrik Cluster accesses the Kubernetes API server using the specified kubeconfig file.
    Manually import a manifest file Downloads and applies a manifest file that is generated on the Kubernetes cluster.

    You can copy the manifest file or download it locally. You can then apply it to create the required resources on the Kubernetes cluster, including an api-handler job and a reverse proxy deployment.

    Click Generate Manifest and follow the steps on the Set Up with a Manifest wizard.
    Selecting Node Port as the transport method requires manually entering the node IP address. If the IP address no longer exists on the Kubernetes cluster, you can edit the cluster to provide the new IP address.
  14. Click Done.

Results

You have added a Kubernetes cluster to RSC.

What to do next

Refresh the Kubernetes cluster, as described in Refreshing a Kubernetes cluster, before you can start protecting it.
