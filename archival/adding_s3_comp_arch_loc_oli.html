      • Instant tiering of snapshots

        The instant tiering feature archives data immediately to the specified cold storage tier in the cloud. This approach is ideal for optimizing storage costs when data restoration is a rare requirement.

      • Dell ECS archival location

        RSC supports Dell ECS Object Store as an archival location to configure immutability for archiving encrypted data.

      • NetApp StorageGRID archival locations

        An NFS archival location is storage on an NFS server where you can archive snapshots from your Rubrik cluster. You must identify a specific location on the NFS server to serve as the archival location.

      • Archival network throttling

        Archival network throttling is a feature by which you can limit the bandwidth of the network used by a Rubrik cluster to archive data so that more bandwidth is available for other backup and recovery operations on the Rubrik cluster.

        Create a Rubrik Cloud Vault location on AWS to archive the backups of cloud native workloads.

    • Cascading archival

Adding S3 compatible archives with object-level immutability

Add an Amazon S3 compatible archival location with object-level immutability to enable integration between the object storage and RSC.

Before you begin

Ensure the following:
  • The Rubrik cluster is running Rubrik CDM version 9.4.0-p1 or later.
  • The version of the selected object storage supports immutability.

    The list of object storage types that support object-level immutability is available in the Compatibility Matrix guide.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Archival & Storage and select Data Center Archival Locations.
  4. Click Create Archival Location.
    The Create Archival Location wizard appears.
  5. Select Object storage (S3 Compatible) and click Next.
    The Storage Details page appears.
  6. Configure the storage details.
    The Amazon S3 compatible archival location settings topic provides information about the archival location configuration.
  7. Turn on Immutability.
    Important:
    To prevent data loss, you must ensure that the selected storage supports object-level immutability. The list of object storage types that support object-level immutability is available in the Compatibility Matrix guide.
  8. Click Next.
  9. Review the summary information in the confirmation wizard and click Create.

Results

The Rubrik cluster tests the keys and connection information. After a successful test, the Rubrik cluster adds the archival location with object-level immutability.
