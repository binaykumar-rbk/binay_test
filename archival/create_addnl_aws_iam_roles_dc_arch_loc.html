        • Amazon S3 archival locations with immutable storage

          Amazon S3 archival locations with immutable storage prevent modification or deletion of archived data. Rubrik supports immutable storage on Amazon S3 archival locations both at the object-level and at the bucket-level.

        • Storage class

          You can choose the storage class depending on the data access, resiliency, and cost requirements of your workload. The storage class can be edited after the archival location is added and the Rubrik cluster applies the new storage class to data that is archived after a change.

        • Multipart uploads

          Incomplete partial uploads to Amazon S3 count toward the total storage used by an account.

        • AWS archive tiering

          Archive tiering with AWS is used to tier data to the Glacier Flexible Retrieval Storage Class or Glacier Deep Archive Storage Class based on an SLA Domain policy.

        • Amazon S3 archival location settings

          Information about the settings required to create an Amazon S3 archival location for a Rubrik cluster.

        • Amazon S3 cloud compute settings

          Information about the Amazon S3 cloud compute advanced settings required to launch a transient instance in an AWS Virtual Private Cloud.

        • Amazon S3 immutability settings

          The advanced settings configuration of an Amazon S3 archival location includes an option to enable bucket-level immutability. Amazon S3 archival locations created for Rubrik clusters running Rubrik CDM version 9.3.1 and later support object-level immutability, which is always enabled by default.

      • Verifying cloud compute connectivity

        Use the cloud compute connectivity check in RSC to verify connectivity between cloud compute instances, a Rubrik cluster, and AWS or Azure cloud storage services using Archival Consolidation.

      • Instant tiering of snapshots

        The instant tiering feature archives data immediately to the specified cold storage tier in the cloud. This approach is ideal for optimizing storage costs when data restoration is a rare requirement.

      • Google Cloud Platform archival locations

        Rubrik CDM supports Google Cloud Platform as an archival location.

      • Amazon S3 compatible archival locations

        RSC supports archival locations that support standard S3 APIs.

      • IBM COS archival locations

        RSC supports IBM Cloud Object Storage as an archival location for which you can configure the immutability.

      • Dell ECS archival location

        RSC supports Dell ECS Object Store as an archival location to configure immutability for archiving encrypted data.

      • NetApp StorageGRID archival locations

        RSC supports using NetApp StorageGRID as archival locations for archiving encrypted data. For data stored in NetApp StorageGRID archival locations, Rubrik provides bucket-level immutability.

      • NFS archival locations

        An NFS archival location is storage on an NFS server where you can archive snapshots from your Rubrik cluster. You must identify a specific location on the NFS server to serve as the archival location.

      • QStar tape archive

        QStar tape archive enables Rubrik cluster to store data on tape for long-term retention and regulatory compliance using QStar Archive Manager on Windows server.

      • Disaster recovery using an archival location

        Rubrik CDM supports recovering data from an existing archival location using a Rubrik cluster that is different from the Rubrik cluster that archives data to the location.

      • Archival network throttling

        Archival network throttling is a feature by which you can limit the bandwidth of the network used by a Rubrik cluster to archive data so that more bandwidth is available for other backup and recovery operations on the Rubrik cluster.

      • Creating RCV archival locations on AWS

        Create a Rubrik Cloud Vault location on AWS to archive the backups of cloud native workloads.

    • Rubrik Cloud Vault

      Rubrik Cloud Vault is a Rubrik managed cloud service that provides isolated and immutable copies of protected data, enhancing cyber resiliency.

    • Cascading archival

      Using cascading archival, you can increase replicated data redundancy and decrease data storage costs by archiving replicated snapshots from a replication target Rubrik cluster to a remote archival location.

    • Replication

      The replication feature directs the Rubrik cluster to send replicas of source snapshots or backups to a target Rubrik cluster and defines the maximum time to keep the replica on each cluster.

Creating additional AWS IAM roles for data center archival locations

For an AWS cloud account in RSC, create different IAM roles corresponding to each data center to isolate access to your Amazon S3 archival locations.

About this task

You can create different IAM roles corresponding to each data center to isolate access to your Amazon S3 archival locations.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Cloud Accounts and select AWS.
  4. From the dropdown menu, select Archive and Storage.
  5. Click and select Create IAM Role for an AWS cloud account.
  6. In Role Name, enter a name for an IAM role.
  7. Optional: In Additional Permissions, select the required additional permissions for the IAM role and click Next.
    Additional PermissionsDescription
    Key Management Service Manages AWS Key Management Service (KMS) keys. Using KMS requires you to encrypt archived data.
    Consolidation Manages cloud compute instances when you use the archival consolidation feature.
    Immutability Configures and manages bucket-level immutability for archival locations.
    The Authorize Access page appears.

    RSC generates a CloudFormation template to create an IAM role in your AWS account with the required permissions for RSC to manage the archival location.

  8. Click Launch CloudFormation Stack.
    Alternatively, to create a CloudFormation stack in AWS manually, you can click Download to download the CloudFormation template.
    The login page for AWS Management Console appears.
  9. Log in to the AWS Management Console using the login credentials of the AWS account you are adding.
    The AWS CloudFormation service page appears.
  10. Click Next and follow the prompt to submit the default inputs for the template in AWS Management Console.
    Alternatively, if you downloaded the CloudFormation template, you can manually create a new CloudFormation stack in any one of the regions using the downloaded template.

Results

The AWS Management Console creates the IAM role in AWS for the AWS cloud account using the template that RSC creates.
