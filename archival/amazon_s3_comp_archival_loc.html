    • Archiving of data center workloads

      With Rubrik Security Cloud, you can create and manage archival locations to archive the disaster recovery and long-term retention data of data center workloads.

      • Archival data encryption

        The Rubrik cluster supports and enforces client-side data encryption for all archival storage types.

      • Archival bucket exclusivity

        Only one Rubrik cluster can archive data to a specific archival bucket.

      • Archival process

        The process of archiving data follows a standard sequence of tasks. During the archival process, the Rubrik cluster determines whether to upload an incremental or full copy of the archival snapshot.

      • Archival Locations

        The Archival Locations page provides summary information about existing archival locations.

      • Archiving data center workloads in Azure

        Archiving data center workloads from a Rubrik cluster to Microsoft Azure requires specific configuration on Azure and RSC before archiving can begin.

      • Amazon S3 archival locations

        Rubrik CDM supports Amazon S3 as an archival location. You can use RSC to create an Amazon S3 archival location.

      • Verifying cloud compute connectivity

        Use the cloud compute connectivity check in RSC to verify connectivity between cloud compute instances, a Rubrik cluster, and AWS or Azure cloud storage services using Archival Consolidation.

      • Instant tiering of snapshots

        The instant tiering feature archives data immediately to the specified cold storage tier in the cloud. This approach is ideal for optimizing storage costs when data restoration is a rare requirement.

      • Google Cloud Platform archival locations

        Rubrik CDM supports Google Cloud Platform as an archival location.

      • Amazon S3 compatible archival locations

        RSC supports archival locations that support standard S3 APIs.

        • Amazon S3 compatible archival location settings

          Information about the settings required for creating an Amazon S3 compatible object store as an archival location without immutability.

        • Amazon S3 compatible archives with bucket-level immutability

          Amazon S3 compatible archival locations on Rubrik clusters running Rubrik CDM versions earlier than 9.4.0-p1 can use bucket-level immutability.

        • Amazon S3 compatible archives with object-level immutability

          RSC provides object-level immutability for Amazon S3 compatible archival locations to provide a highly flexible and cost-effective method for securing backups.

      • IBM COS archival locations

        RSC supports IBM Cloud Object Storage as an archival location for which you can configure the immutability.

      • Dell ECS archival location

        RSC supports Dell ECS Object Store as an archival location to configure immutability for archiving encrypted data.

      • NetApp StorageGRID archival locations

        RSC supports using NetApp StorageGRID as archival locations for archiving encrypted data. For data stored in NetApp StorageGRID archival locations, Rubrik provides bucket-level immutability.

      • NFS archival locations

        An NFS archival location is storage on an NFS server where you can archive snapshots from your Rubrik cluster. You must identify a specific location on the NFS server to serve as the archival location.

      • QStar tape archive

        QStar tape archive enables Rubrik cluster to store data on tape for long-term retention and regulatory compliance using QStar Archive Manager on Windows server.

      • Disaster recovery using an archival location

        Rubrik CDM supports recovering data from an existing archival location using a Rubrik cluster that is different from the Rubrik cluster that archives data to the location.

      • Archival network throttling

        Archival network throttling is a feature by which you can limit the bandwidth of the network used by a Rubrik cluster to archive data so that more bandwidth is available for other backup and recovery operations on the Rubrik cluster.

      • Creating RCV archival locations on AWS

        Create a Rubrik Cloud Vault location on AWS to archive the backups of cloud native workloads.

    • Rubrik Cloud Vault

      Rubrik Cloud Vault is a Rubrik managed cloud service that provides isolated and immutable copies of protected data, enhancing cyber resiliency.

    • Cascading archival

      Using cascading archival, you can increase replicated data redundancy and decrease data storage costs by archiving replicated snapshots from a replication target Rubrik cluster to a remote archival location.

    • Replication

      The replication feature directs the Rubrik cluster to send replicas of source snapshots or backups to a target Rubrik cluster and defines the maximum time to keep the replica on each cluster.

Amazon S3 compatible archival locations

RSC supports archival locations that support standard S3 APIs.

With RSC, you can use object stores compatible with Amazon S3 as an archival location for archiving encrypted data. You can add any object store that supports the standard S3 APIs. Depending on your data retention requirements, you can configure the archival locations with or without immutability. You must, however, make the choice about immutability at the time of creating the archival location. You cannot change the immutability setting later.

You can use the following encryption mechanisms:
  • RSA key
  • Encryption password
    Note:
    The encryption password method is applicable only for archival locations with bucket-level immutability.
Depending on the Rubrik cluster version, RSC supports either bucket-level or object-level immutability for Amazon S3 compatible archival locations.
  • When the Rubrik cluster is running CDM version 9.4.0-p1 or later, you can enable object-level immutability.
  • When the Rubrik cluster is running CDM version earlier than 9.4.0-p1, you can enable bucket-level immutability.
Note:
The immutable archive feature is not enabled by default. Contact Rubrik Support to enable the immutable archive feature for Amazon S3 compatible archival locations.
