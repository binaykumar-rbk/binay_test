      • QStar tape archive

        QStar tape archive enables Rubrik cluster to store data on tape for long-term retention and regulatory compliance using QStar Archive Manager on Windows server.

      • Disaster recovery using an archival location

        Rubrik CDM supports recovering data from an existing archival location using a Rubrik cluster that is different from the Rubrik cluster that archives data to the location.

      • Archival network throttling

        Archival network throttling is a feature by which you can limit the bandwidth of the network used by a Rubrik cluster to archive data so that more bandwidth is available for other backup and recovery operations on the Rubrik cluster.

Adding a NetApp StorageGRID archival location

Add the NetApp StorageGRID archival location by configuring the settings and parameters in RSC. By adding the NetApp StorageGRID archival location, you can integrate NetApp StorageGRID and RSC.

Before you begin

Ensure the following:
  • Add a NetApp StorageGRID archival location by following the general steps for creating a data center archival location, except for storage type selection. When prompted for storage type, perform the steps outlined in Creating a data center archival location.
  • Obtain the access key ID and secret key for the NetApp StorageGRID archival location associated with the Rubrik cluster by contacting your account administrator.

Procedure

  1. In Select storage type, select Object Store (S3 Compatible) and click Next.
  2. In Select object store vendor, select NetApp StorageGRID and click Next.
    The Add storage information wizard appears.
  3. In Archival Location Name, enter a display name for the archival location.
  4. In Rubrik Cluster, select a Rubrik cluster.
  5. In Access Key, enter the access key ID associated with your account or service.
  6. In Secret Key, enter the secret key provided to you by the account administrator for authentication.
  7. In Host Name, enter the resolvable hostname or IP address of the NetApp StorageGRID host.
  8. In Bucket Name, enter a unique name for the bucket.
  9. In Encryption Type, select an encryption type.
    • RSA Key for non-immutable
    • Encryption Password for mutable location
  10. Optional: Select Enable Archive Consolidation.
    Selecting this option frees archival storage and reduces the snapshot chain length by merging each expired snapshot with the next live snapshot.
  11. Optional: Select Use System Proxy.
    Selecting this option enables the system to use the configured proxy settings for network connections. It allows the application to channel its network traffic through the system-configured proxy.

    If you choose not to select this option, direct network communication without involving the system proxy will be enabled.

  12. Click Next.
  13. Optional: Turn on Immutable Archive.
  14. In Immutability Lock Period (Days), enter the number of days.
  15. Select I have read the NetApp StorageGRID immutability documentation and agree to all associated costs. and click Next.
  16. Review the summary information in the confirmation page and click Create.

Results

The Rubrik cluster tests the keys and connection information. After a successful test, the Rubrik cluster adds the NetApp StorageGRID archival location and stores the configuration.
