            Rubrik supports the Live Mount operation for AHV virtual machine snapshots from Rubrik clusters on to Nutanix clusters.

            • AHV virtual machine Live Mount operations

              The operations available for a Live Mounted AHV virtual machine on the Live Mounts page vary depending on the type of the Live Mount.

            • Creating a live mount for AHV

              Live mount an AHV virtual machine on a Nutanix cluster from a snapshot on the Rubrik cluster. The live mounted virtual machine uses the Rubrik cluster as its datastore.

            • Live Mounts page for Nutanix virtual machines

              The Live Mounts page for Nutanix virtual machines displays details about the mounted virtual machines and virtual disks and provides options to unmount virtual machines and virtual disks, change the power status, and migrate the data source.

            • Unmounting an AHV virtual machine

              Unmount the live mounted AHV virtual machine from the Rubrik cluster.

            • Powering off an AHV virtual machine after live mount

              Power off a mounted AHV virtual machine from the Live Mounts page.

            • Migrating a live mounted AHV virtual machine

              Migrate the data source of a live mounted AHV virtual machine from a Rubrik cluster to a Nutanix storage container.

          • Exporting an AHV virtual machine snapshot

            Perform an export operation to create a virtual machine from a point-in-time copy of the source virtual machine.

          • AHV virtual disk recovery by mounting

            You can use RSC to recover specific virtual disks in an AHV virtual machine by mounting the disks from a snapshot on to a target AHV virtual machine.

          • Downloading an AHV virtual disk to a CDM cluster

            Download a specific virtual disk from an AHV virtual machine snapshot to a CDM cluster.

          • Downloading an AHV virtual disk to a local machine

            After you have downloaded the AHV virtual disks to your CDM cluster, download the virtual disks to a local machine for offline analysis.

          • Recovery of a batch of AHV virtual machines

            You can recover a batch of AHV virtual machines in a single operation from the Nutanix Virtual Machines inventory page.

          • AHV virtual machine in-place recovery

            Nutanix AHV virtual machine in-place recovery provides a faster and easier option for recovering an AHV virtual machine from snapshots, compared to other recovery options.

          • AHV virtual machine instant recovery

            You can use the instant recovery option to quickly replace the source virtual machine with a new virtual machine created from a snapshot.

AHV virtual machine recovery using Live Mount

Rubrik supports the Live Mount operation for AHV virtual machine snapshots from Rubrik clusters on to Nutanix clusters.

A virtual machine recovery using the Live Mount operation involves using a snapshot to create and mount a new virtual machine on a Nutanix cluster. Rubrik supports the Live Mount operation on Nutanix clusters running AOS version 6.0.2 and higher.

Following are data migration options for the Live Mount recovery operation:
  • Live Mount without migration does not allow data migration after the mounting is complete. Consequently, specifying a storage container at the time of the mount operation is not required. The local Rubrik cluster serves as the data store for the mounted virtual machine.
  • Live Mount with optional migration provides a choice to migrate the data immediately after the mounting is complete or at a later time. This option requires specification of a storage container at the time of the mount operation. Successful migration unmounts the Live Mount.

After the successful mounting of virtual machines, the Live Mounts page for AHV VMs displays all the mounted virtual machines.

With the option to migrate the data source immediately, the Rubrik cluster unmounts the recovered virtual machine as soon as the migration is complete. If you do not choose to migrate the data source immediately, you can initiate the migration manually from the Live Mounts page of the AHV virtual machines after the mounting completes.
