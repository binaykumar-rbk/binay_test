  • Multitenancy

    Logically isolate shared resources to ensure organizations can only access their specific data.

    • Multitenant organizations

      Multitenancy in RSC refers to the logical isolation of shared compute, storage, and network resources in such a way that each organization can only see and access data that belongs to that organization, especially for shared infrastructure platforms.

    • Multitenancy and Rubrik Envoy

      With Rubrik Envoy, you can enable secure data movement between Rubrik clusters and a tenant organization network.

      • Envoy features

        The Rubrik Envoy virtual appliance connects tenant networks to the Rubrik cluster. Multiple features make the data movement possible.

      • Envoy topology

        Rubrik Envoy allows tenant organizations to connect to Rubrik services on a shared Rubrik cluster.

        • Envoy network requirements

          Rubrik Envoy registration has specific network configuration requirements.

        • Envoy network interfaces

          Rubrik Envoy can connect with a single Ethernet interface or with an optional, second Ethernet interface.

        • Supported Envoy network assignments

          Rubrik Envoy supports both static IP and DHCP network assignments.

        • Envoy dual-NIC support

          Rubrik Envoy supports up to two virtual NICs.

      • Envoy security

        Rubrik Envoy provides secure multitenancy.

      • IP address changes in Rubrik Envoy

        The effect of changes to the IP address of a Rubrik Envoy virtual appliance.

      • Organization networks

        An organization network provides a secure tunnel for point-to-point communication with organization resources. An organization network also provides secure connectivity when NAT is used between a network segment and a shared Rubrik cluster that is not directly routable.

    • Enabling stringent policies for organizations

      An user with administrator privileges for a global organization can enable global enforcement settings to enforce stricter policies for tenant organizations.

Envoy network interfaces

Rubrik Envoy can connect with a single Ethernet interface or with an optional, second Ethernet interface.

Interface Description
Single interface In the default single-NIC mode, Envoy connects to the backup targets in the tenant network as well as to the Rubrik cluster through a single-NIC (eth0) interface.
Dual interface In the dual-NIC bridge mode, eth0 must connect to the backup targets in the tenant network, while eth1 must connect to the Rubrik cluster.
