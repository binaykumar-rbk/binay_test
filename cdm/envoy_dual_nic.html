Envoy dual-NIC support

Rubrik Envoy supports up to two virtual NICs.

From Rubrik CDM version 8.0.2 and later, Rubrik Envoy supports both single and dual Ethernet interfaces. The default mode supports a single eth0 interface, which is sufficient for most use cases. For a network design that requires two interfaces, you can configure an optional second NIC as eth1. Configuring a second NIC allows Envoy to serve as a bridge between the two interfaces (bridge mode).

In bridge mode, one NIC connects to the backup targets in the tenant network, while the other NIC connects to the Rubrik cluster. This makes it possible to assign specific IP addresses and different security policies and firewall rules to each interface.

In bridge mode, the Envoy virtual appliance must connect to the correct virtual network. The required dual-NIC sequence is as follows:
  1. eth0 → Tenant network
  2. eth1 → Rubrik cluster network

You can configure an optional second NIC on any deployed Envoy virtual machine that has a second NIC installed on the vSphere Web Client. You can switch from default mode to bridge mode and from bridge mode to default mode. The Envoy virtual appliance requires reconfiguring after each mode switch.
