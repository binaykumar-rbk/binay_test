        • Creating SLA Domains for many-to-one replication

          Create an SLA Domain for many-to-one replication, the traditional replication mode supported by RSC SLA Domains. This method consolidates data protection from multiple source clusters onto a single target cluster, ensuring centralized recovery and protection.

        • Creating SLA Domains for one-to-one replication

          Create an SLA Domain for one-to-one replication when there are more than two replication targets and the source and replication clusters are in a dedicated replication configuration. With one-to-one replication, you can configure multiple replication pairs, but a source cluster can replicate to a maximum of two target clusters.

SLA Domains with CDP enabled

SLA Domains with Continuous Data Protection enabled are used for near-term recovery.

In order for Continuous Data Protection (CDP) to be functional, a CDP Filter must be installed on an ESXi host (version 6.7 or later). You can configure the Keep Recovery Points For (Hours) settings for SLA Domains with CDP enabled. The maximum values for configuration are:

  • Four hours for source or replication target clusters running Rubrik CDM version 5.1 or later
  • 24 hours for replication target clusters running Rubrik CDM version 5.3 or later
  • 24 hours for source or replication target clusters running Rubrik CDM version 6.0 or later

After the Keep Recovery Points For (Hours) value is configured, you must configure the Take Snapshots Every (Hours) field value to be equal to or less than Keep Recovery Points For Hours.

After CDP is configured for the SLA Domain, configure the rest of the SLA Domain settings.
