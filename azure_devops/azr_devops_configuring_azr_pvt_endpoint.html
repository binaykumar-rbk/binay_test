Configuring an Azure private endpoint for Azure DevOps

To back up snapshots to an archival location with a region-specific setting, configure an Azure private endpoint.

Before you begin

This task assumes that all Azure Virtual Networks (VNets) requiring resolution of the Fully Qualified Domain Name (FQDN) for the configured private endpoint uses the default integrated DNS server setting provided by Azure rather than a custom DNS. If a VNet is using a custom DNS, configure additional settings for private endpoint name resolution within the custom DNS. The Azure Private Endpoint DNS integration topic in the Azure documentation provides more information about the custom DNS integration.
Note:
For Azure DevOps organizations, you configure Azure private endpoints to store the snapshots to an Azure backup location.

Procedure

  1. From RSC, retrieve the Azure storage account name associated with the archival location.
    You can navigate to the Cloud Archival Locations page and find the storage account name under Storage Account Prefix.
  2. In the Azure portal, navigate to the storage account and then go to the Networking page.
    The Firewalls and virtual networks tab appears.
  3. Under Public network access, select Enabled from selected virtual networks and IP addresses.
  4. Under Firewall > Address range, enter the RSC IP addresses.
  5. Click Save.
  6. Click Private endpoint connections.
  7. Create a private endpoint.
    The new private endpoint connects the storage account to the VNet used for the Exocompute instance. The Create a private endpoint topic in the Azure documentation provides more information about creating a private endpoint.

    Rubrik recommends that the region and VNet for the private endpoint match the region and VNet for the Exocompute instance. This is because the Exocompute VNet has default access to the private DNS zone. However, you can use any VNet for the private endpoint if it has peer connectivity to the VNet for the Exocompute instance.

  8. (Private endpoint created in a VNet different from the Exocompute VNet) Link the VNet to the private endpoint by using the private DNS zone.
    You can search for Private DNS zones in the Azure portal and select the privatelink.blob.core.windows.net DNS zone. When adding a virtual network link, select the VNet used for the private endpoint.

    The Link the virtual network topic in the Azure documentation provides more information about adding a virtual network link.

Results

Configuration of an Azure private endpoint for Azure DevOps organization backup is complete.
