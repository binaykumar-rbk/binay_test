Common protection terms for Azure DevOps

It is important to understand some common terms when protecting an Azure DevOps organization with RSC.

By familiarizing yourself with the common terms, you can better understand and navigate the various steps and components involved in protecting an Azure DevOps organization.
Term Description
Azure DevOps A unified platform offering a comprehensive suite of tools, including project management, version control, testing, and CI/CD capabilities. Azure DevOps integrates seamlessly with Azure services, reducing the need for extensive setup and maintenance.
Backup location A location that you configure within your own Azure environment where the data associated with Azure DevOps organizations, projects, and repositories is stored. You configure the backup location while onboarding the Azure DevOps organization. In this context, archival location and backup location are used to refer to the same storage destination.
Organization The top-level container holding multiple projects. Organization represents a company, department, or team managing software development.
Project A logical grouping within an organization where teams collaborate on software development. A project includes repos, pipelines, work items, artifacts, and settings.
Repos Shared storage space where you keep all the code for a project.
Pipelines Automated processes for Continuous Integration (CI) and Continuous Deployment (CD), including build and release pipelines.
Continuous Integration (CI) The practice of testing and building code whenever a change is made to catch issues early.
Continuous Deployment (CD) The practice of automatically deploying a new version to production if all tests pass.
Git utility Git utilities such as cloning, pushing, fetching, bundling, and unbundling drive all repository backup and recovery operations.
Exocompute The Azure DevOps onboarding process includes configuring Exocompute as its final step. For Azure DevOps protection, RSC uses the existing Cloud Native Protection (CNP) Exocompute setup.
Storage RSC uses Azure Blob Storage as the storage layer for Azure DevOps projects and repository protection. RSC supports Azure archival location, restricted to a specific region only.
Entra ID app registration The process of registering an application with Microsoft Entra ID to grant the application an identity, enabling the application to access resources.

RSC creates an Entra ID app registration within the Rubrik Azure tenant and configures the application for multi-tenant access. When you initiate Azure DevOps onboarding, your tenant grants consent to this application. This consent automatically provisions a corresponding Service Principal in your Entra ID tenant. RSC assigns this Service Principal the required Azure DevOps permissions and adds this Service Principal to the appropriate Azure DevOps groups, enabling secure authentication and correct execution of backup jobs on your behalf.
