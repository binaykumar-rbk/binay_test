Set up backup location for Azure DevOps

RSC protects your Azure DevOps organizations, projects, and repositories. You can set up a backup location in Azure for storing the snapshots of Azure DevOps organizations, projects, and repositories.

You can set up a backup location in Azure to store the snapshots of Azure DevOps organizations, projects, and repositories.

When configuring the backup location for your Azure DevOps organizations, you have the option to use one of the following storage access tiers: Hot, Cool, and Cold.

Tiered snapshots immutability

RSC leverages the Azure time-based retention policy for immutable blob data to protect your archived Azure DevOps backups.
  • WORM lock policy: This policy locks the blob data with a Write-Once, Read-Many (WORM) lock for the specified retention interval. After you set this policy, you can read and create blobs, but you cannot modify or delete them until the retention period expires.
  • SLA Domain management: RSC manages the snapshot retention period through the SLA Domains assigned to your Azure DevOps organization, projects, or repositories.
  • Blob versioning: Azure immutability requires Blob versioning to be enabled on the Azure storage account. RSC enables blob versioning when creating the storage account.

Immutability advantages

The Azure immutability feature provides essential security benefits for your backed-up data as this feature prevents the deletion or modification of snapshots from the Azure storage account.

These advantages ensure data retrieval, even following a malicious activity or ransomware attack, until the WORM lock or the SLA Domain retention period expires.
