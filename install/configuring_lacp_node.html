Configuring LACP for nodes

Configures Link Aggregation Control Protocol for a node using the admin CLI.

About this task

The default bond mode is Active-Backup. Configure the bond mode from the CLI to enable Link Aggregation Control Protocol (LACP) for nodes. Rubrik CDM version 9.3.1 and later supports setting the LACP bond mode to bond0 or bond1.

Procedure

  1. Open an SSH session on the node, or access the console using the Intelligent Platform Management Interface (IPMI) or crash cart.
  2. Enter the following command to show the current bond mode. 
    network bond_mode get
    The set bond mode prompt appears.
  3. Enter one of the following bond interface names.
    • bond0
    • bond1
  4. Enter the following command to configure the bond for LACP. 
    network bond_mode set
    The set bond mode prompt appears.
  5. Enter yes.
    Alternatively, to keep the current mode, enter no.
  6. Enter LACP.
    The node sets the bond mode to LACP. This changes the bond mode only on the local node and reboots the node.
  7. Optional: Repeat these steps on other nodes in the Rubrik cluster.
    This task configures only one node to use LACP. Repeat the task on each node that should use LACP.

Results

The node is now configured for LACP.

What to do next

Perform the bootstrap procedure to form the cluster. The bootstrap procedure is described in the Rubrik CDM Install and Upgrade Guide.
