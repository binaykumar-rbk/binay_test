Evaluate SLA Domain options for NAS CD

You can use SLA Domains to unify data protection policies, providing a unified approach to manage data protection of NAS CD data sets.

SLA Domains simplify data protection of the NAS CD data sets with a policy-based approach of protecting data. RSC provides an SLA policy management service, which also unifies SLA policies from multiple clusters into a single view. SLA Domains provide a configurable set of policies that you can apply to a NAS CD object to achieve specific data protection objectives.

Benefits of SLA Domains

SLA Domains in RSC provides multiple benefits when you plan to protect your data by backing it up using RSC.
The following table lists some of the features and benefits of using SLA Domains to protect NAS CD data sets:
Feature Description
Backup automation You can automate backup jobs for NAS CD data sets by configuring and scheduling the backup jobs.
Directory Rule You can create a directory rule for a share or bucket to automatically assign a specific SLA Domain to a subset of NAS CD data sets within that share or bucket.
Backup exclusion You can specify directory paths and file patterns that you do not want RSC to back up.
Management of backup frequency You can specify how often you want RSC to take backup snapshots through the SLA Domain configuration.
Configuration of target You can specify the target destination where you want to back up the NAS CD data sets.
Retention of your backup data You can specify the retention period of your backed-up data and apply retention lock to protect your data from deletion until the expiry of the retention period.
On-demand snapshot You can back up your NAS data at any time by taking an on-demand snapshot of the data.
