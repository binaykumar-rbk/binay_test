Data protection and Recovery using RSC and NAS CD

You can manage your NAS CD datasets depending on whether you import or set up the NAS systems in NAS CD or RSC.

If you set up your NAS system through RSC, see Set up NAS CD data sets from RSC section.

If you set up your NAS system through NAS CD, see Set up NAS CD datasets from NAS CD section.

The following table summarizes the two set up options that you can choose to protect your NAS CD data sets using RSC:
UI platform Import or Set up NAS system Summary
RSC only RSC
You can use RSC for the following actions:
  • Importing NAS systems
  • Creating targets
  • Creating and assigning SLA Domains
  • Monitoring data sets using events, reports, webhooks
  • Recovering data
This option provides you a unified platform to import, manage, protect, monitor, and recover all your NAS CD data sets from a single user interface for all Rubrik products. The SLA Domains provide more functionality and flexibility when protecting your data sets using RSC.
NAS CD and selectively RSC NAS CD
You use NAS CD for the following actions:
  • Importing NAS systems
  • Creating targets
  • Creating and assigning NAS CD Policies

You use RSC for recovery.

This option involves using NAS CD for setting up your data and using RSC capabilities for seamless management and recovery of the NAS CD data sets. You can use this option for your existing NAS CD data sets that you have set up using NAS CD.
