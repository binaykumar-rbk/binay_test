Multi-valued data recovery rules

Information about how to handle conflicts for multi-valued data, both attributes and relationships, during recovery. RSC provides two options, overwrite or merge values, enabling you to select the option that best meets your organizational requirements for data consistency.

Multi-valued data

During recovery, a conflict can arise when a recovered Okta object has multi-valued data, which is the data where an object can hold more than one value for a specific field.

Each Okta object has two types of values associated with it:
  • Relationships with other objects: For example, a User being a member of multiple Groups.
  • The attributes associated with objects: For example, a custom attribute on a User profile that enables multiple entries, such as multiple addresses or phone numbers.

You must select how RSC resolves these conflicts for both relationships and attributes when the data in the live tenant differs from the data in the snapshot. You can select from the recovery options to handle the relationships of an object and the attributes associated with the selected objects that best suit your organizational needs.

Recovery options for conflict resolution

RSC provides you with two options to handle recovery for objects that have multiple values:

Option Description
Overwrite live values Recovers the object values from the backup copy and overwrites the live copy with the object values in the backup copy.

Effect on live data: RSC removes any new values that were added after the snapshot was taken. The object returns to the exact state it was in at the time of the backup.
Merge with live values Recovers the object values from the backup copy and merges them with the existing values of the live copy.

Effect on live data: RSC retains the object values that were added after the snapshot was taken, combining the historical state with the current state.

Example use case

To illustrate the difference between the Overwrite live values and Merge with live values recovery options, consider a scenario involving User Group membership:
  • At snapshot time: At the time when RSC took a scheduled snapshot according to the assigned SLA Domain, the User is a member of Group 1 and Group 2.
  • In live tenant now: The User was removed from Group 1 and Group 2, but was newly added to Group 3 and Group 4.
Recovery option Result on user membership in live tenant
Overwrite live values RSC recovers the user to the exact state of the snapshot. The User is a member of Group 1 and Group 2. Membership in Group 3 and Group 4 is removed from the live tenant.
Merge with live values RSC recovers and merges the values. The User is a member of Group 1, Group 2, Group 3, and Group 4. The historical and new memberships are retained.
