Adding RSC application in Okta

Add the RSC application within your Okta Admin Console and generate secure credentials required for the Okta integration with RSC.

Procedure

  1. Log in to your Okta Admin Console.
  2. On the left menu, navigate to Applications > API Service Integrations.
  3. Click Browse App Catalog.
  4. In the search field, enter Rubrik and locate the Rubrik Security Cloud or similarly named API Service Integration.
  5. Click the Rubrik Security Cloud application tile and then click Add Integration.
    RSC requests certain Okta permissions for syncing data for backup and recovery. For more information, see Okta permissions.
  6. In General Settings, provide a unique application label for the integration or use the default label and click Done.
    The RSC application is created in your Okta tenant. The application configuration page appears, displaying the secure credentials in the Client Credentials section.
  7. In Client Credentials, immediately copy the Client ID and Client Secret to a secure location.
    Important:
    The Client Secret appears only once when you first add the RSC application in Okta. If you navigate away from this page, you will lose the secret and must generate a new one.
    The Client ID and Client Secret are essential when you add your Okta tenant in RSC.

Results

You have successfully added the RSC application in your Okta tenant and generated the secure credentials required to connect your Okta tenant with RSC.
