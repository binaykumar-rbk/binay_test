Creating SLA Domain for Okta tenants

Create a custom SLA Domain to define and apply customized data protection and management policies for your Okta tenants.

Procedure

  1. Log in to RSC.
  2. Click and select Identity Resilience.
    The Identity Resilience dashboard appears.
  3. On the navigation, click Protection > SLA Domains.
    The SLA Domains page appears.
  4. Click Create SLA Domain.
    The Create an SLA Domain wizard appears, providing introductory information about SLA Domains.
  5. Click Next.
    The Select IAM Object Types page appears.
  6. Select Okta and then click Next.
    The Frequency and Retention page appears.
  7. In Take Snapshots, enter the frequency at which you want RSC to take snapshots.
  8. In Retain snapshots, enter the required duration to retain the snapshot.
    For example, number of days, months, or years.
  9. Click Show More to view optional advanced settings.
  10. Optional: In Snapshot Window, enter a time window during which you want RSC to take snapshots.
    When you specify a snapshot window, RSC takes the snapshot during the specified time frame. Workloads managed by a Rubrik cluster use the timezone of that Rubrik cluster. Workloads managed by RSC use the UTC timezone.
  11. In First full snapshot, select a start type.
    OptionDescription
    First opportunity The first full backup occurs by default when a data source is added.
    Custom range The first full backup occurs within a specified window.
  12. Click Next.
    The Archival and Replication page appears.

    RSC does not support archival and replication for Okta. You can skip this page.

  13. Click Next.
    The Confirmation page appears.
  14. In SLA Domain name, enter a name to identify the SLA Domain.
  15. Optional: In Add Description, enter a description to provide additional context for the SLA Domain.
  16. Review the configuration details and click Create.
    A message that you have successfully created an SLA Domain appears.

Results

You have successfully created a custom SLA Domain for your Okta tenants.
