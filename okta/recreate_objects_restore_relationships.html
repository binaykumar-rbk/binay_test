Recreate objects and restore relationships

RSC enhances granular recovery of critical Okta objects by discovering, displaying, and simplifying the complex, interlinked object structure and dependencies in Okta. This enables you to precisely select and manage the recovery of critical Okta objects and their associated dependencies.

Discovery and granularity

The granular recovery capability of RSC for critical Okta objects extends to their related objects and the relationships between them.
  • Dependency discovery: RSC discovers all dependencies of the selected Okta object, simplifying the complex interlinked object structure in the Okta configuration. RSC then displays the metadata of these related objects, providing the flexibility to select those related objects for recovery.
  • Reviewing details: During the recovery process, you can view the details of these discovered objects. By hovering over an object and clicking the View option, a separate tray appears, displaying the detailed metadata of the related objects. This flexibility enables you to select exactly which related objects to include in the recovery along with your primary object.

Rules for recreating versus re-establishing relationships

RSC recreates objects that are directly related to the object being recovered. In some cases, for certain object types, RSC only re-establishes the relationship but does not recreate the related object.

RSC does not recover recursively related objects. This means if you recover Object A, and it recreates missing Object B, RSC will not automatically check and recreate missing dependencies of Object B.
