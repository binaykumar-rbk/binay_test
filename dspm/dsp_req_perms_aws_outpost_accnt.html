    • Data Security Posture

      Data Security Posture provides agile data security across multi-cloud environments for posture management and access governance. It also includes threat monitoring and response to protect your data.

      • Data Security Posture overview

        Using Data Security Posture, you can leverage extensive information about data living in your cloud, SaaS, and on-premises environments to easily prevent data breaches. Additionally, you can create effective data security guardrails, and meet compliance and regulatory needs.

      • RBAC for Data Security Posture

        Information about the privileges required for Data Security Posture access. These privileges help in configuring Role-Based Access Control for users.

      • Data Discovery and Classification

        The Data Discovery and Classification feature in Data Security Posture discovers sensitive information in structured and unstructured data that is subject to regulation.

      • Data Security Posture management dashboard

        The Data Security Posture management dashboard provides a unified view of where sensitive data resides across cloud, datacenter, and SaaS environments. Interactive charts and detailed breakdowns allow you to quickly identify, investigate, and address data most at risk to strengthen overall data security.

      • Data Security Posture Enterprise Edition dashboard

        The Data Security Posture dashboard for the Rubrik Enterprise Edition provides a high-level view of your sensitive data, across protected workloads configured for Data Security Posture.

      • Violations in Data Security Posture

        Policy violations in Data Security Posture provide detailed information and analysis of why a data security or compliance violation is indicated.

      • Object Inventory

        The Object inventory provides complete data and insights into the objects in your cloud information. The Object Inventory page provides an overview of the different objects and their type, the sensitive hit details, the impacted data categories, and active violations.

      • Identity Inventory

        The Identity Inventory provides visibility into data that identities in your environment can access.

      • Threats feature in Data Security Posture

        You can use the Threats feature in Data Security Posture to detect external and internal data breaches in your environment.

      • Data Security Posture policies

        Data Security Posture policies deliver specific data security frameworks for protecting data and simplifying the management of data breaches in complex cloud infrastructures.

      • Data classification management

        You can manage data categories and data types, assign resources, and start on-demand classification according to your requirements in Data Security Posture.

      • Data Security Posture events

        The Data Security Posture Event Logs pageprovides information about the status of classification jobs.

Required permissions for the AWS outpost account

Information about the permissions that the outpost account requires to access the AWS account.

The following table describes the permissions that the outpost account requires to access the AWS account.

Permission Access level Use case
cloudformation:* Full access To allow full access to CloudFormation stacks and stack sets with the following prefix: Laminar
lambda:* Full access To allow full access to Lambda functions with the following prefix: lmnr-
logs:* Full access To allow full access to the log streams related to Lambda functions with the following prefix: lmnr-
s3:* Full access To allow full access to S3 buckets and their contents with the following prefix: lmnr-
kms:* Full access To allow full access to all KMS keys.
secretsmanager:GetSecretValue Read To read secrets with the following prefixed: lmnr-
secretsmanager:DescribeSecret Read To describe secrets with the following prefix: lmnr-
secretsmanager:List* List To list secrets with the following prefix: lmnr-
secretsmanager:CreateSecret Write To create secrets with the following prefix: lmnr-
secretsmanager:Update* Write To update secrets with the following prefix: lmnr-
secretsmanager:Delete* Write To delete secrets with the following prefix: lmnr-
secretsmanager:TagResource Write To add tags to secrets with the following prefix: lmnr-
secretsmanager:UntagResource Write To remove tags from secrets with the following prefix: lmnr-
account:GetAccountInformation Read To read account information.
account:GetRegionOptStatus Read To read region opt status information.
account:ListRegions List To list AWS regions.
iam:CreateServiceLinkedRole Write To create service-linked roles.
sts:DecodeAuthorizationMessage Write To decode authorization messages.
iam:CreateInstanceProfile Write To create instance profiles with the following prefix: lmnr-
iam:DeleteInstanceProfile Write To delete instance profiles with the following prefix: lmnr-
iam:AddRoleToInstanceProfile Write To add roles to instance profiles with the following prefix: lmnr-
iam:RemoveRoleFromInstanceProfile Write To remove roles from instance profiles with the following prefix: lmnr-
iam:PassRole Write To pass roles to Lambda and EC2 services.
iam:DetachRolePolicy Write To detach policies from roles with the following prefix: Laminar
iam:DeletePolicy Write To delete policies with the following prefix: Laminar
iam:DeleteRole Write To delete roles with the following prefix: Laminar
cur:DescribeReportDefinitions Read To read CUR report definitions.
cur:* Full access To allow full access to CUR definitions with the following prefix: lmnr-
