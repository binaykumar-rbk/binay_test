Restoring M365 SharePoint sites using Microsoft 365 Backup Storage

Restore a Microsoft 365 SharePoint site to mitigate data loss events such as accidental deletion, ransomware attacks, or other unforeseen incidents using Microsoft 365 Backup Storage. You can also restore data using express restore points that are either the most recent or closest to a specific date or time.

Procedure

  1. Log in to RSC.
  2. Click and select SaaS App Protection.
    The SaaS App Protection dashboard appears.
  3. Click Microsoft 365 Backup Storage SharePoint.
    The SharePoint dashboard appears.
  4. Click Restore.
  5. Under Restore a Site, click Get Started.
    The Restore a Site wizard appears with a list of SharePoint sites.
  6. Select the preferred data location to filter the objects by a specific location.
  7. Select Active or Relic to filter the objects by status.
  8. Select the SharePoint site and click Next.
  9. Select one of the following options to choose a restore point:
    OptionDescription
    Use the express restore point closest to a particular date and time Recover data using the express restore points that are either the most recent or closest to a specific date or time.

    Select a date and time.
    View all restore points and choose one View all restore points and select a point to recover the data. To do this, follow these steps:
    • From the Time Range filter, select the time.
    • Select a restore point.
  10. Click Next.
  11. Select the option that you want to use to recover the objects:
    OptionDescription
    In-place recovery Recover objects to their original location and overwrite the duplicate objects.
    Recover to a new SharePoint site Recover objects to a new URL.
    A Confirmation window appears.
  12. Click Restore.

Results

RSC restores the objects associated with the SharePoint site protected using Microsoft 365 Backup Storage.
