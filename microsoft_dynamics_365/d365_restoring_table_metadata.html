Restoring Dynamics 365 table metadata

Restore Microsoft Dynamics 365 table metadata from a snapshot to a Microsoft Dynamics 365 source environment.

Procedure

  1. Log in to RSC.
  2. Click and select SaaS App Protection.
    The SaaS Apps dashboard appears.
  3. Click the Dynamics 365 workload.
    The Dynamics 365 dashboard appears.
  4. Click Restore.
  5. On Restore Metadata, click Get Started.
    The Restore Metadata wizard appears.
  6. Click Next.
    The Restore Metadata page appears.
  7. Select Table metadata, and click Next.
  8. Select tables for which you want to restore the associated metadata.
    You can filter the tables based on the following table types: Standard, Activity, or Elastic. Standard tables store structured data, Activity tables store information about activities or events, and Elastic tables store information that changes frequently and can accommodate various data types and formats.

    You can use the search field to search for a specific table.

    RSC displays the All metadata selected option.
  9. Click All metadata selected to determine whether you want to restore all metadata or specific metadata.
    The Select metadata for the selected table appears.
  10. Select one of the following:
    OptionDescription
    Restore all metadata including columns, relationships, keys, forms, and views Select this option and click Save when you want to restore all metadata related to a table. For example, columns, relationships, keys, forms, and views.
    Restore specific metadata Select this option, and then select specific Forms, Columns, Relationships, Keys, and Views, and click Save.
  11. Click Next.
  12. In Destination Environment, select the Microsoft Dynamics 365 environment.
    You can restore the metadata only to the source environment.
  13. In Bypass Dynamics 365 Automation, select the Bypass all power automate flows, custom plugins and workflows check box.
    For seamless restoration, Rubrik recommends not enabling automated Dynamics 365 processes. Bypassing these processes prevents unnecessary jobs from running on the restored metadata, thereby minimizing restoration failures and enhancing performance.
    RSC temporarily deactivates the automated Dynamics 365 processes, such as power automate flows, custom plugins, and workflows associated with the selected metadata. After restoration, RSC enables the automated processes for the selected metadata again.
  14. Click Next.
    The Select a Snapshot page appears displaying a list of snapshots for restoring the selected metadata.
  15. Select the snapshot for recovering metadata.
    You can filter the snapshots based on the Time Range filter.

    While restoring specific metadata, you have the option to compare the snapshot data with the data available in the destination environment.

  16. Hover over the snapshot and click Compare with Destination to compare the snapshot data with the data available in the previously selected destination environment.
    On the Compare with Destination page, you can also change the date to select a different snapshot taken on a specific date. The dashboard then shows the differences between the data of the newly selected snapshot and the destination environment data.
    The Compare with Destination page displays details regarding the data that will be deleted, updated, or added from the selected snapshot to the destination environment.
  17. Click Done
    The Confirmation page appears.
  18. Review the restore options and metadata, and click Restore.

Results

RSC starts restoring the metadata to the selected Microsoft Dynamics 365 environment.

You can monitor the progress of the restore job on the Events page. After completing the restore job, RSC generates a log file containing additional details about the restore operation, such as the mapping between the old ID and the new ID for deleted metadata. You can download the log file by clicking the Additional logs link in the restore event. The link expires in 24 hours.
