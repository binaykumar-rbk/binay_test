Restoring Dynamics 365 data

Restore Dynamics 365 data from a snapshot to a Microsoft Dynamics 365 source environment.

About this task

After analyzing the data changes, you can selectively restore the required table rows from a snapshot taken before the update.

Procedure

  1. Log in to RSC.
  2. Click and select SaaS App Protection.
    The SaaS Apps dashboard appears.
  3. Click the Dynamics 365 subscription.
    The details page for the Dynamics 365 subscription appears.
  4. Click Analyze.
  5. From the Analyze Tables pane, select a table.
    A graph appears, displaying three distinct lines representing rows that were added, deleted, and modified within a seven-day period.
  6. Select a date in the graph.
    A list of rows changed on that particular date appears.
  7. Select the rows that you want to restore, and then click Restore.
    The Restore Rows wizard appears.
  8. Click Next.
  9. In Destination Environment, select the Dynamics 365 environment to which you want to restore the rows.
    RSC supports restoring the rows only to the source environment.
  10. Optional: Select the Bypass all power automate flows, custom plugins and workflows check box.
    For seamless restoration, Rubrik recommends deactivating automated Dynamics 365 processes. Bypassing these processes prevents unnecessary jobs from running on the restored rows, thereby minimizing restoration failures and enhancing performance.
    RSC temporarily deactivates the automated Dynamics 365 processes, such as power automate flows, custom plugins, and workflows associated with the selected rows. After restoration, RSC enables the automated processes for the selected rows again.
  11. Click Next.
    The Select a Snapshot page appears and displays a list of snapshots for restoring the selected rows.
  12. Select a snapshot for recovering rows.
  13. Hover over the snapshot and click Compare with Destination to compare the snapshot data with the data available in the previously selected destination environment.
    The Compare with Destination page appears with the details related to the data that will be deleted, updated, or added from the snapshot.

    You can also change the date to select a different snapshot taken on a specific date. The dashboard then shows the differences between the data of the newly selected snapshot and the destination environment data.

  14. Click Done.
    The Restore Rows page appears.
  15. Click Next.
    The Restore Related Rows page appears and displays the associated parent and child rows.

    You can review the hierarchy of parent and child rows to determine the depth of the child rows, which helps you analyze and identify deleted or orphaned rows.

  16. Select the number of levels of the child tables that you want to analyze and click Analyze Impact.
    For example, to analyze three levels of child rows, you must select 3 as the depth.
    The impact analysis starts. You can view the progress of the job on the Events page.
  17. Do one of the following:
    OptionDescription
    Wait for the analysis to complete Wait for the analysis to complete, which may take some time. Complete the remaining steps to restore the rows after the analysis is complete.
    Save and exit Click Save and Exit to save the configuration and close the window when you do not want to wait for the analysis to complete. You can view the status of the analysis job on the Events page. After the analysis completes, you can click to view the notifications and resume the restore operation.
  18. On the Restore Related Rows page, select the child rows to restore, and click Next.
    You can view the number of deleted and orphaned rows. When you choose to restore a row, RSC recreates the deleted row and associated links of the orphaned row.
    The Confirmation page appears.
  19. Review the restore options and rows, and click Restore.

Results

RSC starts restoring the rows to the selected Microsoft Dynamics 365 environment.

You can monitor the progress of the restore job on the Events page. After completing the restore job, RSC generates a log file containing additional details about the restore operation, such as the mapping between the old ID and the new ID for deleted rows. You can download the log file by clicking the Additional logs link in the restore event. The link expires in 24 hours.
