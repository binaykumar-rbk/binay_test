        • High-level implementation workflow for GitHub

        • Quick links for GitHub

          Immediate access to important tasks and information, enabling quick setup, daily operations, and critical recovery.

        • Plan your GitHub protection

          Before setting up a GitHub organization in RSC, you need to plan user and access management for designated RSC users. This involves reviewing the permissions that RSC requires for GitHub, as well as in your Azure or AWS cloud provider, to install Exocompute and create storage accounts for protecting GitHub organizations.

        • Set up GitHub organization

          You can add a GitHub organization in RSC to start protecting GitHub resources.

          • Permission categories for GitHub

            For the backup and recovery tasks associated with GitHub repositories, RSC requires specific permissions. RSC groups these permissions into categories for granular management so that you can include or exclude these categories as required.

            • Permission model for GitHub

              RSC provides a least permission model for GitHub organizations, granting minimal necessary permissions starting with read-only access. You can add the GitHub Backup app while onboarding to establish data protection and install the GitHub Restore app to elevate permissions for recovery operations when required.

Managing granular permissions for GitHub organization

Update granular permissions for a GitHub organization when needed.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click GitHub.
    The GitHub inventory page appears.
  5. Click Organizations.
    A list of organizations appears.
  6. Click for an organization and click Manage Permissions.
    The Manage Permissions wizard appears.
  7. Click Install App for the Rubrik GitHub Restore app.
    You can uninstall the GitHub Restore app after the recovery is complete or if it is no longer needed.

Results

RSC updates the granular permissions for the GitHub organization.
