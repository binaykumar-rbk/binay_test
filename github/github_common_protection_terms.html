Common protection terms for GitHub

It is important to understand some common terms when protecting a GitHub organization with RSC.

By familiarizing yourself with the common terms, you can better understand and navigate the various steps and components involved in protecting a GitHub organization.
Term Description
GitHub A web-based platform used for version control and collaborative software development using Git. RSC specifically supports the SaaS version of GitHub for data protection.
Organization The top-level container in GitHub that allows groups of projects to be managed collectively. An organization acts as the primary entity for onboarding and policy application in RSC.
Repository The fundamental unit of storage in GitHub containing all project files and the entire revision history.

RSC protects repositories including files, commits, branches, and YAML-based actions. RSC supports applying SLA Domain policies at the repository level.
Backup location A customer-managed location within an Azure or AWS environment where the data associated with GitHub organizations and repositories is stored. This is also referred to as the archival location.
Exocompute A Rubrik-managed infrastructure that runs within your Azure or AWS environment to provide secure, sovereign data processing. For GitHub protection, Exocompute performs git clone and git fetch operations to move repository data directly to your archival storage. During recovery, it manages git push commands to restore data and metadata, including branches and commits, preserving your full development history.
GitHub app The integration mechanism used by RSC to securely access GitHub resources. During onboarding, you can install a Rubrik GitHub Backup app and a Rubrik GitHub Restore app on the organization to grant RSC the necessary permissions.

The term GitHub app refers to both the Rubrik GitHub Backup app and the Rubrik GitHub Restore app.
Snapshot immutability A security feature ensuring that once a backup snapshot of a repository is created, it cannot be modified or deleted, protecting the data against ransomware or accidental deletion.
Storage account An Azure Blob Storage or an Amazon S3 bucket used as the underlying storage layer for GitHub backups. RSC requires this to be configured in the same region as the Exocompute setup to minimize data transfer costs.
