Set up Exocompute for GitHub on Azure

With the Exocompute for GitHub on Azure feature, RSC launches an Azure Kubernetes Service (AKS) cluster to perform data protection operations for your GitHub repositories. RSC uses this AKS cluster as a dedicated compute resource within your Azure subscription to ensure data sovereignty and efficient processing.

Key operations for GitHub Exocompute

Exocompute uses an ephemeral AKS cluster to perform the following GitHub-specific operations:
  • Repository backups: The AKS nodes perform standard git clone for initial full backups and git fetch for incremental updates from GitHub servers.
  • Data transfer to Azure Blob Storage: Exocompute worker nodes coordinate the movement of repository data from the compute layer to your designated Azure Blob Storage containers.
  • Snapshot metadata management: During data transfer, the AKS cluster collects essential metadata such as version details and timestamps, and transmits it to RSC for management and tracking.
  • Scale-on-demand: Based on the workload volume, Exocompute automatically scales AKS nodes to optimize performance and terminates them after the task completes to minimize Azure costs.
  • Recovery: When you initiate a recovery, Exocompute reads snapshot data from Azure Blob Storage and pushes it to the target GitHub organization or repository.

Networking setup methods

RSC provides two methods for configuring the network required for GitHub Exocompute.
Method Overview Advantages Requirements
Automated networking setup (Recommended) RSC automatically creates and manages the VNet, subnet, and Network Security Groups (NSGs). Minimizes configuration errors and accelerates onboarding. Requires the Automated networking setup permission group during Azure cloud account addition while onboarding GitHub organization.
Manual networking setup You can manually create and provide a pre-configured VNet and subnet. Provides full control for organizations with strict security or custom IP requirements. Requires manual alignment with specific CIDR and NSG requirements of Rubrik.

How to select a method

You can select your networking method when adding a GitHub organization to RSC using the Manage Granular Permissions page.

  • Default selection: The Automated networking setup permission category check box is selected by default.
  • To use manual setup: You must manually clear the Automated networking setup permission category check box when adding the Azure cloud account.

Important: After you add an Azure cloud account for the Exocompute use case, the permission category selection becomes permanent. You cannot select or clear the Automated networking setup check box later. To change the networking method, you must delete the Azure cloud account and add it again with the required permission category.

For more information, see Automated versus manual networking setup.

Regional requirements

To minimize data transfer costs and latency, Rubrik recommends setting up Exocompute in the same Azure region as your Azure Blob Storage archival location. For GitHub protection, the Exocompute cluster serves as the gateway in your environment, bridging the RSC control plane and the GitHub SaaS environment.

Exocompute configuration tasks

For configuring Exocompute, you must perform the following prerequisites and tasks:
  1. Evaluating the Exocompute on Azure options before configuring Exocompute. For more information, see Localized versus centralized Exocompute on Azure and Network choices for Exocompute on Azure.
  2. Creating an Azure Virtual Network (VNet) and its subnets or identifying an existing VNet for Exocompute, as described in Creating a VNet.
  3. Ensuring all networking requirements for Exocompute are met, as described in Set up network for Exocompute on Azure.
  4. Adding an Azure subscription in RSC for the Exocompute host and Rubrik-managed AKS cluster use cases, as described in Set up Azure cloud accounts. For steps on creating an Azure cloud account, see Creating Azure cloud accounts with OAuth and Azure cloud account addition without OAuth.
  5. Configuring the Azure Exocompute host subscription for a Rubrik-managed AKS cluster, as described in Configuring Exocompute for Rubrik-managed clusters.
  6. To configure Exocompute while onboarding a GitHub organization, see Adding GitHub organization.
Note:
After configuring Exocompute on Azure, the next step is to set up a backup location, as described in Set up backup location for GitHub.
