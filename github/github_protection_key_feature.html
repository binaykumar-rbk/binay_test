Key features for protecting GitHub

RSC provides unified data management, automated global data protection, and cyber resilience features to protect your GitHub organization.

By leveraging RSC, you can overcome the limitations of native GitHub backup options, ensuring that source code, metadata, and development workflows are secure and recoverable.

The key features for GitHub organization protection include:
  • Unified data management
  • Automated global data protection through SLA Domains
  • Flexible recovery options for GitHub repositories
  • Immutable backup copies
  • Supported components

Unified data management

Feature Description
RSC Provides a SaaS platform for GitHub organization protection. RSC acts as a single, unified management plane, reducing complexity and operational overhead by providing a centralized view of your data protection posture.
Consolidated monitoring and reporting Provides unified reporting to track SLA Domain assignment, protection status, and recovery activity across all GitHub repositories.
Enhanced compliance and governance Enables you to meet regulatory mandates by providing auditable backup and recovery processes for critical intellectual property.

Automated global data protection

Feature Description
SLA Domain protection Uses Rubrik SLA Domains to automate the protection of GitHub repositories. You can apply a single policy to manage backup frequency and retention periods.
Policy-driven automation Automatically discovers new repositories within a GitHub organization and applies protection based on predefined SLA Domain.
Point-in-time snapshots Captures snapshots of repositories, including files, commits, branches, tags, and YAML-based GitHub actions, ensuring a consistent state for recovery.

Recovery options

Feature Description
Organization recovery Restores a repository to a new location within the GitHub organization without overwriting the existing data.
Cross-organization recovery Supports recovering a repository to a different GitHub organization, which is essential for migration scenarios or legal discovery.
Granular metadata recovery Beyond the source code, RSC recovers critical metadata such as branches and commits to maintain the integrity of the development history.

Security and control

Feature Description
Immutable backups Ensures that GitHub backup copies are immutable. Once stored, they cannot be modified or deleted by unauthorized users or ransomware.
Air-gapped isolation Backups are stored separately from the GitHub SaaS production environment, reducing the risk of failure or security breaches.
Role-based Access Control (RBAC) Defines specific permissions for backup administrators versus developers. Elevated access is reserved strictly for recovery operations to prevent accidental data loss.
Data sovereignty Supports the Bring Your Own Storage (BYOS) model using Azure Blob Storage and Amazon S3, enabling organizations to maintain full control over where their GitHub backup data resides.

Supported components

RSC supports the protection of the following GitHub components:

  • Repositories
    • Files
    • Commits and pushes
    • Branches and tags
  • YAML-based GitHub actions residing within the repository
