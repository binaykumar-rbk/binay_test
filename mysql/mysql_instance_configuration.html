Configuration of MySQL instance

To configure MySQL instances with RSC, integrate the MySQL environment by registering Linux servers with a Rubrik cluster and installing the Rubrik Backup Service agent. Using this setup, RSC discovers MySQL instances, requiring configuration details and credentials for effective backup and restore management.

To configure MySQL instances with RSC, you need to integrate your MySQL environment. This involves registering the Linux servers hosting your MySQL instances with a Rubrik cluster. A crucial part of this is installing the Rubrik Backup Service (RBS) agent on each MySQL host. Using RBS, the Rubrik cluster and the host communicate and transfer data, ensuring effective management of backup operations by RSC. This initial setup establishes the necessary connection for RSC to interact with your MySQL servers.

After the host is integrated and RBS is active, RSC discovers the MySQL instances running on it. You can then add each instance to RSC by providing specific configuration details. This includes an instance name, the designated Rubrik cluster for management, and network information like IP address and port. You are also required to provide Linux OS credentials of the OS user running the MySQL instance and MySQL user credentials with sufficient privileges for backup and restore operations. With this configuration, RSC logically onboards the MySQL service, discovers its databases, and applies protection policies.
