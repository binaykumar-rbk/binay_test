Restoring MySQL snapshots to the original instance

Restore MySQL snapshots to the original MySQL instance from which the backup was taken.

Before you begin

Ensure the following:
  • Create a mount point for restoring your MySQL instance snapshots, as described in Creating a restore mount point for MySQL instance snapshots.
  • Install the nfs-utils package on MySQL hosts where you want to restore the snapshots.

Procedure

  1. Stop the MySQL server.
  2. Copy the existing data directory of the MySQL instance to a safe location in case recovery fails.
  3. Delete all files in the data directory of the server.
  4. Enter the following command to set up the MySQL instance on the data directory: 
    mysqld --initialize --user=<mysql_os_user> --datadir <restore_dir>
  5. Enter the following command to copy data from the mounted path to the target data directory: 
    /etc/rubrik/scripts/mysqldb/mysqldb_restore.sh --backup-dir <mount_path> --datadir <restore_dir> --copy-back
  6. Run the following command to change the owner attribute and the group attribute to the MySQL system user: 
    chown -R mysql:mysql <restore_dir>
  7. Verify your data and restart the MySQL server.
  8. Apply the binlogs using the following restore script: 
    /etc/rubrik/scripts/mysqldb/mysqldb_restore.sh --backup-dir <mount_path> --apply-binlog --uuid <instance_cdm_id> -â€“recovery-time <Point-in-time-recovery> --user <mysql_user> --password <mysql_password> --host <mysql_hostname> { --socket-file <socket_file> | --port <mysql_port> [--ssl-ca-cert-file <ssl_ca_cert_file> --ssl-cert-file <ssl_cert_file> --ssl-key-file <ssl_key_file>] }
    Replace <Point-in-time-recovery> => with UTC timestamp of the point-in-time selected by the user or UTC timestamp of the snapshot.

Results

You have successfully restored MySQL snapshots to the original MySQL instance from which the backup was taken.
