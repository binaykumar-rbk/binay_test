Creating custom SLA Domains for MySQL

Create a custom SLA Domain for MySQL to streamline and customize protection and recovery plans for MySQL instances.

Before you begin

Do the following:
  • If you plan to enable archiving, configure an archival location for each Rubrik cluster to which you assign the SLA Domain, as described in Creating a data center archival location.
  • If you plan to enable replication, make sure replication pairing exists between the replication source and target, as described in Replication target setup.

About this task

You can create an SLA Domain at MySQL instance level only. You cannot create an SLA Domain at the database level.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click SLA Domains.
    The SLA Domains page appears.
  4. Click Create SLA Domain.
    Ensure that you understand the information available in the Create SLA Domain wizard, including snapshot frequency and retention policies, before creating one.
    The Create SLA Domain wizard appears.
  5. Click Start on the Create SLA Domain page that describes SLA Domains.
    Selecting Don't show again skips this page the next time you create SLA Domains.
  6. In the Databases and Apps section, select MySQL and click Next.
  7. Optional: Turn on Retention Lock.
    Turning on Retention lock prevents accidental or malicious modification or deletion of data during the retention period.
    You can only use Compliance mode for retention lock in the SLA Domains created for the MySQL workload. You cannot assign or reduce the retention of these SLA Domains, and RSC does not allow snapshots to expire before the scheduled expiration date.
  8. Read the acknowledgment statement and select the checkbox.
  9. Click Next.
    The Frequency and Retention page appears.
  10. Enter a snapshot frequency and a corresponding retention period.
    The snapshot frequency categories are optional, but at least one category must have a value.
  11. Optional: In Snapshot window, enter a time window for the snapshot.
    When you specify a snapshot window, RSC takes the snapshot during the specified time frame. Workloads managed by a Rubrik cluster use the time zone of that Rubrik cluster. Workloads managed by RSC use UTC.
  12. In First full snapshot, select a start type.
    OptionDescription
    First opportunity The first full backup is taken by default when you add a data source.
    Custom range The first full backup occurs within a specified window.
  13. Click Next.
    The Archival and Replication page appears.
  14. Optional: Select Archival.
    Selecting this option configures archival locations for disaster recovery and long-term retention. You can click Learn more to learn about archival locations for disaster recovery and long-term retention.
  15. Optional: Select Replication, then select a replication mode radio button.
    Replication modeDescription
    Many-to-one

    Replicates data from multiple source clusters to a single target cluster, centralizing data protection and simplifying recovery efforts. Use this mode when you have many sources replicating to one central target. It requires configuring at least one replication pairing between the replication target and any replication source.
    One-to-one

    Replicates data between dedicated pairs of source and target clusters. Use this mode when replication sources and targets are configured one-to-one as part of the disaster recovery plan.
    Selecting this option configures replication between source and target clusters for data protection and disaster recovery. You can click Learn more to learn about setting up replication pairings between source and target clusters.
  16. Click Select Source Clusters.
    This option is available only in SLA Domains with archival policy and replication policy configured with one-to-one mode.
    The Select Clusters window appears.
  17. Select the clusters to use for archival and replication policies, and click Save.
  18. Click Next.
    Based on your selection on the Archival and Replication page, a page with the archival or replication parameters appears. If you have not enabled Archival or Replication, the Object-Specific Settings page appears.
  19. If you have enabled Archival or Replication, specify the archival and replication parameters.
    The Configuring an archival policy and Configuring replication policy topics provide more information on how to configure the archiving and replication policies.
  20. On the Object-Specific Settings page, select the frequency and retention of MySQL binary log backup files, and click Next.
    The Confirmation page appears.
  21. In SLA Domain name, enter a name to identify the SLA Domain.
  22. Optional: In Description, enter a description to provide additional context for the SLA Domain.
  23. Review the configuration details and click Create.
    A message appears, stating that you have successfully created an SLA Domain.

Results

You have created a custom SLA Domain for MySQL.
