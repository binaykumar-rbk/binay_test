Retention lock for MySQL SLA Domains

A retention lock on an RSC SLA Domain is a security feature that prevents premature deletion of MySQL snapshots.

The retention lock is a security feature that enhances data protection by ensuring that no one can alter or delete data in the snapshot for a defined period. This provides an additional layer of security for essential business data.

Configuration

Enabling the retention lock feature requires configuring an RSC SLA Domain for MySQL, and then activating the retention lock. When you assign this SLA Domain to MySQL instances, RSC takes snapshots according to the defined schedule and retention policies.

Considerations

Some important points to consider when enabling the retention lock feature for MySQL are that:
  • RSC supports the retention lock feature only in the compliance mode, not in the governance mode.
  • RSC assigns SLA Domains to workloads asynchronously, with tasks running independently, without waiting for others to finish.

Benefits

The following table describes the key benefits of configuring the retention lock feature for MySQL SLA Domains:
Benefit Description
Enhanced data protection Safeguards critical data against accidental or malicious alterations and deletions, ensuring the integrity of essential business information.
Regulatory compliance Supports adherence to regulatory requirements by maintaining unaltered records for the mandated duration, essential for audits and compliance reporting.
