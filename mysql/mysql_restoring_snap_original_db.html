Restoring MySQL snapshots to the original database

Restore the MySQL snapshots to the original MySQL database from which the backup was taken.

Before you begin

Ensure the following:
  • The target MySQL server is using the same page size that was used on the server on which the backup was made.
  • You have enabled the innodb_file_per_table option on the target server.
  • The selected database tables already exist on the target server in the same table definition.
  • The MySQL instance username has CREATE/DROP database, CREATE/ALTER TABLE, FLUSH TABLE WITH READ LOCK, CHECKSUM TABLE privileges.
  • A mount point for restoring your MySQL database snapshots is available. The Creating a restore mount point for MySQL database snapshots topic describes how to create a mount point.
  • You have installed the nfs-utils package on MySQL hosts where you want to restore the snapshots.

About this task

Database recovery only recovers partial data and might result in inconsistencies. For consistent data recovery, recover the instance.

Procedure

  1. Run the following command to copy data from the mounted path to the target data directory: 
    /etc/rubrik/scripts/mysqldb/mysqldb_restore.sh --backup-dir <mount_path> --datadir
        <restore_dir> --copy-back --include-dbs <databases_to_restore> --apply-binlog --uuid <cdm id>
        -–recovery-time <Point-in-time-recovery> --user <mysql_username> --password <mysql_password>
        --host <mysql_hostname> { --socket-file <socket_file> | --port <mysql_port> [--ssl-ca-cert-file
        <ssl_ca_cert_file> --ssl-cert-file <ssl_cert_file> --ssl-key-file <ssl_key_file>]
    }
    Replace <Point-in-time-recovery> => with UTC timestamp of the point in time selected by the user or UTC timestamp of the snapshot.
  2. After recovery is complete, query the database.

Results

You have successfully restored MySQL snapshots to the original MySQL database from which the backup was taken.
