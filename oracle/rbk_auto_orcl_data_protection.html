Automated Oracle Data Protection

Automatically discover and protect Oracle databases.

When RBS is installed on an Oracle host, or on one of the nodes of an Oracle RAC, the Rubrik cluster automatically discovers that Oracle host, or Oracle RAC, and all Oracle database instances running on it. The Rubrik cluster can then create database snapshots, and manage retention, replication, and archiving of those snapshots. These tasks are performed according to the SLA Domain policies assigned to, or inherited by, each discovered database.

DBAs and backup administrators can manage multiple point-in-time copies of Oracle databases, and perform instant recovery, live migration, and restore tasks.

Oracle discovery derives the ORACLE_SID and ORACLE_HOME values for each database from the pmon environment variables. These values identify the system identifier (SID) of the database and map to the directory where the Oracle database client software is installed.

Note:
For Oracle hosts that include a running ASM instance, the OSASM group members must be granted the SYSASM system privileges to administer storage. The OSASM group is named asmadmin. The Oracle Home Owner must be a member of the OSASM group to ensure successful discovery of the Oracle host.
