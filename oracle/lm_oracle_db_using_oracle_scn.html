Live Mounting an Oracle database using an Oracle SCN

Recover Oracle snapshots using the system change number instead of the date and time the snapshot occurs, providing a more accurate point of recovery.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click the Oracle Databases inventory card.
    The Oracle Databases page appears.
  5. Click All Databases.
  6. In the Name column, click the name of a database.
    Alternatively, enter a name in the search field or use the filters in the left pane.
    The Local page for the database appears, with the Snapshots card showing the month view.
  7. Open and select Recover by SCN.
    The Oracle Database Recovery wizard opens.
  8. In Enter SCN, enter the system change number (SCN) for the database you are recovering.
  9. Select Live Mount and click Next.
  10. In Name, select the name of the host or cluster to host the live mount.
  11. Select a restore option.
    OptionDescription
    In case of partial or failed restore, discard the recovery. Deletes a partially restored database, or deletes a database restore that failed, from the Oracle host.
    In case of partial restore, leave partially restored database on the host. Retains the Live Mount on the Rubrik cluster and retains the partially restored database on the Oracle host.
    Do not restore, make RMAN backup files available for manual recovery. Mounts the recovery point so you can recover the files using RMAN on the target host. The files-only mount option mounts the files to the designated backup image path.

    In Backup Image Path, enter a full path on the target host. Use this field to specify where to place the datafiles during the mount operation.

    With this option, you can click Live Mount to complete the task. The remaining steps in this task are not required.
    Live mounts to the source require that you accept the checkbox titled The Live Mount will be performed on the source host.
  12. Click Next.
    The Advanced Cloning Options page appears.

    The Advanced Settings page displays several optional parameters to define.

  13. Optional: Enter the Advanced Cloning Options information.
  14. Click Live Mount.

Results

You have live mounted an Oracle database using an SCN.
