          Rubrik Backup Service is required in order to perform automated discovery of Oracle hosts, Oracle RAC systems, and Oracle databases.

        • Oracle configuration

          Initial configuration of the Rubrik cluster for Oracle requires at least one open or mounted Oracle database and a user account with SYSDBA privileges.

          • System requirements for Oracle databases

            Information about the system requirements for Oracle databases on the Rubrik cluster.

            • Direct NFS support

              Rubrik CDM supports dNFS-enabled Oracle systems.

          • Oracle support requirements on Windows

            RSC includes support requirements to protect Oracle databases on Windows.

          • Oracle 23ai known problems and workarounds

            RSC workload protection for Oracle 23ai databases running with on-premises Oracle Exadata has known issues, but there are available workarounds.

          • Creating an empty oratab file

            If the Oracle host has no databases running, create an empty /etc/oratab file before installing RBS software.

          • Bigfile tablespace restore for Oracle databases

            The Rubrik cluster supports restoring Oracle databases that are configured for bigfile tablespaces, which are used use to create very large data or temporary files.

          • Supported data transfer mechanisms for Oracle databases

            For Oracle databases, the Rubrik cluster supports the NFS and Oracle Secure Thrift data transfer mechanisms. By using these mechanisms, you can transfer data securely between the Rubrik cluster and the Oracle host.

        • Role based access

          Rubrik provides role-based access control, along with options for integrating with existing directory services.

        • Oracle SEPS wallet

          The Oracle Secure External Password Store wallet feature provides additional security for connections by not exposing your username and password on the command line when you log in. The feature stores the password credentials in the client-side Oracle wallet.

        • Oracle database management

          Manage discovered Oracle hosts and Oracle RAC systems from the Rubrik Oracle Databases page.

        • Export Oracle database logs to a USB device

          You can use Rubrik REST APIs to export logs from multiple Oracle databases to USB devices.

        • Oracle Data Guard on Rubrik clusters

          Rubrik clusters support Oracle Data Guard configurations to back up primary and standby databases and to discover new Data Guard groups.

        • Backups and archived redo logs

          The Rubrik cluster protects Oracle databases with backups and archived redo log backups, running as separate jobs and at different frequencies in the Rubrik cluster.

        • Archive log restore

          Archive log restore reinstates archived logs from a specified time range or log sequence period.

        • Oracle pluggable databases

          You can restore, mount, and clone Oracle pluggable databases on the Rubrik cluster.

        • Taking an on-demand snapshot

          Manually initiate a database snapshot in addition to the policy-based snapshots of the database. Assign an SLA Domain to manage that snapshot.

        • Point-in-time recovery

          Point-in-time recovery restores database and log snapshots to the defined point in time.

        • Number of channels in recovery

          The value for the number of channels in recovery can be configured for Live Mount, Clone, Restore, Instant Recovery, and Validation operations.

        • Recovery from RAC to standalone Oracle hosts

          For RSC running Rubrik CDM version 9.2 or later, you can perform live mounts or clones of Oracle RAC snapshots to an Oracle standalone host.

        • Oracle recovery using SCNs

          You can recover Oracle snapshots and mount archive logs using the system change number instead of the date and time when the snapshot occurs, providing a more accurate point of recovery.

        • Advanced Cloning Operations and custom PFILES

          You can use the Advanced Cloning Operations or a custom PFILE options when you require the target recovery operation to use a different configuration than the source database backup.

        • Oracle Live Mount

          An Oracle Live Mount creates a database from a point-in-time copy of the source database to speed up recovery time.

        • Instant Recovery for Oracle

          Instant Recovery for Oracle replaces an existing database with a point-in-time copy, requiring specific configurations and directory setup.

        • Database clones for Oracle

          You can clone an Oracle database by selecting a recovery point to clone to a target location.

        • Advanced Oracle DB clone and mount parameters

          A parameters file allows you to specify advanced recovery options for live mount and cloning tasks.

        • Same Host Recovery

          Recover a database to the source Oracle host or Oracle RAC.

        • Tablespace recovery

          Restore a tablespace in place to the same database.

System requirements for Oracle databases

Information about the system requirements for Oracle databases on the Rubrik cluster.

Requirement Description
CDM auto discovery Hosts must have at least one database in the OPEN or MOUNTED state, or an empty /etc/oratab file for CDM discovery.
Database mode configuration To enable Oracle database, Rubrik CDM requires the database to be in the ARCHIVELOG mode.
dNFS Depending on the Oracle version, Oracle patch 20720667 might be required. This is available for review in the patch version requirements described for direct NFS support.
Daylight savings time (DST) patch You must install the latest Oracle DST patch on the database managed by the Rubrik cluster, depending on the version of the Oracle database.

Oracle document ID 412160.1 provides more information about installing the patch and version requirements.
Linux hosts For Oracle instances on Linux hosts, the ~/.bashrc file is processed each time Rubrik CDM interacts with the Oracle data source. The host ~/.bashrc file should not contain processes that cannot be run at high frequency.
Maximum user processes You must set the minimum value of maxuproc to 16384.
NFS share with read/write size = 512 KB To perform Oracle backups, Rubrik CDM mounts multiple NFS shares with a default read and write size of 512 KB. The Oracle host must have the NFS client software installed and available. In an AIX environment, the default maximum size is 64 KB, which is not sufficient. To avoid performance degradation, increase the maximum size to 512 KB by installing the IBM provided patch.
Oracle Home Owner Rubrik CDM supports a single Oracle Home Owner for each host with SYSDBA privileges. For Oracle 12.1 and later, the Rubrik cluster supports using a single Oracle Home Owner for each host with SYSDBA and SYSBACKUP privileges.
RMAN Rubrik clusters use RMAN to perform backup and recovery of Oracle databases. The required RMAN scripts are generated automatically.

Databases must be in ARCHIVELOG mode and in an OPEN state for the Rubrik cluster to perform backups. Oracle databases in a MOUNTED state are not scheduled for database or log backups.

RMAN does not support backing up Oracle databases with data files larger than 16 TB hosted on the AIX operating system. However, if you are using secure thrift as the transport protocol for backing up Oracle databases, this limitation does not apply. If you are not using secure thrift, you can use Managed Volumes to protect Oracle databases with data files larger than 16 TB hosted on the AIX operating system.
Shared storage For Oracle RAC systems, Rubrik CDM supports shared storage configuration only on Automatic Storage Management (ASM). Archived redo logs must also be on shared storage ASM.
Storage system

Rubrik CDM protection supports the following restore types:

  • Oracle data files in Oracle file systems but not Oracle RAC systems that have storage on the file system.
  • ASM, which manages Oracle datafiles on standalone Oracle and Oracle RAC systems.

For successful restore of Oracle datafiles, the restore type must be the same as the original backup type.
User Credentials

You must configure the Oracle instance with a user account that has SYSDBA or SYSBACKUP privileges.

You can customize the default Oracle user account, on a per-host basis, while adding the Oracle host or Oracle RAC node in the Rubrik cluster. You can change the user credentials when editing the physical hosts associated with the Oracle host or RAC.
Note:
For Oracle instances on Linux hosts, the ~/.bashrc file is processed each time the Rubrik cluster interacts with the Oracle data source. Ensuring that the host ~/.bashrc file does not contain processes that cannot be run at high frequency is mandatory. Rubrik jobs might become unresponsive if the ~/.bashrc file contains or sources a file containing an interactive prompt.

IBM document IV24594 provides more information about NFS patches for IBM AIX version 6.1. IBM document IV24688 provides more information about NFS patches for IBM AIX version 7.1.
