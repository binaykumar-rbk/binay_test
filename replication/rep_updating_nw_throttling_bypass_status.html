Updating the network throttling bypass status for a target cluster

For a replication source Rubrik cluster, update the network throttling bypass status for a specific replication target Rubrik cluster by using a Rubrik GraphQL API mutation.

Before you begin

Determine the UUID of the source and target replication Rubrik clusters, as described in Determining the UUID of a Rubrik cluster.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click API Playground.
    The API Playground page appears.
  4. Click Open API Playground.
    API Playground opens in a new browser tab.
  5. In the query editor, enter the UpdateReplicationNetworkThrottleBypass mutation.
    The following code uses the UpdateReplicationNetworkThrottleBypass mutation to update the network throttling bypass status for the replication target cluster specified in the input:
    query UpdateReplicationNetworkThrottleBypass($input: UpdateReplicationNetworkThrottleBypassInput!) {
  updateReplicationNetworkThrottleBypass(input: $input) {
    success
  }
}
  6. In the variables editor, enter the input required by the mutation.
    To update the network throttling bypass status for a specific replication target cluster, specify the following: the boolean value to update the bypass status, the UUID of the target cluster, and the UUID of the source cluster.
    {
  "input": {
    "clusterUuid": "source_cluster_uuid",
    "config": {
      "shouldBypassReplicationThrottle": nw_throttle_bypass_status
    },
    "id": "target_cluster_uuid"
  }
}
    Where,
    • source_cluster_uuid is the UUID of the replication source Rubrik cluster.
    • nw_throttle_bypass_status is a boolean value.

      true sets the replication target cluster to bypass network throttling.

      false sets the replication target cluster to honor network throttling.

    • target_cluster_uuid is the UUID of the replication target Rubrik cluster.
  7. Click the play button.
    API Playground runs the mutation and displays the response. The response in this example indicates that the mutation operation was successful.
    {
  "data": {
    "updateReplicationNetworkThrottleBypass": {
          "success": true
        }
    }
}

Results

The Rubrik GraphQL API server responds with the result of the mutation operation to update the network throttling bypass status of the specified replication target cluster.
