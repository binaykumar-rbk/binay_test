Editing a MongoDB source for physical backup and recovery using Ops Manager

Edit the Ops Manager details such as the CA certificate and API keys for a MongoDB source cluster.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click the MongoDB inventory card.
    The MongoDB page appears.
  5. Click for a source and select Edit Source.
    The Edit Source wizard appears.
  6. Optional: Turn on Exclude Specific Nodes, and enter the hostname and port number and click Next.
    Turning on Exclude Specific Nodes excludes the specified hostname and port number from protection tasks, including Oplog backups. You can add multiple hostnames and ports to exclude additional nodes from protection tasks. However, you will require these nodes for recovery and must have RBS installed to perform recovery. Ops Manager recommends recovery for all the nodes.
  7. Optional: In CA Certificate ID, select a valid CA certificate.
    Note:
    Add the CA certificate that issued the host certificate to the Rubrik cluster.
    RSC supports importing CA certificate on Add a Source page from Rubrik CDM version 9.4.1 and later. You can import valid and unexpired CA certificates using Certificate Management.
  8. In API Keys, enter the public key and the private key, separated by a colon.
  9. Click Next.
    The Confirm Configuration page appears.
  10. Review the configuration and click Save.

Results

You have edited the Ops Manager details for the MongoDB source cluster.
