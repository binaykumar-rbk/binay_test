Adding a MongoDB host for physical backup and recovery using Ops Manager

Add a MongoDB host for physical backup and recovery of MongoDB sources using Ops Manager by specifying the required configuration details to establish a connection to the port and enable data storage and retrieval on the MongoDB server.

Before you begin

You must register all the MongoDB hosts, including the config server hosts for a sharded setup with the RBS.
Note:
The MongoDB hosts require RBS. The server hosting the Ops Manager does not require RBS installation. In a sharded setup, the config server requires RBS, but the Mongos node does not require RBS installation.

Procedure

  1. Log in to RSC.
  2. Click and select Data Protection.
    The Data Protection dashboard appears.
  3. On the navigation bar, click Inventory.
    The Inventory page appears.
  4. Click the MongoDB inventory card.
    The MongoDB page appears.
  5. Click Manage Hosts.
    The Settings page appears.
  6. Click Add Hosts.
    The Add Hosts dialog box appears.
  7. Select a Rubrik cluster.
  8. Select an option for using the RBS.
    OptionDescription
    I have already installed and registered the RBS software Provide the IP address or the hostname of the host and click Add.
    I want to install and register the RBS software now Select the platform and version for deploying RBS.
  9. In Host IP addresses or FQDNs, enter the IP addresses or hosts.
  10. Click Add.

Results

RSC checks connectivity with the specified MongoDB hosts and adds a single host or multiple hosts.
