MongoDB authentication mechanisms

Rubrik supports Salted Challenge Response Authentication Mechanism and Lightweight Directory Access Protocol authentication mechanisms that MongoDB provides for database access.

Rubrik supports Salted Challenge Response Authentication Mechanism (SCRAM) and Lightweight Directory Access Protocol (LDAP) for securing database access.

MongoDB supports the SCRAM and LDAP authentication mechanisms.

SCRAM:
  • SCRAM is the default and most commonly used secure authentication mechanism, making it a preferred choice for many deployments.
  • Verifies identity during MongoDB database connections and ensures secure transmission and storage of credentials.
  • MongoDB authenticates users by comparing provided credentials with username, password, and other information in the authentication database.
  • Rubrik supports SCRAM authentication on CDM versions 9.0.1 and later.
LDAP:
  • MongoDB uses credentials stored in an LDAP directory, such as a username and password, to authenticate users.
  • LDAP integration with MongoDB includes existing directory services such as Active Directory, which simplifies user management.
  • LDAP authentication centralizes user credentials in a directory, reducing complexity in managing accounts and passwords across systems.
  • Enabling LDAP authentication in MongoDB requires configuration to allow authentication to the LDAP server.
  • MongoDB connects to the LDAP server during authentication using the provided username and retrieves the password from the directory server, ensuring seamless integration.
  • Rubrik supports LDAP authentication on CDM versions 9.0.2 and later.

MongoDB documentation provides information about authentication.
