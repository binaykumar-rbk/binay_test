Ops Manager details retrieval

To add a MongoDB source with Physical Backup and Recovery via Ops Manager integration method, you need details such as the Group/Project ID and Cluster/Deployment ID from Ops Manager, which you can retrieve.

Requirements for retrieving the Ops Manager details

To retrieve the Group/Project ID and Cluster/Deployment ID from Ops Manager, you need the following:
  • An API key, which consists of a public key and a private key, separated by a colon.
  • The IP address and port number of the Ops Manager host.

For more information on creating an API key, you can refer to Creating API key to protect MongoDB sources using Ops Manager.

Retrieving a Group/Project ID for Ops Manager

To retrieve the Group/Project ID, you must send a curl request from any host that can communicate with Ops Manager.

Example request:
curl --user "{PUBLIC-KEY}:{PRIVATE-KEY}" --digest \
 --header "Accept: application/json" \
 --include \
 --request GET "https://{opsManagerHost}:{port}/api/public/v1.0/groups?pretty=true"

The response contains all the details of the group. You can copy the Group/Project ID from the response and use it to fill in the details for creating the MongoDB source in RSC.

For more information on the request and response parameters, you can refer to the MongoDB documentation on how to get all projects for the current user.

Retrieving a Cluster/Deployment ID for Ops Manager

To retrieve the Cluster/Deployment ID, you must use the Group/Project ID and send a curl request from any host that can communicate with Ops Manager.

Example request:
curl --user "{PUBLIC-KEY}:{PRIVATE-KEY}" --digest \
     --header "Accept: application/json" \
     --include \
     --request GET "https://<OpsManagerHost>:<Port>/api/public/v1.0/groups/{PROJECT-ID}/clusters?pretty=true"

The response contains a list of clusters. You can copy the required Cluster/Deployment ID from the response and use it to fill in the details for creating the MongoDB source in RSC.

For more information on the request and response parameters, you can refer to the MongoDB documentation on how to get all clusters in one project.
