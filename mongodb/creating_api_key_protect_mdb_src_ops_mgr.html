Creating API key to protect MongoDB sources using Ops Manager

Create an API key using Ops Manager to help you add your MongoDB sources to RSC.

Before you begin

Ensure you have the Project User Admin role assigned to you in Ops Manager.

Procedure

  1. Log in to your Ops Manager console.
  2. In the Organizations menu of the navigation bar, select the organization that contains your project.
  3. In Access Manager, select Project Access.
  4. Go to the API Keys tab and click Create API Key.
    Enter all the details in the API key information fields.
  5. Select Project Backup Admin for the project permissions and click Next.
    Copy the public and private keys to the clipboard, as Ops Manager displays these keys only once.
  6. Optional: Create an API key with Global Backup Admin permission. Go to the Admin section of your Ops Manager console and navigate to General > API Keys > Create API Key. Create a key with the permission Global Backup Admin.
    Note:
    This is an alternate method to create an API key.
  7. In Private Key & Access List, click Add Access List Entry.
  8. Enter the IPv4 addresses of all the nodes in the cluster you want to use to protect the MongoDB source and click Save.

Results

The obtained public and private keys form the API key. You must secure this API key to retrieve the Project/Group ID and Cluster/Deployment ID for adding a MongoDB source with Physical Backup and Recovery via Ops Manager integration method.
