Managed Volumes protection key features

RSC provides unified data management, automated global data protection, backup and recovery options for Managed Volumes, and advanced security features to protect critical data.

The RSC features for Managed Volumes and SLA Managed Volumes offer comprehensive capabilities for protection and management.

Unified data management

The following table provides details about the unified data management features for Managed Volumes.
Feature Supported in Description
RSC data protection RSC Provides a Software as a Service (SaaS) platform to manage and protect Managed Volumes across multiple deployment environments.
Consolidated reporting and monitoring RSC Offers unified reporting to track SLA Domain assignments, protection and recovery activities, and SLA Domain policy compliance for Managed Volumes workloads.
Rubrik Cloud Cluster Elastic Storage (ES)
  • Microsoft Azure: Rubrik CDM version 8.1 or later
  • AWS: Rubrik CDM version 8.1 or later
  • Google Cloud Platform (GCP): Rubrik CDM version 9.4 or later
  • Oracle Cloud Infrastructure (OCI): Rubrik CDM version 9.5 or later
 Extends protection capabilities for Managed Volumes deployed on cloud environments.

Automated global data protection through SLA Domains

The following table provides details about the automated global data protection features for Managed Volumes.
Feature Supported in Description
Protection policies RSC Assigns SLA Domains to SLA Managed Volumes to automatically ensure that backup operations meet defined business needs, Recovery Time Objectives (RTO), or Recovery Point Objectives (RPO) requirements.
Automated backups RSC Uses SLA Domains to automate backups of SLA Managed Volumes based on a defined schedule.
Pause/Resume protection Rubrik CDM version 5.2 or later Enables administrators to temporarily pause or restart protection activities for specific volumes.

Backup and recovery

The following table provides details about the backup and recovery features for Managed Volumes.
Feature Supported in Description
On-demand snapshots Rubrik CDM version 5.2 or later Supports taking manual snapshots of Managed Volumes as needed, outside of predefined schedules.
Archival Rubrik CDM version 5.2 or later Enables long-term retention of snapshots by tiering them to lower-cost archival storage, such as cloud object storage for compliance and cost efficiency.
Multiple archival locations Rubrik CDM version 9.4 or later Supports the configuration of multiple archival targets for a single Managed Volume to enhance data redundancy and compliance.
Legal hold Rubrik CDM version 7.2 or later Ensures specific snapshots are retained indefinitely for legal or compliance reasons, overriding standard SLA Domain expiration policies.
Replication Rubrik CDM version 5.2 or later Facilitates the replication of snapshots to other Rubrik clusters for disaster recovery purposes.
Replication reader co-existence with archival Rubrik CDM version 9.4.0 or later Maintains an active archival connection for a cluster while simultaneously serving as a replication reader for another cluster.
Retention lock Rubrik CDM version 7.2 or later Provides compliance and legal requirement mechanisms to enforce immutable retention periods for backups, preventing unauthorized deletion or modification of data.
Luna UI support Rubrik CDM version 9.3.0 or later Provides modern user interface support for network and volume configuration management.
Cross SMB domain recovery RSC Provides flexibility for live mount recoveries to hosts configured with different SMB domains or user accounts by specifying distinct SMB details, including domain name, username, and Active Directory (AD) groups, during live mount recoveries.

Security and access control

The following table provides details about the security and access control features for Managed Volumes.
Feature Supported in Description
RBAC RSC Offers granular permissions to administrators to manage data protection operations for Managed Volumes workloads.
Quorum authorization Rubrik CDM version 9.3.1 or later Enforces authorized approvals to modify data protection decisions, such as policy changes and deletion requests, to prevent unauthorized actions and protect data integrity.
NFSv4 without TLS Rubrik CDM version 9.3.0 or later Supports the NFS version 4 protocol for environments where NFSv3 is restricted for security reasons.
Note:
TLS encryption for data in transit is currently not supported for Managed Volumes. For more information, see Rubrik Compatibility Matrix.
User notes RSC Enables you to add descriptive notes to Managed Volumes objects for audit and tracking purposes.

Networking

The following table provides details about the networking features for Managed Volumes.
Feature Supported in Description
VLAN support Rubrik CDM version 5.2 or later Supports explicit assignment of a Subnet, in Classless Inter-Domain Routing (CIDR) format, to restrict the network traffic of Managed Volumes to a specific VLAN.
IPv6 support Rubrik CDM version 9.0 or later Supports communication and protection over IPv6 networks.
