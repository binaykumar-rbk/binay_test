Scripted backups for snapshots with assigned SLA Domains

Client-side scripted backups manage the frequency and retention settings of snapshots and those of the assigned SLA Domain to protect data sources associated with a Managed Volume.

The base frequency of an SLA Domain is the frequency at which snapshots are created to comply with all the rules specified for the SLA Domain. RSC recommends configuring scripted third-party backups of the data source associated with a Managed Volume to match the base frequency of the assigned SLA Domain.

Setting Description
Scripted frequency Determines how often the third-party script creates a backup of the specified database for Managed Volumes.
Scripted retention Specifies how long a snapshot is retained in the Managed Volume at any point.
SLA Domain base frequency

For Managed Volumes, SLA Domain base frequency represents how often the Rubrik cluster selects an available Managed Volume snapshot to ensure compliance with the SLA policies.

  • When the base frequency is the same as the scripted frequency, the Rubrik cluster manages each backup snapshot according to the SLA policies.
  • When the base frequency is less than the scripted frequency, the Rubrik cluster selects the most recent backup snapshot from the current period, and expires the remaining snapshots from that period.
  • When the base frequency is more than the scripted frequency, gaps in available backup snapshots occur and the Managed Volume is out of compliance.
SLA Domain retention Determines how long the Rubrik cluster manages the backup snapshots.

The frequency of scripted backups should normally be the same as the base frequency of the SLA Domain. The scripted frequency may exceed but cannot be less than the SLA Domain base frequency.

To enable the Rubrik cluster to provide restore points outside of the scripted retention period, SLA Domains used with scripted backups must provide longer retention periods than the scripted backups.

Well matched scripted backup and SLA Domain settings

Scripted backup settings appropriately matched to SLA Domain settings.

  • Scripted frequency and retention: One backup per day with seven days retention.
  • SLA Domain base frequency and retention: One snapshot per day with 31 days retention.
  • Result: The Rubrik cluster retains each daily snapshot for 31 days. Each daily snapshot contains seven days of scripted backups.

Unmatched scripted backup and SLA Domain settings

RMAN frequency is greater than the base frequency of the SLA Domain.

  • Scripted frequency and retention: One backup per hour with seven days retention.
  • SLA Domain base frequency and retention: One snapshot every four hours with 14 days retention.
  • The Rubrik cluster selects the latest of the four backups taken by the third-party backup script during the four hour base frequency period. The Rubrik cluster retains each selected snapshot for the 14 day retention period. Each snapshot contains seven days of scripted backups.
