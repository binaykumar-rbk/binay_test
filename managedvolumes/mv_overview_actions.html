Managed Volumes overview actions

Actions available on the Overview tab for SLA Managed Volumes and Managed Volumes in the Managed Volume object details page.

This table provides additional information on the actions available on the Overview tab of the Managed Volume object details page.

Action Description
Manage snapshot operations When the Managed Volume is in a read-only state, this helps to change the state to writable in order to receive backup data. When the Managed Volume is in a writable state, this helps to take a snapshot and changes the state to read-only.
Manage Protection Opens the Manage Protection dialog box to assign a Managed Volume to an SLA Domain.
Begin Snapshot The Rubrik cluster sets the Managed Volume to read-write and the Managed Snapshot Operations dialog box changes.
Take On Demand Snapshot Takes snapshots of always mounted Managed Volumes. The Take On Demand Snapshot dialog box appears a short while after clicking Begin Snapshot. After the snapshot is taken, the Managed Volume returns to a read-only state.
Ellipsis > Edit Open the ellipsis menu to edit the Managed Volume.
Ellipsis > Delete Open the ellipsis menu to delete the Managed Volume.
