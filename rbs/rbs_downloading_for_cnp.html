    • Rubrik clusters

    • Security

      Rubrik Security Cloud provides end-to-end security for data management information.

    • Outbound connection through ports

      RSC requires you to open an outbound connection through port 443 for a Rubrik cluster to access specific sites.

    • Rubrik Backup Service

      The Rubrik Backup Service is software installed on a host that coordinates activity between the Rubrik cluster and the host.

      • Obtaining Rubrik Backup Service connector

        Download the Rubrik Backup Service connector to enable the connected Rubrik clusters to restore folders and files over corrupted data in the file system on the host.

      • Downloading Rubrik Backup Service for cloud-native protection

        Download the Rubrik Backup Service for cloud-native protection to coordinate backup activity between the Exocompute instance and RSC.

      • Minimum RBS privileges for Windows

        RBS requires certain privileges and permissions on Windows for both installation and regular operations.

      • Rubrik Backup Service registration to multiple Rubrik clusters

        A single Rubrik Backup Service host can connect to multiple Rubrik clusters.

      • Host registration with RBS on multiple clusters

        A Rubrik Backup Service host can be registered to multiple Rubrik clusters by adding the host to both source and target clusters.

      • Changing the primary Rubrik cluster of a host

        When RBS installed on a host is registered with multiple Rubrik clusters, you can use RSC to change the primary Rubrik cluster of a host.

      • Installing RBS remotely on multiple hosts

        From RSC, you can install RBS remotely to manage and protect multiple Linux, Solaris, HP-UX, AIX, or Windows hosts.

      • RBS network throttling

        RBS manages data backups and may consume all network bandwidth, affecting other applications. Network throttling helps to limit the usage of RBS bandwidth and ensures that essential services remain unaffected.

    • SMB security

      With the SMB security feature in RSC, a Rubrik cluster provides security for SMB shares by requiring Active Directory authentication and enforcing SMB security.

    • Rubrik cluster deployment

      Deploying a Rubrik cluster running on Rubrik appliance nodes involves installing the CDM software on the nodes and setting up that software for your environment.

    • Cluster registration

      Rubrik offers two modes for registering clusters with Rubrik Security Cloud, online and offline.

    • Rubrik MFA with TOTP

      The Rubrik two-step verification feature uses time-based one-time passwords in addition to the user login credentials to enforce multi-factor authentication on user accounts.

    • CLI administrator role

      As a CLI administrator, a Rubrik cluster user can connect to nodes in Rubrik clusters through the CLI Secure Shell (SSH) protocol.

    • Jumbo frame support

      For Rubrik CDM versions 9.4 and later, you can set the value for the maximum transmission unit to a maximum value of 10000, instead of the default value of 1500. This larger unit size is called jumbo frames.

    • System shutdown

      A system shutdown can be done from the Rubrik CLI.

Downloading Rubrik Backup Service for cloud-native protection

Download the Rubrik Backup Service for cloud-native protection to coordinate backup activity between the Exocompute instance and RSC.

Procedure

  1. Log in to RSC.
  2. Click and select Settings.
    The Settings menu appears.
  3. Click Data Source and select Connectors.
    The Download Rubrik Backup Service (RBS) page appears.
  4. In Location, select Rubrik - Cloud.
    The corresponding connectors appear under the RBS for the selected location area.
  5. Click the download icon for the connector type that matches the host.

Results

The browser initiates a download of the RBS software.
Related concepts
  • Application-consistent protection for Azure virtual machines
    Application-consistent backups preserve the working state of applications running on a protected Azure virtual machine.
  • Application-consistent snapshot for AWS EC2 instances
    Application-consistent snapshot is a feature that you can use to take snapshots that include data from pending I/O operations. You can use this feature with AWS to take snapshots that include data from pending I/O operations between the applications running on the AWS EC2 instance and the disk attached to the AWS EC2 instance.
