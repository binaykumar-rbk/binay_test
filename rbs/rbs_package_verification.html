RBS package verification for platforms based on Linux RPM

RBS package verification confirms the authenticity and integrity of the downloaded package for platforms based on Linux RPM.

After downloading RBS, you can verify the RBS package to ensure that it works as intended. You can use these commands to confirm the authenticity and integrity of the RPM package.

  • You can run rpm checksig to verify the RPM signatures.
    ubuntu@vm-machine-abc123-abc1234:~$ rpm --checksig rubrik-agent.x86_64.rpm
rubrik-agent.x86_64.rpm: digests signatures OK
  • You can run rpm -Kv to verify the integrity of the RPM packages.
    ubuntu@vm-machine-abc123-abc1234:~$ rpm -Kv rubrik-agent.x86_64.rpm
rubrik-agent.x86_64.rpm:
    Header V4 RSA/SHA512 Signature, key ID abcd1234: OK
    Header SHA1 digest: OK
    Header SHA256 digest: OK
    Payload SHA256 digest: OK
    V4 RSA/SHA512 Signature, key ID abcd1234: OK
    MD5 digest: OK

From Rubrik CDM version 7.0.3 and later, except Rubrik CDM version 8.0, each Rubrik cluster signs the RPM package for RBS with a private key of a public-private key pair. You can use the public key to trust the RBS package.

When the system does not trust the signature, the command rpm --checksig rubrik-agent.x86_64.rpm produces the following sample output:
ubuntu@vm-machine-abc123-abc1234:~$ sudo rpm --checksig rubrik-agent.x86_64.rpm
rubrik-agent.x86_64.rpm: digests SIGNATURES NOT OK
When the system trusts the signature, the command rpm --checksig rubrik-agent.x86_64.rpm produces the following sample output:
ubuntu@vm-machine-abc123-abc1234:~$ rpm --checksig rubrik-agent.x86_64.rpm
rubrik-agent.x86_64.rpm: digests signatures OK
