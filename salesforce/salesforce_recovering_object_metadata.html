Recovering Salesforce object metadata

Recover Salesforce custom object metadata associated with individual Salesforce objects from a snapshot to a Salesforce production or sandbox environment. You can recover all metadata related to individual objects, or you can recover specific metadata. Object metadata that you can restore includes fields, validation rules, record types, and page layouts.

About this task

You can recover Salesforce custom object metadata from a snapshot to a Salesforce production or sandbox environment.

Procedure

  1. Log in to RSC.
  2. Click and select SaaS App Protection.
    The SaaS Apps Dashboard appears.
  3. Click the Salesforce inventory card.
    The details page for the Salesforce instance appears.
  4. Click Restore.
  5. Click Get Started in the Restore Metadata section.
    Selecting Don't show again skips this page the next time you restore metadata.
    The Restore Metadata wizard appears.
  6. Select Object metadata and click Next.
  7. Select the objects with the metadata to restore.
  8. Click Select metadata and select the type of metadata to recover.
    OptionDescription
    Restore all metadata Restores all metadata associated with the Salesforce object.
    Restore specific metadata Restores specific metadata, for example, a custom field of a Salesforce object.

    Select the metadata to recover.
  9. Click Close and then click Next.
  10. Specify the Salesforce environment to which to restore the object metadata.
    ActionDescription
    Use an existing destination To use an existing Salesforce environment, select the environment from Destination Environment and then click Next.
    Add a new destination To add a new destination, perform the following steps:
    • Click .
    • Enter details about the Salesforce organization to use as the recovery destination.
    • Log in to Salesforce to grant RSC access to manage your Salesforce data.
    • Click Done.
    When recovering metadata to an existing destination, by default, RSC takes the source destination as the recovery destination. This list contains the source organization and the associated sandbox organizations. RSC does not list the sandbox organizations associated with any other production organization.

    When you add a new destination, the user account you use to connect to the Salesforce organization must have write permission. RSC does not back up the Salesforce organization added from this window. When adding a backup environment, the Salesforce organization must be associated with the source. It should also have the same schema, to ensure data integrity and avoid backup issues.

  11. Click Next.
  12. Select the snapshot to use to restore object metadata.
  13. Optional: To compare snapshot data with the live environment data, hover over the snapshot, and then click Compare.
    The Compare with Live Environment dialog box appears, displaying the modified, added, and deleted records. The following table explains the different types of records:
    Record type Description
    Added The data is available in the snapshot, but not available in the live environment. When you restore the record, the snapshot data is added to the recovery destination.
    Modified The data available in the live environment and in the snapshot is different. You can hover over a modified cell to see the difference. When you restore the record, the snapshot data overrides the data in the recovery destination.
    Deleted The data is available in the live environment, but not available in the snapshot. When you restore the record, the live environment data is deleted from the recovery destination.
  14. After comparing the metadata, click Close.
  15. Click Next.
  16. Verify the recovery options and metadata, and then click Restore.

Results

RSC starts restoring the metadata to the selected Salesforce environment. You can view the progress of the recover job on the Events page.
